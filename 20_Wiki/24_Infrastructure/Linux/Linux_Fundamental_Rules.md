## 개념 한 줄 요약

**리눅스는 윈도우와는 다른 자신만의 엄격한 규칙을 가지고 있다."**
대소문자를 구분하고, 공백을 싫어하며, 강력한 권한을 가진 '뿌리(Root)'가 존재하는 시스템이다.
이 규칙들을 먼저 이해해야 터미널에서 당황하지 않는다.

---
## 왜 필요한가? (Why)

**문제점:**
- 윈도우처럼 파일명에 공백을 넣거나 대소문자를 섞어 쓰면, 리눅스 쉘(Shell)은 이를 명령어로 오해하거나 파일을 찾지 못해 "Command not found" 에러를 뿜어낸다.

**해결책:**
- 리눅스의 고유 규칙을 이해하면 터미널 환경에서 당황하지 않고 정확하게 명령을 내릴 수 있으며, 특히 보안상 중요한 '루트 권한'을 안전하게 다룰 수 있다.

---
## 실전 맥락 (Practical Context)

데이터 엔지니어는 맥북 터미널에서 로컬 작업을 하다가도, 곧바로 Docker 컨테이너나 AWS EC2 같은 리눅스 서버로 넘어가 작업해야 한다.
이때 **프롬프트의 모양(`$` vs `#`)** 만 보고도 내가 지금 사고를 칠 수 있는 권한인지 아닌지 즉시 판단하는 능력이 필수적이다.

---
## Code Core Points (핵심 규칙 & 기호)

리눅스 내비게이션의 나침반이 되는 핵심 기호들이다.

- **`/`**: 뿌리(Root). 모든 경로의 시작점.
- **`~`**: 홈 디렉토리. 현재 로그인한 사용자의 개인 방.
- **`.`**: 현재 폴더 (Relative Path 시작점).
- **`..`**: 부모(상위) 폴더.
- **`$`**: 일반 사용자 (안전함).
- **`#`**: 루트 사용자 (위험함, 무엇이든 지울 수 있음).

## 핵심 배경 지식 (Core Knowledge)

### ① 대소문자 구분 (Case Sensitive) ⭐️ 중요

- **규칙:** 리눅스는 `File.txt`와 `file.txt`를 완전히 다른 파일로 인식한다.
- **주의:** 명령어, 파일 이름, 환경 변수 모두에 적용된다.

### ② 공백 사용 금지 (Don't use space)

- **규칙:** 파일이나 폴더 이름에 공백(띄어쓰기)을 쓰는 것은 매우 위험하다.
- **이유:** 쉘은 공백을 "명령어의 구분자"로 인식하기 때문에, 공백이 들어간 파일은 명령을 내리기가 매우 까다롭다.

### ③ 루트 계정 (Root Account): 슈퍼유저

- **정의:** 시스템의 모든 권한을 가진 가장 강력한 계정이다.
- **특징:** 모든 파일과 리소스에 무제한으로 접근할 수 있다.
- **보안 수칙:**  평소에는 일반 계정으로 활동한다. 관리자 권한이 필요할 때만 **`sudo`** (Superuser Do) 명령어를 사용한다.
    - _FYI:_ 루트 디렉토리는 `/`이고, 루트 계정 전용 홈 디렉토리는 `/root`이다.
    - 맥북(macOS)에서도 시스템 설정이나 Docker 권한 설정 시 동일하게 사용한다.

>"맥북에서도 `brew install` 같은 명령어를 쓸 때 가끔 비밀번호를 요구하지? 
>그게 내부적으로는 `sudo`의 힘을 빌려 시스템 영역에 프로그램을 설치하는 거야. 
> Docker 컨테이너 안에서 `chown`을 쓸 때 권한 에러가 났던 것도 이 **루트 권한**을 제대로 다루지 못해서였지. 


### ④ 특수문자의 두 얼굴 (Special Characters) ⭐️ 실무 꿀팁

리눅스에서 `-`(하이픈)이나 `~`(물결)은 단순한 글자가 아니다.
상황에 따라 다르게 변신하기 때문에 주의해야 한다.

**1. 하이픈(`-`)의 딜레마**

- **문제점:** 리눅스 명령어는 `-`를 보면 **"옵션(Option)"** 이라고 생각한다. (예: `ls -al`)
- **상황:** 만약 파일 이름이 `-data.txt`라면?
    - `cat -data.txt` (X) -> 컴퓨터는 `-d`라는 옵션을 찾다가 에러를 낸다.
        
- **해결책:** "이건 옵션이 아니라 파일이야!"라고 알려주기 위해 **경로(`src`)** 를 명시해야 한다.
    - `cat ./-data.txt` (O) -> "현재 폴더(`.`) 안에 있는 `-data.txt`야" 라고 정확히 지목.


**2. 물결(`~`)과 뒤로가기(`-`)**

- **`~` (Tilde):** 내 **홈 디렉토리**로 가는 절대 반지만능 키다.
    - `cd ~/study` -> (O) "내 방(Home)에 있는 study 폴더로 가줘."
    - `cd study` -> (X) "지금 서 있는 곳(Current)에 study가 없으면 에러 발생."

- **`cd -` (Dash):** TV 리모컨의 **"이전 채널"** 버튼과 같다.
    - 방금 있던 폴더로 다시 돌아가고 싶을 때 유용하다. (실수로 엉뚱한 곳에 갔을 때 `cd -` 치면 복귀!)

>"경로를 적을 때 귀찮아도 **`~/`** 를 습관적으로 붙이는 게 좋아.
> `cd Downloads`는 '여기'에 다운로드 폴더가 없으면 에러가 나지만, `cd ~/Downloads`는 내가 우주 어디에 있든 **무조건 내 방의 다운로드 폴더**를 찾아가거든. 이것만 알아도 경로 에러의 80%는 줄일 수 있어!"

## 파일 시스템 경로 (File System Path)

리눅스에서 파일의 위치를 나타내는 방식은 두 가지이다.

| **종류**               | **설명**                                 | **예시**               |
| -------------------- | -------------------------------------- | -------------------- |
| **절대 경로 (Absolute)** | 루트(`/`)부터 시작하는 전체 경로. 현재 위치와 상관없이 일정함. | `/home/user/doc.txt` |
| **상대 경로 (Relative)** | 현재 내가 있는 위치를 기준으로 한 경로.                | `./doc.txt` (현재 폴더)  |

## 내 정체 확인하기 (`whoami`)

프롬프트의 기호(`$` 또는 `#`)로도 알 수 있지만, 시스템에게 "나 지금 누구야?"라고 직접 물어보는 가장 확실한 방법이다.

### 왜 필요한가? (Why)

- **확실한 검증:** 터미널 설정에 따라 프롬프트에 유저 이름이 안 나오는 경우도 있다. 이때 `whoami`는 거짓말을 하지 않는다.
- **스크립트 제어:** 쉘 스크립트를 짤 때 "이 스크립트는 `root`만 실행할 수 있게 해라" 같은 조건을 걸 때 필수적으로 사용된다.

### 상세 실습 (Detailed Analysis)

우리가 구축한 Docker 환경에서 직접 확인해보자.

```bash
# 1. 컨테이너 내부로 입장 (이미 ubuntu 유저로 설정됨)
docker exec -it linux-server bash

# 2. 나의 정체 확인하기
whoami
# 출력: ubuntu 
# (해석: Dockerfile에서 설정한 대로 ubuntu 유저로 로그인되었음 성공!)

# 3. 내 소속 확인하기 (더 자세히)
id
# 출력: uid=1000(ubuntu) gid=1000(ubuntu) groups=1000(ubuntu),27(sudo)
# (해석: 나는 1000번 유저이고, 'sudo' 그룹에 속해 있어서 관리자 권한을 빌릴 수 있구나!)

# 4. 관리자 권한 테스트
sudo whoami
# 출력: root
# (해석: 'sudo'를 붙인 순간만큼은 내가 root로서 명령을 내린 것이다.)
```

## 커맨드 프롬프트 (Command Prompt)

### 컨테이너 내부 

터미널을 켰을 때 우리를 반기는 한 줄의 텍스트이다.

`ubuntu@b3e02011b13b:~$`

- **user:** 현재 접속한 사용자 이름 (`ubuntu`)
- **host:** 현재 접속한 컴퓨터(서버) 이름 (`b3e02011...`)
- **prompt symbol:** `$`는 일반 사용자, `#`은 루트 사용자를 의미한다.


### 예시: 내 맥북 프롬프트 분석 

> `root@6baab77f1293:/#` 

- **`root`**: 현재 사용자 이름 (최고 관리자)
- **`6baab77f1293`**: 호스트 이름 (Docker가 자동으로 부여한 컨테이너 ID)
- **`/`**: 현재 위치 (최상위 루트 디렉토리)
- **`#`**: **가장 중요한 기호!** 현재 관리자(`root`) 권한임을 뜻합니다. (일반 유저는 `$`)


|**구성 요소**|**실제 내용**|**의미**|
|---|---|---|
|**User**|`gong`|현재 로그인한 **사용자 이름**입니다.|
|**Host**|`gimhangyeong-ui-MacBookPro`|현재 명령어를 입력받는 **컴퓨터(내 맥북)의 이름**입니다.|
|**Location**|`~`|현재 내가 있는 **폴더 위치**입니다. `~`는 내 방(홈 디렉토리)을 뜻합니다.|
|**Separator**|`@`|"사용자 **at** 컴퓨터"라는 뜻으로 유저와 호스트를 구분합니다.|
|**Symbol**|`` 또는 `$`|명령어를 입력할 준비가 되었다는 신호입니다. (주로 일반 사용자는 `$`, 루트는 `#`을 씁니다.)|

## 초보자가 자주 착각하는 포인트

- **"맥북 터미널에서 쳤는데 제 이름(`gong`)이 나와요!"**
    - 컨테이너 안으로 **입장(`docker exec`)** 하지 않았기 때문이다. 
    - 터미널 창은 하나지만, 내가 서 있는 땅이 '맥북(Host)'인지 '리눅스(Container)'인지 항상 구분해야 한다.
        
- **"`sudo whoami`를 치면 유저가 바뀌나요?"**
    - 아니, 영구적으로 바뀌는 게 아니라 **그 명령어 한 줄을 실행하는 동안만** 잠시 `root`의 가면을 쓰는 것이다. 
    - 영구적으로 바뀌려면 `su - root` 같은 명령어를 써야 한다.
