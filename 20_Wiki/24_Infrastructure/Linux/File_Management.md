## 핵심 개념 요약

리눅스 시스템의 가장 기본 단위인 **파일과 디렉토리(폴더)를 생성, 복사, 이동, 삭제하는 필수 명령어 모음**입니다.

---
## 이것이 왜 필요한가 (Why)

**문제:** 
- 서버 환경(CLI)에는 마우스로 드래그 앤 드롭할 수 있는 '폴더 창'이나 실수로 지운 파일을 살려내는 '휴지통'이 없습니다.

**해결:**
- 정확한 명령어로 파일을 다루는 법을 익혀야만, 실수 없이 데이터를 백업(`cp`)하거나 이동(`mv`)하고 불필요한 파일을 정리(`rm`)할 수 있습니다.

---
## 실무 적용 사례

데이터 엔지니어는 매일 이런 작업을 합니다:
1.  **데이터 수집:** 오늘의 데이터를 저장할 날짜별 폴더 생성 (`mkdir`)
2.  **설정 파일 생성:** 간단한 환경 설정이나 테스트용 데이터를 빠르게 생성 (`echo`, `touch`)
3.  **데이터 이동:** 처리가 끝난 CSV 파일을 '완료(processed)' 폴더로 이동 (`mv`)
4.  **청소:** 1년 지난 오래된 로그 파일 삭제 (`rm`)

---
## 코드 핵심 포인트

1.  **생성:** `mkdir`(폴더), `touch`(빈 파일), **`echo`**(내용 있는 파일)
2.  **관리:** `cp`(복사), `mv`(이동/이름변경)
3.  **삭제:** `rm`(삭제) - **가장 주의해야 함!**

---
## 상세 코드 분석

### 파일과 폴더 만들기 (Create)

```bash
# 1. 빈 파일 만들기 (내용 없음, 시간 갱신용)
touch data.csv

# 2. 내용이 있는 파일 만들기 (가장 빠름!)
# echo "내용" > 파일명 : 화면에 출력할 내용을 파일로 저장(덮어쓰기)
echo "user_id, name" > headers.csv

# 3. 폴더 만들기 (-p: 중간 폴더까지 한 번에 생성)
# 2026 폴더가 없어도 에러 없이 2026 -> 01 -> data 순으로 다 만들어줌
mkdir -p 2026/01/data
```

### 복사하고 이동하기 (Copy & Move)

```bash
# 1. 파일 복사 (cp 원본 복사본)
cp data.csv data_backup.csv

# 2. 폴더 복사 (-r: Recursive, 폴더 내부 내용물까지 재귀적으로 복사)
# 폴더를 복사할 땐 -r 옵션이 없으면 에러가 남!
cp -r 2026/01 2026/01_backup

# 3. 파일 이동 (mv 원본 목적지)
mv data.csv ./archive/

# 4. 이름 변경 (이동할 곳이 같은 자리면 이름이 바뀜)
mv old_name.txt new_name.txt
```

### 삭제하기 (Delete - 휴지통 없음 🚨)

```bash
# 1. 파일 삭제
rm data.csv

# 2. 폴더 삭제 (-r: 폴더 내부까지, -f: 묻지 말고 강제로)
# 실무에서 가장 조심해서 써야 하는 명령어 1위
rm -rf ./archive
```

## 초보자가 자주 하는 실수

1.  **`echo` vs `touch`:**
    - `touch`는 0바이트짜리 **완전 빈 껍데기**만 만듭니다.
    - 간단한 글자라도 적어서 저장하고 싶다면 **`echo "내용" > 파일명`**이 훨씬 편합니다.

2. **`mv`의 두 얼굴:**
    - `mv`는 파일을 다른 폴더로 옮길 때도 쓰지만, **이름을 바꿀 때(Rename)** 도 씁니다.
    - 리눅스에는 `rename`이라는 별도 명령어가 잘 안 쓰입니다.

- **`cp` 에러:**
    - 폴더를 복사하려고 할 때 그냥 `cp`만 치면 "omitting directory"라며 거부합니다. 
    - 폴더는 무조건 **`-r`** (Recursive) 옵션을 붙여야 합니다.
