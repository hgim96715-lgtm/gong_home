## 개념 한 줄 요약

**"엑셀의 '개수 세기(Count)', '정렬(Sort)', '중복 제거(Remove Duplicates)' 기능을 터미널 명령어로 옮겨놓은 것이다."**

---
## 왜 필요한가? (Why)

**문제점:**
- 10GB가 넘는 대용량 CSV 파일이나 로그 파일은 엑셀에서 열리지도 않는다.
- "오늘 가장 많이 에러를 낸 사용자 IP TOP 5를 뽑아줘" 같은 요청을 받았을 때, 파일을 일일이 눈으로 셀 수는 없다.

**해결책:**
- **`wc`** 로 데이터의 규모(행 수)를 파악하고,
- **`sort`** 와 **`uniq`** 를 조합해서 데이터의 빈도수와 순위를 0.1초 만에 뽑아낸다.

---
## 실무 적용 사례 (Practical Context)

데이터 엔지니어의 아침 루틴:
1.  **데이터 양 확인:** "어제 수집된 로그가 몇 줄이지?" -> `wc -l`
2.  **이상치 탐지:** "혹시 중복 데이터가 들어왔나?" -> `uniq -d`
3.  **랭킹 분석:** "가장 많이 접속한 국가는 어디지?" -> `sort | uniq -c | sort -nr` (국룰 콤보)

## `wc` (Word Count): 데이터 양 측정기

파일 내용을 다 읽을 필요 없이, **"이 파일 얼마나 커?"** 를 숫자로 딱 보여준다.

### **CASE 1. 파일 하나만 검사할** 때

**(`wc file.txt`)** 결과는 항상 `[줄 수] [단어 수] [용량] [파일명]` 순서로 나온다.

```bash
wc file.txt
# 출력 결과 해석:
#  8      14      82      file.txt
#  ↑       ↑       ↑         ↑
# (줄)   (단어)  (바이트)   (이름)
```

- **8 (Lines):** 엔터키를 8번 쳤다는 뜻. (행의 개수)
- **14 (Words):** 띄어쓰기로 구분된 단어 덩어리가 14개라는 뜻.
- **82 (Bytes):** 파일 크기(용량)가 82 바이트라는 뜻.

### **CASE 2. 여러 파일을 한 번에 검사할 때 (`wc file file1`)**


파일을 여러 개 나열하면, 각각의 통계를 보여주고 **맨 마지막에 합계(total)** 를 구해준다.

```bash
# file.txt와 file1.txt 두 개를 동시에 세어줘!
wc file.txt file1.txt

# 출력 결과:
#  8      14      84      file.txt   (첫 번째 파일)
#  4      12      45      file1.txt  (두 번째 파일)
# 12      26     129      total      (★ 합계: 두 파일의 총합)
```

**활용:** 쪼개진 로그 파일(`log_part1`, `log_part2`)의 전체 데이터 양을 계산할 때 계산기 두드릴 필요 없이 이 기능을 쓰면 된다.

### **CASE 3. 자주 쓰는 옵션 들 **

 - 숫자가 너무 많아서 헷갈리면 옵션으로 하나만 콕 집어서 본다.

```bash
# "다 필요 없고, 몇 줄인지만 알려줘." (가장 많이 씀)
wc -l file.txt
# 출력: 8 file.txt
```

- **파일 없이 즉석에서 셀 때 (파이프 `|` 사용)**

방금 내가 친 명령어의 결과(`Standard Input`)를 바로 세어볼 수도 있다.

```bash
echo "hello world" | wc
# 출력: 1       2      12
```

**결과 분석 (1, 2, 12의 비밀)**

- **1 (줄):** "hello world" 한 줄이니까 1.
- **2 (단어):** 공백을 기준으로 "hello", "world" 두 단어니까 2.
- **12 (바이트):** 글자 수는 10개 같지만, 컴퓨터는 **보이지 않는 문자**까지 센다.
    - `h` `e` `l` `l` `o` (5글자)
    - (공백 1글자)
    - `w` `o` `r` `l` `d` (5글자)
    - **`\n` (엔터키 1글자)** 👈 `echo`는 마지막에 무조건 줄바꿈을 포함한다!
    - **합계: 5 + 1 + 5 + 1 = 12 byte**

 **주요 옵션:**
 - `-l`: 줄 수 (Lines) 
 - `-w`: 단어 수 (Words)
 - `-c`: 바이트 수 (Bytes)
 - `-m`: 문자 수 (Characters)

##  `sort`: 줄 세우기 반장

데이터를 오름차순(abc..) 혹은 내림차순(zYX..)으로 정렬한다. `sort [options] [file...]`

```bash
# 기본: 텍스트 기준 오름차순 정렬 (1, 10, 2 순서로 나옴 주의!)
sort data.txt

# [실무] 숫자 크기대로 내림차순 정렬 (Top 10 뽑을 때)
sort -nr data.txt
```

### 주요 옵션 전체 정리

|옵션|의미|언제 쓰나|
|:--|:--|:--|
|`-n`|숫자 크기로 정렬 (Numeric)|숫자 데이터 정렬할 때 필수|
|`-r`|거꾸로 정렬 (Reverse)|내림차순|
|`-k`|특정 열 기준 정렬 (Key)|CSV, TSV 특정 컬럼 정렬|
|`-t`|구분자 지정 (Token)|쉼표/탭/파이프 구분 파일|
|`-u`|중복 제거하며 정렬 (Unique)|`sort \| uniq` 대신 한방에|
|`-f`|대소문자 무시 정렬 (Fold)|알파벳 혼용 데이터|
|`-b`|앞쪽 공백 무시 (Blank)|들여쓰기 섞인 데이터|
|`-s`|동일값은 원래 순서 유지 (Stable)|정렬 순서 보장이 필요할 때|
|`-h`|사람이 읽는 크기 단위 정렬|1K, 2M, 3G 같은 용량 정렬|
|`-o`|결과를 파일로 저장 (Output)|리다이렉트(`>`) 대신 사용|


### [심화] CSV / TSV / 파이프 구분 파일 정렬 (`-t`, `-k`)

데이터 엔지니어 필수. 데이터가 띄어쓰기가 아니라 **쉼표(`,`)**, **탭(`\t`)**, **파이프(`|`)** 로 구분되어 있다면 구분자를 명시해야 한다.

> **암기 꿀팁:** 구분자로 잘라낸 데이터 조각 하나하나를 컴퓨터 공학에서는 **토큰(Token)** 이라고 불러. 그래서 **"데이터를 Token(-t)으로 쪼개는 기준선!"** 이라고 외우면 절대 안 까먹어.

```bash
# 예시 파일 (s1.txt): 이름,점수
# Kim,80
# Lee,100
# Park,50

# "쉼표(,)로 나누고, 2번째 컬럼 숫자 크기로 정렬"
sort -t, -k2 -n s1.txt
```

### 구분자 종류별 사용법

```bash
# ① 쉼표(,) 구분 — CSV
sort -t',' -k2 -n data.csv

# ② 탭(\t) 구분 — TSV  ← 가장 많이 빠뜨리는 것!
sort -t$'\t' -k2 -n data.tsv
#    ^^^^^^^^
#    $'\t' 는 쉘에게 "이건 탭 문자야" 라고 알려주는 문법
#    그냥 -t'\t' 나 -t"\t" 로 쓰면 탭이 아니라 \t 문자 그대로 인식해서 작동 안 함!

# ③ 파이프(|) 구분
sort -t'|' -k2 -n data.txt

# ④ 콜론(:) 구분 — /etc/passwd 같은 시스템 파일 형태
sort -t':' -k3 -n /etc/passwd
```

> **`$'\t'` 왜 이렇게 쓰나?** 탭은 눈에 안 보이는 특수문자라서 쉘이 직접 인식하려면 `$'...'` 문법이 필요하다. `$'...'` = "안에 있는 이스케이프 문자를 실제 문자로 변환해줘" 라는 쉘 전용 표기법.

###  `-k` 옵션 함정: 범위를 명시해라

```bash
# ⚠️ -k2 는 "2번째 컬럼부터 줄 끝까지" 전부를 기준으로 삼는다
sort -t',' -k2 data.csv      # 2번째 컬럼 + 그 이후 전부 기준

# ✅ -k2,2 는 "정확히 2번째 컬럼만" 기준으로 삼는다 (실무 권장)
sort -t',' -k2,2 data.csv    # 2번째 컬럼만 기준

# ✅ 다중 정렬: 2번째 컬럼으로 먼저 정렬, 같으면 3번째 컬럼으로 정렬
sort -t',' -k2,2 -k3,3n data.csv
```

---

### 자주 쓰는 실무 조합 패턴

```bash
# 상위 10개만 뽑기 (Top N)
sort -nr data.txt | head -10

# 중복 제거하면서 정렬 (-u)
sort -u names.txt

# 결과를 파일로 저장 (-o) — 입력파일 덮어쓸 때도 안전함
sort -nr data.txt -o result.txt
# 리다이렉트(>)로는 같은 파일 덮어쓰기가 위험하지만 -o는 안전

# 대소문자 무시하고 알파벳 정렬 (-f)
sort -f names.txt

# 사람이 읽는 용량 단위로 정렬 (-h)  ← du 명령어랑 자주 조합
du -sh * | sort -h
# 1K, 500M, 2G 같은 단위를 크기 순서로 제대로 정렬해줌
```

## `uniq`: 중복 제거기

`uniq [options] [input_file] [output_file]`

중복된 줄을 걸러내어 하나만 보여주거나 개수를 센다. 

**⚠️ 주의:** 반드시 **정렬(`sort`)된 상태**에서 써야 한다! (인접한 줄만 비교하기 때문)

**문법 (Syntax)**

파이프 없이도 입출력 파일을 직접 지정할 수 있다.

```bash
# 문법: uniq [옵션] [읽을파일] [저장할파일]
uniq input.txt output.txt
```

### **빈도수 세기 (`-c`)** 데이터가 각각 몇 번 나왔는지 센다.

```bash
uniq -c u1.txt
# 결과:
#   2 1  (1은 두 번 나옴)
#   1 2  (2는 한 번 나옴)
#   2 3
#   1 4
#   1 5
#   2 6
```

###  **중복된 놈만 체포해 (`-d`)** 2번 이상 등장한 데이터(Repeated)만 보여준다. **(중복 데이터 찾기)**

```bash
uniq -d u1.txt
# 결과:
# 1
# 3
# 6
```

### **혼자 온 손님만 입장해 (`-u`)** 딱 1번만 등장한 데이터(Unique)만 남기고 중복된 건 다 버린다.

```bash
uniq -u u1.txt
# 결과:
# 2
# 4
# 5
```

### sort 랑 uniq 같이 

```bash
# 중복 제거하고 하나씩만 보기
sort data.txt | uniq

# [실무] 중복된 횟수(빈도) 세기 ⭐️
sort data.txt | uniq -c
```

**주요 옵션 요약:**
- `-c`: 중복된 횟수를 앞에 표시 (Count).
- `-d`: 중복된 데이터만 출력 (Duplicate).
- `-u`: 중복되지 않은(유일한) 데이터만 출력 (Unique).

## 상세 분석: 최강의 콤보 (The Ultimate Pipeline) 

데이터 엔지니어라면 숨 쉬듯이 치게 될 명령어 조합이다. 
**목표: "가장 많이 등장한 단어 Top 3 뽑기"**

```bash
# 명령어 흐름
cat access.log | sort | uniq -c | sort -nr | head -3
```

- **`cat`**: 파일 내용을 읽는다.
- **`sort`**: 같은 단어끼리 붙어있게 정렬한다. (uniq를 쓰기 위한 준비)
- **`uniq -c`**: 겹치는 단어를 하나로 합치고, 왼쪽에 **횟수**를 적어준다.
- **`sort -nr`**: 횟수(숫자 `-n`)를 기준으로 많은 순서대로(`-r`) 다시 정렬한다.
- **`head -3`**: 위에서 3등까지만 자른다.

## 초보자가 자주 하는 실수 (Misconceptions)

### ① "uniq를 썼는데 중복 제거가 안 돼요!"

- **원인:** 정렬(`sort`)을 먼저 안 해서 그렇다.
- `uniq`는 바로 윗줄이랑 아랫줄만 비교하는 단순한 친구다. `sort`와 `uniq`는 항상 **바늘과 실**처럼 붙어 다녀야 한다.

### ② "숫자 정렬이 이상해요 (1, 10, 2...)"

- **원인:** 컴퓨터는 기본적으로 숫자를 '글자'로 취급해서 사전 순서로 정렬하기 때문이다.    
- 반드시 **`-n` (Numeric)** 옵션을 줘야 `1, 2, 10` 순서로 올바르게 정렬된다.
