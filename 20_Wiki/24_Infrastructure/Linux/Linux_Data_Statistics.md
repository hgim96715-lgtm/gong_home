# Linux Data Statistics — `wc` · `sort` · `uniq`

## 개념 한 줄 요약

> **"엑셀의 개수 세기·정렬·중복 제거 기능을 터미널에서 쓰는 것이다."**

|도구|역할|
|---|---|
|`wc`|데이터 규모 파악 (줄 수 · 단어 수 · 용량)|
|`sort`|데이터 정렬 (오름차순 · 내림차순 · 컬럼 기준)|
|`uniq`|중복 제거 · 빈도 집계|

---

## 왜 필요한가?

- 10GB 넘는 로그 파일은 엑셀에서 열리지도 않는다.
- "오늘 에러 가장 많이 낸 IP TOP 5" 를 눈으로 세는 건 불가능하다.

**데이터 엔지니어의 아침 루틴:**

```bash
wc -l access.log          # 1. 어제 로그가 몇 줄 쌓였지?
sort access.log | uniq -d # 2. 혹시 중복 데이터 들어왔나?
sort access.log | uniq -c | sort -nr | head -5  # 3. 가장 많이 접속한 IP TOP 5
```

# ① `wc` — 데이터 양 측정기

파일을 다 읽을 필요 없이 **"이 파일 얼마나 커?"** 를 숫자로 보여준다.

**출력 순서:** `[줄 수] [단어 수] [바이트] [파일명]`

```bash
wc file.txt
#  8      14      82      file.txt
#  ↑       ↑       ↑
# (줄)   (단어)  (바이트)
```

---

## A. 파일 하나

```bash
wc file.txt
#  8   14   82   file.txt
# 줄   단어  바이트
```

---

## B. 여러 파일 한 번에

각각의 통계를 보여주고 **마지막에 합계(total)** 를 자동으로 구해준다.

```bash
wc file.txt file1.txt
#  8    14    84    file.txt
#  4    12    45    file1.txt
# 12    26   129    total      ← 자동 합산 ✅
```

> **활용:** 쪼개진 로그 파일(`log_part1`, `log_part2`)의 전체 데이터 양을 계산할 때 계산기 필요 없음.

---

## C. 파이프로 즉석에서 세기

```bash
echo "hello world" | wc
#  1       2      12
#  줄      단어   바이트
```

**왜 바이트가 12인가?** 글자는 10개인데 12인 이유:

```
h e l l o   w o r l d \n
↑         ↑           ↑
5글자    공백 1개    echo가 자동으로 붙이는 줄바꿈

5 + 1 + 5 + 1(\n) = 12 byte
```

> `echo`는 마지막에 무조건 `\n`(줄바꿈)을 붙인다.

---

## D. 주요 옵션

|옵션|의미|예시|
|---|---|---|
|`-l`|줄 수 (Lines) ⭐️가장 많이 씀|`wc -l file.txt`|
|`-w`|단어 수 (Words)|`wc -w file.txt`|
|`-c`|바이트 수 (Bytes)|`wc -c file.txt`|
|`-m`|문자 수 (Characters)|`wc -m file.txt`|

```bash
# 가장 자주 쓰는 패턴
wc -l file.txt
# → 8 file.txt
```

# ② `sort` — 줄 세우기 반장

데이터를 오름차순 또는 내림차순으로 정렬한다.

```bash
# 기본: 텍스트 사전 순 정렬
# ⚠️ 숫자도 문자로 취급 → 1, 10, 2 순서로 나옴!
sort data.txt

# 실무: 숫자 크기 내림차순 (Top N 뽑을 때)
sort -nr data.txt
```

---

## A. 주요 옵션 전체 정리

|옵션|의미|언제 쓰나|
|---|---|---|
|`-n`|숫자 크기로 정렬 (Numeric)|숫자 데이터 정렬할 때 **필수**|
|`-r`|역순 정렬 (Reverse)|내림차순 (큰 것부터)|
|`-k`|특정 열 기준 정렬 (Key)|CSV·TSV 특정 컬럼 정렬|
|`-t`|구분자 지정 (Token)|쉼표·탭·파이프 구분 파일|
|`-u`|중복 제거하며 정렬 (Unique)|`sort \| uniq` 대신 한 방에|
|`-f`|대소문자 무시 (Fold)|알파벳 혼용 데이터|
|`-b`|앞쪽 공백 무시 (Blank)|들여쓰기 섞인 데이터|
|`-s`|동일값은 원래 순서 유지 (Stable)|정렬 순서 보장이 필요할 때|
|`-h`|사람이 읽는 크기 단위 정렬|1K, 2M, 3G 같은 용량 정렬|
|`-o`|결과를 파일로 저장 (Output)|리다이렉트(`>`) 대신 사용|

---

## B. CSV · TSV · 파이프 구분 파일 정렬 (`-t`, `-k`) ⭐️

데이터가 공백이 아니라 쉼표·탭·파이프로 구분되어 있다면 구분자를 명시해야 한다.

> **암기 팁:** 구분자로 잘라낸 데이터 조각을 **토큰(Token)** 이라고 한다. `-t` = "Token으로 쪼개는 기준선"

```bash
# 예시 데이터 (s1.txt)
# Kim,80
# Lee,100
# Park,50

# "쉼표(,)로 나누고, 2번째 컬럼 숫자 크기로 정렬"
sort -t, -k2 -n s1.txt
```

### 구분자 종류별 사용법

```bash
# ① 쉼표(,) — CSV
sort -t',' -k2 -n data.csv

# ② 탭(\t) — TSV  ⚠️ 가장 많이 실수하는 부분!
sort -t$'\t' -k2 -n data.tsv
#    ^^^^^^^^
#    그냥 -t'\t' 로 쓰면 탭이 아니라 \t 문자 그대로 인식해서 작동 안 함!

# ③ 파이프(|)
sort -t'|' -k2 -n data.txt

# ④ 콜론(:) — /etc/passwd 같은 시스템 파일
sort -t':' -k3 -n /etc/passwd
```

> **`$'\t'` 왜 이렇게 쓰나?** 탭은 눈에 안 보이는 특수문자라서 쉘이 직접 인식하려면 `$'...'` 문법이 필요하다.
>  `$'...'` = "안에 있는 이스케이프 문자를 실제 문자로 변환해줘" 라는 쉘 전용 표기법.

---

## C. `-k` 옵션 함정 ⚠️

```bash
# ⚠️ -k2 : "2번째 컬럼부터 줄 끝까지" 전부를 기준으로 삼음
sort -t',' -k2 data.csv

# ✅ -k2,2 : "정확히 2번째 컬럼만" 기준 (실무 권장)
sort -t',' -k2,2 data.csv

# ✅ 다중 정렬: 2번째 컬럼 우선, 같으면 3번째 컬럼으로
sort -t',' -k2,2 -k3,3n data.csv
```

---

## D. 자주 쓰는 실무 조합

```bash
# Top N 뽑기
sort -nr data.txt | head -10

# 중복 제거하며 정렬 (-u)
sort -u names.txt

# 결과를 파일로 저장 (-o)
# ⚠️ 리다이렉트(>)로 같은 파일 덮어쓰기는 위험하지만 -o는 안전
sort -nr data.txt -o result.txt

# 대소문자 무시하고 알파벳 정렬 (-f)
sort -f names.txt

# 용량 단위 정렬 (-h) — du 랑 자주 조합
du -sh * | sort -h
# 1K, 500M, 2G 같은 단위를 크기 순서로 올바르게 정렬
```

# ③ `uniq` — 중복 제거기

중복된 줄을 걸러내어 하나만 보여주거나 개수를 센다.

**문법:** `uniq [옵션] [읽을파일] [저장할파일]`

> **⚠️ 핵심 주의사항:** 반드시 **`sort` 된 상태**에서 써야 한다! `uniq`는 바로 위아래 줄만 비교하기 때문에, 정렬 안 하면 중복 제거가 제대로 안 된다.

```bash
# 파이프 없이 파일 직접 지정도 가능
uniq input.txt output.txt
```

---

## A. 기본 옵션들

```bash
# 기본 (옵션 없음): 인접한 중복 제거, 각 값 한 번씩만 출력
uniq u1.txt
# 1 / 2 / 3 / 4 / 5 / 6

# -c : 각 값이 몇 번 나왔는지 횟수 표시 (Count)
uniq -c u1.txt
#   2 1   ← 1이 두 번 나옴
#   1 2
#   2 3
#   1 4

# -d : 2번 이상 등장한 중복 데이터만 출력 (Duplicate)
uniq -d u1.txt
# 1 / 3 / 6

# -u : 딱 1번만 나온 고유 데이터만 출력 (Unique)
uniq -u u1.txt
# 2 / 4 / 5
```

| 옵션   | 의미                 | 용도         |
| ---- | ------------------ | ---------- |
| `-c` | Count — 횟수 표시      | 빈도 분석      |
| `-d` | Duplicate — 중복만 출력 | 중복 데이터 탐지  |
| `-u` | Unique — 고유값만 출력   | 유일한 데이터 추출 |

---

## B. `sort` 와 함께 쓰기

```bash
# 중복 제거하고 하나씩만 보기
sort data.txt | uniq

# 빈도 분석 ⭐️ 실무 핵심
sort data.txt | uniq -c
```

---

## C. `-c` 옵션의 공백 문제 ⚠️

`uniq -c` 는 숫자를 **오른쪽 정렬** 로 출력하기 때문에 앞에 공백이 붙는다.

```bash
uniq -c u1.txt
# 실제 출력:
#       2 1   ← 앞에 공백이 있다!
#       1 2
```

이 공백이 왜 붙냐면, 숫자가 커졌을 때 세로로 보기 좋게 맞추려는 포맷 때문이다.

```
# 공백 없으면 → 읽기 어렵다
100 apple
1 banana
20 cherry

# 공백 있으면 → 세로 정렬이 맞아서 읽기 쉽다
100 apple
  1 banana
 20 cherry
```

**문제:** 채점 문제나 파이프라인에서 `"2 apple"` 형식을 기대할 때, 앞의 공백 때문에 **포맷 불일치로 오답** 처리된다.

**해결: `awk '{$1=$1; print}'`**

```bash
sort data.txt | uniq -c | awk '{$1=$1; print}'
#       2 apple   →   2 apple  ✅ 공백 제거!
#       1 banana  →   1 banana
```

`$1=$1` 원리: awk가 필드를 **재조합(rebuild)** 할 때 OFS(기본값: 공백 1개)로 다시 이어붙이면서 앞뒤 불필요한 공백이 사라진다.

> `awk` 상세 내용 → [[Linux_Stream_Editor#C. 공백 제거 트릭 — `$1=$1` ⭐️|awk 공백 제거 트릭]] 참고

# 최강의 콤보 파이프라인 ⭐️

> **"가장 많이 등장한 항목 Top 3 뽑기"**

```bash
cat access.log | sort | uniq -c | sort -nr | head -3
```

각 단계가 하는 일:

```
cat access.log   → 파일 내용을 읽는다
     ↓
sort             → 같은 항목끼리 붙어있게 정렬한다 (uniq 사용 준비)
     ↓
uniq -c          → 중복을 묶고 횟수를 앞에 적는다
     ↓              예: "      3 192.168.0.1"
sort -nr         → 횟수(숫자) 기준으로 많은 순서대로 재정렬
     ↓
head -3          → 상위 3개만 잘라낸다
```

**공백까지 제거하려면:**

```bash
cat access.log | sort | uniq -c | sort -nr | head -3 | awk '{$1=$1; print}'
```

# 초보자가 자주 하는 실수

|실수|원인|해결|
|---|---|---|
|`uniq` 썼는데 중복 제거가 안 됨|`sort` 없이 `uniq` 단독 사용|반드시 `sort \| uniq` 조합으로|
|숫자 정렬이 1, 10, 2 순서로 나옴|기본 정렬은 문자(사전) 순|`-n` 옵션 추가|
|TSV 파일 구분자 인식 안 됨|`-t'\t'` 로 쓰면 탭이 아니라 문자 그대로 인식|`-t$'\t'` 로 써야 함|
|`uniq -c` 결과에 공백이 붙음|숫자 오른쪽 정렬 포맷|`awk '{$1=$1; print}'` 추가|
