## 개념 한 줄 요약

**"집 전체 평수(`df`)를 볼 것인가, 방 안의 가구 크기(`du`)를 잴 것인가?"**

* **`df` (Disk Free):** 시스템 전체의 **남은 용량(부동산 평수)** 을 확인한다. (숲을 봄)
* **`du` (Disk Usage):** 특정 폴더나 파일이 **실제로 차지하는 용량(가구 크기)** 을 확인한다. (나무를 봄)

---
## 왜 필요한가? (Why)

**문제점:**
- "서버에 파일을 저장하려는데 `No space left on device` 에러가 떠요!"
- "도대체 뭐가 내 하드디스크를 100GB나 잡아먹고 있는 거지?"

**해결책:**
- **`df -h`** 로 어느 파티션이 꽉 찼는지 확인한다. (예: `/` 파티션이 99% 사용 중)
- **`du -sh`** 로 범인이 되는 거대 파일을 찾아내서 지운다.

---

##  Code Core Points: ① `df` (전체 현황 파악) 🏠

파일 시스템 전체의 마운트 상태와 여유 공간을 보여준다.

```bash
# -h (Human-readable): 사람이 읽기 편하게 GB, MB 단위로 보여줌 (필수 옵션!)
df -h
```

**결과**

```text
Filesystem            Size  Used Avail Use% Mounted on
overlay               1.8T   64G  1.7T   4% /
tmpfs                  64M     0   64M   0% /dev
shm                    64M     0   64M   0% /dev/shm
/run/host_mark/Users  1.9T  204G  1.7T  11% /example
/dev/vda1             1.8T   64G  1.7T   4% /etc/hosts
```

**결과 해석**

|컬럼|의미|실무 포인트|
|---|---|---|
|**Filesystem**|장치 이름 (예: `/dev/vda1`)|어떤 디스크인지 식별|
|**Size**|전체 디스크 크기 (예: 59G)|서버 용량 한계|
|**Used**|사용 중인 용량 (예: 40G)|증가 추이 체크|
|**Avail**|사용 가능한 남은 용량 (예: 16G)|**0이면 서비스 장애** 🚨|
|**Use%**|디스크 사용률 (예: 72%)|**80~90%부터 경고** ⚠️|
|**Mounted on**|디스크가 연결된 위치 (예: `/`)|`/` 가 가장 중요|

## Code Core Points: ② `du` (범인 색출)

특정 디렉토리나 파일의 용량을 계산한다.

### A. 자주 쓰는 옵션 3대장

- **`-h`**: 보기 좋게 출력 (K, M, G)
- **`-s` (Summarize)**: 구질구질하게 다 보여주지 말고 **합계(Total)** 만 보여줘.
- **`-a` (All)**: 폴더뿐만 아니라 **파일**까지 다 보여줘.

### B. 실전 명령어 조합

```bash
# 1. 현재 폴더의 총 용량만 딱 보고 싶을 때 (가장 많이 씀)
du -sh

# 2. "현재 위치 바로 아래(1단계 깊이)" 폴더들의 용량을 각각 보고 싶을 때 ⭐️
# 이거 안 쓰면 하위 폴더 수천 개가 다 출력돼서 터미널 터짐.
du -h --max-depth=1 .
```

**결과**

```text
1. du -sh 결과
52K	.

2. du -h --max-depth=1 . 결과
8.0K	./.config
12K	./.ssh
52K	.
```

**결과 해석**

| 항목              | 의미                          |
| --------------- | --------------------------- |
| `du -sh`        | 현재 디렉터리(`.`) 전체 용량을 한 줄로 요약 |
| `52K .`         | 현재 디렉터리 총 사용량이 **52KB**     |
| `--max-depth=1` | 바로 아래 1단계 디렉터리까지만 표시        |
| `./.config`     | `.config` 디렉터리가 **8KB** 사용  |
| `./.ssh`        | `.ssh` 디렉터리가 **12KB** 사용    |
| `.`             | 하위 디렉터리 포함한 현재 디렉터리 전체 용량   |

## 실무 적용 시나리오 (Workflow)

**미션: "서버 용량이 꽉 찼다! 범인을 찾아라!"**

### Step 1. 전체 스캔 (`df`)

```bash
df -h
# 결과: / (루트) 파티션이 98% 찼음 확인.
```

### **Step 2. 범인 추적 (`du`)** 루트(`/`)에서 시작해서 용량이 큰 폴더를 찾아 들어간다.

```bash
# 1. 루트에서 가장 용량 큰 폴더 찾기
cd /
sudo du -h --max-depth=1 | sort -hr
# 결과: /var 폴더가 50GB네? 범인은 저기다!

# 2. /var 로 이동해서 다시 추적
cd /var
sudo du -h --max-depth=1 | sort -hr
# 결과: /var/log 가 48GB네? 로그 파일이 범인이다!
```

## 초보자가 자주 하는 실수 (Misconceptions)

### ① "`du` 명령어가 멈춘 것 같아요!"

- `du`는 실제로 파일 하나하나를 방문해서 크기를 잽니다. 
- 파일이 수십만 개인 폴더에서 그냥 `du`를 치면 계산하느라 한참 걸립니다.
- 반드시 **`-s` (요약)** 나 **`--max-depth=1`** 옵션을 써야 합니다.

### ② "파일을 지웠는데 용량이 안 늘어나요! (`df` vs `du` 차이)"

- `rm`으로 로그 파일을 지웠는데 `df` 해보면 여전히 100%일 때가 있습니다.
- **이유:** 어떤 프로그램이 그 파일을 아직 잡고(실행 중) 있어서 완전히 삭제가 안 된 상태(Zombie file)입니다.
- **해결:** 해당 프로그램을 재시작하거나 프로세스를 종료(`kill`)해야 용량이 반환됩니다.

### ③ "`Permission denied` 에러가 나요."

- 시스템 폴더(`/var`, `/root` 등)의 용량을 재려면 권한이 필요합니다.
- 앞에 **`sudo`**를 붙이세요. (`sudo du -sh ...`)
