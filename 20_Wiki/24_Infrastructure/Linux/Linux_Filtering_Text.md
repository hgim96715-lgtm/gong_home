## 개념 한 줄 요약

**"산더미처럼 쌓인 텍스트 속에서 내가 원하는 단어(패턴)가 들어있는 줄(Line)만 쏙쏙 뽑아주는 거름망(Filter)이다."**
(이름의 유래: **G**lobal **R**egular **E**xpression **P**rint)

---
## 왜 필요한가? (Why)

**문제점:**
- 서버 로그 파일(`server.log`)은 수십만 줄이 넘는다. 여기서 "Error"가 발생한 부분만 눈으로 찾는 건 불가능하다.
- `ls -al`을 쳤는데 파일이 1,000개가 넘으면, 내가 찾는 파일이 어디 있는지 스크롤 하다가 놓친다.

**해결책:**
- **`grep`** 을 사용하면 수만 줄의 데이터 중 내가 궁금한 단어가 포함된 줄만 화면에 출력해준다.
- 불필요한 정보는 버리고 핵심 정보만 남기는 **데이터 엔지니어의 필수 생존 도구**다.

---
## 실무 적용 사례 (Practical Context)

1.  **로그 분석:** "오늘 발생한 에러만 보여줘."
    - `cat server.log | grep "ERROR"`
2.  **프로세스 찾기:** "지금 파이썬 프로그램 실행 중이야?"
    - `ps aux | grep python`
3.  **코드 검색:** "이 프로젝트 전체에서 'TODO'라고 적어놓은 주석 다 찾아줘."
    - `grep -r "TODO" .`

---
## Code Core Points (핵심 문법)

가장 강력한 조합은 **파이프(`|`)** 와 함께 쓰는 것이다.

```bash
# 기본 문법
# grep [옵션] "찾을패턴" [파일]

# 1. 파일에서 찾기
grep "apple" fruits.txt

# 2. 다른 명령어의 결과에서 건져내기 (Pipe)
ls -al | grep "txt"
```

**파이프(`|`)란?**

- 앞 명령어(`ls`)의 실행 결과를 텍스트 파일로 저장하지 않고, 즉시 뒤 명령어(`grep`)의 입력으로 넘겨주는 **연결 통로**다.
- **장점:** 임시 파일을 만들지 않아서 처리가 빠르고 깔끔하다.


### Tip

- **`ps aux | grep ...`** (실행 중인 프로그램 찾기)
- **`cat ... | grep "Error"`**(에러 로그 찾기)

는 제일 많이 쓰는 명령어

## 상세 분석: 자주 쓰는 옵션 (Common Options)

실무에서는 그냥 `grep`만 쓰기보다 옵션을 섞어 쓸 때가 더 많다.

|**옵션**|**설명**|**예시**|**비고**|
|---|---|---|---|
|**`-i`**|**I**nsensitive|대소문자 구분 없이 찾기|`grep -i "error"` (Error, error 다 찾음)|
|**`-v`**|In**v**ert|해당 패턴을 **제외하고** 출력|`grep -v "INFO"` (정보성 로그 빼고 보기)|
|**`-r`**|**R**ecursive|하위 폴더까지 싹 다 뒤지기|`grep -r "def" .` (함수 정의 찾기)|
|**`-n`**|**N**umber|몇 번째 줄인지 줄 번호 표시|`grep -n "main" app.py`|
|**`-c`**|**C**ount|찾은 줄의 **개수**만 세기|`grep -c "FAIL" test.log`|


---
## 심화: 정규표현식 (Regular Expressions)

`grep`은 단순 단어 찾기를 넘어, **패턴(Pattern)** 으로 검색할 때 진가를 발휘한다.

```bash
# -E 옵션: 확장 정규표현식 사용 (Extended Regex)

# 1. "Error" 또는 "Fail"이 들어간 줄 찾기 (OR 조건)
grep -E "Error|Fail" server.log

# 2. 숫자로 시작하는 줄 찾기
grep "^[0-9]" data.txt
```

## 초보자가 자주 하는 실수 (Misconceptions)

### ① "파일 내용이 지워지는 거 아닌가요?"

- `grep -v "text"`를 쓰면 화면에서만 안 보일 뿐, **원본 파일은 절대 변하지 않는다.**
- 안심하고 막 써도 된다. (Read Only)

### ② "와일드카드(*)랑 헷갈려요"

- 쉘에서 `*.txt`는 "모든 텍스트 파일"을 뜻하지만,
- `grep` 안에서의 정규표현식 `*`는 "앞 글자가 0번 이상 반복됨"이라는 뜻이다. (완전히 다름!)
- 그냥 단어 찾기 할 때는 복잡하게 생각 말고 **따옴표(`" "`) 안에 정확한 단어**를 넣는 습관을 들이자.
