## 개념 한 줄 요약

**"일반 사용자가 잠시동안 '최고 관리자(Root)'의 가면을 쓰고 강력한 명령을 내리게 해주는 마법의 명령어다."**
(**S**uper**U**ser **DO** = 슈퍼유저가 하란다!)

---
##  왜 필요한가? (Why)

**문제점:**
- 리눅스는 보안을 위해 평소에는 힘없는 **일반 유저(`ubuntu`)** 로 로그인해서 쓴다.
- 실수로 중요 시스템 파일을 지우는 대참사를 막기 위해서다.

**해결책:**
- 하지만 프로그램을 깔거나 설정을 바꿀 때는 강력한 힘이 필요하다.
- 이때 로그아웃하지 않고, 명령어 앞에 `sudo`만 붙여서 **"임시로 관리자 승인"** 을 받는다.

---
## 실무 적용 사례 (Practical Context)

**"Permission denied"** 에러를 만났을 때, 90%는 `sudo`가 필요하다.

1.  **프로그램 설치:** `apt-get install python3` -> (그냥 하면 거부당함) -> `sudo apt-get ...`
2.  **시스템 파일 수정:** `vi /etc/hosts` -> (저장이 안 됨) -> `sudo vi /etc/hosts`
3.  **Docker 실행:** (설정에 따라) `docker ps`가 안 될 때 `sudo docker ps`

---
## 상세 분석: Root vs Sudo (Detailed Analysis)

### ① Root (절대 권력자)

- 윈도우의 `Administrator`와 같다.
- 시스템의 모든 것을 파괴할 수 있는 신(God)이다.
- 프롬프트가 `$` 대신 `#`으로 표시된다.

### ② Sudo (대리인)

- 일반 유저가 잠시 Root의 권한을 빌려 쓰는 것.
- 비밀번호를 물어보면 **내 계정(ubuntu)의 비밀번호**를 입력하면 된다. (화면에 글자가 안 찍혀도 입력되고 있는 것이니 당황 금지!)

---
## Code Core Points (핵심 꿀팁)

### ① `sudo !!`  ⭐️ 실무 필수

방금 입력한 명령어가 "권한 거부" 당했을 때, 다시 타이핑할 필요 없이 이걸 치면 된다.

```bash
apt-get update
# [에러] Permission denied... (아 맞다 sudo!)

sudo !!
# -> 자동으로 'sudo apt-get update'가 실행됨
```

### ② `sudo -i` (아예 관리자로 변신)

명령어마다 `sudo` 치기 귀찮을 때, 아예 Root 계정으로 로그인해버리는 것.

```bash
sudo -i
# 프롬프트가 'root@server:~#' 로 바뀜.
# 작업 끝나면 'exit' 쳐서 일반인으로 돌아와야 함.
```

---
## 초보자가 자주 하는 실수 (Misconceptions)

### ① "비밀번호를 쳤는데 화면에 안 나와요!"

- 리눅스 보안 정책상 **비밀번호 입력은 화면에 아무것도 표시되지 않는다.** (별표 `*`도 안 나옴)
- 고장 난 게 아니니 침착하게 치고 엔터를 누르면 된다.

### ② "모든 파일 생성에 sudo를 쓴다"

- 내 홈 디렉토리(`~`)에서 작업할 때는 `sudo`를 쓰면 안 된다.
- 이유: `sudo`로 만든 파일은 주인이 `root`가 되어버려서, 나중에 내가 수정/삭제를 못 하게 된다. (결국 `chown`으로 되돌려야 하는 악순환 발생)

> "내 집(Home)에서는 그냥 하고, 공공장소(시스템 폴더)를 건드릴 때만 `sudo`를 꺼내자."
