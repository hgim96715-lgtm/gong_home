## 핵심 개념 요약

시스템이 터미널에 입력된 **명령어(실행 파일)를 찾으러 다니는 경로들의 목록**입니다.

---
## 이것이 왜 필요한가 (Why)

**문제점:**
- `python`이나 `docker`를 실행할 때마다 `/usr/local/bin/python`처럼 전체 경로를 매번 다 입력하는 것은 매우 번거롭고 불가능에 가깝습니다.

 **해결책:**
- 자주 사용하는 프로그램들이 모여 있는 폴더 경로를 `PATH` 변수에 미리 등록해두면, 이름만 입력해도 시스템이 해당 폴더들을 뒤져서 프로그램을 실행해줍니다.

---
## 실무 적용 사례

터미널에서 **`command not found`** 에러가 발생했다면 대부분 `PATH` 설정 문제입니다. 
새로운 도구(Java, Go, DB 클라이언트 등)를 설치한 후 명령어 실행이 안 될 때, 해당 설치 폴더를 `PATH`에 추가하여 문제를 해결합니다.

---
## 코드 핵심 포인트

1. **구분자:** 여러 개의 경로를 등록할 때는 **콜론(`:`)** 으로 구분합니다.
2. **우선순위:** 변수에 적힌 **왼쪽 경로**부터 순서대로 찾아봅니다. (먼저 발견된 것이 실행됨)
3. **업데이트:** 기존의 소중한 경로들을 날리지 않으려면 반드시 `$PATH`를 포함해서 덧붙여야 합니다.

---
## 상세 코드 분석

###  현재 내 컴퓨터에 설정된 전체 경로 확인하기

```bash
echo $PATH
# 결과 예시: /usr/local/bin:/usr/bin:/bin:/usr/sbin:/sbin
```

### 새로운 프로그램 경로 추가하기 (예: /my/tools/bin 폴더) `:`

```bash
# 기존 경로($PATH) 뒤에 콜론(:)을 찍고 새 경로를 붙여줍니다.
export PATH="$PATH:/Users/jason/my_tools/bin"
```

###  관련 있는 다른 경로 변수들 확인

```bash
echo $PYTHONPATH      # 파이썬 모듈을 찾는 경로
echo $LD_LIBRARY_PATH # 공유 라이브러리를 찾는 경로
```

---
## 초보자가 자주 하는 실수

1. **기존 경로 덮어쓰기:** 
	- `export PATH="/new/path"`라고만 적으면 기존의 `/bin`, `/usr/bin` 경로가 사라져서 `ls`, `cd` 같은 기본 명령어도 작동하지 않게 됩니다.
	- 반드시 `$PATH`를 경로에 포함하세요.

2.  **오타 주의:** 
	- 경로 중간에 공백이 들어가거나 콜론(`:`)을 빠뜨리면 경로 인식이 되지 않습니다.
