##  개념 한 줄 요약

**"터미널을 켤 때마다 자동으로 실행되어, 나의 설정(환경 변수, 단축키 등)을 영구적으로 적용해주는 스크립트 파일이다."**
(파일명 뒤의 `rc`는 **R**un **C**ommand의 약자다.)

---
## 왜 필요한가? (Why)

**문제점:**
- 터미널에서 열심히 `export PATH=...`를 입력해도, **창을 닫거나 컴퓨터를 껐다 켜면 설정이 물거품처럼 사라진다.** (휘발성)

**해결책:**
- 터미널이 켜지는 순간 자동으로 읽어가는 파일(`rc` 파일)에 설정을 적어두면, 매번 입력할 필요 없이 **설정이 영구적으로 유지된다.**

---
## 진단: "나는 지금 어떤 쉘을 쓰고 있을까?" (`echo $SHELL`)

설정 파일을 건드리기 전에, **현재 내가 어떤 쉘 위에 서 있는지** 확인하는 것이 1순위다.

```bash
echo $SHELL
```

|**출력 결과**|**현재 쉘**|**수정해야 할 파일 (Target)**|**비고**|
|---|---|---|---|
|`/bin/zsh`|**zsh**|**`~/.zshrc`**|최신 맥북(macOS) 표준|
|`/bin/bash`|**bash**|**`~/.bashrc`**|리눅스(Ubuntu/CentOS) 표준|

---
## 심화: 두 개의 평행 우주 (Host vs Container)

터미널 창은 하나지만, 나는 지금 **문 하나를 사이에 두고 두 세계를 오가고 있다**는 사실을 잊으면 안 된다. 
이걸 모르면 "맥북인데 왜 bash지?" 하며 혼란에 빠진다.

|**구분**|**현실 세계 (Host)**|**가상 세계 (Container)**|
|---|---|---|
|**위치**|내 맥북 (MacBook Pro)|도커 컨테이너 (`linux-server`)|
|**프롬프트**|`gong@MacBook...`|**`ubuntu@b3e0...`** (👈 현재 위치)|
|**기본 쉘**|**zsh**|**bash**|
|**행동 요령**|`~/.zshrc` 수정|**`~/.bashrc` 수정**|

**[결론]** `echo $SHELL`을 쳤을 때 `/bin/bash`가 나왔다면, 내 맥북 설정이 꼬인 게 아니다. 
**"내가 지금 리눅스 서버(Container) 안에 정상적으로 들어와 있구나!"** 라고 안심하고 **`.bashrc`** 를 수정하면 된다.

---
## 실무 가이드: "나는 무슨 파일을 수정해야 해?" (Practical Context)

이 표 하나면 종결된다. 
복잡하게 생각하지 말고 운영체제에 따라 하나만 파면 된다.

| 내 환경 | 사용하는 쉘(Shell) | **수정해야 할 파일 (Target)** | 비고 |
| :--- | :--- | :--- | :--- |
| **Mac (맥북)** | **zsh** (최신 맥 표준) | **`~/.zshrc`** | 맥북 유저는 여기만 고치면 99% 해결됨. |
| **Linux (서버)** | **bash** (서버 표준) | **`~/.bashrc`** | Docker 컨테이너나 EC2 서버는 여기. |
| **Legacy Mac** | bash (구형 맥) | `~/.bash_profile` | 구형 맥북일 경우만 해당. |

##  상세 코드 분석 (Detailed Analysis)

### ① 설정 파일 열고 수정하기 (Standard Workflow)

맥북(`zsh`) 기준으로 실습해보자. (리눅스라면 파일명만 `.bashrc`로 바꾸면 됨)

```bash
# 1. 편집기로 설정 파일 열기 (파일이 없으면 새로 만들어짐)
vi ~/.zshrc

# 2. [vi 내부] 맨 아래로 이동(Shift+G)해서 설정 추가 (o)
# 예: Python 경로 추가, Airflow 홈 설정 등
export AIRFLOW_HOME=~/airflow
alias air="docker exec -it airflow-webserver bash"

# 3. 저장하고 나오기
# ESC -> :wq -> 엔터
```

### ② 수정 사항 즉시 적용하기 (`source`) ⭐️ 필수

파일을 저장했다고 끝난 게 아니다.
컴퓨터에게 **"방금 고친 내용 다시 읽어와!"** 라고 명령해야 한다.

```bash
# 방법 A: 명령어 사용 (권장)
# # 문법: source <파일경로> 
source ~/.zshrc

# 방법 B: 그냥 터미널 껐다 켜기
# (귀찮으니까 source를 쓰자)
```

### ③ 심화: `profile` vs `rc` (몰라도 되지만 알면 좋은 차이)

- **`.bash_profile` (로그인용):** 컴퓨터 켜고 아이디/비번 치고 들어올 때 **딱 1번** 실행됨. (주로 전체 환경 변수)
- **`.bashrc` (실행용):** 이미 로그인된 상태에서 **새 터미널 창을 열 때마다** 실행됨. (주로 별칭 alias, 프롬프트 꾸미기)
- _실무 팁:_ 요즘은 귀찮아서 `.bashrc`에 다 몰아넣고, `.bash_profile`에서 `.bashrc`를 불러오게 만드는 추세다.

## 초보자가 자주 하는 실수 (Misconceptions)

1. **"수정했는데 왜 안 바뀌죠?"**
    - 십중팔구 **`source ~/.zshrc`** 를 안 했기 때문이다. 
    - 리눅스는 시키지 않으면 파일을 다시 읽지 않는다.

2. **"파일이 없어요!"**
    - `ls`만 치면 안 보인다. 
    - 점(`.`)으로 시작하는 파일은 숨김 파일이므로 **`ls -a`** 로 찾아야 한다. 
    - 없으면 `vi`로 열 때 자동으로 생성되니 걱정 말자.

3. **"제 맥북에 `.bash_profile` 만들었는데 안 돼요."**
    - 최신 맥북(Catalina 이후)은 `zsh`을 쓴다.
    - `bash` 설정 파일은 쳐다보지도 않는다. 
    - **`echo $SHELL`** 명령어로 내가 지금 무슨 쉘을 쓰는지 확인해보자.
        - `/bin/zsh` 나오면 -> `.zshrc` 수정
        - `/bin/bash` 나오면 -> `.bashrc` 수정
