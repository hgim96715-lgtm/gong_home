## 핵심 개념 요약

터미널이 시작될 때마다 자동으로 실행되어 **환경 변수나 사용자 설정을 영구적으로 적용해주는 스크립트 파일**입니다.

---
## 이것이 왜 필요한가 (Why)

**문제점:**
- 터미널 창에서 직접 입력한 `export` 명령어는 현재 창을 닫거나 컴퓨터를 재부팅하면 깨끗이 사라집니다 (휘발성).

**해결책:**
- 로그인할 때나 새 터미널 창을 열 때 시스템이 가장 먼저 읽어가는 파일(`rc` 또는 `profile`)에 설정을 기록해두면, 매번 번거롭게 입력할 필요 없이 설정을 영구적으로 유지할 수 있습니다.

---
## 실무 적용 사례

1. **OS별 차이:** 
	- 맥북(macOS) 사용자라면 주로 **`~/.zshrc`** 파일을 편집하여 `alias`(단축어)를 만들거나 `PATH`를 수정합니다. 
	- 서버 환경(Ubuntu 등)에서는 **`~/.bashrc`** 를 주로 사용합니다.

2. **범위 설정:**
	- 모든 사용자에게 공통 설정을 적용할 때는(관리자 권한) `/etc/profile`을 수정합니다.

3. **데이터 엔지니어링 표준:**
    - **`bash_profile`**: 서버에 처음 접속(Login)할 때 Java나 Python의 경로를 잡아주는 무거운 작업.
    - **`bashrc`**: 터미널에서 자주 쓰는 나만의 단축키(alias)나 프롬프트 꾸미기 설정.

---
## 코드 핵심 포인트

1. **위치:** 대부분 사용자 홈 디렉토리(`~`)에 **숨김 파일**(파일명이 `.`으로 시작)로 존재합니다.
2. **반영:** 파일을 수정한 뒤에는 반드시 **`source`** 명령어를 써야 터미널을 껐다 켜지 않고도 즉시 적용됩니다.
3. **구분:** 로그인할 때 한 번 읽는 파일(`profile`)과 새 창을 열 때마다 읽는 파일(`rc`)의 차이를 이해해야 합니다.

---
## 상세 코드 분석

### 설정 파일 편집 및 적용 (기본)

가장 많이 쓰는 패턴입니다. (맥북 기준 `zshrc`, 리눅스 기준 `bashrc`)

```bash
# 1. 편집기로 파일 열기
vi ~/.zshrc

# 2. 파일 맨 아래에 영구 저장할 변수 추가
export AIRFLOW_HOME="~/airflow"

# 3. (중요) 수정한 내용을 현재 터미널에 즉시 적용하기
source ~/.zshrc
```

### 파일별 역할 구분 (Bash 기준)

리눅스 서버 환경에서 마주치게 될 파일들입니다.

```bash
# [범위 1] 시스템 전체 설정 (관리자만 수정 권장)
/etc/profile        # 모든 사용자가 로그인할 때 실행되는 공통 설정

# [범위 2] 개별 사용자 설정 (내 홈 디렉토리 ~ 에 위치)
~/.bash_profile     # Login Shell: 내가 로그인할 때 '처음 한 번'만 읽는 파일
~/.bashrc           # Non-Login Shell: 이미 로그인한 상태에서 '새 창'을 열 때마다 읽는 파일
~/.profile          # bash_profile이 없을 때 대신 읽는 범용 설정 파일
```

### 맥북(zsh)과 리눅스(bash)의 차이점

맥북은 기본적으로 `zsh`을 쓰기 때문에 파일 이름이 매칭되는 것이 따로 있습니다.

```bash
# 맥북(zsh) 유저라면?
~/.zprofile         # bash_profile 역할 (로그인 시 1회)
~/.zshrc            # bashrc 역할 (새 창 열 때마다 - 맥북은 주로 여기에 다 적음)
```

### 실무 꿀팁: '호출' 기법 (Sourcing)

로그인할 때(`profile`) 터미널 설정(`rc`)도 자동으로 먹히게 하기 위해, 프로파일 안에 아래 코드를 넣어둡니다.

```bash
# ~/.bash_profile (또는 ~/.zprofile) 파일 내부
if [ -f ~/.bashrc ]; then
    . ~/.bashrc   # "로그인할 때 .bashrc도 같이 읽어와라!" 라는 뜻
fi
```

## 초보자가 자주 하는 실수

1. **수정 후 무반응 ("왜 안 바뀌죠?"):**
    - 파일을 저장만 하고 **`source ~/.zshrc`** 명령어를 실행하지 않았기 때문입니다. 
    - 컴퓨터는 파일을 저장만 했다고 해서 설정을 바로 바꾸지 않습니다.

2. **잘못된 파일 수정 (맥북 유저 주의):**
    - 사용하는 쉘이 `zsh`인데 `.bashrc`만 열심히 고치고 있으면 아무런 변화가 없습니다. 
    - 터미널 상단 타이틀바에 `zsh`이라고 써있다면 반드시 **`~/.zshrc`** 를 수정하세요.

3. **파일 우선순위 혼동:**
    - `~/.bash_profile`이 존재하면 시스템은 `~/.profile`을 무시하고 지나갑니다. 내가 어떤 파일을 메인으로 쓰고 있는지 확인해야 합니다.
