## Concept Summary

**"Vertical Slicing Tool."**
파일의 각 줄(Line)을 읽으면서, 특정 문자(구분자)를 기준으로 열(Column)을 수직으로 잘라내는 명령어.

---
##  Why: 왜 필요한가?

* **문제:** `awk`는 강력하지만 문법이 길고(`awk '{print $1}'`), 단순한 CSV(콤마 구분) 파일이나 `/etc/passwd`(콜론 구분) 파일에서 특정 컬럼만 뽑을 때는 타이핑이 귀찮다.

* **해결:** `cut`을 쓰면 옵션 두 개(`-d`, `-f`)만으로 아주 심플하고 빠르게 데이터를 추출할 수 있다. 대용량 로그 파일에서 ID만 추출할 때 `awk`보다 미세하게 더 빠르다.

---
##  Practical Context (실무 활용)

* **CSV 파일 처리:** 콤마(`,`)로 구분된 파일에서 고객 ID(첫 번째 열)만 추출할 때.
* **시스템 사용자 확인:** `/etc/passwd` 파일에서 사용자 이름만 뽑아낼 때.
* **날짜/시간 파싱:** 로그 파일의 고정된 위치(예: 앞의 10글자 날짜)만 잘라낼 때.

---
##  Code Core Points

가장 중요한 옵션은 **"어디를(Bytes/Char/Fields)"**, **"무엇으로(Delimiter)"** 자를지 정하는 것입니다.

### A. 구분자로 자르기 (`-d`, `-f`) ⭐️ (가장 많이 씀)

엑셀의 "텍스트 나누기(구분 기호로 분리)"와 같습니다.

- **문법:** `{bash}cut -d "구분자" -f "가져올 필드 번호" 파일명`

```bash
# 문법: cut -d "구분자" -f "가져올 필드 번호" 파일명

# 예제: 콜론(:)으로 구분된 /etc/passwd 파일에서 1번째 필드(유저명)만 가져오기
cut -d ":" -f 1 /etc/passwd

# 여러 필드 가져오기 (1번과 7번 필드)
cut -d ":" -f 1,7 /etc/passwd

# 범위 가져오기 (1번부터 3번까지)
cut -d ":" -f 1-3 /etc/passwd
```

### B. 글자 위치로 자르기 (`-c`)

고정 길이 데이터(Fixed Width)를 다룰 때 유용합니다.

- **문법:** `{bash}cut -c "시작위치-끝위치" 파일명`

```bash
# 문법: cut -c "위치" 파일명

# 1. 범위 지정: 1번째부터 5번째 글자까지만 출력
cut -c 1-5 data.txt

# 2. 끝까지 지정: 10번째 글자부터 줄 끝까지 출력
cut -c 10- data.txt

# 3. 딱 한 글자만: 3번째 글자 하나만 쏙 뽑기
cut -c 3 data.txt

# 4. [추가] 콕콕 집어서: 2번째랑 7번째 글자만 (콤마 사용)
# "범위(-)가 아니라 콤마(,)를 쓰면 떨어진 글자도 가져옵니다."
cut -c 2,7 data.txt
```

- **`1-5`** (하이픈): 1번부터 5번까지 **쭉 이어짐** (범위)
- **`2,7`** (콤마): 2번과 7번 **딱 두 개만** (개별 선택)
- **`1-3,7`** (혼합): 1~3번 범위랑, 7번 글자 (섞어 쓰기 가능!)

## Detailed Analysis (한 줄 해석)

**상황:** `user_info.csv` 파일이 `id,name,email,age` 순서로 저장되어 있다고 가정.

```bash
# CSV 파일에서 이름(2번째)과 이메일(3번째)만 뽑고 싶다.
cat user_info.csv | cut -d "," -f 2,3
```

- `|` (Pipe): `cat`으로 읽은 내용을 `cut`에게 넘겨줍니다.
- `-d ","`: "야, 이 데이터는 콤마(`,`)로 칸이 나뉘어 있어." (Delimiter 지정)
- `-f 2,3`: "그중에서 2번째랑 3번째 칸만 화면에 보여줘." (Field 지정)

## 흔한실수 (주의사항)

### `awk`와의 결정적 차이 (공백 처리) 🚨

초보자가 가장 많이 하는 실수입니다.

- **`awk`:** 스페이스바가 1개든 10개든 알아서 "공백"으로 처리해 줍니다. (`ls -l` 결과 처리에 적합)
- **`cut`:** 스페이스바 1개를 정확히 문자 1개로 취급합니다. 공백이 불규칙한 데이터(예: `ls -l` 출력 결과)에는 **절대 쓰면 안 됩니다.**

```bash
# [상황] ls -l 결과: "drwxr-xr-x  5  root  root ..." (공백이 여러 개 섞여 있음)

# ❌ 실패: cut은 공백 1개만 구분자로 인식해서 엉뚱한 결과가 나옴
ls -l | cut -d " " -f 5 

# ✅ 성공: awk는 공백이 여러 개여도 알아서 처리함
ls -l | awk '{print $5}'
```

### Default Delimiter

`-d` 옵션을 안 쓰면 기본 구분자는 **TAB(탭)** 입니다. 
스페이스로 구분된 파일인데 `-d`를 안 쓰면 아무것도 안 잘리고 통째로 출력됩니다.
