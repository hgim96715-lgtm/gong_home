## 개념 한 줄 요약

**Terminal Editor(터미널 에디터)** 는 윈도우의 '메모장'이나 맥의 'TextEdit' 같은 프로그램을, **마우스 없이 키보드만으로 조작하는 검은 화면 속의 문서 편집기**다.
대표적으로 **`vi`(빔)** 과 **`nano`(나노)** 가 있다.

---
##  왜 필요한가? (Why)

* **GUI가 없는 세상:** 우리가 다루는 서버(Docker 컨테이너, AWS EC2)는 대부분 그래픽 화면이 없는 **Linux 환경**이다.
* **생존 필수:** 설정 파일(`airflow.cfg`, `.zshrc`)을 고쳐야 하는데, 에디터를 다룰 줄 모르면 **파일을 열어놓고 저장도 못한 채 갇혀버리는 끔찍한 상황**이 발생한다.
* **표준 도구:** 화려한 VS Code가 없어도, 지구상 모든 리눅스 서버에는 **`vi`** 가 무조건 깔려 있다.

---

## Code Core Points (vi의 3가지 모드)

`vi`는 **"모드(Mode)"** 라는 독특한 개념이 있다. 이걸 모르면 글자가 안 써져서 당황하게 된다.

1.  **일반 모드 (Normal Mode):** 처음 켰을 때 상태. (글자 입력 불가 , 이동/삭제만 가능)
2.  **입력 모드 (Insert Mode):** **`i`** 를 눌러야 진입. (이제야 메모장처럼 글 써짐 )
3.  **명령 모드 (Command Mode):** **`:`** 를 눌러야 진입. (저장하거나 끄는 명령 내림 )

---
## Detailed Analysis (상세 가이드)

### ① vi (Vim) : 리눅스의 표준

어렵지만 반드시 익혀야 하는 기본기.

**Step 1. 파일 열기**

```bash
vi ~/.zshrc
# 파일이 열리면 글자가 안 써진다. (정상! 일반 모드니까)
```

**Step 2. 이동하기 (Navigation) ⭐️

입력 모드(`i`)가 아닌 상태(ESC 누른 상태)에서 써야 한다.

* **위아래 (수직 이동)**
    * **`gg`** (Go to Top): 문서의 **가장 첫 줄**로 이동.
    * **`Shift + G`** (Go to Bottom): 문서의 **가장 마지막 줄**로 이동. (설정 추가할 때 필수!)

* **양옆 (수평 이동)**
    * **`0`** (숫자 0): 현재 줄의 **맨 앞**으로 이동.
    * **`$`** (Shift + 4): 현재 줄의 **맨 뒤**로 이동. (긴 코드 끝부분 수정할 때 꿀팁!)

**Step 3. 수정하기 (Edit Mode)**

- **`i`** (Insert): 이걸 누르면 화면 아래에 `-- INSERT --`라고 뜬다. 이제 수정 가능하다. 현재 커서 자리에서 입력 시작.
- **`o`** (Open Line): 현재 줄 **아래에 빈 줄을 새로 만들면서** 입력 시작. (엔터 칠 필요 없이 바로 새 줄!)
- **`dd`**: 현재 줄 **삭제** (잘못 썼을 때 지우개처럼 사용)
- **`u`**: **실행 취소** (Undo) -> 실수했을 때 `ESC` 누르고 `u` 누르면 되살아남!

Step 4. 저장하고 끄기 (Save & Quit) ⭐️ 공식처럼 암기

- **`ESC`**: 입력 모드 탈출 (무조건 연타!)
- **`:` (콜론)**: 명령 대기 상태 (화면 맨 아래 커서 생김) 
- **`wq`**: **W**rite(저장) + **Q**uit(종료). 엔터 치면 저장되고 나옴.

|**상황**|**명령어 순서**|**설명**|
|---|---|---|
|**저장하고 끄기**|`ESC` -> `:wq` -> `Enter`|국룰. 가장 많이 씀.|
|**그냥 끄기**|`ESC` -> `:q` -> `Enter`|수정한 게 없을 때.|
|**강제 종료 (비상탈출)**|`ESC` -> `:q!` -> `Enter`|**"망했다! 저장 안 하고 나갈래!"** 할 때 사용.|

----
### ② nano : 초보자의 구세주

`vi`가 너무 어렵다면 `nano`를 써라. 화면 아래에 단축키가 다 써져 있어서 직관적이다.

>혹시 `vi`가 너무 답답하면 `nano`를 설치(`sudo apt-get install nano`)해서 써보세요!

**Step 1. 파일 열기**

```bash
nano ~/.zshrc
```

**Step 2. 조작법 (화면 하단 참고)**

- 리눅스에서 `^` 기호는 `Ctrl` 키를 뜻한다.  
- **저장하기:** `Ctrl + O` (Write Out) -> `Enter` (파일명 확인)
- **나가기:** `Ctrl + X` (Exit)

----
## 초보자가 자주 하는 실수 (Misconceptions)

1. **"Ctrl+C, Ctrl+V가 안 돼요!"**
    - 터미널에서 `Ctrl+C`는 복사가 아니라 **"프로그램 강제 종료(Kill)"** 명령이다.
    - 복붙 하려면 마우스 우클릭을 쓰거나 `Cmd+C / Cmd+V`를 써야 한다.
        
2. **"도무지 빠져나갈 수가 없어요 ㅠㅠ"**
    - `vi`에 갇히는 건 전 세계 개발자 밈(Meme) 1위다.
    - 당황하지 말고 **`ESC`를 3번 연타**한 다음 **`:q!`** 를 치면 해방될 수 있다.
        
3. **"방향키가 안 먹혀요!"**
    - 옛날 `vi`는 `h, j, k, l` 키로 이동했지만, 요즘 `vi`(사실은 `vim`)는 방향키도 잘 먹힌다. 
    - 안 먹힌다면 호환성 모드 문제니 그냥 `nano`를 쓰는 게 정신건강에 좋다.
