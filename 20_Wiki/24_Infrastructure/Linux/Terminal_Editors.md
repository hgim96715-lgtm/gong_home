## 개념 한 줄 요약

**Terminal Editor(터미널 에디터)** 는 윈도우의 '메모장'이나 맥의 'TextEdit' 같은 프로그램을, **마우스 없이 키보드만으로 조작하는 검은 화면 속의 문서 편집기**야.

----
##  왜 필요한가 (Why)

* **GUI가 없는 세상:** 우리가 다루는 서버(Docker 컨테이너, AWS EC2)는 대부분 그래픽 화면이 없는 **Linux 환경**이야.
* **생존 필수:** 설정 파일(`airflow.cfg`, `.zshrc`)을 고쳐야 하는데, 에디터를 다룰 줄 모르면 **파일을 열어놓고 저장도 못한 채 갇혀버리는 끔찍한 상황**이 발생해.
* **표준 도구:** 화려한 VS Code가 없어도, 지구상 모든 리눅스 서버에는 **`vi`** 가 깔려 있어.

##  Code Core Points (vi의 3가지 모드)

`vi`는 **"모드(Mode)"** 라는 독특한 개념이 있어. 
이걸 모르면 글자가 안 써져서 당황해.

1.  **일반 모드 (Normal Mode):** 처음 켰을 때 상태. (글자 입력 불가 , 이동/삭제만 가능)
2.  **입력 모드 (Insert Mode):** `i`를 눌러야 진입. (이제야 메모장처럼 글 써짐 )
3.  **명령 모드 (Command Mode):** `:`를 눌러야 진입. (저장하거나 끄는 명령 내림)

---
##  Detailed Analysis (필수 단축키)

### Step 1. 파일 열기

```bash
vi ~/.zshrc
```

- 파일이 열리면 아무리 키보드를 쳐도 글자가 안 써질 거야. (정상이야! **일반 모드**거든.)

### Step 2. 수정하기 (Edit)

- **`i`**: Insert(입력)의 약자. 
- 이걸 누르면 화면 아래에 `-- INSERT --`라고 뜸. 
- 이제 마음껏 수정해.

### Step 3. 저장하고 끄기 (Save & Quit)

이 순서를 무조건 외워! (손가락이 기억하게 해야 함)

1. **`ESC`**: 입력 모드 탈출 (다시 일반 모드로 복귀)
2. **`:` (콜론)**: 명령을 내리겠다! (화면 맨 아래 커서가 생김)
3. **`wq`**: Write(저장) + Quit(종료). 엔터 치면 저장되고 터미널로 빠져나옴.
	- **비상탈출 (저장 안 하고 끄기)** "아, 실수로 중요한 거 지웠다! 저장되면 안 돼!"
	- `ESC` -> `:q!` (Quit + 강제(!)) -> 엔터

---
## 초보자가 자주 착각하는 포인트

1. "Ctrl+C, Ctrl+V가 안 돼요!"
	- 터미널에서 `Ctrl+C`는 **"강제 종료(Kill)"** 명령어야. 
	- 복붙 하려면 마우스 우클릭을 쓰거나 `Cmd+C / Cmd+V`를 써야 해.

2. "도무지 빠져나갈 수가 없어요 ㅠㅠ"
	- `vi`에 갇히는 건 전 세계 개발자 밈(Meme) 1위야.
	- 당황하지 말고 **`ESC`를 연타**한 다음 **`:q!`** 를 치면 해방될 수 있어.
