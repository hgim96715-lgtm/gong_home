## 핵심 개념 요약

"내가 잘 때도 컴퓨터가 일하게 만드는 알람 시계"

리눅스에서 사용자가 지정한 **시간(Time)** 이나 **주기(Interval)** 에 맞춰 특정 명령어(Script)를 **자동으로 실행**해주는 예약 시스템입니다. 
에어플로우(Airflow)의 스케줄링 개념도 바로 이 Crontab 문법에서 시작되었습니다.

---
## 이것이 왜 필요한가 (Why)

 데이터 엔지니어는 24시간 깨어있을 수 없지만, 데이터 파이프라인은 멈추면 안 됩니다.
 - **배치 작업(Batch):** "매일 새벽 2시에 어제 쌓인 로그를 가져와서 DB에 넣어줘."
 - **모니터링:** "10분마다 서버가 살아있는지 체크해줘."
 - **자동화:** "매주 월요일 아침 9시에 주간 보고서를 메일로 보내줘."

---
## 실무 적용 사례(Airflow와의 관계)

Airflow에서 DAG를 짤 때 `schedule_interval`에 `'@daily'`나 `'0 9 * * *'` 같은 값을 넣게 되는데, 이게 바로 **Crontab 문법**입니다. 
즉, **Crontab을 이해하면 Airflow 스케줄링도 마스터**하게 됩니다.

---
## Code Core Points: 문법 "별 5개의 법칙" ⭐

순서가 생명입니다. (`분-시-일-월-요`) 이 순서만 외우면 됩니다.

```scss
  *  *  *  *  * command_to_execute
  ┬  ┬  ┬  ┬  ┬
  │  │  │  │  │
  │  │  │  │  └─ 요일 (0-7, 0과 7은 일요일)
  │  │  │  └── 월 (1-12)
  │  │  └── 날짜 (1-31)
  │  └───── 시 (0-23)
  └────────── 분 (0-59)
```

### 족집게 예제 (Copy & Paste)

|**주기**|**코드 (분 시 일 월 요)**|**설명**|
|---|---|---|
|**매분 실행**|`* * * * *`|1분마다 무조건 실행 (테스트용)|
|**매일 새벽 2:30**|`30 2 * * *`|가장 흔한 배치 작업 시간|
|**매주 월요일 아침 9시**|`0 9 * * 1`|주간 보고서, 업무 시작 알림|
|**평일 근무시간만**|`0 9-18 * * 1-5`|월~금, 9시부터 18시까지 정각마다|
|**10분마다**|`*/10 * * * *`|`/`는 '간격'을 의미함|

## Step-by-Step Guide: 1분 만에 실습하기

이론보다 중요한 건 **"진짜 돌아가는지 확인하는 것"** 입니다. 
도커 컨테이너나 리눅스 터미널에서 딱 한 번만 따라 해보세요.

### Step 1. 알바생 출근시키기 (서비스 켜기)

설정을 아무리 잘해도 실행해 줄 데몬(Cron 서비스)이 꺼져 있으면 아무 일도 안 일어납니다.

```bash
# 1. 서비스 시작
service cron start
# 결과: * Starting periodic command scheduler cron ... [ OK ] 가 떠야 함!

# 2. 잘 살아있는지 확인 (Check)
ps -ef | grep cron
# 결과: cron 프로세스가 한 줄이라도 보이면 성공!
```

### Step 2. 업무 지시서 작성하기 (Edit)

```bash
crontab -e
```

- `vi` 에디터가 열리면 `i`를 눌러 입력 모드로 들어갑니다.
- 맨 아래 줄에 다음 내용을 붙여넣으세요. (1분마다 현재 시간을 파일에 기록하는 명령)

```bash
* * * * * echo "Cron is alive: $(date)" >> /tmp/cron_test.log
```

- `ESC` -> `:wq` -> `Enter`로 저장하고 나옵니다.

### Step 3. 결과 확인하기 (Verify)

1분을 기다린 뒤, 실제로 파일이 생겼는지 확인합니다.

```bash
# 로그 파일 내용 보기
cat /tmp/cron_test.log

# 실시간으로 쌓이는 거 구경하기 (끄려면 Ctrl + C)
tail -f /tmp/cron_test.log
```

- `Cron is alive: ...` 메시지가 보이면 **성공**입니다!

## Troubleshooting (멈추는 법 & 삭제하는 법)

### 1. 로그 화면이 계속 올라가요! (`tail -f` 중단)

- **해결:** **`Ctrl + C`**
- 화면 출력만 멈춘 것이고, 뒤에서 Cron은 계속 일하고 있습니다.

### 2. 이제 그만 실행하고 싶어요 (일감 삭제)

- 테스트가 끝났는데 계속 로그가 쌓이면 안 되겠죠?

```bash
# 방법 A: 편집기로 들어가서 해당 줄 지우기 (권장)
crontab -e  -> 해당 줄에서 'dd' 입력 -> :wq 저장

# 방법 B: 모든 스케줄 한 방에 날리기 (주의!)
crontab -r
```

### 3. "경로를 못 찾아요" (Path Issue)

- **증상:** 터미널에선 잘 되는데 Cron으로만 돌리면 에러가 남.
- **원인:** Cron은 바보라서 `python`이 어디 있는지 모릅니다.
- **해결:** 무조건 **절대 경로**를 쓰세요.
    - `python main.py` (X)
    - `/usr/bin/python3 /home/user/main.py` (O)
