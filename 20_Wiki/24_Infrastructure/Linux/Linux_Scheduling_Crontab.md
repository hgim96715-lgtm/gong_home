## 핵심 개념 요약

**"리눅스의 '알람 시계'. 내가 자는 동안에도 정해진 시간에 명령어를 실행해 주는 자동화 도구."**

**Cron:** 백그라운드에서 돌면서 예정된 작업을 실행하는 데몬(서비스)입니다
**Crontab (Cron Table):** 그 Cron에게 "언제 무엇을 할지" 적어주는 **시간표(Table)** 이자 명령어입니다

에어플로우(Airflow)의 스케줄링 개념도 바로 이 Crontab 문법에서 시작되었습니다.

---
## 이것이 왜 필요한가 (Why)

**문제점:** 
- "매일 밤 12시에 서버 로그를 압축해서 백업해야 하는데, 매번 야근할 수는 없잖아?"
- "10분마다 서버가 살아있는지 체크하는 스크립트를 돌리고 싶어."

**해결책:**
- **`crontab`** 에 한 줄만 적어두면, 리눅스가 알아서 **지정된 시간, 날짜, 간격**에 맞춰 명령어를 실행합니다
- 사용자별로 서로 다른 스케줄을 만들 수도 있습니다

---
## 실무 적용 사례(Airflow와의 관계)

Airflow에서 DAG를 짤 때 `schedule_interval`에 `'@daily'`나 `'0 9 * * *'` 같은 값을 넣게 되는데, 이게 바로 **Crontab 문법**입니다. 
즉, **Crontab을 이해하면 Airflow 스케줄링도 마스터**하게 됩니다.

---
## Code Core Points: 필수 명령어 3총사

가장 먼저 외워야 할 기본 명령어입니다.

```bash
# 1. 스케줄 편집 (Edit)
# "시간표 좀 짜볼까?" (vi 에디터가 열림)
crontab -e

# 2. 스케줄 확인 (List)
# "내가 무슨 예약을 걸어놨더라?"
crontab -l

# 3. 스케줄 삭제 (Remove)
# "모든 예약 취소해!" (주의: 묻지도 않고 다 지움)
crontab -r
```

> (참고: 관리자 권한으로 다른 유저의 크론탭을 볼 때는 `sudo crontab -u username -e`를 씁니다.)

---
## Code Core Points: 문법 "별 5개의 법칙" 

순서가 생명입니다. (`분-시-일-월-요`) 이 순서만 외우면 됩니다.

**공식:** `분 시 일 월 요일 명령어`

```scss
  *  *  *  *  * command_to_execute(실행할 파일 경로 또는 명령어)
  ┬  ┬  ┬  ┬  ┬
  │  │  │  │  │
  │  │  │  │  └─ 요일 (0-7, 0,7=일, 1=월...)
  │  │  │  └── 월 (1-12)
  │  │  └── 날짜 (1-31)
  │  └───── 시 (0-23)
  └────────── 분 (0-59)
```


### 족집게 예제 (Copy & Paste)

|**주기**|**코드 (분 시 일 월 요)**|**설명**|
|---|---|---|
|**매분 실행**|`* * * * *`|1분마다 무조건 실행 (테스트용)|
|**매일 새벽 2:30**|`30 2 * * *`|가장 흔한 배치 작업 시간|
|**매주 월요일 아침 9시**|`0 9 * * 1`|주간 보고서, 업무 시작 알림|
|**평일 근무시간만**|`0 9-18 * * 1-5`|월~금, 9시부터 18시까지 정각마다|
|**10분마다**|`*/10 * * * *`|`/`는 '간격'을 의미함|

### 특수 기호 (Wildcards)

- `*`: **매번** (Every). "매분, 매시, 매일"
- `/`: **간격** (Step). `*/10` = 10분마다
- `,`: **목록** (List). `1,2,3` = 1일, 2일, 3일에
- `-`: **범위** (Range). `1-5` = 월요일부터 금요일까지

## 실전 예제 모음 (Copy & Paste)

가장 많이 쓰는 패턴들입니다.

### ① 주기적인 실행

```bash
# 1분마다 실행 (가장 기본)
* * * * * echo "Run every minute" 

# 10분 간격으로 실행 (0분, 10분, 20분...)
*/10 * * * * /path/to/script.sh 
```

### ② 특정 시간 실행

```bash
# 매일 오후 2시 30분에 실행
30 14 * * * /path/to/backup.sh 

# 매주 일요일 자정(00:00)에 실행
0 0 * * 0 /path/to/weekly_backup.sh 
```

### ③ 업무 시간(Working Hour)에만 실행

```bash
# 월~금, 아침 9시부터 오후 5시까지, 매 정각마다 실행
0 9-17 * * 1-5 /path/to/work_task.sh
```

## Troubleshooting (멈추는 법 & 삭제하는 법)

### 1. 로그 화면이 계속 올라가요! (`tail -f` 중단)

- **해결:** **`Ctrl + C`**
- 화면 출력만 멈춘 것이고, 뒤에서 Cron은 계속 일하고 있습니다.

### 2. 이제 그만 실행하고 싶어요 (일감 삭제)

- 테스트가 끝났는데 계속 로그가 쌓이면 안 되겠죠?

```bash
# 방법 A: 편집기로 들어가서 해당 줄 지우기 (권장)
crontab -e  -> 해당 줄에서 'dd' 입력 -> :wq 저장

# 방법 B: 모든 스케줄 한 방에 날리기 (주의!)
crontab -r
```

## 초보자가 자주 하는 실수 (Misconceptions)

### ① "스크립트가 안 돌아가요!" (경로 문제)

- **Crontab은 바보입니다.** 환경변수(`$PATH`)를 잘 몰라요.
- `python script.py` 라고 쓰면 못 찾습니다.
- 무조건 **절대 경로**를 쓰세요! 👉 `/usr/bin/python3 /home/user/script.py`

### ② "로그가 어디 찍히는지 모르겠어요."

- Crontab은 기본적으로 출력을 화면에 보여주지 않습니다.
- 실행 결과를 보고 싶다면 **리다이렉션(`>>`)** 을 꼭 써야 합니다.
- 예: `* * * * * /home/my_script.sh >> /home/cron.log 2>&1`
    

### ③ "권한 문제 (Permission Denied)"

- 실행하려는 쉘 스크립트 파일(`.sh`)에 **실행 권한(`+x`)** 이 있는지 확인하세요.
- `chmod +x /path/to/script.sh`
