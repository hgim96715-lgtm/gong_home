## 핵심 개념 요약

리눅스 시스템에서 프로세스가 실행되는 **'주변 환경'을 정의하는 Key-Value(키-값) 쌍의 데이터**입니다.

---
## 이것이 왜 필요한가 (Why)

 **문제점:** 
- 프로그램을 실행할 때마다 매번 설정값(DB 주소, 비밀번호, 경로 등)을 일일이 입력하거나 코드에 직접 적어두면(Hard-coding) 보안상 위험하고 관리가 매우 힘듭니다.

**해결책:**
- 변수 이름은 유지하되 실제 값만 바꿔주면, 코드 수정 없이도 개발 서버와 운영 서버의 설정을 다르게 가져갈 수 있어 **유연성과 보안**을 동시에 챙길 수 있습니다.

---
## 실무 적용 사례

데이터 엔지니어링 실무에서는 **DB 접속 정보(Connection String), API Key, Airflow 설정** 등을 환경 변수로 관리합니다. 
맥북 터미널을 열었을 때 `HOME`(내 홈 폴더 경로), `USER`(내 계정 아이디) 등이 자동으로 잡히는 것도 모두 시스템이 환경 변수를 참조하기 때문입니다.

---
## 코드 핵심 포인트

1. **조회:** `env`나 `printenv` 명령어를 사용하여 현재 설정된 변수들을 확인합니다.
2. **참조:** 변수에 담긴 값을 쓸 때는 이름 앞에 **`$`** 기호를 붙입니다. (예: `echo $HOME`)
3. **설정:** **`export`** 명령어를 사용해 변수를 선언해야만, 내가 실행할 프로그램(파이썬, 도커 등)까지 해당 변수를 읽을 수 있습니다.

---
## 상세 코드 분석

### 모든 환경 변수 목록을 한눈에 확인하기

```bash
env
```

###  특정 변수의 값만 확인하기 (참조할 땐 반드시 앞에 $ 표시)

```bash
echo $USER
# 출력 예시: jason_data
```

### 새로운 환경 변수 생성 및 내보내기 (Export)

```bash
# 현재 터미널 세션과 여기서 실행될 모든 프로그램이 이 변수를 인식하게 됩니다.
export DB_PORT="5432"
```

---
## 초보자가 자주 하는 실수

1. **`$` 기호 사용 시점:** 
	- 변수를 새로 만들거나 값을 대입할 때는 쓰지 않고, 저장된 값을 불러와서 사용할 때만 앞에 붙입니다.

2. Export 명령어 누락:
	- `export` 없이 `VAR="123"`이라고만 선언하면, 현재 터미널 창에서는 변수가 보이지만 여기서 실행한 파이썬 스크립트나 도커 컨테이너는 이 변수를 찾지 못합니다.

3. **설정의 휘발성:**
	- 터미널에 직접 입력한 `export`는 창을 닫으면 사라집니다. 
	- 영구적으로 유지하려면 `.zshrc` 같은 설정 파일에 기록해야 합니다.
