# Linux_Vim_Nano — 텍스트 편집기

## 한 줄 요약

```
vim  → 모드 기반 / 강력하고 빠름 / 서버 어디서나 있음
nano → 모드 없음 / 바로 타이핑 / 초보자 친화적
```

# ① vim 모드 구조 ⭐️

```
vim 의 핵심 = 모드
다른 편집기와 가장 다른 점
모드마다 키의 역할이 완전히 달라짐
```

```
Normal 모드   → 기본 시작 상태 / 명령어 입력
Insert 모드   → 텍스트 타이핑
Command 모드  → : 로 저장·종료 등 실행
Visual 모드   → 텍스트 선택 (범위 지정)
```

## 모드 전환 흐름

```
        i / a / o
Normal ──────────→ Insert
  ↑                  ↓
  └──────── ESC ─────┘

        :
Normal ──→ Command
           :w / :wq / :q!
```

## Normal → Insert 진입

```bash
i    # 커서 앞에서 입력 시작
a    # 커서 뒤에서 입력 시작
o    # 아래 새 줄 만들고 입력
I    # 줄 맨 앞에서 입력
A    # 줄 맨 끝에서 입력
O    # 위에 새 줄 만들고 입력
```

```
가장 많이 쓰는 것:
  i  → 현재 위치에서 바로 입력
  o  → 다음 줄에 새로 입력
```

## Insert → Normal 복귀

```bash
ESC        # Insert → Normal
Ctrl + [   # ESC 와 동일
```

# ② 파일 열기 / 저장 / 종료

## 파일 열기

```bash
vim 파일명        # 파일 열기 (없으면 새로 생성)
vim              # 빈 화면으로 시작
```

## 저장 / 종료 — Command 모드에서

```bash
:w        # 저장 (종료 안 함)
:wq       # 저장 후 종료
:q        # 종료 (변경 없을 때만)
:q!       # 저장 없이 강제 종료 ← 실수 복구
:x        # 변경 있을 때만 저장 후 종료 (:wq 와 거의 동일)
```

```bash
ZZ        # :wq 단축키 (Normal 모드에서 대문자 ZZ)
ZQ        # :q! 단축키
```

```
⚠️ 가장 많이 하는 실수:
  Insert 모드에서 :wq 치면 그냥 문자로 입력됨
  → 반드시 ESC 눌러 Normal 로 돌아온 뒤 :wq
```

# ③ 이동 — hjkl ⭐️

```
Normal 모드에서 방향키 대신 hjkl 사용
방향키도 되지만 hjkl 이 더 빠름 (손이 홈 포지션에서 안 벗어남)
```

## 기본 이동

```bash
h    # ← 왼쪽
j    # ↓ 아래
k    # ↑ 위
l    # → 오른쪽
```

## 단위 이동

```bash
w    # 다음 단어 앞으로
b    # 이전 단어 앞으로
e    # 현재 단어 끝으로

0    # 줄 맨 앞
$    # 줄 맨 끝
^    # 줄 첫 번째 문자 (공백 제외)

gg   # 파일 맨 위 (1번 줄)
G    # 파일 맨 아래
5G   # 5번 줄로 이동
```

## 화면 이동

```bash
Ctrl + f   # 한 페이지 아래
Ctrl + b   # 한 페이지 위
Ctrl + d   # 반 페이지 아래
Ctrl + u   # 반 페이지 위
```

# ⑤ 편집

## 삭제

```bash
x      # 커서 위 문자 1개 삭제
dd     # 현재 줄 삭제
dw     # 커서부터 단어 끝까지 삭제
d$     # 커서부터 줄 끝까지 삭제
d0     # 커서부터 줄 앞까지 삭제
dgg    # 커서부터 파일 맨 위까지 삭제
dG     # 커서부터 파일 맨 아래까지 삭제
3dd    # 3줄 삭제 (숫자 앞에 붙이면 반복)
```

## 복사 / 붙여넣기

```bash
yy     # 현재 줄 복사 (yank)
yw     # 단어 복사
y$     # 줄 끝까지 복사
p      # 커서 아래에 붙여넣기
P      # 커서 위에 붙여넣기
```

## 실행 취소 / 다시 실행

```bash
u        # undo (되돌리기)
Ctrl+r   # redo (다시 실행)
.        # 마지막 명령어 반복
```

## 치환

```bash
:s/old/new/      # 현재 줄 첫 번째 old → new
:s/old/new/g     # 현재 줄 전체
:%s/old/new/g    # 파일 전체
:%s/old/new/gc   # 파일 전체 + 확인하면서
```

# ⑥ nano

```
모드 없음 → 열자마자 바로 타이핑
하단에 단축키 항상 표시 (^ = Ctrl)
외울 필요 없어서 초보자 친화적
```

## 파일 열기

```bash
nano 파일명
```

## 주요 단축키

```bash
Ctrl + O    # 저장 (Write Out) → Enter 로 확인
Ctrl + X    # 종료 (변경 있으면 저장 여부 물어봄)
Ctrl + W    # 검색
Ctrl + K    # 줄 잘라내기 (Cut)
Ctrl + U    # 붙여넣기 (Uncut)
Ctrl + G    # 도움말
Ctrl + /    # 줄 번호로 이동
```

# ⑦ 상황별 편집기 선택

```
nano 쓸 때:
  빠르게 설정 파일 한두 줄 수정
  vim 아직 익숙하지 않을 때
  모드 전환 없이 바로 편집하고 싶을 때

vim 쓸 때:
  대용량 파일 편집 (빠른 이동 + 검색)
  반복 편집 자동화 (매크로 / . 반복)
  SSH 원격 서버 작업 (대부분 vim 설치돼 있음)
  코드 / 스크립트 편집
```

|기준|vim|nano|
|---|---|---|
|진입 장벽|높음 (모드 익혀야 함)|낮음 (바로 사용)|
|속도|빠름|보통|
|서버 기본 탑재|대부분 있음|없는 경우도 있음|
|대용량 파일|강함|보통|
|원격 서버 작업|추천|가능|

```
결론:
  빠른 수정   → nano
  진지한 편집 → vim
  서버 작업   → vim (nano 없을 수도 있음)
```

# 자주 하는 실수

|실수|원인|해결|
|---|---|---|
|`:wq` 쳤는데 문자로 입력됨|Insert 모드 상태|ESC 눌러 Normal 로 먼저|
|나가지지 않음|`:q` 만 치면 변경 있으면 거부|`:q!` 강제 종료 또는 `:wq`|
|수정했는데 저장 안 됨|`:w` 안 씀|`:w` 또는 `:wq`|
|hjkl 이 문자로 입력됨|Insert 모드 상태|ESC 눌러 Normal 모드로|
|실수로 뭔가 삭제됨|`d` 명령어|`u` 로 undo|
