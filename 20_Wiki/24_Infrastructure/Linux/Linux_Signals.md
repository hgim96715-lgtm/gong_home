## 개념 한 줄 요약

**"프로세스에게 보내는 '경고장' 또는 '사형 선고'."**
실행 중인 프로그램(Process)에게 "멈춰", "종료해", "다시 시작해" 같은 신호를 보내는 통신 방법이다.

---
## Code Core Points: 필수 시그널 3대장

`kill` 명령어는 죽이는 게 아니라 **신호를 보내는(Send Signal)** 도구다. 
번호를 알고 있어야 제대로 다룰 수 있다.

|   번호   | 이름          | 의미                    | 비유                            |
| :----: | :---------- | :-------------------- | :---------------------------- |
| **15** | **SIGTERM** | **Terminate (종료 요청)** | "작업 마무리하고 퇴근하세요~" (정중함)       |
| **9**  | **SIGKILL** | **Kill (강제 종료)**      | "당장 짐 싸서 나가!!" (강제 집행, 거부 불가) |
| **2**  | **SIGINT**  | **Interrupt (인터럽트)**  | **Ctrl + C** (키보드로 멈추는 것)     |

---
## 실무 적용 사례 (Practical Context)

### ① 우아한 종료 (Soft Kill) - 권장

먼저 정중하게 종료를 요청한다. 
프로그램이 저장할 거 저장하고 스스로 꺼지게 둔다.

```bash
# kill -15 [PID]
kill -15 1234 
# 또는 그냥 kill 1234 (기본값이 15번임)

# PID 모를 때는 이름으로 보낼수도 있다 
pkill -15 nginx 
```

### ② 강제 종료 (Hard Kill) - 최후의 수단 

먹통이 돼서 말을 안 들으면 9번으로 날려버린다. 
단, 저장 안 된 데이터는 날아간다.

```bash
# kill -9 [PID]
kill -9 1234
```

### ③ 키보드로 취소하기

터미널에서 뭐 돌리다가 멈추고 싶을 때 누르는 그 키.

```bash
Ctrl + C
# (내부적으로 SIGINT(2번) 신호가 날아감)
```
