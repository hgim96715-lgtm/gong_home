# 1️⃣ 사용자(User) – 모든 것의 시작

### User

**“명령을 입력하는 사람”**

- 키보드로 `ls`, `grep`, `python`, `docker` 같은 명령을 입력하는 주체
    
- 서버 관리자, 개발자, 자동화 시스템 모두 포함
    

👉 **리눅스는 항상 “누가 이 명령을 했는가?”를 기준으로 동작**

---

# 2️⃣ 신분증 & 보안 (Identity & Security)

## ID / whoami (신분증)

whoami

id

- 리눅스는 모든 사용자를 **UID / GID**로 관리
    
- 파일 접근, 프로세스 실행, 네트워크 사용 여부 전부 여기서 결정
    

📌 비유

> “리눅스 세계의 주민등록증”

---

## sudo (관리자 권한)

sudo apt install nginx

- 일반 사용자가 **일시적으로 root 권한** 획득
    
- 로그가 남고, 허용된 명령만 가능
    

📌 핵심

> “신분증이 있어야 권한 검사 가능”

➡️ 이게 나중에 **Permission(rwx)** 과 연결됨

---

# 3️⃣ 자동화 & 스크립팅 영역

## Shell Script (.sh)

backup.sh

deploy.sh

- 사람이 치던 명령을 **파일로 묶어서 실행**
    
- 결국 내부에서는 **Shell이 한 줄씩 해석**
    

---

## Cron (Crontab)

0 3 * * * backup.sh

- 사람이 없어도 **정해진 시간에 자동 실행**
    
- 새벽 백업, 로그 정리, 배치 작업
    

📌 중요한 점

> Cron도 결국 **Shell을 호출해서 명령 실행**

---

# 4️⃣ Shell – 통역사 역할

### Shell (bash, zsh)

- 사용자의 **텍스트 명령을 해석**
    
- 변수, 파이프, 리다이렉션 처리
    
- 프로그램 실행 요청을 커널에 전달
    

📌 비유

> “사람 말 → 커널이 이해하는 요청으로 번역”

---

# 5️⃣ 파이프라인 (Pipeline) – 데이터 엔지니어링

cat file | grep error | sort | uniq | awk '{print $2}'

### Pipeline (Data Flow)

- 명령과 명령을 `|` 로 연결
    
- **출력 → 입력**으로 데이터가 흐름
    

---

### Cmd1: 읽기

- `cat`, `head`, `tail`
    
- 파일 또는 스트림을 읽음
    

### Cmd2: 검색

- `grep`, `find`
    
- 조건에 맞는 데이터만 필터링
    

### Cmd3: 정렬/중복 제거

- `sort`, `uniq`
    
- 데이터 정제
    

### Cmd4: 가공

- `awk`, `sed`, `jq`
    
- 컬럼 처리, 패턴 변경, JSON 파싱
    

📌 핵심

> 리눅스 쉘은 **텍스트 기반 데이터 처리 파이프라인**

---

# 6️⃣ Kernel – 리눅스의 심장

### Linux Kernel

- **하드웨어와 소프트웨어 사이의 중재자**
    
- 직접 CPU, RAM, 디스크를 만질 수 있는 유일한 존재
    

📌 Shell은 요청만 함  
📌 실제 실행은 **Kernel이 담당**

---

# 7️⃣ 커널 서브시스템

## Process Management

- 프로세스 생성 (`fork`, `exec`)
    
- 스케줄링
    
- 종료 관리
    

---

## Memory Management

- RAM 할당
    
- 가상 메모리
    
- swap
    

---

## File System (VFS)

- ext4, xfs, nfs를 **통합 인터페이스**로 제공
    
- 파일 권한 검사 수행
    

---

## Network Stack

- TCP/IP
    
- 소켓
    
- 포트 관리
    

---

## Device Driver

- 키보드, 디스크, 네트워크 카드 제어
    

---

# 8️⃣ 프로세스 생명주기

## Job Control

- 쉘이 관리하는 작업 단위
    

---

### Foreground

python app.py

- 터미널 점유
    

---

### Background

python app.py &

nohup python app.py

- 터미널 종료 후에도 실행
    

---

### Daemon

- systemd가 관리
    
- nginx, sshd, docker
    

📌 흐름

> Job → Foreground / Background → (장기 실행되면) Daemon

# 9️⃣ 시스템 모니터링

## uptime

## top / htop

- CPU / 메모리 / 프로세스 감시
    

## netstat / ss

chmod

chown

📌 그래서

# 1️⃣1️⃣ 네트워크 흐름

DNS → TCP/IP → Port → NIC

- DNS: 주소 해석
    
- TCP/IP: 통신 규칙
    
- Port/Socket: 서비스 식별
    
- NIC: 물리적 전송
    

📌 모니터링 도구가 포트 상태를 감시

---

# 1️⃣2️⃣ 하드웨어

### CPU, RAM, Disk

- 커널 → 드라이버 → 하드웨어
    
- 사용자는 **직접 접근 불가**

# 🐧 리눅스는 “큰 회사”다 – 신입용 설명

이 그림을 한 문장으로 바꾸면 이거예요:

> **리눅스는 회사이고,  
> 사용자는 직원,  
> Shell은 접수창구,  
> Kernel은 본사 관리자,  
> 하드웨어는 공장이다.**

이제 출근부터 일 처리까지 하나씩 가볼게요.

---

## 1️⃣ 사용자(User) = 직원

👤 **사람이 컴퓨터에게 일을 시킴**

ls

python app.py

- 사용자는 “이거 해줘”라고 **글자로 요청**
    
- 컴퓨터는 아직 이 말을 이해 못함
    

📌 그래서 중간에 **통역사**가 필요함

---

## 2️⃣ 신분증 확인 (Identity)

### whoami / id = 사원증

whoami

id

- 리눅스는 항상 먼저 묻습니다  
    👉 **“너 누구야?”**
    

📌 이유

- 누구냐에 따라 할 수 있는 일이 다름
    
- 남의 파일 마음대로 못 건드리게 하기 위해
    

---

### sudo = 팀장 권한 빌리기

sudo apt install nginx

- 평소엔 일반 직원
    
- 필요할 때만 **잠깐 팀장 권한**
    

📌 중요한 점

> 관리자 권한은 **항상 제한적이고 기록됨**

---

## 3️⃣ 자동화 = 시키는 일 대신 해주는 로봇

### Shell Script (.sh)

- 매번 타이핑 귀찮음
    
- 여러 명령을 **한 파일에 저장**
    

👉 “이거 순서대로 실행해줘”

---

### Cron = 알람 맞춰서 자동 실행

- 새벽 백업
    
- 매일 로그 정리
    

📌 사람이 없어도 실행됨  
📌 하지만 결국 **Shell에게 시킴**

---

## 4️⃣ Shell = 접수창구 직원

💬 “사람 말 → 컴퓨터 말”로 번역

- 사용자가 친 글자 해석
    
- 틀린 명령인지 확인
    
- 실행 가능한 프로그램인지 판단
    

📌 Shell은 **일을 직접 하지 않음**  
📌 **관리자(Kernel)에게 요청만 전달**

---

## 5️⃣ 파이프라인 = 공장 조립 라인

cat log | grep error | sort | uniq

### 이렇게 생각하면 쉬움 👇

- 1번 직원: 파일 읽기
    
- 2번 직원: 필요한 줄만 골라내기
    
- 3번 직원: 정렬
    
- 4번 직원: 중복 제거
    

📌 앞사람 결과를 뒷사람에게 전달  
📌 리눅스가 **텍스트를 잘 다루는 이유**

---

## 6️⃣ Kernel = 본사 관리자 (절대 권력)

👑 **리눅스의 진짜 주인**

- CPU 사용
    
- 메모리 할당
    
- 파일 접근 허용/거부
    
- 네트워크 통신
    

📌 사용자는 직접 접근 불가  
📌 Shell을 통해서만 요청 가능

---

## 7️⃣ 커널 내부 부서들

### 🧩 프로세스 관리

- 프로그램 실행
    
- 동시에 여러 개 돌리기
    

### 🧠 메모리 관리

- RAM 나눠 쓰기
    
- 부족하면 디스크 활용
    

### 📁 파일 시스템

- 파일 저장
    
- 권한 확인 (읽기/쓰기/실행)
    

### 🌐 네트워크

- 인터넷 통신
    
- 포트 관리
    

### 🔌 드라이버

- 키보드, 디스크, 랜카드 제어
    

---

## 8️⃣ 프로그램 실행 방식

### Foreground (앞에서 일함)

python app.py

- 터미널 붙잡고 있음
    

---

### Background (뒤에서 일함)

python app.py &

- 다른 작업 가능
    

---

### Daemon (회사 설비)

- 항상 켜져 있음
    
- 웹 서버, DB
    

📌 systemd가 관리

---

## 9️⃣ 상태 확인 = CCTV 보기

👀 **일 잘 되고 있나 확인만**

- uptime → 회사 바쁜지
    
- top → 누가 일 많이 하는지
    
- netstat → 어떤 문 열려 있는지
    

📌 직접 손대지는 않음

---

## 🔟 파일 구조 = 회사 건물 구조

/

├─ bin   → 도구 창고

├─ etc   → 규칙 문서

├─ home  → 개인 사무실

├─ var   → 기록 보관실

---

### 권한 (Permission) = 출입 카드

- 읽기 / 쓰기 / 실행
    
- 신분증(UID) 기준으로 검사
    

📌 그래서 **권한 오류 = 출입 불가**

---

## 1️⃣1️⃣ 네트워크 = 택배 시스템

- DNS → 주소 찾기
    
- TCP/IP → 전달 규칙
    
- Port → 받는 창구
    
- NIC → 실제 차량
    

---

## 1️⃣2️⃣ 하드웨어 = 공장

🏭 CPU / RAM / Disk

- 커널만 직접 제어 가능
    
- 우리는 **명령만 요청**
    

---

# 🎯 신입이 이 그림으로 꼭 이해해야 할 5가지

1️⃣ 명령어는 바로 실행되지 않는다  
2️⃣ 항상 **누가 했는지**를 확인한다  
3️⃣ Shell은 통역사일 뿐  
4️⃣ Kernel이 모든 결정권자  
5️⃣ 권한과 자동화가 서버의 핵심

---

## 🔥 이 정도 이해면?

- 리눅스 **무섭지 않음**
    
- 에러 메시지 해석 가능
    
- 서버가 “왜 이러는지” 감이 옴
