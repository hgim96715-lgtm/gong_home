## 개념 한 줄 요약

**`source` 명령어**는 설정 파일(`.zshrc` 등)을 수정했을 때, 터미널을 껐다 켜지 않고 **변경된 내용을 현재 터미널에 즉시 반영(Reload)** 하는 명령어이야.

---
## 왜 필요한가 (Why)

**귀차니즘 해결:**
- `.zshrc`에 Alias 하나 추가했는데, 적용하려고 터미널 창을 닫았다가 다시 열면 작업 흐름이 끊기잖아?

**즉시성:**
- "저장했으니까 바로 써보자!" 할 때 딜레이 없이 바로 테스트해 볼 수 있어.

---
## Practical Context (언제 쓰는가?)

- **Alias 등록 후:** `vi ~/.zshrc`로 `alias air='...'`를 추가하고 저장했어.
- **바로 실행 시도:** `air`를 치면? -> `command not found` (아직 몰라).
- **Source 실행:** `source ~/.zshrc` 입력.
- **재시도:** 이제 `air`를 치면 잘 됨!

---
## Code Core Points

```bash
# 문법: source <파일경로> 
source ~/.zshrc
```

- **동의어:** `.` (점 하나)도 `source`와 같은 뜻이야.
	- `. ~/.zshrc` (이렇게 쓰는 고인물들도 있음)

---
## Detailed Analysis (작동 원리)

사실 리눅스에는 **"자식 프로세스(Child Process)"** 라는 개념이 있어.

- 보통 스크립트 실행(`./script.sh`)은 **새로운 창(자식)** 을 몰래 열어서 실행하고 닫아버려. 
- 그래서 거기서 변수를 바꿔도 **원래 창(부모)** 에는 영향이 없어.
- 하지만 **`source`** 는 **"지금 이 창(Current Shell)"** 에서 코드를 한 줄 한 줄 직접 읽어서 실행해
- 그래서 설정 파일의 변수나 Alias가 **내 현재 터미널**에 쏙 박히는 거야.

---
## 초보자가 자주 착각하는 포인트

1. "매번 칠 때마다 `source` 해야 하나요?"
	- 아니! 한 번 설정해두면 다음부터 **새로 여는 터미널**은 알아서 적용돼 있어.
	-  `source`는 **"방금 수정한 걸 지금 당장 이 창에서 쓰고 싶을 때"** 만 하는 거야.

2. "`source` 했는데 에러가 나요."
	- 십중팔구 `.zshrc` 파일 안에 **오타(문법 오류)** 가 있는 거야.
	- 다시 `vi ~/.zshrc` 해서 따옴표(`'`)가 빠졌거나 괄호가 안 닫힌 곳이 없는지 확인해 봐.
