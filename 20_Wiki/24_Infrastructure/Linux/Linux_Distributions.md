## 개념 한 줄 요약

"리눅스 커널이라는 엔진 위에 어떤 시트와 옵션을 넣었느냐에 따라 이름이 달라지는 자동차 모델들과 같다."
리눅스 커널에 GNU 도구, 관리 소프트웨어, 데스크톱 환경 등을 묶어서 사용자가 바로 설치해 쓸 수 있게 만든 것을 **배포판(Distribution, 줄여서 Distro)** 이라고 한다.

---
## 주요 계층 및 가문별 특징 (The Big Families)

리눅스 배포판은 크게 **데비안 계열**과 **레드햇 계열** 두 지붕으로 나뉜다. 
데이터 엔지니어라면 이 두 가문의 '패키지 매니저(명령어)' 차이를 아는 것이 핵심이다.

### ① 데비안 계열 (Debian Family)

가장 대중적이고 사용자 친화적인 가문이다.

- **대표 주자:** **Ubuntu(우분투)**, Debian, Linux Mint
- **패키지 매니저:** **`apt`**, `dpkg`
- **특징:** 설치가 쉽고 커뮤니티가 거대하다. 우리가 아는 대부분의 클라우드 서비스나 학습용 튜토리얼의 기본값이다.

### ② 레드햇 계열 (Red Hat Family)

기업용 서버 시장의 절대 강자 가문이다.

- **대표 주자:** **CentOS**, RHEL(Red Hat Enterprise Linux), Fedora, Rocky Linux
- **패키지 매니저:** **`yum`**, **`dnf`**, `rpm`
- **특징:** 보수적이고 안정적이다. 금융권이나 대기업 서버에서 주로 사용하며, 보안 설정(SELinux 등)이 까다로운 편이다.

## 상세 비교 분석 (Detailed Analysis)

|**구분**|**데비안/우분투 계열**|**레드햇/CentOS 계열**|
|---|---|---|
|**철학**|최신 기능과 편의성|극강의 안정성과 기업 지원|
|**패키지 확장자**|`.deb`|`.rpm`|
|**설치 명령어**|`apt install 패키지명`|`yum install 패키지명`|
|**서버 시장 위치**|스타트업, 클라우드, 개인 개발자|대기업, 금융, 공공기관|

## 실전 맥락 (Practical Context)

### 데이터 엔지니어는 어떤 배포판을 써야 할까?

1. **도커(Docker) 컨테이너**: 우리가 아는 대부분의 도커 이미지는 **Ubuntu**나 **Debian**을 기반으로 만들어진다. 패키지 설치가 빠르고 가볍기 때문이다.
2. **Spark/Hadoop 클러스터**: 예전에는 **CentOS**가 표준이었으나, CentOS의 정책 변화로 인해 최근에는 **Ubuntu**나 **Rocky Linux**로 넘어가는 추세이다.
3. **학습용**: 무조건 **Ubuntu**를 추천한다. 구글링했을 때 나오는 해결책의 90%가 우분투 기준이다.

## Common Misconceptions (주의사항)

- **❌ 우분투에서 `yum`을 치면 안 되나요?**
    - 응, 안 돼. 
    - 집집마다 쓰는 언어가 다르듯이 우분투는 `apt`만 알아듣고, CentOS는 `yum`만 알아들어.
        
- **❌ 배포판마다 커널이 완전히 다른가요?**
    - 아니야. 
    - 뿌리인 **리눅스 커널**은 공유해. 
    - 다만 각 배포판이 커널 버전을 최신으로 유지하느냐(우분투), 검증된 구버전을 쓰느냐(레드햇)의 차이야.



>Airflow 로그에서 `Debian-1deb12u1`이라는 글자를 봤지? 
>그건 우리가 쓰는 Airflow 도커 이미지가 **데비안(Debian) 가문**의 리눅스 위에서 만들어졌다는 뜻이야. 
>그래서 우리가 Java를 깔 때 `yum`이 아니라 **`apt-get install`** 을 썼던 거지!
