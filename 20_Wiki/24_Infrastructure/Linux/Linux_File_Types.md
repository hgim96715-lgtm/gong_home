## 개념 한 줄 요약

**"리눅스에서는 '모든 것이 파일(Everything is a file)'이다."**
텍스트 문서뿐만 아니라, **폴더(Directory), 하드디스크, 마우스, 심지어 인터넷 연결(Socket)** 까지도 리눅스는 전부 '파일'로 취급해서 관리한다.

---
## 왜 필요한가? (Why)

**문제점:**
- `ls -l`을 쳤는데 맨 앞에 `-rw-r--r--`도 있고 `drwxr-xr-x`도 있고 `srw-rw----`도 있다.
- "도대체 맨 앞의 알파벳(`-`, `d`, `l`, `s`...)이 무슨 뜻이지?"

**해결책:**
- 이 알파벳(File Type)을 알아야 이것이 **들어갈 수 있는 방(폴더)** 인지, **읽을 수 있는 문서**인지, 아니면 **건드리면 안 되는 하드웨어 장치**인지 구분할 수 있다.

---
## 실무 적용 사례 (Practical Context)

데이터 엔지니어가 주로 마주치는 건 4가지다.

1.  **`-` (일반 파일):** 설정 파일(`config.yaml`), 소스 코드(`.py`) 수정할 때.
2.  **`d` (디렉토리):** 데이터 저장할 폴더 만들고 이동할 때.
3.  **`l` (링크):** 파이썬 버전 관리할 때. (`python` 명령어를 치면 실제론 `python3.9`가 실행되게 연결)
4.  **`s` (소켓):** **도커(Docker)** 가 죽었는지 살았는지 확인할 때 (`/var/run/docker.sock`).

---
## Code Core Points: 리눅스 7대 파일 타입

`ls -l` 명령어를 쳤을 때 **가장 맨 앞글자(Symbol)** 를 보면 된다.

### ① 가장 흔한 3대장 (99% 마주침)

1.  **`-` (Regular File): 일반 파일**
    - 텍스트, 이미지, 실행 파일, 압축 파일 등 데이터를 담고 있는 평범한 파일

2.  **`d` (Directory): 디렉토리(폴더)**
    - 다른 파일들의 목록을 담고 있는 파일. 리눅스에선 폴더도 파일의 일종이다.

3.  **`l` (Symbolic Link): 심볼릭 링크(바로가기)**
    - 윈도우의 '바로가기 아이콘'과 같다. 원본 파일을 가리키는 이정표.
    - 예: `/usr/bin/python -> python3.8`

### ② 특수 목적 (가끔 마주침)

4.  **`s` (Socket): 소켓**
    - 프로세스끼리 대화하는 전화기. 네트워크 통신이나 도커 데몬과의 통신에 쓰임.
    - 예: `/var/run/docker.sock`

5.  **`p` (Named Pipe / FIFO): 파이프**
    - 프로세스 간에 줄 서서 데이터를 주고받는 통로(Queue). 먼저 넣은 게 먼저 나온다.

### ③ 하드웨어 장치 (건드리지 말 것)

6.  **`c` (Character Device): 문자 장치**
    - 데이터를 한 글자씩(Byte stream) 처리하는 장치.
    - 예: 키보드, 마우스, 터미널 화면(`/dev/tty`)

7.  **`b` (Block Device): 블록 장치**
    - 데이터를 덩어리(Block)째로 저장하고 읽는 장치.
    - 예: 하드디스크, SSD, USB (`/dev/sda`)

---
## 상세 분석 (Detailed Analysis)

직접 확인해보자.

```bash
# ls -l 명령어로 /dev (장치 폴더)를 조회해보면 신기한 게 많다.
ls -l /dev

# 출력 예시:
# c rw-rw-rw-  1 root tty     /dev/tty   (맨 앞 c: 키보드/화면 같은 문자 장치)
# b rw-rw----  1 root disk    /dev/sda   (맨 앞 b: 하드디스크)
# s rwxrwxrwx  1 root docker  /var/run/docker.sock (맨 앞 s: 도커 소켓)
```

## 초보자가 자주 하는 실수 (Misconceptions)

### ① "확장자(.txt, .exe)가 중요하나요?"

- **윈도우:** 확장자로 파일 종류를 판단한다. `.txt`를 지우면 메모장이 못 연다.
- **리눅스:** **확장자는 장식일 뿐이다.** 파일의 **내용(Content)** 과 **권한(Permission)** 이 중요하다.
- 이름이 `image.png`여도 실제 내용이 텍스트면 그냥 텍스트 파일이다.
- 실행 파일(`example.sh`)에 `.sh`를 떼고 `example`이라고 해도, 실행 권한(`x`)만 있으면 잘 돌아간다.
