## 개념 한줄 요약

**`tr` (Translate)** 명령어는 텍스트 스트림에서 문자를 **1:1로 치환하거나 삭제**할 때 사용합니다. 
`sed`보다 기능은 단순하지만 훨씬 가볍고 빠릅니다. 
주로 대소문자 변환이나 특수문자 제거에 쓰입니다.

----
## 언제 쓰는가?

* 사용자 입력이 대문자로 오든 소문자로 오든 똑같이 처리하고 싶을 때.
* 윈도우에서 작성된 파일의 `^M` (Carriage Return) 문자를 지우고 싶을 때.
* 쉼표(`,`)로 구분된 데이터를 공백으로 바꾸고 싶을 때.

---
##  Code Core Points

> `tr`은 파일을 직접 읽지 못합니다! 반드시 **파이프(`|`)** 나 **리다이렉션(`<`)** 을 통해 데이터를 받아야 합니다.

```bash
# 기본 문법
cat file.txt | tr [바꿀문자셋] [바뀔문자셋]
echo "텍스트" | tr [옵션] [문자]
```

---
## Detailed Analysis (코드 해석)

코드를 한 줄씩 뜯어봅시다.

```bash
# 1. 사용자의 입력을 받음 (예: "Yes", "Y", "YES" 등)
read STDIN 

# 2. 핵심 로직: 대문자 ➡️ 소문자 강제 변환
# echo "$STDIN" : 변수 내용을 출력해서 파이프(|)로 넘김
# tr 'A-Z' 'a-z' : A부터 Z까지(대문자)를 a부터 z까지(소문자)로 1:1 매핑하여 치환
STDIN=$(echo "$STDIN" | tr 'A-Z' 'a-z')

# 3. 이제 사용자가 "Yes"라고 치든 "YES"라고 치든 "yes"로 비교 가능
if [ "$STDIN" = "y" ]; then
    echo "YES"
...
```

### 주요 옵션

- `tr 'A-Z' 'a-z'`: 대문자를 소문자로 (가장 많이 씀)
- `tr -d '문자'`: 해당 문자를 **삭제(delete)** (예: `tr -d '-'` 전화번호 하이픈 제거)
- `tr -s '문자'`: 중복된 문자를 **하나로 압축(squeeze)** (예: `tr -s ' '` 공백 여러 개를 하나로)
- `tr '()' '[]'`: 괄호 종류 변환 (소괄호 → 박스 브래킷)

> 💡 **tr은 숫자/영문자뿐만 아니라 특수문자도 1:1 치환된다!**
>  단, **따옴표(`' '`)는 필수** — 괄호 같은 특수문자는 반드시 따옴표로 감싸야 한다.

```bash
# 모든 소괄호를 박스 브래킷으로 바꾸기 
echo "주어진 텍스트 (조각)" | tr '()' '[]' 
# 결과: 주어진 텍스트 [조각] 

# 지정 문자 삭제 (소문자 알파벳 제거)
tr -d 'a-z'
```

---
## 초보자들이 많이 하는 실수 포인트!

- **파일 직접 입력 불가:** `tr file.txt 'a' 'b'` (❌ 에러 발생)
	- 정답: `cat file.txt | tr 'a' 'b'` (⭕️) 또는 `tr 'a' 'b' < file.txt` (⭕️)
- **단어 치환 불가:** `tr`은 **문자(Character)** 단위입니다. "Apple"을 "Banana"로 바꾸려면 `sed`를 써야지 `tr`로는 안 됩니다.
