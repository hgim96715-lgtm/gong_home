## 개념 한 줄 요약

**"리눅스의 모든 파일과 폴더에는 '누구(Who)'에게 '어떤 행동(What)'을 허락할지 적혀 있는 신분증이 붙어 있다."**

---
## 왜 필요한가? (Why)

**문제점:**
- 리눅스는 여러 사람이 동시에 쓰는 **멀티 유저 시스템**이다.
- 내 일기장(파일)을 다른 사람이 마음대로 읽거나 지우면 안 된다.

**해결책:**
- 파일마다 **주인(Owner)**, **그룹(Group)**, **제3자(Others)** 를 나누고, 각각에게 읽기/쓰기/실행 권한을 차등적으로 부여한다.

---
## 실무 적용 사례 (Practical Context)

가장 흔한 에러: **`Permission denied` (권한 거부)** 🚫

1.  **스크립트 실행 불가:** 쉘 스크립트(`run.sh`)를 만들었는데 실행이 안 될 때 -> `chmod +x`로 해결.
2.  **파일 수정 불가:** 분명 내 서버인데 설정 파일 저장이 안 될 때 -> `sudo`를 쓰거나 `chown`으로 주인 바꾸기.
3.  **웹 서버 오류:** 웹사이트 이미지가 안 뜰 때 -> 외부인(Others)에게 읽기 권한(`r`)이 없어서 발생.

---
## 상세 분석: 권한 해부하기 (Detailed Analysis)

`ls -l`을 입력했을 때 나오는 외계어(`drwxr-xr-x`)를 해석할 줄 알아야 한다.

### ① 대상 (Who)

권한은 3가지 그룹으로 나뉜다.

1.  **User (u):** 파일의 **소유자** (가장 강력한 권한).
2.  **Group (g):** 소유자와 같은 팀(그룹)에 속한 사람들.
3.  **Others (o):** 그 외 나머지 모든 사람들 (제3자).

### ② 행동 (What)

각 대상에게 부여할 수 있는 3가지 능력치다.
권한이 없으면 대시(`-`)로 표시된다.

|**기호**|**의미**|**숫자값(Score)**|**설명**|
|---|---|---|---|
|**r**|**R**ead|**4**|파일을 읽을 수 있다. (cat, vi 읽기)|
|**w**|**W**rite|**2**|파일을 수정/삭제할 수 있다. (save)|
|**x**|e**X**ecute|**1**|파일을 실행할 수 있다. (프로그램 실행)|
|**-**|**None**|**0**|**권한 없음 (금지)**|

### ③ 실전 해석: 이 긴 줄은 도대체 뭔가요?

터미널에 나온 이 한 줄을 **7개 구역**으로 쪼개서 해석해보자.

```bash
# 예시 데이터
-rw-r--r--  1  root  root   681  Mar 23 2022  xattr.conf
```

| **순서** | **실제 값**         | **이름**         | **설명 (해석)**                                            |
| ------ | ---------------- | -------------- | ------------------------------------------------------ |
| **1**  | **`-rw-r--r--`** | **권한 (Mode)**  | **[가장 중요]** 맨 앞글자가 `-`면 파일, `d`면 폴더. 그 뒤는 3개씩 끊어서 읽는다. |
| **2**  | `1`              | 링크 수           | 이 파일에 연결된 바로가기 수 (몰라도 됨).                              |
| **3**  | **`root`**       | **소유자 (User)** | 이 파일의 주인님.                                             |
| **4**  | **`root`**       | **그룹 (Group)** | 이 파일에 접근 가능한 팀(그룹).                                    |
| **5**  | `681`            | 용량 (Size)      | 파일 크기 (byte 단위).                                       |
| **6**  | `Mar 23...`      | 수정 날짜          | 마지막으로 건드린 시간.                                          |
| **7**  | `xattr.conf`     | 파일 이름          | 파일이나 폴더의 이름.                                           |

### ④ 연습: 암호 해독해보기 (Case Study) 

#### **CASE 1. 폴더일 때 (`d`로 시작)**

맨 앞글자가 `d`면 폴더. 그 뒤는 3개씩 끊어서 읽는다

```bash
drwxr-xr-x  4  root  root  4096  Jan 25 12:52  ssl
```

- **`d`**: 아, 이건 **폴더(Directory)** 구나!
- **`rwx` (주인)**: Root는 읽고(4), 쓰고(2), 들어가기(1) 다 할 수 있네. (7)
- **`r-x` (그룹)**: Root 그룹 팀원들은 읽고 들어가기만 되고, **삭제는 안 되네.** (5)
- **`r-x` (남들)**: 다른 사람들도 구경은 할 수 있네. (5)
- **결론:** 권한 점수는 **755**.

#### **CASE 2. 파일일 때 (`-`로 시작)**

```bash
-rw-r--r--  1  root  root  20  Jan 25 12:52  subgid
```

- **`-`**: 아, 이건 **파일**이구나!
- **`rw-` (주인)**: Root는 읽고(4) 수정(2) 할 수 있네. (6)
- **`r--` (그룹)**: 팀원들은 **읽기만(4)** 되네. (수정 금지!)
- **`r--` (남들)**: 남들도 **읽기만(4)** 되네.
- **결론:** 권한 점수는 **644**.

## Code Core Points (명령어 3대장)

### ① `chmod`: 권한 변경 (Change Mode)

권한을 바꾸는 명령어다. **숫자 모드**와 **기호 모드** 두 가지 방법이 있다.

**[방법 A] 숫자 모드 (실무 선호도 1위)** 

`r(4) + w(2) + x(1)` 점수를 더해서 3자리 숫자로 만든다.

```bash
# 755 = 주인(7) / 그룹(5) / 남(5)
# 주인: 4+2+1=7 (다 됨)
# 그룹: 4+0+1=5 (읽고 실행만)
# 남들: 4+0+1=5 (읽고 실행만)
chmod 755 file.txt
```

**[방법 B] 기호 모드 (직관적)** `u/g/o`에게 `+/-`로 권한을 준다.

```bash
# 그룹(g)에게 쓰기(w) 권한 추가(+)
chmod g+w file.txt

# 모든 사람(a)에게 실행(x) 권한 추가
chmod +x run.sh
```

### ② `chown`: 소유자 변경 (Change Owner)

파일의 명의를 이전한다. _"이 파일 이제부터 `ubuntu` 꺼 아니고 `john` 꺼야."_

```bash
# 문법: chown [새주인]:[새그룹] [파일]

# 1. 주인만 변경
chown john file.txt

# 2. 주인과 그룹 동시에 변경 (가장 많이 씀)
chown john:developers file.txt

# 3. 폴더 내부까지 싹 다 변경 (-R 옵션)
chown -R john:developers /var/www/html
```

### ③ `chgrp`: 그룹만 변경 (Change Group)

소유자는 그대로 두고 그룹만 바꿀 때 쓴다. (하지만 실무에선 `chown :group`으로 대체 가능해서 잘 안 쓴다.)
**"콜론(`:`) 앞에 아무것도 안 적는 것"** 

```bash
# [기존 방법] chgrp 사용 
chgrp -R developers /project

# [실무 방법] chown 사용 (추천 ⭐️)
# 해석: "주인은 건드리지 말고(공백), 그룹만(:developers) 바꿔라"
chown -R :developers /project
```

> **꿀팁 (Tip):** > 굳이 `chgrp` 명령어를 외울 필요 없다. 
> **`chown :새그룹 파일명`** 처럼 콜론(`:`) 앞에 주인을 비워두면, 주인은 그대로 두고 **그룹만 변경**된다.
> (예: `chown :developers file.txt`)

## 초보자가 자주 하는 실수 (Misconceptions)

### ① "귀찮으니까 `777` 주면 안 되나요?" 

- **`chmod 777`** 은 "전 세계 누구든 이 파일을 지우고 수정하고 바이러스를 심을 수 있다"는 뜻이다.
- **절대 금지.** 보통은 `755`(실행파일)나 `644`(일반문서)를 쓴다.

### ② "`-R` 옵션을 남발하지 말자"

- `chmod -R 777 /` 같은 명령어를 치는 순간, 시스템 전체가 망가져서 **서버를 포맷해야 할 수도 있다.** (복구 불가능)
- `-R`(재귀, Recursive)은 항상 폴더 경로를 3번 확인하고 써야 한다.

---
### 뇌에 새기기 (암산 연습)

- **`rwx`** = 4 + 2 + 1 = **7** (슈퍼 권한)
- **`rw-`** = 4 + 2 + 0 = **6** (읽고 쓰기) 
- **`r-x`** = 4 + 0 + 1 = **5** (읽고 실행)
- **`r--`** = 4 + 0 + 0 = **4** (읽기 전용)
