## 막대그래프 색깔별 의미 (판독법) 

그래프의 색깔이 곧 **"시간을 어디에 썼는가?"** 를 나타냅니다.

### 🔵 Scheduler Delay (파란색) = "대기 시간"

* **의미:** 드라이버가 일꾼(Executor)에게 "일 해라!" 하고 명령을 보내고, 일꾼이 받을 때까지 걸린 시간.
* **진단:** 이게 길다면? -> **"일감이 배달되기를 기다리는 중."**
* **원인:** 파티션(Task) 개수가 너무 많아서(Small Task Problem), 스케줄러가 과부하 걸림.

### 🔴 Task Deserialization Time (빨간색/살구색) = "택배 까는 시간"

* **의미:** 일꾼이 받은 명령(Task) 봉투를 뜯어서 내용을 이해하는 시간.
* **진단:** 이게 길다면? -> **"준비 운동만 하다가 지침."**
* **원인:** 전송된 데이터나 클로저(함수) 사이즈가 너무 큼.

### 🟢 Executor Computing Time (연두색) = "진짜 일한 시간" 👍

* **의미:** 순수하게 데이터를 지지고 볶는(`map`, `filter`, `join`) 시간.
* **목표:** **전체 막대의 90% 이상이 연두색이어야 건강한 상태입니다.**

### 💎 Getting Result Time (하늘색) = "결과 배송 시간"

* **의미:** 작업이 끝난 결과물을 드라이버가 가져오는 시간.
* **진단:** 이게 길다면? -> **"결과 데이터가 너무 커서 받는 데 오래 걸림."**
* **원인:** `collect()` 등으로 큰 데이터를 한 번에 당겨올 때 발생.

### 🟣 Result Serialization Time (보라색) = "결과 포장 시간"

* **의미:** 작업 결과를 드라이버에게 보내기 위해 다시 포장하는 시간.
* **진단:** 이게 길다면? -> **"결과물이 너무 커서 보내기 힘듦."**
* **원인:** `collect()`로 너무 큰 데이터를 당기려 할 때 발생.

### 🟠 Shuffle Read (황토색) / 🟡 Shuffle Write (노란색)

* **의미:** 데이터를 다른 노드와 주고받는(Shuffle) 시간.

## 실전 진단 예시 🩺

### ❌ 나쁜 그래프 (배보다 배꼽이 더 큼)

* **현상:** 파란색(🔵), 빨간색(🔴), 하늘색(💎) 등 **잡다한 색**이 전체의 30~50%를 차지함.
* **해석:** "일하는 시간보다 준비하고 정리하는 시간이 더 김."
* **처방:** 파티션 줄이기(`coalesce`), `collect()` 자제하기.

### ✅ 좋은 그래프 (이상적인 상태)

* **현상:** 전체가 **연두색(🟢)** 으로 가득 차 있음.
* **해석:** "낭비 없이 열심히 일만 하고 있음."

###  Tip
"튜닝의 시작은 **Event Timeline**을 보는 거야.
만약 **초록색(일하는 시간)** 은 짧은데 전체 시간이 오래 걸린다면, 코드를 고칠 게 아니라 **파티션 설정(Configuration)** 을 의심해봐야 해!"
