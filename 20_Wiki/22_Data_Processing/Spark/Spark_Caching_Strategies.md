## 개념 한 줄 요약 

**"비싼 돈(CPU/시간) 들여 계산한 중간 결과를 메모리나 디스크에 저장해두고, 다음에 필요할 때 꺼내 쓰는 기술."**

* 스파크는 기본적으로 **Lazy Evaluation(지연 연산)** 입니다. Action을 호출할 때마다 **처음부터 다시 계산(Recompute)** 합니다
* 하지만 `cache()`를 쓰면 계산 결과를 저장해두므로, 반복 작업 시 속도가 비약적으로 빨라집니다.

---

## 언제 써야 하나요? (Timing) 

무조건 다 캐시한다고 좋은 게 아닙니다. (메모리는 비싸니까요!) 

1.  **반복 사용:** 동일한 DataFrame을 **여러 번(2회 이상) Action**에 사용할 때.
2.  **복잡한 연산:** 엄청난 `map`, `filter`, `join`을 거친 결과를 재사용해야 할 때.
3.  **반복 알고리즘:** 머신러닝(Iterative Algorithm)처럼 같은 데이터를 계속 뺑뺑이 돌릴 때.

> **⚠️ 주의:** 데이터가 메모리보다 너무 크거나, 한 번만 쓰고 버릴 거라면 캐싱하지 마세요. 오히려 쓰는 비용(Overhead)이 더 듭니다.

## `cache()` vs `persist()` 차이점 

둘은 사실상 같습니다. `cache()`는 `persist()`의 기본 옵션일 뿐입니다.

| 메서드             | 저장 위치 (Storage Level) | 설명                                                |
| :-------------- | :-------------------- | :------------------------------------------------ |
| **`cache()`**   | `MEMORY_AND_DISK`     | (Spark 3.x 기준) 메모리에 넣다가 꽉 차면 디스크로 보냅니다. 가장 무난합니다. |
| **`persist()`** | **선택 가능**             | 메모리만 쓸지, 디스크만 쓸지, 직렬화할지 등을 세밀하게 조정할 수 있습니다        |

###  주요 Storage Level 옵션 

| 옵션명 | 메모리 사용 | CPU 사용 | 특징 | 추천 상황 |
| :--- | :--- | :--- | :--- | :--- |
| `MEMORY_ONLY` | 높음 | 낮음 | 가장 빠름. 꽉 차면 에러 나거나 쫓겨남. | 메모리가 넉넉할 때 |
| `MEMORY_AND_DISK` | 높음 | 중간 | **(기본값)** 메모리 부족하면 디스크에 저장. | 일반적인 상황 |
| `DISK_ONLY` | 아주 낮음 | 높음 | 메모리 안 씀. 디스크 I/O 발생. | 데이터가 너무 클 때 |
| `MEMORY_ONLY_SER` | 낮음 | 높음 | 데이터를 압축(직렬화)해서 저장. CPU를 더 씀. | 메모리 아껴야 할 때 |

## 관리 방법 (Life Cycle) 

캐시는 **빌린 돈**과 같습니다. 다 썼으면 반드시 갚아야(반환해야) 합니다.

```python
# 1. 캐시 등록 (Lazy - 아직 실행 안 됨)
df.cache() 

# 2. 캐시 적재 (Action 호출 시점에 메모리에 올라감) 
df.count()

# ... 여러 작업 수행 ...

# 3. 캐시 비우기 (중요!) 
df.unpersist()
```

### 흔한 실수 (Common Pitfalls)

- **메모리 누수 (Memory Leak):** `unpersist()`를 안 하고 계속 새로운 데이터프레임을 캐시하면 **OOM(Out Of Memory)** 이 발생합니다.
- **전체 캐싱:** 1TB짜리 데이터를 100GB 메모리에 캐싱하려고 하면? 당연히 터집니다. 필요한 컬럼만 `select` 한 뒤에 캐시하세요

### Tip

"초보들이 제일 많이 하는 실수가 뭔지 알아? **`df = df.cache()`라고 쓰는 거야.** 
캐시는 변환(Transformation)이 아니라 상태를 바꾸는 거야. 
그냥 **`df.cache()`** 라고만 써도 적용돼. 
그리고 제발! 다 쓴 캐시는 **`df.unpersist()`** 로 지워줘. 안 그러면 다음 작업들이 메모리 없어서 빌빌댄다!"
