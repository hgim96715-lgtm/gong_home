## 개념 요약 (Concept Summary)

위에서 아래로 무조건 다시 실행(Rerun)되는 Streamlit의 스크립트 특성을 제어하여,
**특정 조건이 만족되지 않으면 더 이상 아래쪽 코드를 실행하지 못하게 차단(`st.stop`)** 하거나 강제로 새로고침(`st.rerun`)하는 흐름 제어 도구입니다.

---

## 왜 필요한가 (Why is it needed)

 **문제점:** 
- 사용자가 드롭다운에서 아직 아무것도 선택하지 않았는데(값=`None`), 아래쪽 코드에서는 그 값을 가지고 DB를 조회하거나 그래프를 그리려고 시도하다가 무조건 에러(빨간 화면)가 터집니다.

 **해결책:**
 -  `st.stop()`을 사용하면, 에러가 터지기 전에 **"조건이 안 맞으면 여기서 코드를 조기 종료(Early Return)해버려라"** 라고 명령하여 앱을 안전하고 예쁘게 유지할 수 있습니다.

---

## 실전 맥락 (Practical Context)

대시보드 실무에서 **"필수 입력값 검증(Validation)"** 을 할 때 숨 쉬듯 사용합니다. 
예를 들어, "날짜를 먼저 선택하세요", "파일을 먼저 업로드하세요" 같은 안내 문구를 띄운 뒤, 사용자가 행동을 취할 때까지 무거운 데이터 처리 로직(DB 쿼리 등)이 돌아가지 않도록 보호하는 '방어막' 역할을 합니다.

---

## Code Core Points

### 실행 강제 중지 (`st.stop`)

- **문법:** `st.stop()`
- **동작 원리:** 파이썬 코드 실행기가 이 줄을 만나는 순간, **이 줄 아래에 있는 모든 코드를 무시하고 화면 그리기를 즉시 멈춥니다.**
- **코드 해석:** 사용자가 월을 선택하지 않아서 `selected_month_idx`가 `None`인 상태라면, 안내 메시지(`st.info`)를 띄우고 `st.stop()`을 걸어버립니다. 사용자가 월을 선택해야만 비로소 차단기가 풀리고 아래의 핵심 로직이 실행됩니다.

```python
import streamlit as st

# 1. 월 선택 드롭다운 (초기값은 None)
selected_month_idx = st.selectbox(
    label="월 선택",
    options=[0, 1, 2],
    format_func=lambda i: f"{i+1}월",
    index=None # 아무것도 선택되지 않은 상태로 시작
)

# 2. 필수 값 검증 및 차단기 (방어 로직)
if selected_month_idx is None:
    st.info("👈 왼쪽에서 월을 먼저 선택해주세요~~!!")
    st.stop() # 🚨 여기서 멈춤! 아래 코드는 절대 실행 안 됨!

# ==========================================
# ⬇️ 차단기가 풀려야만(선택을 해야만) 실행되는 안전 구역 ⬇️
# ==========================================

st.success(f"{selected_month_idx + 1}월 데이터 조회를 시작합니다!")
# 무거운 DB 쿼리 실행...
# 예쁜 그래프 그리기...
```

### 강제 새로고침 (`st.rerun`)

- **문법:** `st.rerun()`
- **동작 원리:** 이 줄을 만나는 순간, 즉시 스크립트를 **맨 위(1번째 줄)부터 다시 실행**합니다.
- **실무 맥락:** 사용자가 특정 버튼을 눌러서 데이터를 삭제하거나 초기화했을 때, 화면에 바뀐 상태를 즉각 반영하기 위해 강제로 F5(새로고침)를 누르는 것과 같은 효과를 냅니다.

## 초보자가 자주 하는 실수

#### "코드를 멈추고 싶으면 파이썬 내장 함수인 `sys.exit()`나 `quit()`을 쓰면 안 되나요?"

- **절대 안 됩니다!** `sys.exit()`을 쓰면 사용자 화면만 멈추는 게 아니라, **Streamlit 서버(엔진) 자체가 죽어버려서 앱이 완전히 다운**됩니다
- 웹 대시보드에서는 반드시 Streamlit 전용 차단기인 `st.stop()`을 써야 현재 접속한 유저의 화면 렌더링만 얌전하게 멈출 수 있습니다.

#### "`st.stop()`을 썼는데 화면이 하얗게 멈췄어요."

- `st.stop()`은 "아무것도 안 하겠다"는 뜻입니다. 
- 그래서 `st.stop()`을 부르기 직전에 반드시 `st.info()`나 `st.warning()` 같은 알림창을 띄워서 사용자에게 **"왜 멈췄는지, 무엇을 해야 하는지"** 알려줘야 합니다. 
- 그렇지 않으면 사용자는 앱이 고장 난 줄 압니다.
