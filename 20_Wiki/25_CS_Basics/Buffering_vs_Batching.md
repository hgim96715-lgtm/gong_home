## 쉬운 비유로 이해하기 (개념)

데이터가 발생할 때마다 즉시 DB나 파일에 꽂아 넣지 않고, **임시 바구니(Buffer)** 에 일정량(**Batch Size**)을 모았다가 **한 번에 묶어서 처리**하는 컴퓨터 공학의 핵심 기법입니다.

- **버퍼(Buffer):** 데이터를 잠시 모아두는 '빈 택배 상자' (메모리, RAM)
- **배치(Batch):** 상자에 물건이 꽉 차서 '한 번에 트럭에 싣고 출발'하는 행위

## 왜 이렇게 해야 할까? (안 하면 생기는 대참사 🚨)

왜 이렇게 해야 할까? (안 하면 생기는 대참사 🚨)

**Disk I/O 비용 폭발 (너무 잦은 배달):** 
컴퓨터 입장에서 하드디스크(SSD)에 파일을 쓰거나 DB와 네트워크를 연결하는 작업은 '비행기 띄우는 것'만큼 느리고 무거운(비싼) 작업입니다. 100번 비행기를 띄울 것을, 1번으로 줄여야 서버가 안 터집니다.

**Small File Problem (자잘한 쪼가리 파일 문제):**
데이터 1건 들어올 때마다 파일 1개를 만들면, 나중에 하둡(Hadoop)이나 스파크(Spark) 같은 빅데이터 툴이 수백만 개의 쪼가리 파일을 하나씩 열어보느라 과부하로 죽어버립니다. (빅데이터 제1원칙: "파일은 무조건 크고 묵직하게 뭉쳐라!")

## 파이썬 실전 코드 패턴 (해결책)

해결책은 아주 간단합니다. 파이썬의 **빈 리스트(`[]`)** 를 메모리 바구니로 삼고, 일정 개수가 차면 한 번에 처리한 뒤 바구니를 싹 비워주면 됩니다!

```python
# 1. 택배 상자 준비
buffer = []            # 데이터를 임시로 모아둘 '빈 바구니' (메모리 공간)
BATCH_SIZE = 100       # 상자 크기 (100개가 모이면 한 번에 포장해서 보내겠다는 규칙!)

# 2. 컨베이어 벨트(Kafka)에서 물건 줍기
for message in consumer:
    buffer.append(message.value)  # 데이터가 들어올 때마다 일단 바구니에 쏙 넣음 (엄청 빠름)
    
    # 3. 상자가 꽉 찼나요?
    if len(buffer) >= BATCH_SIZE:
        
        # [여기서 모인 100개를 DB에 넣거나 Parquet 파일로 압축 저장함]
        print("📦 100개 포장 완료! 창고로 보냅니다!")
        
        # 4. 상자 비우기 (가장 중요 ⭐️)
        buffer.clear() # 다음 100개를 담기 위해 바구니를 깨끗하게 비워줌
```

## 요약 공식

- **한 개씩 바로 저장 =** 하수 (서버 터짐, DB 뻗음)
- **바구니(`[]`)에 모았다가 한 번에(`if len == 100`) 저장 =** 고수 (초고속, 안정적)
