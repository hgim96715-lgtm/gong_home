## CSV란 무엇인가? 

**"쉼표(,)로 칸을 나누고, 줄바꿈(Enter)으로 줄을 나눈 가장 단순한 텍스트 파일."**

* **정의:** **C**omma **S**eparated **V**alues (쉼표로 구분된 값들)
* **특징:** 메모장, 엑셀, 파이썬, 자바 등 **세상 모든 프로그램이 읽을 수 있는 만국 공용어**입니다.

---
## CSV의 해부도 

복잡한 서식(색깔, 폰트, 수식)이 전혀 없는 **순수한 글자(Plain Text)** 입니다.

###  실제 파일 내용 (Raw Text)

```text
Name,Age,City          <-- 1. 헤더 (Header): 첫 줄은 보통 컬럼 제목
James,30,New York      <-- 2. 레코드 (Record): 한 줄이 데이터 한 건
Anna,25,"Seoul, Korea" <-- 3. 특수 상황: 데이터 안에 쉼표가 있으면 쌍따옴표("")로 감쌈
```

- **행 (Row):** 줄바꿈(`\n`)으로 구분.
- **열 (Column):** 구분자(Delimiter)로 구분. (기본은 쉼표`,`)

## CSV vs Excel (.xlsx) 비교

|**구분**|**CSV (.csv)**|**Excel (.xlsx)**|
|---|---|---|
|**형식**|단순 텍스트 파일|압축된 바이너리 파일 (XML 집합)|
|**용량**|**가벼움** (글자만 저장)|무거움 (서식, 차트, 수식 포함)|
|**호환성**|**최강** (모든 언어/툴 호환)|엑셀/전용 라이브러리 필요|
|**타입 정보**|**없음** (모두 문자로 취급)|있음 (날짜, 숫자 구분됨)|
|**단점**|"010" -> 10으로 변할 수 있음|무겁고 읽는 속도가 느림|

## 데이터 엔지니어링의 2대 난제 (주의사항) 

판다스든 스파크든 CSV를 다룰 때 **무조건 마주치는 문제**입니다.

### ① 구분자 (Delimiter/Separator)

모든 CSV가 쉼표로만 되어 있지는 않습니다.

- **TSV (Tab Separated Values):** 탭(`\t`)으로 구분.
- **PSV (Pipe Separated Values):** 파이프(`|`)로 구분.
- **해결:** 데이터를 읽을 때 "이 파일은 탭으로 나눠져 있어!"라고 옵션을 줘야 합니다.

```python
# 탭으로 구분된 파일 읽기
df = pd.read_csv("data.tsv", sep='\t')
# 파이프로 구분된 파일 읽기
df = pd.read_csv("data.tsv", sep='|')
```

### ② 한글이 깨질 때 (`encoding`) 깩뛟쀍?!

CSV는 **저장된 인코딩 방식**과 **읽는 방식**이 다르면 글자가 깨집니다.

- **`utf-8`:** 전 세계 표준 (맥, 리눅스, 최신 윈도우)
- **`cp949` (또는 `euc-kr`):** 옛날 윈도우 엑셀에서 만든 한글 파일

```python
# 한글이 깨지면 이걸 넣어보세요!
df = pd.read_csv("korean_data.csv", encoding='cp949')
```

## [비교] Pandas vs Spark 처리 방식

|**구분**|**Pandas (pd.read_csv) 🐼**|**Spark (spark.read.csv) ⚡️**|
|---|---|---|
|**헤더 처리**|`header=0` (기본값: 첫 줄을 헤더로 인식)|`header=False` (**기본값: 첫 줄도 데이터로 인식!**)|
|**타입 추론**|자동 추론 (기본)|`inferSchema=False` (**기본값: 전부 문자로 읽음**)|
|**속도**|빠름 (단일 파일)|느림 (타입 추론 시 전체 스캔 필요)|
|**병렬 처리**|불가 (한 번에 다 읽음)|**가능** (파일을 쪼개서 여러 노드가 읽음)|
|**주 사용처**|10GB 미만 데이터 분석|TB 단위 빅데이터 처리|

>"현업에서는 CSV가 **가장 흔하지만, 가장 위험한 포맷**이기도 해.
>구분자가 꼬이거나 인코딩이 안 맞아서 에러가 터지는 경우가 다반사거든.
>그래서 데이터 파이프라인을 짤 때는 CSV로 들어온 데이터를 정제해서 **Parquet(파케이)** 같은 튼튼한 포맷으로 변환하는 작업을 제일 먼저 한단다!"
