##  개념 한 줄 요약

**"데이터가 쏟아지는 속도와 처리하는 속도가 다를 때, 시스템(DB, 서버)이 과부하로 죽지 않도록 중간에 신호등을 설치해 흐름을 통제하는 기술."**

---
##  왜 필요한가? (문제 상황)

- **Kafka(생산자)**는 1초에 수천 개의 메시지를 미친 듯이 쏟아낼 수 있다.
- 반면 **Database(소비자)** 나 **API 서버**는 1초에 처리할 수 있는 연결(Connection) 수에 한계가 있다.
- 아무런 제어 없이 데이터를 그대로 DB에 꽂아 넣으면? 👉 **DB 커넥션 풀 고갈 및 서버 다운 (DDoS 공격과 같은 효과)**

##  핵심 해결책 (실전 예시)

Streamlit 대시보드에서 Kafka 데이터를 받으며 1분마다 DB를 조회해야 할 때의 안전장치.

```python
# 루프가 1번 돌 때마다 0.05초 대기 (time.sleep(0.05))
# 루프가 20번 돌면? -> 딱 1초!

loop_count += 1

# [쓰로틀링 적용] 카프카 메시지가 올 때마다 DB를 찌르지 않고, 20번에 한 번(1초 간격)만 찌른다!
if loop_count % 20 == 0: 
    df = pd.read_sql(query, engine)
```
