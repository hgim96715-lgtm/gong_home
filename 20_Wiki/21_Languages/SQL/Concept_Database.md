# 데이터베이스(DB)란 무엇인가?

>"엑셀 파일 100개로 수만 명의 고객 데이터를 관리할 수 있을까요? 불가능합니다.
> 데이터베이스는 기업의 심장이자, 모든 데이터 엔지니어링의 출발점입니다."

---
## 엑셀(Excel) 놔두고 왜 굳이 DB를 쓸까?

데이터베이스를 한 줄로 정의하면 **'특정 기업이나 조직의 업무를 수행하기 위해 논리적으로 연관된 데이터들의 모임'** 입니다.

우리가 흔히 쓰는 엑셀과 비교해 보면 DB의 존재 이유가 명확해집니다.

**동시성:** 
엑셀은 누군가 파일을 열고 있으면 "읽기 전용으로 여시겠습니까?"가 뜹니다. 
DB는 수천 명이 동시에 데이터를 읽고 써도 엉키지 않습니다.

**무결성(정확성):** 
엑셀은 숫자 칸에 실수로 '가나다'를 적어도 저장됩니다. 
DB는 규칙에 어긋나는 쓰레기 데이터가 들어오는 것을 원천 차단합니다.

**대용량 처리:** 
엑셀은 데이터가 100만 줄만 넘어가도 버벅거리지만, 
DB는 수억 건의 데이터도 순식간에 검색해 냅니다.

## 데이터베이스의 4가지 특징

데이터베이스가 되기 위해서는 다음 4가지 조건을 반드시 만족해야 합니다. 
앞 글자를 따서 **'통저운공'** 으로 외우면 절대 안 틀립니다!

| **특징 (키워드)**                         | **의미**       | **상세 설명 (실무 비유)**                                                                                 |
| ------------------------------------ | ------------ | ------------------------------------------------------------------------------------------------- |
| **통**합 데이터<br><br>(Integrated Data)  | **"중복 최소화"** | 똑같은 고객 정보가 A부서 파일, B부서 파일에 따로 존재하면 안 됩니다. 똑같은 데이터가 여러 개 존재하는 **중복(Redundancy)을 최소화**한 데이터의 모임입니다. |
| **저**장 데이터<br><br>(Stored Data)      | **"컴퓨터 저장"** | 종이 장부에 적어둔 내역은 DB가 아닙니다. 컴퓨터 하드디스크, SSD, 클라우드 등 **컴퓨터가 접근할 수 있는 저장 매체에 저장된** 데이터여야 합니다.           |
| **운**영 데이터<br><br>(Operational Data) | **"핵심 필수품"** | 조직의 고유한 업무를 수행하는 데 있어서 **없어서는 안 될 반드시 필요한** 데이터입니다. (단순히 잠깐 쓰고 버리는 임시 메모장 같은 데이터는 DB가 아닙니다.)      |
| **공**유 데이터<br><br>(Shared Data)      | **"동시 접근"**  | 나 혼자 보는 일기장이 아닙니다. 여러 응용 프로그램이나 여러 사용자가 **서로 다른 목적으로 동시에 접근하고 공동으로 이용**할 수 있는 데이터입니다.             |

## 흔히 하는 착각: DB vs DBMS

- **데이터베이스 (DB):** 데이터 그 자체의 집합 (예: 창고 안에 쌓여있는 물건들)
- **데이터베이스 관리 시스템 (DBMS):** 이 DB를 사용자들이 쉽게 조작하고 관리할 수 있도록 도와주는 **소프트웨어 프로그램** (예: 창고 관리 시스템과 관리인)
	- **대표적인 DBMS 종류:** Oracle, MySQL, PostgreSQL, SQL Server 등

>**우리가 SQL 문법을 써서 명령을 내리는 대상은 사실 'DB'가 아니라 'DBMS'입니다!**
>DBMS가 우리의 명령을 듣고 실제 DB(하드디스크)에서 데이터를 꺼내다 주는 것이죠.
