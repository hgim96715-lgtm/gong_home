# SQL의 본질

SQL(Structured Query Language)은 관계형 데이터베이스(RDBMS)와 대화하기 위해 만들어진 **국제 표준 언어**입니다

---
## 비절차적 언어 (Non-Procedural)

프로그래밍 언어는 크게 '절차적 언어'와 '비절차적 언어'로 나뉩니다. 
**SQL은 대표적인 비절차적(선언적) 언어**입니다. 

**절차적 언어 (C, Java, Python 등)**
- **특징:** 컴퓨터에게 **'어떻게(HOW)'** 데이터를 찾을지 그 과정(순서)을 하나하나 지시합니다.
- **비유:** "주소록 첫 번째 줄로 가. 성이 '김'씨인지 확인해. 맞으면 리스트에 넣어. 그다음 두 번째 줄로 가..." (반복문 사용)

**비절차적 언어 (SQL)**
- **특징:** 사용자는 처리 과정은 알 필요 없이, 결과적으로 **'무엇(WHAT)'** 을 원하는지만 선언합니다.
- **비유:** "주소록에서 '김'씨인 사람 다 가져와!" (끝)
- **핵심:** 내가 "무엇"을 원하는지 SQL로 던져주면, 내부적으로 가장 빠른 길("어떻게")을 찾아주는 내비게이션 역할을 하는 녀석이 바로 **'옵티마이저(Optimizer)'** 입니다.

---
## 집합적 사고방식 (Set-Based Thinking)

데이터 엔지니어가 SQL을 잘 짜려면 파이썬의 `for`문(반복문) 뇌를 버리고, **집합(Set) 뇌**를 장착해야 합니다.

**SQL은 한 줄 한 줄(Row-by-Row) 데이터를 처리하지 않습니다.** **테이블 전체를 하나의 **'커다란 집합(덩어리)'** 으로 보고, 조건(`WHERE`)에 맞는 부분 집합을 뚝 떼어내거나, 기준(`GROUP BY`)에 따라 여러 개의 작은 집합으로 쪼개는 방식으로 일합니다.

## SQL의 4가지 명령어 분류 

| **분류**                | **이름**                           | **역할**                                       | **포함되는 명령어**                          |
| --------------------- | -------------------------------- | -------------------------------------------- | ------------------------------------- |
| **DML**<br><br>(조작어)  | Data **Manipulation** Language   | 데이터(행/Row)를 **조회, 추가, 수정, 삭제**합니다. (가장 많이 씀) | `INSERT`, `UPDATE`, `DELETE`          |
| **DDL**<br><br>(정의어)  | Data **Definition** Language     | 테이블(뼈대)을 **만들고, 고치고, 부숩니다.**                 | `CREATE`, `ALTER`, `DROP`, `TRUNCATE` |
| **DCL**<br><br>(제어어)  | Data **Control** Language        | 사용자에게 권한을 **부여하거나 뺏습니다.**                    | `GRANT`, `REVOKE`                     |
| **TCL**<br><br>(트랜잭션) | **Transaction** Control Language | 작업 결과를 **확정짓거나 취소**합니다.                      | `COMMIT`, `ROLLBACK`                  |
