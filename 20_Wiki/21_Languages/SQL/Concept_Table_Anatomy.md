# 테이블 구조 해부

>RDBMS에서 데이터를 저장하는 2차원 표를 **테이블(Table)** 또는 **릴레이션(Relation)** 이라고 부릅니다. 
>이 표를 뜯어보면 가로(행)와 세로(열)로 이루어져 있는데, 바라보는 관점(논리적/물리적)에 따라 부르는 이름이 다릅니다.

---
## 테이블 해부도 (한눈에 보기)

| **회원ID (PK)** | **이름** | **가입일자**   | **등급** |
| ------------- | ------ | ---------- | ------ |
| M001          | 홍길동    | 2023-01-01 | VIP    |
| M002          | 이몽룡    | 2023-05-15 | GOLD   |
| M003          | 성춘향    | 2024-02-10 | BASIC  |

###   세로축 (열): 정보의 종류

-  **열 (Column)** : 물리적 DB에서 부르는 이름
- **속성 (Attribute)** : 논리적 데이터 모델링에서 부르는 이름 (가장 작은 논리적 단위)
- **필드 (Field)** : 프로그래밍이나 엑셀에서 주로 부르는 이름

> 위 테이블에서 속성(Column)은 `회원ID`, `이름`, `가입일자`, `등급` 등 총 4개입니다.

###  가로축 (행): 실제 데이터 1건

- **행 (Row)** : 물리적 DB에서 부르는 이름
- **튜플 (Tuple)** : 관계형 데이터베이스(수학적 모델)에서 부르는 이름
-  **레코드 (Record)** : 일반적인 파일 시스템이나 실무에서 부르는 이름
- **인스턴스 (Instance) / 어커런스 (Occurrence)** : 논리적 모델링에서 하나의 실체를 부르는 이름

>위 테이블에서 튜플(Row)은 홍길동, 이몽룡, 성춘향의 데이터 건수인 총 3개(3줄)입니다.

---
## 카디널리티 vs 차수

테이블의 크기를 나타내는 두 가지 고급 용어입니다.
**"열차 카행" (열은 차수, 카디널리티는 행)** 으로 외우세요!

|**용어**|**의미**|**예시 테이블 기준**|**암기 꿀팁**|
|---|---|---|---|
|**차수 (Degree)**|**열(Column/속성)의 개수**|4개|**열차** (열 = 차수)|
|**카디널리티 (Cardinality)**|**행(Row/튜플)의 개수**|3개|**카행** (카디널리티 = 행)|

---
## 도메인 (Domain) : 데이터의 허용 범위

**도메인이란, 하나의 속성(Column)이 가질 수 있는 '원자값들의 집합(허용된 범위)'**

**예시 1:** `성별` 속성의 도메인은 `['남', '여']` 입니다. 여기에 '외계인'이나 '3'이라는 데이터는 들어올 수 없습니다.
**실무 적용:** DB 테이블을 만들 때 데이터 타입(`VARCHAR`, `INTEGER`)을 지정하거나, 제약 조건(`CHECK (age >= 0)`)을 거는 행위가 바로 도메인을 정의하는 것입니다.
