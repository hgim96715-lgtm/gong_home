## 개념 한 줄 요약

테이블을 구성하는 **가로줄(Row)** 과 **세로줄(Column)** 의 정확한 명칭을 이해하고, 
`AS`를 이용해 컬럼이나 테이블의 이름을 코딩하기 편한 **'별명(Alias)' 으로 바꿔주는 방법입니다.

---
## 왜 필요한가 (Why)

**문제 상황:**
1.  **용어 혼란:** 동료가 "거기 인스턴스가 비었는데요?"라고 했는데, 내가 "인스턴스가 뭐지?" 하고 있으면 대화가 안 됩니다. (속성, 필드, 레코드 등 같은 대상을 부르는 말이 너무 많습니다.)
2.  **AS 부재:** `SELECT price * 0.1` 처럼 계산을 하면, 결과표의 제목이 `price * 0.1`이라고 지저분하게 나옵니다. 나중에 프로그램에서 이 값을 가져다 쓸 때 이름이 없어서 에러가 납니다.

**해결책:**
1.  **용어 통일:** 세로줄=컬럼=속성, 가로줄=로우=인스턴스 임을 머릿속에 매핑합니다.
2.  **AS 사용:** 계산식이나 복잡한 이름에 `AS tax` 처럼 **'별명(Alias)'** 을 붙여주어, 결과표를 깔끔하게 정리합니다.

---
## 실무 맥락에서의 사용 이유

실무 회의에서는 이런 대화가 오갑니다.

* "이번에 '회원등급' **속성(Column)** 하나 추가하죠." -> 테이블 구조(Schema)를 바꾸자는 뜻.
* "이상한 **인스턴스(Row)** 가 하나 들어왔는데요?" -> 불량 데이터가 들어왔다는 뜻.
* "테이블 A와 B의 `id` 컬럼 이름이 겹치니까, 조회할 때는 `user_id`로 **알리어스(AS)** 주세요." -> 코딩할 때 중복 에러 방지.

---
## 코드 핵심 포인트

1.  **동의어 정리:**
    * **세로 ↓ (Column):** 속성(Attribute), 필드(Field) → 데이터의 **'타입(성격)'** 을 정의
    * **가로 → (Row):** 튜플(Tuple), 레코드(Record), 인스턴스(Instance) → 실제 저장된 **'데이터 한 건'**
2.  **AS 문법:** `컬럼명 AS 별명` (AS는 생략 가능하지만, 초보 때는 써주는 게 가독성에 좋습니다.)
* 테이블 별칭: `테이블명 AS 별명` (보통 `AS`를 생략하고 `테이블명 별명`으로 많이 씀)

---
## 상세 분석 (Line-by-Line)

```sql
SELECT 
    p.product_name AS "상품명",    --  p라는 테이블의 컬럼을 가져옴
    p.price,                     --  이것은 '속성(Attribute)' 또는 '컬럼'이라고 부름
    p.price * 0.1 AS tax         --  계산 결과에 별명 부여
    category                    --  컬럼을 선택하는 행위
FROM 
   coupang_products AS p        -- 3. 테이블 이름이 기니까 이제부터 'p'라고 부르겠다!
LIMIT 1;                        -- 6. 데이터 한 줄(Row, 인스턴스)만 가져오기
```

**`p.product_name AS "상품명"`**:

- 원본 데이터베이스에는 `product_name`이라고 저장되어 있지만, 출력된 결과 엑셀 파일에는 **"상품명"** 이라고 표시됩니다.
- 별명에 공백(띄어쓰기)이 있으면 반드시 큰따옴표(`" "`)로 감싸야 합니다.

**`LIMIT 1` (인스턴스/Row/Tuple):**

- 가져온 결과의 **가로줄 한 줄**을 의미합니다.
- 예: `(맥북, 200만원, 20만원, 전자기기)` ← 이 데이터 묶음 하나가 바로 **인스턴스(Row)** 입니다.

## 초보자가 자주 착각하는 포인트

**테이블 별칭 선언 후 '본명' 사용 금지 (가장 중요!)**
- `FROM Users AS u`라고 별명을 지어놓고, `SELECT Users.name`이라고 **본래 테이블 이름을 쓰면 에러**가 납니다.
- **규칙:** 별명을 짓는 순간부터 본명은 잊어야 합니다. 무조건 `u.name`이라고 불러야 합니다.

**AS를 쓰면 DB 원본 이름이 바뀌나요?**
    - 절대 아닙니다! 원본은 그대로 있고, **지금 내 화면에 보이는 결과표(영수증)** 에서만 이름이 잠시 바뀌는 것입니다. 안심하고 쓰세요.

 **Row와 Column이 헷갈려요.**
    - **컬럼(Column)** 은 그리스 신전의 **'기둥'** 을 생각하세요. 기둥은 세로로 서 있죠? (속성, 타입)
    - **로우(Row)**는 노를 젓는(Rowing) 배를 생각하세요. 옆으로 나아갑니다. (데이터 건수)

 **작은따옴표('') vs 큰따옴표("")**
    - `AS`로 별명을 지을 때 띄어쓰기가 있다면 **큰따옴표("")** 가 표준입니다. (`AS "총 매출"`)
    - 데이터 값(문자열)을 표현할 때는 **작은따옴표('')** 를 씁니다. (`WHERE name = '홍길동'`)
    - 이거 반대로 쓰면 에러 납니다.
