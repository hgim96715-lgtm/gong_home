# 숫자 함수

## Concept Summary(한줄 요약)

**"지저분한 소수점을 다듬고, 부호와 나머지를 자유자재로 다루는 수학 도구상자."** 
데이터 분석 결과는 대부분 소수점이 길게 나오거나, 절댓값/나머지 연산이 필요한 경우가 많습니다. 
보고서에 넣기 좋게 데이터를 가공하거나 비즈니스 로직(A/B 테스트 등)을 구현할 때 필수적입니다.

---
## 왜 필요한가? (Why)

- **가독성:** `33.333333...%` 보다는 `33.3%`가 훨씬 읽기 좋습니다.
- **비즈니스 규칙:** "원 단위 절사(버림)" 같은 금전 계산 규칙을 지켜야 합니다.
- **분산/그룹핑:** 나머지를 구하는 함수(`MOD`)를 통해 유저를 균등하게 A/B 그룹으로 나눌 수 있습니다.

---
## 소수점 정리 

### ① 반올림: `ROUND(숫자, 자릿수)`

지정한 자릿수 **까지** 남기기 위해, 그 **다음 자리**에서 반올림합니다.
자릿수 명시하지 않으면 기본값 0 

```sql
SELECT 
    ROUND(123.4567, 1),  -- 123.5  (둘째 자리 5에서 반올림(올림) -> 첫째 자리까지 남김)
    ROUND(123.4567, 2),  -- 123.46 (셋째 자리 6에서 반올림(올림) -> 둘째 자리까지 남김)
    ROUND(123.4567),  -- 123    (소수 첫째 자리 4에서 반올림(버림) -> 정수만 남김)
    
    -- 💡 마이너스(-)는 정수 부분을 건드립니다!
    ROUND(123.4567, -1), -- 120    (1의 자리 3에서 반올림(버림) -> 10의 자리까지 남김)
    ROUND(164.76, -2);   -- 200    (10의 자리 6에서 반올림(올림) -> 100의 자리까지 남김)
```

>**SQLD & 실무 팁:** 자릿수에 **마이너스(`-`)** 를 넣으면 소수점 위(10단위, 100단위)를 정리할 수 있습니다.

### ② 버림(자르기): `TRUNC(숫자, 자릿수)`

반올림 없이 지정한 자릿수 뒤를 칼같이 잘라버립니다.

**DB별 문법 주의:** 
Oracle과 PostgreSQL은 `TRUNC`를 쓰지만, MySQL/SQL Server는 `TRUNCATE`를 씁니다.

```sql
SELECT TRUNC(123.4567, 1); -- 123.4 (5 뒤로 그냥 삭제)
SELECT TRUNC(123.4567, -1); -- 120 (1의 자리 버림)
```

### ③ 무조건 올림: `CEIL()` / `CEILING()`

천장(Ceiling)을 뚫고 올라갑니다. 소수점이 0.1이라도 있으면 **다음 정수**가 됩니다.

**용도:** "페이지 수 계산" (게시글 11개 / 한 페이지 10개 = 1.1 -> 2페이지 필요)

**음수일 때의 작동 원리(수직선 오른쪽으로!)**
`EIL`은 나보다 '크거나 같은' 정수를 찾습니다. 
따라서 `-22.3`의 천장은 `-23`이 아니라, 수직선 오른쪽에 있는 **`-22`** 가 됩니다! 

**DB별 문법 주의**
- **Oracle:** `CEIL()`만 지원
- **SQL Server (MSSQL):** `CEILING()`만 지원
- **PostgreSQL / MySQL:** 관대하게 `CEIL()`, `CEILING()` 둘 다 지원

```sql
-- 양수일 때: 오른쪽(더 큰 수)으로 이동
SELECT CEIL(123.1);     -- 결과: 124
SELECT CEILING(123.1);  -- PostgreSQL은 둘 다 동일하게 작동

-- 음수일 때: 오른쪽(더 큰 수)으로 이동
SELECT CEIL(-22.3);     -- 결과: -22
```

### ④ 무조건 내림: `FLOOR()`

바닥(Floor)으로 내려갑니다. **가장 작은 정수**로 갑니다.

- **용도:** "만 나이 계산" (소수점 무조건 버림)

**음수일때 함정(수직선 왼쪽으로)**
양수일 때는 소수점을 그냥 자르는 것 같지만, 음수일 때는 조심해야 합니다. 
`FLOOR`는 "나보다 작거나 같은 정수"를 찾기 때문에, `-22.3`의 바닥은 `-22`가 아니라 **더 작은 수인 `-23`** 이 됩니다.

```sql
SELECT FLOOR(123.9);  -- 결과: 123
SELECT FLOOR(-22.3);  -- 결과: -23 (⚠️ -22가 아님을 절대 주의!)

-- 💡 헷갈림 방지: TRUNC와의 비교
-- TRUNC는 수직선 방향과 상관없이 소수점 뒤를 그냥 '싹둑' 잘라버립니다.
SELECT TRUNC(-22.3);  -- 결과: -22
```

## 부호와 크기 (Sign & Magnitude)

### ⑤ 절댓값: `ABS(숫자)`

부호(+, -)를 떼고 무조건 양수로 만듭니다. (Absolute)

- **용도:** 두 값의 '차이(격차)' 자체를 구하고 싶을 때 (누가 더 큰지 상관없을 때).

```sql
SELECT ABS(-10); -- 결과: 10
SELECT ABS(15);  -- 결과: 15
```

### ⑥ 부호 판별: `SIGN(숫자)`

숫자가 양수인지, 음수인지, 0인지 판별해줍니다.

- **결과값:** 양수면 `1`, 0이면 `0`, 음수면 `-1` 반환
- **용도:** 매출 증감률이나 성장 지표가 플러스인지 마이너스인지 빠르게 라벨링할 때.

```sql
SELECT SIGN(500), SIGN(0), SIGN(-25); -- 결과: 1, 0, -1
```

## 나눗셈과 나머지 (Division)

### ⑦ 나머지 연산: `MOD(분자, 분모)`

숫자를 나눈 후의 **나머지(Remainder)** 를 반환합니다.
단, 분모가 0일 경우 분자 반환

> **💡 PostgreSQL 팁:** `MOD(A, B)` 함수 대신, 파이썬처럼 연산자 `A % B`를 써도 완전히 똑같이 작동합니다.

**분모가 0일 때**
- **Oracle (SQLD 시험 기준):** 분모가 0이어도 에러가 나지 않고, **분자(앞의 숫자)를 그대로 반환**합니다.
- **PostgreSQL / MySQL:** 수학적 원칙을 엄격하게 지켜서 **에러(Division by zero)** 를 발생시킵니다. 실무에서는 주의하세요!

**음수가 포함될 때의 부호 결정**
**"나머지의 부호는 무조건 분자(나눠지는 수, 즉 왼쪽 숫자)의 부호를 따라간다!"** 
분모(오른쪽 숫자)의 부호는 결과에 아무런 영향을 주지 못합니다.

```sql
-- 1. 기본 나눗셈
SELECT MOD(10, 3);   -- 결과: 1

-- 2. 분모가 0일 때 (SQLD 주의!)
SELECT MOD(10, 0);   -- 결과: 10 (Oracle 기준 에러 없이 앞의 숫자 반환)

-- 3. 부호가 섞여 있을 때 (결과는 무조건 앞 숫자의 부호 따라감!)
SELECT MOD(-10, 3);  -- 결과: -1 (앞 숫자가 음수이므로 결과도 마이너스)
SELECT MOD(10, -3);  -- 결과: 1  (앞 숫자가 양수이므로 결과도 플러스)
SELECT MOD(-10, -3); -- 결과: -1 (앞 숫자가 음수이므로 결과도 마이너스)

-- 💡 번외: PostgreSQL 기호 연산
SELECT 10 % 3;       -- 결과: 1
```

## PostgreSQL Special Caution (나눗셈 주의!)

PostgreSQL 초보자가 가장 많이 하는 실수입니다. 
**"정수 나누기 정수"는 결과도 무조건 정수(몫)만 나옵니다.** (파이썬의 `//` 연산과 동일)

```sql
-- ❌ 기대와 다른 결과
SELECT 10 / 3; 
-- 결과: 3 (소수점 0.3333이 전부 증발함!)

-- ⭕️ 올바른 방법 (둘 중 하나를 실수형으로 만들어야 함)
SELECT 10.0 / 3;        -- 결과: 3.3333...
SELECT 10 / 3::numeric; -- 결과: 3.3333... (PostgreSQL 특화 형변환)

-- 실무 활용 (나눈 뒤 반올림까지 한방에)
SELECT ROUND(10.0 / 3, 2); -- 결과: 3.33
```

## Practical Context (실무 & SQLD 활용)

### 상황 1: A/B 테스트 유저 버킷 분배 (`MOD`)

사용자 ID를 기반으로 짝수/홀수 혹은 10등분 하여 A/B 테스트 그룹을 나눌 때 `MOD`가 핵심 역할을 합니다.

```sql
SELECT 
    customer_id,
    -- customer_id를 10으로 나눈 나머지가 0이면 A, 아니면 B 버킷
    (CASE WHEN MOD(customer_id, 10) = 0 THEN 'A' ELSE 'B' END) AS test_bucket
FROM users;
```

### 상황 2: 평점 평균 구하기 (`ROUND` + 형변환)

Postgres에서 `AVG()` 함수는 입력값이 정수면 결과도 정수로 나올 수 있으니 형변환 후 반올림해야 합니다.

```sql
SELECT 
    product_name,
    -- rating을 numeric으로 바꿔서 평균 내고 -> 소수 첫째 자리까지 반올림
    ROUND(AVG(rating::numeric), 1) AS avg_score 
FROM reviews
GROUP BY product_name;
```

### 상황 3: 금전 단위 절사 (`TRUNC`)

매출액이나 포인트를 10원, 100원 단위에서 절사(버림)할 때 마이너스 자릿수를 활용합니다.

```sql
-- 54,321원 -> 54,000원으로 100원 단위 절사
SELECT TRUNC(54321, -3);
```
