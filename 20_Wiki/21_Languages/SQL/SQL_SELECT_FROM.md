## 개념 한 줄 요약

데이터베이스라는 거대한 엑셀 파일에서 **"어떤 표(Table)의 어느 열(Column)을 가져올지"** 지정하는 가장 기초적인 명령어입니다.

---
## 왜 필요한가 (Why)

**문제 상황:** 데이터베이스에는 수백 개의 테이블이 있고, 하나의 테이블에 수백 개의 컬럼이 있을 수 있습니다. 
"그냥 다 보여줘"라고 하면, 컴퓨터가 수억 건의 데이터를 한 번에 화면에 뿌리려다 멈춰버리거나(OOM), 클라우드 비용 폭탄을 맞게 됩니다.

**해결책:** `SELECT` 뒤에 **'내가 진짜 필요한 컬럼 이름'** 만 콕 집어서 적어줌으로써, 컴퓨터가 처리해야 할 데이터 양을 획기적으로 줄여줍니다.

---
## 실무 맥락에서의 사용 이유

현업(특히 BigQuery 같은 클라우드 환경)에서 **`SELECT *` (전체 조회)는 '해고 사유'** 라는 농담이 있을 정도로 위험합니다. 
데이터 엔지니어는 쿼리를 짤 때 항상 **"최소한의 컬럼만 가져오고 있는가?"** 를 고민해야 합니다. 
1TB짜리 테이블에서 `*`를 쓰면 1TB 비용이 나가지만, 컬럼 하나만 지정하면 10MB 비용으로 끝날 수 있기 때문입니다.

---
## 코드 핵심 포인트

1. **순서 준수:** 무조건 `SELECT`가 먼저고 `FROM`이 뒤에 옵니다. (영어 어순: 가져와라(SELECT) ~로부터(FROM))
2. **쉼표(,) 주의:** 컬럼이 여러 개일 때는 쉼표로 구분하지만, **맨 마지막 컬럼 뒤에는 쉼표를 찍으면 안 됩니다.** (초보자가 제일 많이 내는 에러)
3. **세미콜론(;):** 쿼리의 마침표입니다. 이거 안 찍으면 툴에 따라 에러가 날 수 있습니다.

---
## 상세 분석 (Line-by-Line)

```sql
SELECT 
    product_name,    -- 1. 상품명 컬럼 가져오기
    price,           -- 2. 가격 컬럼 가져오기
    category         -- 3. 카테고리 컬럼 가져오기 (마지막엔 쉼표 금지!)
FROM 
    coupang_products -- 4. 어느 테이블에서 가져올지 지정
;                    -- 5. 쿼리 끝
```

## 초보자가 자주 착각하는 포인트

**`SELECT *`이 편하니까 계속 써도 된다?**

- 학습용 DB(행 100개)에서는 괜찮지만, 습관 들면 실무 가서 사고 칩니다. 귀찮아도 컬럼명을 일일이 적는 버릇을 들이세요.
- (Tip: 실무 툴들은 테이블 이름 옆의 `...`을 누르면 컬럼명을 자동으로 쫙 입력해 주는 기능이 있습니다.)

**대소문자 구별해야 하나요? (`select` vs `SELECT`)**

- SQL 문법(명령어)은 대소문자를 가리지 않습니다. `select`, `Select`, `SELECT` 다 똑같습니다.
- 하지만 **'가독성'** 을 위해 명령어는 대문자(`SELECT`), 테이블/컬럼명은 소문자(`user_id`)로 쓰는 게 전 세계 국룰입니다.

**컬럼 순서가 중요한가요?**

- 데이터 자체에는 영향이 없지만, **결과표가 보이는 순서**가 바뀝니다. 
- `price`를 먼저 적으면 가격이 맨 앞 열에 나옵니다. 보고 싶은 순서대로 적으세요.
