## 개념 한 줄 요약

데이터베이스라는 거대한 엑셀 파일에서 **"어떤 표(Table)의 어느 열(Column)을 가져올지"** 지정하는 가장 기초적인 명령어이자, 데이터를 **가져오면서 동시에 가공(계산, 문자열 합성)까지 해주는 만능 도구**입니다.

---
## 왜 필요한가 (Why)

**문제 상황:** 데이터베이스에는 수백 개의 테이블이 있고, 하나의 테이블에 수백 개의 컬럼이 있을 수 있습니다. 
"그냥 다 보여줘"라고 하면, 컴퓨터가 수억 건의 데이터를 한 번에 화면에 뿌리려다 멈춰버리거나(OOM), 클라우드 비용 폭탄을 맞게 됩니다.

**해결책:** `SELECT` 뒤에 **'내가 진짜 필요한 컬럼 이름'** 만 콕 집어서 적어줌으로써, 컴퓨터가 처리해야 할 데이터 양을 획기적으로 줄여줍니다.

---
## 실무 맥락에서의 사용 이유 (SELECT * 의 위험성)

현업(특히 BigQuery 같은 클라우드 환경)에서 **`SELECT *` (전체 조회)는 '해고 사유'** 라는 농담이 있을 정도로 위험합니다. 
데이터 엔지니어는 쿼리를 짤 때 항상 **"최소한의 컬럼만 가져오고 있는가?"** 를 고민해야 합니다. 
1TB짜리 테이블에서 `*`를 쓰면 1TB 비용이 나가지만, 컬럼 하나만 지정하면 10MB 비용으로 끝날 수 있기 때문입니다.

---
## 가공과 별칭 (Alias)

`SELECT` 절은 단순히 데이터를 있는 그대로 가져오는 것을 넘어, **숫자를 계산하고 문자를 이어 붙일 수 있습니다.** 
이때 파생된 데이터의 이름이 지저분해지므로 '별칭'을 함께 사용합니다.

### ① 산술 연산자 (Arithmetic Operators)

숫자형 데이터가 들어있는 컬럼은 `SELECT` 절이나 `WHERE` 절에서 바로 사칙연산(`+`, `-`, `*`, `/`, **`%`**)이 가능합니다.

- **실무 활용:** '원가' 컬럼을 가져와서 10% 할인된 '할인가'를 즉석에서 계산해 보여줄 때 사용합니다.
- **SQLD 주의점:** 수학과 똑같이 **곱하기/나누기가 더하기/빼기보다 먼저 계산**됩니다. 우선순위를 바꾸려면 괄호 `()`를 쳐야 합니다
- 컬럼끼리 산술 연산을 할 때 **식에 `NULL`이 단 하나라도 껴있다면, 그 연산의 결과는 무조건 `NULL` 이 되어버립니다.

| **순위**  | **연산자 종류**         | **기호 및 키워드**                       | **설명 (SQLD 암기 팁)**                                   |
| ------- | ------------------ | ---------------------------------- | ---------------------------------------------------- |
| **1순위** | **괄호**             | `( )`                              | **무조건 0순위.** 우선순위를 강제로 끌어올리고 싶을 때 무조건 괄호부터 칩니다.      |
| **2순위** | **산술 (곱/나누기/나머지)** | `*`, `/`, **`%`**                  | **나머지를 구하는 `%`**는 곱하기/나누기와 동급입니다. 더하기/빼기보다 먼저 계산됩니다. |
| **3순위** | **산술 (더하기/빼기)**    | `+`, `-`                           | -                                                    |
| **4순위** | **비교 연산자**         | `=`, `>`, `<`, `>=`, `<=`, `<>`    | 양쪽의 값이 큰지 작은지 같은지 비교합니다.                             |
| **5순위** | **SQL 전용 연산자**     | `BETWEEN`, `IN`, `LIKE`, `IS NULL` | SQL에서만 쓰는 특수한 필터링 연산자들입니다.                           |
| **6순위** | **논리 (부정)**        | `NOT`                              | 결과의 반대를 뒤집습니다. (예: `NOT IN`)                         |
| **7순위** | **논리 (AND)**       | `AND`                              | **🚨 함정 주의!** `AND`가 `OR`보다 무조건 먼저 계산됩니다.            |
| **8순위** | **논리 (OR)**        | `OR`                               | 가장 꼴찌로 계산됩니다.                                        |

### ② 합성 연산자 (Concatenation Operators)

두 개 이상의 문자열이나 컬럼을 **하나로 길게 이어 붙일 때** 사용합니다.
DBMS(오라클, SQL Server 등)마다 쓰는 기호가 다르다.

- **Oracle / PostgreSQL:** `||` (수직선 두 개)
- **SQL Server:** `+`
- **표준 / MySQL:** `CONCAT()` 함수 사용

### ③ 별칭 (Alias) 붙이기

연산을 하고 나면 결과표의 컬럼명이 `price * 0.9`처럼 수식 그대로 나와서 아주 지저분합니다. 
이때 **`AS`** 키워드를 써서 예쁜 새 이름표를 달아줍니다.

| **별칭 작성 규칙**            | **예시 코드**        | **설명**                                                                          |
| ----------------------- | ---------------- | ------------------------------------------------------------------------------- |
| **기본 작성법**              | `컬럼명 AS 별칭`      | `AS`는 생략해도 되지만 가독성을 위해 쓰는 것을 권장합니다.                                             |
| **🚨 쌍따옴표(`""`) 필수 조건** | `컬럼명 AS "할인 가격"` | 1) 별칭에 **공백(띄어쓰기)** 이 있을 때<br><br>2) 특수문자가 포함될 때<br><br>3) 대소문자를 엄격하게 구분하고 싶을 때 |

## 상세 분석 (Line-by-Line)

```sql
SELECT 
    -- 1. 단순 조회 및 별칭 지정 (AS 생략 가능)
    product_name AS name,
    
    -- 2. 산술 연산: 원래 가격에서 10% 할인된 가격을 즉석에서 계산
    price * 0.9 AS discounted_price,
    
    -- 3. 합성 연산: "카테고리명_상품명" 형태로 문자열 이어 붙이기 (Oracle 기준)
    category || '_' || product_name AS full_product_code
    
FROM 
    coupang_products -- 4. 어느 테이블에서 가져올지 지정
;                    -- 5. 쿼리의 마침표
```

## 초보자가 자주 착각하는 포인트

### **대소문자 구별해야 하나요? (`select` vs `SELECT`)**

- SQL 문법(명령어) 자체는 대소문자를 가리지 않습니다. `select`나 `SELECT`나 똑같이 작동합니다.
- 하지만 **'가독성'** 을 위해 명령어는 대문자(`SELECT`), 테이블/컬럼명은 소문자(`user_id`)로 쓰는 것이 전 세계 데이터 엔지니어들의 국룰입니다.


### **컬럼 순서가 중요한가요?**

- 데이터 자체에는 영향이 없지만, **결과표(Grid)가 화면에 출력되는 순서**가 바뀝니다. 
- `price`를 먼저 적으면 가격이 맨 앞 열에 나옵니다. 보고 싶은 순서대로 적으세요.

### **`AS '할인 가격'` (홑따옴표) 하면 안 되나요?**

- **절대 안 됩니다! (SQLD 핵심 오답 함정)**
- SQL에서 홑따옴표 `' '`는 **'문자열 데이터 그 자체(값)'** 를 의미하고, 쌍따옴표 `" "`는 **'컬럼명이나 테이블명(객체 이름)'** 을 의미합니다. 
- 별칭은 컬럼의 이름표이므로 반드시 쌍따옴표 `" "`를 써야 합니다.
