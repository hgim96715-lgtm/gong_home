## Concept Summary

**"지저분한 소수점을 깔끔하게 다듬는 미용실."**
데이터 분석 결과(평균 등)는 대부분 소수점이 길게 나오기 때문에, 보고서에 넣기 좋게 반올림하거나 잘라내는 과정이 필수입니다.

---
## 왜 필요한가? (Why)

* **가독성:** `33.333333...%` 보다는 `33.3%`가 훨씬 읽기 좋습니다.
* **비즈니스 규칙:** "원 단위 절사(버림)" 같은 금전 계산 규칙을 지켜야 합니다.
* **데이터 정합성:** 비교 연산 시 미세한 소수점 차이로 `False`가 뜨는 것을 방지합니다.

## Core Functions (핵심 4인방)

### ① 반올림: `ROUND(숫자, 자릿수)`  (가장 중요)

가장 많이 씁니다. 
지정한 자릿수 **까지** 남기기 위해, 그 **다음 자리**에서 반올림합니다.

```sql
-- PostgreSQL에서는 FROM dual이 필요 없습니다. 그냥 SELECT만 쓰세요.
SELECT 
    ROUND(123.4567, 1),  -- 123.5  (둘째 자리에서 판단 -> 첫째 자리까지 남김)
    ROUND(123.4567, 2),  -- 123.46 (셋째 자리에서 판단 -> 둘째 자리까지 남김)
    ROUND(123.4567, 0),  -- 123    (소수 첫째에서 판단 -> 정수만 남김)
    ROUND(123.4567, -1); -- 120    (1의 자리에서 판단 -> 10의 자리까지 남김)
```

**Tip:** 자릿수에 **마이너스(`-`)** 를 넣으면 소수점 위(10단위, 100단위)를 정리할 수 있습니다. (예: 19,800원 만들기)

### ② 버림(자르기): `TRUNC(숫자, 자릿수)`

**MySQL은 `TRUNCATE`이지만, Postgres는 `TRUNC`입니다.** 
반올림 없이 지정한 자릿수 뒤를 칼같이 잘라버립니다.

```sql
SELECT 
    TRUNC(123.4567, 1); -- 123.4 (5 뒤로 그냥 삭제)
```

### ③ 무조건 올림: `CEIL()` / `CEILING()`

천장(Ceiling)을 뚫고 올라갑니다. 소수점이 0.1이라도 있으면 다음 정수가 됩니다.

- **용도:** "페이지 수 계산" (게시글 11개면 2페이지 필요)

```sql
SELECT CEIL(123.1); -- 결과: 124
```

### ④ 무조건 내림: `FLOOR()`

바닥(Floor)으로 내려갑니다. 정수 기준으로 가장 작은 수로 갑니다.

- **용도:** "나이 계산" (만 나이, 소수점 버림)

```sql
SELECT FLOOR(123.9); -- 결과: 123
```

## PostgreSQL Special Caution (나눗셈 주의!) 

PostgreSQL 초보자가 가장 많이 하는 실수입니다. 
**"정수 나누기 정수"는 결과도 무조건 정수(몫)만 나옵니다.**

```sql
-- ❌ 기대와 다른 결과
SELECT 10 / 3; 
-- 결과: 3 (소수점 0.3333이 사라짐!)

-- ⭕️ 올바른 방법 (둘 중 하나를 실수형으로 만들어야 함)
SELECT 10.0 / 3;        -- 결과: 3.3333...
SELECT 10 / 3::numeric; -- 결과: 3.3333... (형변환 사용)

-- 실무 활용 (반올림까지)
SELECT ROUND(10.0 / 3, 2); -- 결과: 3.33
```

## Practical Context (실무)

### 상황 1: 매출액 천 원 단위 절사 (TRUNC)

```sql
-- 54,321원 -> 54,000원
SELECT TRUNC(54321, -3);
```

### 상황 2: 평점 평균 구하기

Postgres에서 `AVG()` 함수는 입력값이 정수면 결과도 정수로 나올 수 있으니 주의해야 합니다.

```sql
SELECT 
    product_name,
    -- rating을 numeric으로 바꿔서 평균 내고 -> 반올림
    ROUND(AVG(rating::numeric), 1) AS avg_score 
FROM reviews
GROUP BY product_name;
```
