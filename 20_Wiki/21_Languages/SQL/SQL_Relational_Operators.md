# 순수관계 연산자 (Pure Relational Operators)

## 개념 한 줄 요약

> **"관계형 데이터베이스의 이론적 뿌리. E.F. Codd 가 정의한 관계 대수(Relational Algebra)에서 나온 4가지 연산자."** SQL 의 SELECT · JOIN · WHERE 등 모든 문법은 이 관계 대수 이론을 구현한 것이다.

---

## 왜 배우는가?

SQL 은 "어떻게(How)" 가 아니라 **"무엇을(What)"** 원하는지 선언하는 언어다. 
그 이면에서 DB 엔진이 실제로 수행하는 연산이 바로 **관계 대수**이고, 순수관계 연산자는 그 핵심 4가지다.

> SQLD 과목 1 시험 범위 — 연산자 이름과 SQL 문법 매핑이 출제된다.

---

---

# ① 4가지 순수관계 연산자 한눈에 보기

|연산자|기호|핵심 의미|SQL 대응|
|---|:-:|---|---|
|**선택 (SELECT)**|σ|행(Row) 을 골라낸다|`WHERE` 절|
|**추영 (PROJECT)**|π|열(Column) 을 골라낸다|`SELECT` 절|
|**조인 (JOIN)**|⋈|두 테이블을 연결한다|`JOIN` 절|
|**나누기 (DIVIDE)**|÷|조건을 모두 만족하는 행을 찾는다|서브쿼리로 구현|

> **주의:** 여기서 말하는 SELECT 는 SQL 의 `SELECT` 절이 아니다. 
> 관계 대수의 **선택(σ)** 은 행을 필터링하는 연산 → SQL 에서는 `WHERE` 로 구현된다.

---

---

# ② 선택 (SELECT) — σ (시그마)

> **"조건을 만족하는 행(Row)만 골라낸다."** SQL 의 `WHERE` 절에 해당한다.

```
관계 대수 표기:
σ 급여>5000 (직원)
= 직원 테이블에서 급여가 5000 초과인 행만 선택

SQL 구현:
SELECT * FROM 직원
WHERE 급여 > 5000;
```

```
원본 직원 테이블:
┌────┬────────┬──────┐
│ 사번 │ 이름   │ 급여 │
├────┼────────┼──────┤
│ 001 │ 홍길동 │ 3000 │
│ 002 │ 이순신 │ 6000 │ ← 선택
│ 003 │ 강감찬 │ 8000 │ ← 선택
└────┴────────┴──────┘

결과:
┌────┬────────┬──────┐
│ 002 │ 이순신 │ 6000 │
│ 003 │ 강감찬 │ 8000 │
└────┴────────┴──────┘
```

# ③ 추영 (PROJECT) — π (파이)

> **"필요한 열(Column)만 골라낸다. 자동으로 중복 행이 제거된다."** SQL 의 `SELECT 컬럼명` 에 해당한다.

```
관계 대수 표기:
π 이름,급여 (직원)
= 직원 테이블에서 이름과 급여 컬럼만 추출

SQL 구현:
SELECT 이름, 급여 FROM 직원;
```

```
원본 직원 테이블:
┌────┬────────┬──────┬────────┐
│ 사번 │ 이름   │ 급여 │ 부서코드 │
├────┼────────┼──────┼────────┤
│ 001 │ 홍길동 │ 3000 │ D01    │
│ 002 │ 이순신 │ 6000 │ D02    │
└────┴────────┴──────┴────────┘

결과 (이름, 급여만 추영):
┌────────┬──────┐
│ 이름   │ 급여 │
├────────┼──────┤
│ 홍길동 │ 3000 │
│ 이순신 │ 6000 │
└────────┴──────┘
```

> **추영의 중복 제거:** PROJECT 는 결과에서 완전히 동일한 행이 나오면 자동으로 제거한다.
>  SQL 로 치면 `SELECT DISTINCT 컬럼명` 과 동일한 개념이다.

# ④ 조인 (JOIN) — ⋈ (나비 넥타이)

> **"두 테이블을 공통 속성(컬럼)을 기준으로 연결해서 하나의 테이블로 만든다."** SQL 의 `JOIN` 절에 해당한다.

```
관계 대수 표기:
직원 ⋈ 부서코드=부서코드 부서
= 직원 테이블과 부서 테이블을 부서코드로 연결

SQL 구현:
SELECT *
FROM 직원
JOIN 부서 ON 직원.부서코드 = 부서.부서코드;
```

```
직원 테이블:              부서 테이블:
┌────┬────────┬──────┐   ┌──────┬────────┐
│ 사번 │ 이름 │부서코드│   │부서코드│ 부서명 │
├────┼────────┼──────┤   ├──────┼────────┤
│ 001 │홍길동 │ D01  │   │ D01  │ 영업팀 │
│ 002 │이순신 │ D02  │   │ D02  │ 인사팀 │
└────┴────────┴──────┘   └──────┴────────┘

JOIN 결과:
┌────┬────────┬──────┬────────┐
│ 사번 │ 이름 │부서코드│ 부서명 │
├────┼────────┼──────┼────────┤
│ 001 │홍길동 │ D01  │ 영업팀 │
│ 002 │이순신 │ D02  │ 인사팀 │
└────┴────────┴──────┴────────┘
```

## 조인의 종류 (관계 대수 → SQL 매핑)

|관계 대수|의미|SQL|
|---|---|---|
|자연 조인 (Natural Join)|같은 이름의 컬럼으로 자동 연결|`NATURAL JOIN`|
|세타 조인 (Theta Join)|임의의 조건으로 연결 (`=` `>` `<` 등)|`JOIN ON 조건`|
|동등 조인 (Equi Join)|`=` 조건으로만 연결 (세타 조인의 특수한 경우)|`JOIN ON A.col = B.col`|
|외부 조인 (Outer Join)|일치하지 않는 행도 포함|`LEFT / RIGHT / FULL OUTER JOIN`|

# ⑤ 나누기 (DIVIDE) — ÷

> **"나누는 테이블의 모든 조건을 만족하는 행만 골라낸다."** SQL 에 직접 대응하는 문법이 없어서 **서브쿼리 + NOT EXISTS** 로 구현한다.

```
개념:
R ÷ S = R 에서 S 의 모든 값과 쌍을 이루는 행만 추출

예시 상황:
"모든 필수 과목을 수강한 학생만 찾아라"
```

```
수강 테이블 (R):          필수과목 테이블 (S):
┌────────┬──────┐       ┌──────┐
│ 학생   │ 과목 │         │ 과목 │
├────────┼──────┤      ├──────┤
│ 홍길동 │ 수학 │         │ 수학 │  ← 필수
│ 홍길동 │ 영어 │         │ 영어 │  ← 필수
│ 이순신 │ 수학 │         └──────┘
└────────┴──────┘
이순신은 영어를 안 들었음!

R ÷ S 결과:
┌────────┐
│ 학생   │
├────────┤
│ 홍길동 │  ← 수학 + 영어 모두 수강 ✅
└────────┘
이순신은 영어 미수강 → 제외 ❌
```

## SQL 구현 — NOT EXISTS 패턴

```sql
-- "모든 필수 과목을 수강한 학생"
SELECT DISTINCT 학생
FROM 수강 R
WHERE NOT EXISTS (
    SELECT 과목 FROM 필수과목 S
    WHERE NOT EXISTS (
        SELECT * FROM 수강
        WHERE 학생 = R.학생
        AND 과목 = S.과목
    )
);
-- "필수과목 중 수강하지 않은 과목이 하나도 없는 학생"
-- = "모든 필수과목을 수강한 학생"
```

> DIVIDE 는 이중 NOT EXISTS 패턴으로 구현한다. 
> SQLD 에서는 개념 이해 수준으로 출제되고, 구현 쿼리까지 요구하는 경우는 드물다.

# ⑥ 순수관계 연산자 vs 일반 집합 연산자

> 관계 대수 연산자는 두 종류로 나뉜다. 시험에서 구분 문제로 자주 출제된다.

|구분|연산자|특징|
|---|---|---|
|**순수관계 연산자**|SELECT(σ) · PROJECT(π) · JOIN(⋈) · DIVIDE(÷)|관계형 DB 에만 존재하는 고유 연산|
|**일반 집합 연산자**|UNION · INTERSECTION · DIFFERENCE · CARTESIAN PRODUCT|수학의 집합 이론에서 온 연산|

## 일반 집합 연산자 → SQL 매핑

|집합 연산자|의미|SQL|
|---|---|---|
|UNION (합집합)|두 테이블의 모든 행|`UNION` / `UNION ALL`|
|INTERSECTION (교집합)|두 테이블에 공통으로 있는 행|`INTERSECT`|
|DIFFERENCE (차집합)|한쪽에만 있는 행|`MINUS` (Oracle) / `EXCEPT` (PostgreSQL)|
|CARTESIAN PRODUCT (곱집합)|두 테이블의 모든 조합|`CROSS JOIN`|
