# SQL 정규표현식 :복잡한 문자열 패턴 찾고 발라내기

## Concept Summary

정규표현식(Regular Expression, Regex)은 텍스트 안에서 특정한 '규칙(패턴)'을 가진 문자열을 기호를 통해 찾아내거나, 추출하고, 다른 글자로 바꿔치기하는 아주 강력한 문자열 조작 마법이다.

---
## Why is it needed(왜 필요한가)?

기존 SQL의 `LIKE '%apple%'` 명령어로는 "정확히 숫자 3자리-숫자 4자리-숫자 4자리로 된 전화번호만 찾아줘"라거나 "특수문자가 섞인 유효한 이메일 주소만 찾아줘" 같은 복잡한 조건은 도저히 처리할 수가 없기 때문에, 현미경처럼 정교한 패턴 매칭 도구가 필요하다.

---
## Practical Context(실무맥락)

실무에서는 로그 데이터나 비정형 텍스트를 분석하기 좋게 전처리(Preprocessing)할 때 숨 쉬듯이 쓴다. **SQLD 시험**에서는 오라클의 정규식 함수 3대장(`REGEXP_LIKE`, `REGEXP_SUBSTR`, `REGEXP_REPLACE`)의 문법과, `.`(임의의 문자) 같은 메타 문자의 의미를 정확히 묻는 문제가 출제되므로 각각의 기능을 완벽히 분리해서 알아두어야 한다.

## Code Core Points 1: 정규식 필수 암호(메타 문자) 해독표 

| **기호**                  | **의미**           | **상세 설명 및 예시**                                                 |
| ----------------------- | ---------------- | -------------------------------------------------------------- |
| **`.`**                 | **임의의 문자**       | 줄바꿈을 제외한 아무 글자나 딱 **1글자** (예: `A.C` ➔ ABC, A1C, A@C)           |
| **`^`**                 | **시작**           | 문자열의 **맨 앞**을 의미 (예: `^A` ➔ A로 시작하는 문자열)                       |
| **`$`**                 | **끝**            | 문자열의 **맨 뒤**을 의미 (예: `Z$` ➔ Z로 끝나는 문자열)                        |
| **`[ ]`**               | **문자 집합**        | 괄호 안의 문자 중 **아무거나 1개** (예: `[abc]` ➔ a 또는 b 또는 c)              |
| **`[-]`**               | **범위**           | `[]` 안에서 문자의 **연속된 범위**를 지정 (예: `[0-9]` ➔ 숫자, `[a-z]` ➔ 소문자)   |
| **`[^]`**               | **부정**           | `[]` 안에서 맨 앞에 쓰이며, **뒤에 오는 문자들을 제외**함 (예: `[^0-9]` ➔ 숫자가 아닌 것) |
| **`( )`**               | **그룹**           | 문자들을 하나로 묶음. **추출이나 순서 제어**에 사용 (예: `(abc)+` ➔ abc, abcabc...) |
| **`*`**                 | **0회 이상**        | 앞의 문자가 **0번 이상** 반복 (없어도 됨)                                    |
| **`+`**                 | **1회 이상**        | 앞의 문자가 **1번 이상** 반복 (최소 1개는 있어야 함)                             |
| **`?`**                 | **0회 또는 1회**     | 앞의 문자가 존재하거나 존재하지 않음 (예: `apple?` ➔ appl, apple)               |
| **<code>&#124;</code>** | **OR**           | 기호 왼쪽 혹은 오른쪽 패턴 중 하나 (예: A&#124;B → A 또는 B                     |
| **`\`**                 | **Escape / 단축키** | `\.` 처럼 기호의 마법을 풀거나, `\d`(숫자), `\w`(문자) 처럼 단축키로 사용             |

|`[:digit:]`|숫자|`[0-9]`|
|`[:lower:]`|소문자|`[a-z]`|
|`[:upper:]`|대문자|`[A-Z]`|
|`[:space:]`|공백 문자|`[ \t\r\n\f\v]`|
|`[:blank:]`|빈칸 문자|`[ \t]`|
|`[:punct:]`|구두점 문자|`[!"#$%&'()*+,-./:;<=>?@[\]^_{|
|`[:xdigit:]`|16진수 문자|`[0-9A-Fa-f]`|
|`[:cntrl:]`|제어 문자|`[\x00-\x1F\x7F]`|
|`[:print:]`|출력 가능 문자|`[\x20-\x7E]`|
|`[:graph:]`|출력 가능 문자|`[\x21-\x7E]`|

## 정규식 검색 옵션 & 매칭

|기호 / 옵션|이름|의미|설명 및 예시|
|---|---|---|---|
|`1`|첫 번째 위치|검색 시작 위치|일부 도구에서 **1번째 문자부터 검색 시작**|
|`i`|Ignore Case|대소문자 무시|`abc` = `ABC` = `AbC`|
|`g`|Global|전체 검색|문자열 전체에서 **모든 일치 결과** 검색|
|`m`|Multiline|다중 행|`^`, `$`가 **각 줄 기준**으로 동작|
|`^`|검색 시작 위치|문자열 시작|`^abc` → abc로 시작|
|`$`|검색 끝 위치|문자열 끝|`abc$` → abc로 끝|
|첫 번째 일치|First Match|최초 매칭|기본 동작: **가장 먼저 일치한 패턴 1개**|
|`( )`|Group|그룹|`(ab)` → ab를 하나의 단위로 묶음|
|`\1`|Backreference|첫 번째 그룹 참조|`(ab)\1` → ab**ab**|
|`\2`, `\3`|그룹 참조|n번째 그룹|`(a)(b)\2\1` → ba|
|`?:`|Non-capturing Group|캡처 안 함|`(?:abc)+` → 그룹화만, 추출 ❌|

## 오라클 정규식 함수의 '숨겨진 매개변수' 완벽 해부

오라클의 정규식 함수들은 기본적으로 `(대상, '패턴')`만 써도 동작하지만, 더 정교한 조작을 위해 뒤에 **최대 4개의 옵션(매개변수)** 을 추가로 적을 수 있다.

이때 **순서가 무조건 고정**되어 있으므로, 옵션을 건너뛰고 싶더라도 앞의 자리들은 기본값으로 채워주어야 한다!

**기본 문법**`REGEXP_함수(원본데이터, '패턴', [시작위치], [발생횟수], ['매칭옵션'], [그룹번호])`

| **매개변수 순서** | **이름**     | **의미**           | **상세 설명**                                                                     |
| ----------- | ---------- | ---------------- | ----------------------------------------------------------------------------- |
| **1** (필수)  | **대상 컬럼**  | 무엇을 검색할까?        | 텍스트 원문이나 컬럼명                                                                  |
| **2** (필수)  | **정규식 패턴** | 어떤 패턴을?          | `'\d+'` 같은 정규표현식                                                              |
| **3** (선택)  | **시작 위치**  | **어디서부터** 찾을까?   | `1` 이면 첫 글자부터 검색 (기본값: 1)                                                     |
| **4** (선택)  | **발생 횟수**  | **몇 번째** 일치하는 것? | `2` 이면 패턴과 일치하는 것 중 두 번째 덩어리 (기본값: 1)                                         |
| **5** (선택)  | **매칭 옵션**  | **대소문자** 구분할까?   | `'i'`: 대소문자 무시 (Ignore case)<br><br>`'c'`: 대소문자 구분 (기본값)<br><br>`'m'`: 다중행 모드 |
| **6** (선택)  | **그룹 번호**  | **어느 괄호**를 뽑을까?  | 패턴 내에 `( )`가 여러 개일 때, 추출할 괄호 번호 (기본값: 0 전체)                                   |

### 매개변수 사용 시 주의사항 (함수별 차이점)

- `REGEXP_LIKE` (조건 매칭)
	- 위치나 횟수를 따질 필요 없이 "있냐 없냐"만 보므로 **3번까지만** 쓴다.
	- 문법: `REGEXP_LIKE(컬럼, '패턴', ['매칭옵션'])`
	- 예시: `REGEXP_LIKE(이름, '^a', 'i')` ➔ 대소문자 무시하고 A나 a로 시작하는 사람!

- **`REGEXP_COUNT` (개수 세기)**
    - 횟수를 뱉어내는 함수이므로 '발생 횟수'나 '그룹 번호' 옵션이 필요 없다.
    - 문법: `REGEXP_COUNT(컬럼, '패턴', [시작위치], ['매칭옵션'])`
    - 예시: `REGEXP_COUNT('A-A-a', 'A', 1, 'i')` ➔ 대소문자 무시하고 A가 몇 개인지 세어라! (결과: 3)

-  **`REGEXP_SUBSTR`, `REGEXP_INSTR` (추출 및 위치 찾기)**
    - 이 두 녀석이 **1번부터 6번까지 풀옵션**을 다 쓸 수 있는 끝판왕들이다!
    - 예시: `REGEXP_SUBSTR('text', '패턴', 1, 2, 'i', 1)` ➔ 1번째 글자부터, 2번째로 일치하는 것 중, 대소문자 무시하고, 첫 번째 괄호 추출!

## Code Core Points 2: 정규식 5대장 (Oracle vs PostgreSQL)

### [조건 매칭] "이 패턴을 가진 데이터가 있는 행만 찾아라!"

- **SQLD (Oracle)**: `REGEXP_LIKE(컬럼, '패턴')` 함수를 `WHERE` 절에 쓴다.
- **실무 (PostgreSQL)**: `~` 연산자를 쓴다.

```sql
-- "A로 시작하고, 중간에 아무 글자나 2개 있고, D로 끝나는 단어 찾기" (예: A12D, AxyD)
-- 💡 여기서 '.'은 '아무 문자 1개'를 뜻하므로 '..'은 '아무 문자 2개'다!
SELECT * FROM 단어장 
WHERE REGEXP_LIKE(단어, '^A..D$');  -- Oracle (SQLD)

SELECT * FROM 단어장 
WHERE 단어 ~ '^A..D$';            -- PostgreSQL
```

### [데이터 추출] "긴 텍스트 안에서 이 패턴 부분만 쏙 뽑아내라!"

- **SQLD (Oracle)**: `REGEXP_SUBSTR(컬럼, '패턴')`
- **실무 (PostgreSQL)**: `SUBSTRING(컬럼 FROM '패턴')`

```sql
-- "긴 주소 텍스트 안에서, 숫자 3자리 연속된 우편번호 부분만 쏙 빼와!"
SELECT 
    주소,
    REGEXP_SUBSTR(주소, '[0-9]{3}') AS 추출된_우편번호 -- Oracle (SQLD)
    -- SUBSTRING(주소 FROM '[0-9]{3}')              -- PostgreSQL
FROM 고객정보;
```

### [데이터 치환] "이 패턴을 찾아서, 다른 글자로 바꿔치기해라!"

- **SQLD & 실무 공통**: `REGEXP_REPLACE(컬럼, '찾을패턴', '바꿀문자')`

```sql
-- "주민등록번호 뒷자리(대시 다음 숫자 7개)를 찾아서 *******로 가려라!"
SELECT 
    주민번호,
    REGEXP_REPLACE(주민번호, '-[0-9]{7}', '-*******') AS 마스킹_주민번호
FROM 직원;
```


### [위치 찾기] "이 패턴이 문자열의 몇 번째 글자부터 시작하는지 알려줘!"

- **SQLD (Oracle)**: `REGEXP_INSTR(컬럼, '패턴')` (결과를 **숫자**로 반환!)
- **실무 (PostgreSQL)**: 완벽히 1:1로 매칭되는 내장 함수가 없어서, 보통 정규식 추출 결과의 위치를 찾는 우회 방법(`strpos` 등)을 쓴다. 따라서 이 함수는 **오라클과 SQLD 시험의 핵심 타겟**이다!

```sql
-- "이 텍스트에서 '숫자'가 도대체 몇 번째 글자부터 시작해?"
SELECT 
    원문,
    REGEXP_INSTR('데이터123엔지니어', '[0-9]+') AS 숫자_시작_위치 
    -- 💡 '데'(1) '이'(2) '터'(3) '1'(4) ➔ 숫자가 4번째 글자부터 시작하므로 결과는 4!
FROM DUAL;
```

### [패턴 개수 세기] "이 텍스트 안에 해당 패턴이 총 몇 번 들어가 있어?" 

- **SQLD (Oracle)**: `REGEXP_COUNT(컬럼, '패턴')` (결과를 **숫자**로 반환!)
- **실무 (PostgreSQL)**: 오라클처럼 깔끔하게 딱 떨어지는 1:1 전용 함수가 없어서, 일치하는 덩어리들을 배열로 뽑아낸 뒤 그 배열의 길이를 재는 방식(`array_length`) 등으로 우회해서 쓴다. 즉, 오라클만의 강력한 기능이라 **SQLD 단골 손님**이다!

```sql
-- "이 문장 안에 '연속된 숫자' 덩어리가 도대체 몇 개나 들어있지?"
SELECT 
    원문,
    REGEXP_COUNT('A123-B45-C6789', '\d+') AS 숫자_덩어리_개수 
    -- 💡 '123'(1개), '45'(2개), '6789'(3개) ➔ 숫자 덩어리가 총 3개 있으므로 결과는 3!
FROM DUAL;
```

## (컴퓨터의 해석 흐름: SQLD 기출 응용)

#### 문제: `REGEXP_SUBSTR('데이터베이스 정규표현식', '데.*스')` 의 결과는?

- **`데`** : 일단 '데'라는 글자를 찾는다.
- **`.`** : '데' 바로 뒤에 **아무 글자나 1개** 와야 한다.
- **`*`** : 그런데 그 아무 글자가 **0번 이상 무한히 반복**되어도 좋다! (즉, 글자 수 상관없이 아무 글자나 다 받아주겠다는 뜻)
- **`스`** : 그러다가 '스'라는 글자가 나오면 거기서 멈춘다.
- **결과**: '데'부터 '스'까지 그 사이에 뭐가 있든 다 긁어오므로, **`데이터베이스`**라는 단어가 통째로 추출된다!

## Detailed Analysis (중첩 그룹 번호 매기기)

>"괄호가 겹쳐 있을 땐, 왼쪽에서 오른쪽으로 열리는 괄호`(` 순서대로 번호를 매긴다!"

#### 문제 `REGEXP_SUBSTR('1234567890','(123)(4(56)(78))', 1, 1, 'i', 1)` 의 결과는?

-  **Group 1**: `(123)` ➔ 가장 먼저 열리는 괄호
- **Group 2**: `(4(56)(78))` ➔ 두 번째로 열리는 괄호 (안쪽 괄호를 모두 포함)
- **Group 3**: `(56)` ➔ 세 번째로 열리는 괄호
- **Group 4**: `(78)` ➔ 네 번째로 열리는 괄호

**1, 1, 'i'**: 1번째 글자부터 찾고, 1번째로 일치하는 덩어리에서, 대소문자 무시.
**마지막 인자 `1`**: **"첫 번째 그룹 `(123)`의 내용만 가져와라!"**
**결과**: `123`

##### 만약 마지막 숫자가 바뀐다면? (시험 대비 변형)

- `... , 1, 1, 'i', 2)` ➔ **결과: `45678`** (두 번째 괄호 전체)
- `... , 1, 1, 'i', 3)` ➔ **결과: `56`** (세 번째 괄호)
-  `... , 1, 1, 'i', 4)` ➔ **결과: `78`** (네 번째 괄호)
- `... , 1, 1, 'i', 0)` ➔ **결과: `12345678`** (0은 그룹 상관없이 패턴 전체 매칭 결과)

## Detailed Analysis (SQLD 킬러 함정: 중첩 불가 원칙)

시험에서 `REGEXP_COUNT`를 낼 때 가장 많이 파놓는 함정이 바로 **'중첩(Overlapping) 불가'** 원칙이야.

#### `REGEXP_COUNT('123123123', '123123')` 의 결과는 무엇일까?

- 사람은 "어? 첫 번째 1부터 시작하는 `123123`이 하나 있고, 네 번째 1부터 시작하는 `123123`이 또 있네? 정답 2개!"라고 생각하기 쉬워.
- 하지만 컴퓨터는 맨 앞의 `123123`을 1개로 센 순간, 그 글자들을 꿀꺽 삼켜버려.
- 그럼 남은 텍스트는 뒤에 꼬리처럼 남은 `123`뿐이지? 이건 패턴(6자리)과 일치하지 않으니까 버려져.
- **최종 결과**: **1** (절대 2가 아님!)

## 이건 알자!

**"진짜 마침표(.) 기호 자체를 찾고 싶을 때는 어떻게 하나요?"**

- 정규식에서 `.`은 '임의의 문자 1개'라는 강력한 마법 기호로 쓰이고 있어. 만약 이메일 주소의 `.`처럼 진짜 순수한 마침표 기호 그 자체를 찾고 싶다면, 앞에 **백슬래시(`\`)** 를 붙여서 마법을 풀어줘야 해!
- ❌ `REGEXP_LIKE(이메일, '.com')` ➔ (Acom, 1com 다 찾아버림)
- ⭕️ `REGEXP_LIKE(이메일, '\.com')` ➔ (진짜 .com 만 찾음)


**"`LIKE`랑 `REGEXP_LIKE`는 언제 구분해서 쓰나요?"**
-  조건이 "숫자로 시작하고 중간에 영어가 섞인 사람"처럼 복잡해지면 무조건 `REGEXP_LIKE` 매운맛 도구를 꺼내야 해.
