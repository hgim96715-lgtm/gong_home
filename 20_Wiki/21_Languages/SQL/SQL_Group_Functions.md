# 그룹 함수 (Group Functions): 소계와 총계 내기

## 한줄요약(Concept Summary)

일반적인 `GROUP BY`의 결과에 더하여, 추가적인 중간 소계(Subtotal)와 전체 총계(Grand Total)를 한 번의 쿼리로 자동으로 계산해 주는 다차원 집계 함수이다.

---
## 왜 필요한가(Why is it needed)?

**문제점:**
부서별 합계, 지사별 합계, 전체 총합을 각각 구해서 보려면 `UNION ALL`을 여러 번 덕지덕지 붙여야 하므로 쿼리가 수십 줄로 길어지고 성능도 크게 저하된다.

**해결책:**
그룹 함수를 사용하면 단 한 번의 테이블 스캔(Scan)만으로 원하는 모든 레벨(계층)의 집계 결과를 깔끔하게 뽑아낼 수 있다.

---
## 실무맥락(Practical Context)

대시보드나 경영진 보고용 리포트에서 '연도별/월별 매출 현황'을 보여줄 때, 각 월의 매출뿐만 아니라 '해당 연도의 총매출'과 '전체 누적 총매출'을 하나의 표 안에 모두 표현해야 하는 리포팅 업무에서 필수적으로 사용한다. PostgreSQL과 Oracle 모두 동일한 표준 문법을 지원한다.

---
## Code Core Points (3대 그룹 함수)

모든 그룹 함수는 `GROUP BY` 절 뒤에 괄호와 함께 작성한다. 
컬럼의 개수를 $n$개라고 할 때, 결과가 어떻게 나오는지(경우의 수)를 기억하기

### `ROLLUP` : 지정된 순서대로 계층적 소계 구하기 ($n+1$개)

오른쪽에서 왼쪽 순서로 그룹을 하나씩 벗겨가며(Roll-up) 소계와 총계를 구한다. 
**인자의 순서가 바뀌면 결과도 완전히 달라진다.**

```sql
-- 지역별/상품별 매출, [지역별 소계], [전체 총계]를 구한다.
SELECT 지역, 상품, SUM(가격) AS 총합
FROM 판매
GROUP BY ROLLUP(지역, 상품);

/* 생성되는 집계 그룹 (3가지):
1. (지역, 상품)별 매출
2. (지역)별 매출 소계     <- 상품 컬럼은 NULL로 표시됨
3. () 전체 총계           <- 지역, 상품 모두 NULL로 표시됨
*/
```

| **지역**     | **상품**     | **총합**   | **💡 그룹화 기준 (3가지)**    |
| ---------- | ---------- | -------- | ---------------------- |
| 서울         | A          | 250      | (1) 지역, 상품 그룹화         |
| 서울         | B          | 300      | (1) 지역, 상품 그룹화         |
| **서울**     | **[NULL]** | **550**  | **(2) 지역 그룹화 (서울 소계)** |
| 부산         | A          | 550      | (1) 지역, 상품 그룹화         |
| 부산         | B          | 350      | (1) 지역, 상품 그룹화         |
| **부산**     | **[NULL]** | **900**  | **(2) 지역 그룹화 (부산 소계)** |
| **[NULL]** | **[NULL]** | **1450** | **(3) 전체 그룹화 (총계)**    |


### `CUBE` : 가능한 모든 경우의 수로 교차 집계하기 ($2^n$개)

결합 가능한 모든 그룹에 대한 소계와 총계를 구한다. `ROLLUP`보다 결과 행이 훨씬 많으며, 
**인자의 순서가 바뀌어도 결과는 동일하다.**

```sql
-- 지역별/상품별 매출, [지역별 소계], [상품별 소계], [전체 총계]를 구한다.
SELECT 지역, 상품, SUM(가격) AS 총합
FROM 판매
GROUP BY CUBE(지역, 상품);

/* 생성되는 집계 그룹 (4가지):
1. (지역, 상품)별 매출
2. (지역)별 매출 소계
3. (상품)별 매출 소계     <- ROLLUP에는 없던 결과! (지역 컬럼이 NULL로 표시됨)
4. () 전체 총계
*/
```

| **지역**     | **상품**     | **총합**   | **💡 그룹화 기준 (4가지)**                       |
| ---------- | ---------- | -------- | ----------------------------------------- |
| 서울         | A          | 250      | (1) 지역, 상품 그룹화                            |
| 서울         | B          | 300      | (1) 지역, 상품 그룹화                            |
| **서울**     | **[NULL]** | **550**  | **(2) 지역 그룹화 (서울 소계)**                    |
| 부산         | A          | 550      | (1) 지역, 상품 그룹화                            |
| 부산         | B          | 350      | (1) 지역, 상품 그룹화                            |
| **부산**     | **[NULL]** | **900**  | **(2) 지역 그룹화 (부산 소계)**                    |
| **[NULL]** | **A**      | **800**  | **(3) 상품 그룹화 (A상품 소계) ⬅️ ROLLUP엔 없는 결과!** |
| **[NULL]** | **B**      | **650**  | **(3) 상품 그룹화 (B상품 소계) ⬅️ ROLLUP엔 없는 결과!** |
| **[NULL]** | **[NULL]** | **1450** | **(4) 전체 그룹화 (총계)**                       |

### `GROUPING SETS` : 내가 원하는 조합만 콕 집어서 구하기

총계나 소계를 강제로 전부 계산하는 것이 아니라, 괄호 안에 지정한 특정 그룹의 집계만 평면적으로 구한다.

```sql
-- 지역별 합계와 상품별 합계 딱 2가지만 위아래로 이어 붙여서 구한다.
SELECT 지역, 상품, SUM(가격) AS 총합
FROM 판매
GROUP BY GROUPING SETS(지역, 상품);

/* 생성되는 집계 그룹 (2가지):
1. (지역)별 매출 합계     <- 부서는 NULL
2. (상품)별 매출 합계     <- 지사는 NULL
※ 주의: (지사, 부서) 조합이나 () 전체 총계는 내가 명시하지 않았으므로 절대 출력되지 않는다!
*/
```

| **지역**     | **상품**     | **총합** | **💡 그룹화 기준 (딱 2가지)** |
| ---------- | ---------- | ------ | --------------------- |
| 서울         | **[NULL]** | 550    | (1) 지역 그룹화 (서울 합계)    |
| 부산         | **[NULL]** | 900    | (1) 지역 그룹화 (부산 합계)    |
| **[NULL]** | A          | 800    | (2) 상품 그룹화 (A상품 합계)   |
| **[NULL]** | B          | 650    | (2) 상품 그룹화 (B상품 합계)   |

#### `GROUPING SETS` 안에 `ROLLUP` / `CUBE` 혼합하기

`GROUPING SETS`의 인자값으로 단순 컬럼명뿐만 아니라, `ROLLUP`이나 `CUBE` 함수를 직접 집어넣어 더욱 복잡한 다차원 집계를 한 번에 뽑아낼 수 있다.

```sql
-- 지역별 합계와 함께, 상품에 대한 ROLLUP(상품별 합계 + 전체 총계)을 동시에 구한다.
SELECT 지역, 상품, SUM(가격) AS 총합
FROM 판매
GROUP BY GROUPING SETS(지역, ROLLUP(상품));

/* 생성되는 집계 그룹 (총 3가지):
1. '지역' 인자가 만드는 -> (지역)별 합계
2. 'ROLLUP(상품)' 인자가 만드는 -> (상품)별 합계
3. 'ROLLUP(상품)' 인자가 만드는 -> () 전체 총계
*/
```

|**지역**|**상품**|**총합**|**💡 혼합 시 그룹화 기준 (3가지)**|
|---|---|---|---|
|서울|**[NULL]**|550|(1) 지역 그룹화 (서울 합계)|
|부산|**[NULL]**|900|(1) 지역 그룹화 (부산 합계)|
|**[NULL]**|A|800|(2) 상품 그룹화 (A상품 합계)|
|**[NULL]**|B|650|(2) 상품 그룹화 (B상품 합계)|
|**[NULL]**|**[NULL]**|**1450**|**(3) 전체 총계 (ROLLUP이 만듦)**|

## Code Core Points (짝꿍 함수: `GROUPING`)

소계나 총계로 인해 생성된 행의 빈칸은 모두 `NULL`로 표시된다. 
보고서에 덩그러니 `NULL`이라고 뜨면 보기 안 좋으므로, **이 빈칸을 찾아내어 '총계'나 '합계' 같은 원하는 텍스트로 바꿔 출력하기 위해** `GROUPING` 함수를 사용한다. 
- **작동 원리:** `GROUPING(컬럼명)`은 해당 빈칸이 집계를 위해 시스템이 만든 **가짜 빈칸(총계/소계용)이면 숫자 `1`** 을 반환하고, 원래 있던 **진짜 데이터면 `0`** 을 반환한다.
- **활용법:** `CASE WHEN` 조건문과 결합하면 빈칸이 생기는 위치를 정확히 타겟팅해서 원하는 글자를 덮어씌울 수 있다.

```sql
-- GROUPING 결과가 1(가짜 빈칸)이면 '총합계'나 '소계'라는 글자를 출력하고, 0이면 원래 진짜 데이터를 출력한다.
SELECT 
    CASE WHEN GROUPING(지역) = 1 THEN '총합계' ELSE 지역 END AS 지역,
    CASE WHEN GROUPING(상품) = 1 THEN '소계' ELSE 상품 END AS 상품, 
    SUM(가격) AS 총합
FROM 판매
GROUP BY ROLLUP(지역, 상품);

/* 결과 예시: 보기 흉했던 [NULL]이 예쁜 이름표로 바뀐다!
지역    | 상품  | 총합
----------------------
서울    | A     | 250
서울    | B     | 300
서울    | 소계  | 550   <-- (지역 소계) 상품 컬럼의 GROUPING 값이 1이라서 '소계'가 찍힘
부산    | A     | 550
부산    | B     | 350
부산    | 소계  | 900   <-- (지역 소계) 상품 컬럼의 GROUPING 값이 1이라서 '소계'가 찍힘
총합계  | 소계  | 1450  <-- (전체 총계) 둘 다 GROUPING 값이 1이라서 텍스트가 치환됨
*/
```

## 초보자의 흔한 착각

#### **"`ROLLUP(A, B)`와 `ROLLUP(B, A)`는 같은 결과가 나온다?"**

- **절대 아니다.** `ROLLUP`은 방향성(계층)이 매우 중요하다. `ROLLUP(지사, 부서)`는 '지사별 총합'을 구해주지만, `ROLLUP(부서, 지사)`는 '부서별 총합'을 구해준다.
- 반면 `CUBE`와 `GROUPING SETS`는 순서를 바꿔도 결과 전체 행의 집합은 완전히 동일하다.

#### **"`NULL`이 보이면 무조건 집계 함수가 만든 가짜 빈칸이다?"**

- **아니다.** 원본 데이터베이스 자체에 아직 부서를 배정받지 못해 부서 컬럼이 `NULL`인 신입사원의 매출 데이터가 있을 수 있다. 이 '진짜 NULL'과 집계 함수가 만든 '가짜 NULL'을 완벽하게 구별해 내는 유일한 방법이 바로 위에서 배운 `GROUPING` 함수를 사용하는 것이다.
