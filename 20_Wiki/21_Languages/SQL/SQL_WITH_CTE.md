# SQL_WITH_CTE

## 개념 한 줄 요약

파이썬 스크립트 맨 위에 변수를 선언하듯, 복잡한 SQL 쿼리 상단에 **'이름표를 붙인 1회용 임시 테이블(CTE)'** 을 만들어두고 메인 쿼리에서 깔끔하게 꺼내 쓰는 문법입니다.

---

## (왜 필요한가)Why is it needed

- **문제점:** 괄호 안에 괄호가 계속 들어가는 서브쿼리(인라인 뷰)를 남발하면, 코드가 마치 영화 '인셉션'처럼 깊어져서 안쪽부터 바깥쪽으로 거꾸로 읽어야 하는 고통이 따릅니다.
- **💡 해결책:** `WITH` 구문을 쓰면 데이터의 가공 흐름을 **위에서 아래로(Top-Down)** 자연스럽게 읽을 수 있도록 쿼리를 **모듈화(블록 조립)** 할 수 있습니다. 가독성과 유지보수성이 압도적으로 좋아집니다.

---

## Practical Context(실무맥락)

 **언제 쓰나요?**
- 데이터 엔지니어가 수백, 수천 줄짜리 복잡한 데이터 마트(Data Mart) 생성 쿼리를 짤 때 숨 쉬듯 씁니다. 실무 분석가들이 가장 사랑하는 SQL 문법 1위이기도 합니다.

 **어떻게 쓰나요?** 
 - "1. 부서별 평균 급여 구하기 -> 2. 우수 사원 필터링하기 -> 3. 두 개 조인해서 최종 결과 뽑기" 처럼, 사람의 사고방식(순서) 그대로 코드를 위에서부터 아래로 써 내려갈 때 씁니다.

---
## Code Core Points

###  Before: 서브쿼리 지옥 (가독성 최악)

가장 안쪽 괄호부터 읽어야 해서 "도대체 뭘 하려는 쿼리지?" 하고 한참을 노려봐야 합니다.

```sql
SELECT e.직원명, d.부서명, t.부서평균급여
FROM 직원 e
JOIN 부서 d ON e.부서코드 = d.부서코드
JOIN (
    -- 🚨 여기서부터 읽어야 함 (시선이 위아래로 왔다 갔다 함)
    SELECT 부서코드, AVG(급여) AS 부서평균급여 
    FROM 직원 
    GROUP BY 부서코드
) t ON e.부서코드 = t.부서코드
WHERE e.급여 > t.부서평균급여;
```

### After: WITH 구문 (파이써닉한 모듈화)

마치 파이썬에서 `dept_avg = 계산식` 이라고 변수에 담아두고 나중에 꺼내 쓰는 것과 완벽하게 똑같습니다.

```sql
-- 1. 임시 테이블(CTE)을 명확한 이름으로 선언해 둡니다.
WITH dept_avg AS (
    SELECT 부서코드, AVG(급여) AS 부서평균급여 
    FROM 직원 
    GROUP BY 부서코드
)

-- 2. 위에서 만든 임시 테이블을 메인 쿼리에서 깔끔하게 가져다 씁니다.
SELECT e.직원명, d.부서명, t.부서평균급여
FROM 직원 e
JOIN 부서 d ON e.부서코드 = d.부서코드
JOIN dept_avg t ON e.부서코드 = t.부서코드 -- 💡 괄호 없이 깔끔하게 조인!
WHERE e.급여 > t.부서평균급여;
```

## Detailed Analysis (실무 심화)

### 여러 개의 CTE를 콤마(,)로 연결하기

임시 테이블이 여러 개 필요할 때는 `WITH`를 여러 번 쓰는 게 아니라, **처음에 딱 한 번만 쓰고 콤마(,)로 계속 이어나갑니다.**

```sql
WITH 
top_departments AS ( SELECT 부서코드 FROM 부서 WHERE 매출액 > 10000 ),
-- 💡 두 번째 CTE부터는 WITH를 생략하고 콤마로 연결!
high_salary_emp AS ( SELECT 사원번호, 부서코드 FROM 직원 WHERE 급여 > 8000 )
SELECT * FROM high_salary_emp;
```

### 앞서 만든 CTE를 재활용하기 (Chaining)

방금 위에서 만든 CTE 블록을, 바로 다음 CTE 블록 안에서 재활용할 수도 있습니다! (이것이 인라인 뷰로는 불가능한 `WITH`문만의 초강력 기능입니다.)

```sql
WITH 
step_1 AS ( SELECT * FROM raw_data WHERE date = '2026-02-25' ),
-- 💡 step_1의 결과를 받아서 step_2를 가공 (체인 연결)
step_2 AS ( SELECT id, COUNT(*) AS cnt FROM step_1 GROUP BY id ) 
SELECT * FROM step_2;
```

### CTE 안에서 계산된 컬럼은 반드시 이름을 지어줘야 합니다!

기존 테이블에 있던 일반 컬럼(`부서코드`)은 상관없지만, `COUNT(*)`, `AVG(급여)` 같은 집계 함수나 `급여 * 1.1` 처럼 **가공된 컬럼은 반드시 별칭(Alias)을 명시해야 메인 쿼리에서 꺼내 쓸 수 있습니다.** 이름을 명시하는 방법은 2가지가 있습니다.

#### 방법 A: SELECT 문 안에서 `AS` 쓰기 (가장 흔함)

```sql
WITH dept_avg AS (
    SELECT 부서코드, AVG(급여) AS 부서평균급여 -- 👈 여기서 컬럼명 지정
    FROM 직원 GROUP BY 부서코드
)
```

#### 방법 B: CTE 이름 옆에 괄호로 명시하기 (코드 포맷팅이 깔끔해짐)

```sql
-- 👈 임시 테이블 이름 옆에 (컬럼1, 컬럼2) 형태로 미리 컬럼명을 선언해버림!
WITH dept_avg (부서코드, 부서평균급여) AS (
    SELECT 부서코드, AVG(급여)
    FROM 직원 GROUP BY 부서코드
)
```

## 내가 겪었던 실수!!

#### "원본 테이블에 분명히 있는 컬럼인데, 메인 쿼리에서 없다고 에러가 나요!" 🚨

- CTE는 원본 테이블의 바로가기가 아니라 **"완전히 새롭게 깎아낸 미니 테이블"** 입니다.
- `WITH my_cte AS (SELECT 사원번호, 급여 FROM 직원)` 이라고 적었다면, 이 `my_cte` 안에는 딱 2개의 기둥(컬럼)만 존재합니다.
- 원본 `직원` 테이블에 `직원명`이 있더라도, CTE 안에서 명시적으로 `SELECT` 해주지 않으면 바깥쪽 메인 쿼리에서는 절대 `직원명`을 꺼내 쓸 수 없습니다. **"필요한 컬럼은 CTE의 SELECT 절에 모조리 적어줘야 한다"** 는 것을 잊지 마세요!

```sql
-- ❌ 실패하는 코드 (직원명을 CTE에서 꺼내놓지 않음)
WITH my_cte AS (
    SELECT 사원번호, 급여  -- 🚨 기둥이 딱 2개뿐인 미니 테이블 생성!
    FROM 직원
)
SELECT 사원번호, 직원명    -- 💥 에러 발생! (my_cte 안에는 직원명 컬럼이 아예 존재하지 않음)
FROM my_cte;

-- ⭕ 성공하는 코드 (바깥에서 쓸 건 안에서 다 꺼내줘야 함)
WITH my_cte AS (
    SELECT 사원번호, 급여, 직원명 -- ✅ 밖에서 쓸 예정인 컬럼은 여기서 미리 다 SELECT 해둠!
    FROM 직원
)
SELECT 사원번호, 직원명
FROM my_cte;
```


#### "CTE 안에서 `COUNT(*)`를 썼는데 컬럼을 못 찾는다고 에러가 나요!"

- CTE 괄호 안에서 `SELECT COUNT(*)` 까지만 쓰고 `AS 카운트` 처럼 별칭을 안 주면, 데이터베이스는 그 컬럼의 이름을 알 수 없거나 지멋대로(`count` 혹은 `?column?`) 지어버립니다.
- 결국 메인 쿼리에서 `SELECT 카운트`를 하려고 해도 에러가 터집니다. **무조건 `AS` 로 컬럼명을 명시하는 습관**을 들이세요!
