## 개념 한 줄 요약

**"프로그램이 에러 때문에 '뻑' 하고 죽는 걸 막고, 우아하게 대처(수습)하거나 안전하게 종료시키는 안전장치(Safety Net)"**

---
## 왜 필요한가? (Why) 

코드를 짜다 보면 내 잘못이 아니어도 에러가 납니다.

- **사용자:** 숫자를 입력하라니까 "한글"을 입력함.
- **네트워크:** 와이파이가 갑자기 끊김.
- **파일:** 읽으려는 파일이 삭제됨.

이때 예외 처리가 없으면 프로그램은 **그 즉시 멈추고 꺼집니다(Crash).** 
예외 처리를 하면 **"에러가 났지만, 무시하고 다음 거 진행해!"** 또는 **"일단 저장하고 꺼져!"** 라고 명령할 수 있습니다.

## 기본 문법 (`try` ~ `except`)

가장 기본 구조입니다. **"시도해보고(try), 안 되면 이거 해(except)."** 

```python
try:
    # 1. 에러가 날 수도 있는 위험한 코드
    number = int(input("숫자를 입력하세요: "))
    print(10 / number)

except ValueError:
    # 2. 숫자가 아닌 문자를 넣었을 때 실행됨
    print("❌ 숫자만 입력해야 합니다!")

except ZeroDivisionError:
    # 3. 0을 입력해서 나눌 수 없을 때 실행됨
    print("❌ 0으로는 나눌 수 없습니다!")

except Exception as e:
    # 4. 위에서 안 걸린 나머지 모든 에러 (치트키)
    # 'e'에는 에러 메시지가 담겨 있습니다.
    print(f"❌ 알 수 없는 에러가 났어요: {e}")
```

## 조건 실행 (`finally`) 

**성공하든 실패하든 무조건 실행**해야 하는 코드를 넣습니다. 
데이터 엔지니어링에서 **"DB 연결 종료(`close`)"** 나 **"파일 닫기"** 를 할 때 **필수**입니다.

```python
try:
    print("1. 냉장고 문을 연다.")
    print("2. 코끼리를 넣는다.")
    # 에러 발생! 코끼리가 너무 커서 안 들어감!
    result = 10 / 0 

except Exception:
    print("🚨 에러: 코끼리 넣기 실패!")

finally:
    # 에러가 나도 냉장고 문은 닫아야 음식이 안 상하겠죠?
    print("3. (무조건) 냉장고 문을 닫는다.")
```

**결과:** 에러가 났지만 "3. 냉장고 문을 닫는다"는 무조건 출력됩니다.

## 일부러 에러 내기 (`raise`) 🚨

내가 정한 규칙을 어겼을 때, **강제로 에러를 발생**시킬 때 씁니다.

```python
def register_age(age):
    if age < 0:
        # 나이가 음수일 수는 없음! 강제로 에러 발생시킴
        raise ValueError("나이는 0보다 작을 수 없습니다.")
    
    print(f"{age}세 등록 완료.")

try:
    register_age(-5)
except ValueError as e:
    print(e) # "나이는 0보다 작을 수 없습니다." 출력
```

## 파이썬 에러(Exception) 도감

### 문법/오타 관련 (실행조차 안 됨)

#### ① `SyntaxError` (문법 에러)

- **언제?** : 파이썬 문법 규칙을 어겼을 때. (가장 초보적인 실수)
- **상황:** 괄호를 안 닫았거나, 땡땡(`:`)을 빼먹었을 때.

#### ② `IndentationError` (들여쓰기 에러)

- **언제?** : 띄어쓰기(Tab/Space)를 잘못 맞췄을 때.
- **상황:** `if`문이나 `def` 밑에는 무조건 들여쓰기를 해야 하는데 안 했거나, 줄이 삐뚤빼뚤할 때.

#### ③ `NameError` (이름 에러)

- **언제?** : 없는 변수를 불렀을 때. (99% 오타)
- **상황:** `user_id`라고 만들어놓고 `userid`라고 부를 때.

### 데이터 값/타입 관련 (실행 도중 멈춤)

#### ④ `TypeError` (타입 에러) 

- **언제?** : 서로 계산할 수 없는 애들끼리 붙여놨을 때.
- **상황:** 숫자랑 글자를 더하려고 할 때.

#### ⑤ `ValueError` (값 에러) ️

- **언제?** : 타입은 맞는데, 내용물이 이상할 때. 
- **상황:** 글자를 숫자로 바꾸려고 했는데, 숫자가 아닌 게 들어있을 때.

### 리스트/딕셔너리 관련 (데이터 꺼낼 때)

#### ⑥ `IndexError` (인덱스 에러)

- **언제?** : 리스트의 범위를 넘어갔을 때.
- **상황:** 데이터는 3개밖에 없는데 10번째 놈을 내놓으라고 할 때.

#### ⑦ `KeyError` (키 에러) 

- **언제?** : 딕셔너리에 없는 열쇠(Key)를 찾을 때.
- **상황:** `{'name': 'Gong'}` 인데 `address`를 내놓으라고 할 때.

### 파일/DB/시스템 관련 (DE 필수)

#### ⑧ `FileNotFoundError` (파일 없음)

- **언제?** : 읽으려는 파일이 그 위치에 없을 때.
- **상황:** 파일명 오타나 경로(`.env` 위치 등)가 틀렸을 때.

#### ⑨ `OperationalError` (DB 접속 에러)

- **언제?** : DB가 꺼져있거나, 비밀번호가 틀렸을 때. (`psycopg2` 전용)
- **상황:**  "Connection failed" 같은 상황.

#### ⑩ `KeyboardInterrupt` (사용자 중지)

- **언제?** : 터미널에서 `Ctrl + C`를 눌러서 강제로 껐을 때.
- **상황:** 무한루프(Kafka Consumer)를 내가 직접 끌 때 발생. (이건 에러라기보다 신호입니다.)

## 초보자가 자주 하는 실수 (Bad Practice) & 팁

###  `pass` 남발하지 말기

```python
try:
    # ...
except Exception:
    pass  # 최악! 😡 에러가 났는데 아무 말 없이 넘어가면 나중에 원인을 못 찾음.
```
