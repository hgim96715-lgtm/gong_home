##  개념: 중복을 허용하지 않는 주머니 

`set`(집합)은 두 가지 큰 특징이 있습니다.

1.  **중복이 없다:** 같은 값을 백 번 넣어도 딱 하나만 남습니다.
2.  **순서가 없다:** 인덱싱(`[0]`)이 불가능합니다.

---
## 핵심 활용: 로직 단순화 (Logic Simplification) 

사용자님의 사례처럼 복잡한 비교 연산자(`!=`, `{text}==`) 범벅을 **집합의 크기(`len`)** 하나로 깔끔하게 해결할 수 있습니다.

### ❌ Before: 하드 코딩 (The Hard Way)

변수끼리 일일이 비교하느라 조건문이 길어지고, 오타가 나기 쉽습니다.

```python
def solution(a, b, c):
    answer = 0
    # 3개가 모두 다를 때
    if a != b and a != c and b != c:
        answer = a + b + c
    # 3개가 모두 같을 때
    elif a == b and b == c and a == c:
        answer = (a + b + c) * (a**2 + b**2 + c**2) * (a**3 + b**3 + c**3)
    # 2개만 같을 때 (조건이 가장 복잡함!)
    elif a != b == c or a == b != c or a == c != b:
        answer = (a + b + c) * (a**2 + b**2 + c**2)
    return answer
```


### After: Set 활용 (The Smart Way)

리스트를 집합으로 바꾸면 **중복이 사라진다**는 점을 이용합니다.

- **3개** 남음 (`{1, 2, 3}`) → 모두 다름
- **2개** 남음 (`{1, 2}`) → 두 개는 같음
- **1개** 남음 (`{1}`) → 모두 같음

```python
def solution(a, b, c):
    # 1. 리스트로 묶고 계산 준비
    nums = [a, b, c]
    s = sum(nums)
    
    # 2. 집합의 크기(len)로 상황 판단
    unique_count = len(set(nums))
    
    if unique_count == 3:      # 모두 다름
        return s
    elif unique_count == 2:    # 두 개만 같음
        return s * sum(x**2 for x in nums)
    else:                      # 모두 같음 (unique_count == 1)
        return s * sum(x**2 for x in nums) * sum(x**3 for x in nums)
```

**통찰:** 변수가 `a, b, c, d` 4개로 늘어난다면?

- **Before:** `if`문이 수십 줄로 늘어납니다.
- **After:** `len(set(nums))` 로직은 그대로 유지됩니다. (확장성 최고!)

## 데이터 엔지니어링 필수 연산 (집합 연산) 

데이터 비교 분석(Diff)을 할 때 필수적입니다.

```python
A = {1, 2, 3}
B = {2, 3, 4}

# 교집합 (공통된 데이터 찾기)
print(A & B)  # {2, 3}

# 차집합 (A에는 있는데 B에는 없는 것 = 누락 데이터 찾기)
print(A - B)  # {1}

# 합집합 (모두 합치기, 중복 제거됨)
print(A | B)  # {1, 2, 3, 4}
```

## 집합(Set)에서 값 꺼내기 (Access Items)

`set`은 인덱싱(`[0]`)이 안 되기 때문에, 데이터를 꺼내는 방법이 리스트와 완전히 다릅니다.

### "딱 하나만" 꺼내고 싶을 때 (`next`)

사용자님이 말씀하신 **"이터레이터(Iterator)"** 방식을 씁니다. 
집합은 순서가 없어서 바로 못 꺼내니, **"줄을 세우는 기계(`iter`)"** 에 통과시킨 뒤 **"맨 앞 녀석(`next`)"** 을 부르는 원리입니다.

> **⚠️ 주의:** 매번 실행할 때마다 **어떤 값이 튀어나올지 모릅니다.** (랜덤)

```python
s = {10, 20, 30}

# ❌ 실패: TypeError (집합은 인덱싱 불가)
# print(s[0]) 

# ✅ 성공: 이터레이터로 변환 후 첫 번째 값 호출
first_val = next(iter(s))
print(first_val)  # 10, 20, 30 중 하나가 나옴 (보장 못 함)
```

### "두 개"를 각각 변수에 담고 싶을 때 (Unpacking)

집합 안에 데이터 개수가 확실하다면(예: 딱 2개), **변수 할당(Unpacking)** 을 쓸 수 있습니다.

```python
s = {1, 2}

# 변수 개수 = 데이터 개수 (딱 맞아야 함)
a, b = s

print(a)  # 1 또는 2
print(b)  # 2 또는 1 (누가 어디로 갈지 모름)
```

### "안전하게" 순서대로 꺼내고 싶을 때 (List 변환) 

데이터 엔지니어링에서는 **가장 많이 쓰는 방법**입니다. 
고민하지 말고 **리스트로 바꾼(`list()`) 다음** 인덱싱하세요.

```python
s = {10, 20, 30}

# "야, 일단 줄부터 서봐." (리스트 변환)
safe_list = list(s)

# 이제 마음대로 꺼내면 됨
print(safe_list[0]) 
print(safe_list[1])
```


### 하나씩 뽑으면서 삭제하기 (`pop`)

로또 추첨처럼 **"통 안에 있는 공을 하나씩 꺼내서 없애버릴 때"** 씁니다.

```python
s = {1, 2, 3}

picked = s.pop()  # 하나 툭 튀어나옴 (그리고 s에서는 사라짐!)
print(s)          # 남은 2개만 보임
```

## 요약 (Cheat Sheet)

|**상황**|**코드**|**설명**|
|---|---|---|
|**중복 제거**|`list(set(data))`|리스트를 깨끗하게 청소|
|**다름/같음 판단**|`len(set(data))`|조건문 지옥 탈출 열쇠|
|**공통 값 찾기**|`set_a & set_b`|교집합 연산|


| **상황**     | **코드**          | **특징**              |
| ---------- | --------------- | ------------------- |
| **인덱싱 시도** | `s[0]`          | **절대 불가 (에러 발생)**   |
| **하나만 조회** | `next(iter(s))` | 원본 유지, 랜덤 값 추출      |
| **변수 할당**  | `a, b = s`      | 개수 딱 맞아야 함 (언패킹)    |
| **안전한 접근** | `list(s)[0]`    | **가장 추천 (리스트로 변환)** |
| **꺼내고 삭제** | `s.pop()`       | 원본 데이터 사라짐 (파괴적)    |
