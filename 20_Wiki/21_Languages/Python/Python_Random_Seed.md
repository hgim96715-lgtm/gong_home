##  개념 한 줄 요약

**"컴퓨터의 '가짜 랜덤(Pseudo-random)'을 통제해서, 언제 실행해도 똑같은 '랜덤 결과'가 나오게 만드는 고정핀."**

* **Random:** 무작위로 숫자를 뽑음.
* **Seed (씨앗):** 무작위가 시작되는 **출발점**. 출발점이 같으면 도착점(결과)도 항상 같음.

---
## 왜 필요한가? (Why)

**문제점:**
* 데이터 1,000개 중 랜덤으로 10개를 뽑아서 테스트를 성공함.
* "팀장님, 성공했습니다!" 하고 보여주려고 **다시 실행했는데**, 이번엔 다른 10개가 뽑혀서 **에러가 남**. (대참사)

**해결책:**
* **"우리 이번엔 '42번' 시나리오로 랜덤을 돌리자."** 라고 약속함.
* 코드를 100번 다시 돌려도 **똑같은 데이터 10개**가 뽑힘. -> **재현 가능성(Reproducibility)** 확보!

---
##  완벽한 비유: 마인크래프트 맵 생성 

* 게임을 할 때 **'맵 시드(Seed)'** 에 `1234`를 넣고 게임을 시작하면?
    * 서울에 사는 나도, 부산에 사는 친구도 **똑같은 위치에 산이 있고, 똑같은 위치에 강이 흐르는** 맵이 생성됩니다.
* 컴퓨터의 랜덤은 주사위를 굴리는 게 아니라, **미리 정해진 수억 개의 '난수표 책'** 을 읽는 것입니다.
* **Seed**는 그 책의 **'몇 페이지를 펼칠지'** 정하는 번호입니다.

---
## Code Core Points 

파이썬 기본 `random` 모듈로 실험해봅시다.

### ① Seed가 없을 때 (실행할 때마다 다름)

```python
import random

# seed 설정 없음
print(random.randint(1, 100)) 
# 1회차 실행: 45
# 2회차 실행: 12 (값이 바뀜!)
```

### ② Seed를 고정했을 때 (항상 같음)

```python
import random

# "난수표 42페이지를 펴라!"
random.seed(42)

print(random.randint(1, 100))
# 1회차 실행: 82
# 2회차 실행: 82
# 100회차 실행: 82 (절대 안 바뀜)
```

## [보충] `randint` vs `range` (범위의 함정) 

`{python}random.randint(a, b)`를 쓸 때 초보자가 가장 많이 틀리는 것이 바로 **'끝 숫자의 포함 여부'** 입니다.

### ① `random.randint(1, 3)`

* **의미:** 1부터 3까지의 정수 중 하나.
* **범위:** `1 <= N <= 3` (**3 포함!**)
* **결과:** 1, 2, 3 중 하나가 나옴.

### ② `range(1, 3)
`
* **의미:** 1부터 3 직전까지의 숫자 생성.
* **범위:** `1 <= N < 3` (**3 미포함!**)
* **결과:** 1, 2 만 나옴.

> **💡 왜 헷갈리게 만들었나요?**
> * `range`는 리스트의 인덱스(`0`부터 시작)를 다루기 위해 만들어져서 "마지막은 빼는 게" 국룰입니다. 
> * (데이터 개수 맞추기 편함)
> * `randint`는 "주사위 1~6"처럼 사람이 생각하는 직관적인 범위가 필요해서 "마지막도 포함"시킵니다.

### ③ 실전 예제 (로또 번호 뽑기)

```python
import random

# 1부터 45까지 (45 포함)
lotto_num = random.randint(1, 45) 

# (X) range처럼 생각해서 46이라고 쓰면 46이 나올 수도 있음!
# wrong_num = random.randint(1, 46) -> 46 나오면 꽝!
```

## 실수와 리스트 다루기 (`uniform`, `choice`) ⭐️

`randint`가 정수(Integer) 전용이라면, 이 둘은 **소수점**과 **목록**을 담당합니다. 
데이터 생성할 때 필수입니다.

### ① `random.uniform(a, b)` : 실수 뽑기

**"a와 b 사이의 아무 실수(Float)나 하나 뽑아줘."**

- **용도:** 센서 온도(36.5도), 주식 가격, 확률 등 **소수점이 필요한 데이터**를 만들 때 씁니다.
- **특징:** `randint`와 달리 끝 숫자가 포함될 수도 있고 안 될 수도 있습니다(수학적 확률). 보통은 **범위 내의 실수**라고 생각하면 됩니다.

```python
import random

# 10.0 이상 40.0 이하의 실수 생성
temperature = random.uniform(10.0, 40.0)

print(temperature) 
# 결과 예시: 23.4521... (소수점이 길게 나옴)

# 💡 꿀팁: 보기 좋게 소수점 자르기 (round)
print(round(temperature, 1)) 
# 결과: 23.5
```

### ② `random.choice(seq)` : 제비뽑기

**"이 리스트(목록) 안에서 하나만 골라줘."**

- **용도:** 가위바위보, 점심 메뉴 고르기, **범주형 데이터(Category)** 생성할 때 가장 많이 씁니다.
- **주의:** 빈 리스트(`[]`)를 넣으면 에러(`IndexError`)가 납니다.

```python
import random

states = ["ca", "ny", "az", "tx"]

# 리스트 중 하나를 무작위로 선택
my_state = random.choice(states)

print(my_state)
# 결과: "ny" (실행할 때마다 바뀜)
```

## [비교] `randint` vs `uniform` vs `choice` 한 눈에 보기

| **함수**                   | **뽑는 대상**    | **예시 코드**  | **결과 타입**  |
| ------------------------ | ------------ | ---------- | ---------- |
| **`randint(1, 10)`**     | 정수 (Integer) | 주사위 굴리기    | `5`        |
| **`uniform(1.0, 10.0)`** | 실수 (Float)   | 온도, 키, 몸무게 | `5.239...` |
| **`choice(['A', 'B'])`** | 리스트 요소       | 메뉴 고르기     | `'A'`      |
