## 개념 한 줄 요약

**"순서가 뒤죽박죽인 데이터를, 내가 원하는 '기준(Key)'을 정해서 줄 세우는 기술."**

* **상황:** `{'수학': 50점, '영어': 90점}` 데이터를 점수 높은 순으로 랭킹을 매기고 싶어!
* **문제:** 딕셔너리는 순서가 없음.
* **해결:** 리스트로 바꾼 뒤(`items`), **"점수를 기준으로 봐라!"** 라고 명령(`key`)하고 정렬(`sorted`)함.

---
## `sorted()` 함수 완전 해부 

파이썬의 만능 정렬 함수인 `sorted`는 3가지 재료를 받습니다.

```python
sorted(데이터, key=기준함수, reverse=True/False)
```

|**옵션(Parameter)**|**설명**|**비유**|
|---|---|---|
|**① 데이터 (Iterable)**|정렬할 대상 (리스트, 튜플 등)|줄 세울 학생들|
|**② `key` (정렬 기준)**|**"무엇을 보고 정렬할까요?"** (생략 가능)|"키 순서? 성적 순서? 출석 번호 순서?"|
|**③ `reverse` (방향)**|오름차순(기본값) vs 내림차순|"작은 사람부터? 큰 사람부터?"|

---
## 핵심은 `key` 옵션이다! 

초보자가 제일 헷갈려하는 게 바로 이 `key`입니다.

### Q. `key`가 왜 필요한가요?

컴퓨터는 융통성이 없습니다. `('A', 30)` 이라는 덩어리를 주면, **"A(문자)로 정렬해요? 30(숫자)으로 정렬해요?"** 라고 묻지 않고, 무조건 **맨 앞에 있는 'A'만 보고** 정렬해버립니다.

이때 **"야, 앞에 있는 거 보지 말고, 뒤에 있는 숫자(30)를 기준으로 삼아!"** 라고 콕 집어주는 역할이 `key`입니다.


### Q. `key=lambda...` 해석하기

```python
key = lambda x: x[1]
```

- **의미:** "학생 한 명(`x`)이 들어오면, 그 학생의 두 번째 정보(`x[1]`)를 기준으로 삼으세요."
- **`lambda`:** 거창한 게 아니라, 그냥 **"잠깐 쓸 함수"** 입니다.
    - `def get_score(x): return x[1]` 이렇게 길게 쓰기 귀찮아서 한 줄로 줄인 것입니다.

## 실전 꿀팁: `key=lambda`를 안 쓰고 정렬하는 마술 (튜플의 비밀)

"파이썬은 튜플을 만나면, 무조건 맨 앞(0번째) 데이터부터 차례대로 비교하는 '본능'이 있습니다."

**Q. 파이썬은 튜플을 어떻게 정렬할까요? (사전식 정렬)**

1. "일단 튜플의 **첫 번째 값([0])** 만 보고 줄을 세우자!" ➡️ `1`이 `3`보다 작으니 앞으로 보냄.
2. "어라? `1`이 두 명이라서 **동점(Tie)** 이네?"
3. "동점자들은 튜플의 **두 번째 값([1])** 을 보고 마저 줄을 세우자!" ➡️ `'B'`가 `'C'`보다 빠르니 최종 합격.

>데이터를 묶을 때 **(가장 중요한 정렬 기준, 두 번째 정렬 기준, 그냥 들고 갈 데이터)** 순서로 튜플을 만들고 `sorted()`에 던져주세요. 
>`lambda` 없이도 가장 깔끔하고 빠른 정렬이 완성됩니다.

#### ✅ 튜플의 본능을 이용한 숏코딩 (`(랭크, 인덱스)` 순서)

애초에 튜플을 만들 때 **내가 정렬하고 싶은 기준(랭크)을 튜플의 맨 앞(0번째)에** 두는 겁니다! 
그러면 파이썬이 본능적으로 맨 앞의 랭크를 보고 알아서 정렬해 버립니다. `lambda`가 아예 필요 없어지죠.

```python
# 애초에 (3등, 0번 학생) 순서로 튜플을 만듦
arr = [(x, i) for i, x in enumerate(rank) if attendance[i]]

# 파이썬: "오, 맨 앞이 랭크네? 랭크 순서대로 알아서 줄 세울게!"
arr = sorted(arr)
```

## 실전 코드 분석 (딕셔너리 줄 세우기) 

**목표:** `{'A': 30, 'C': 5, 'B': 15}` -> 값(숫자)이 큰 순서대로 정렬하기.

```python
data = {'A': 30, 'C': 5, 'B': 15}

# Step 1. 리스트로 변환 (items)
# [('A', 30), ('C', 5), ('B', 15)]
items = data.items()

# Step 2. 정렬 (sorted)
result = sorted(
    items,                     # 1. 줄 세울 애들
    key=lambda item: item[1],  # 2. 기준: 튜플의 [1]번(숫자)을 봐라!
    reverse=True               # 3. 방향: 큰 숫자가 1등 (내림차순)
)

print(result)
# 결과: [('A', 30), ('B', 15), ('C', 5)]
```

- **만약 `key`를 안 썼다면?**
- `sorted(items)` 라고만 쓰면, 컴퓨터는 튜플의 **[0]번(알파벳 A, B, C)** 을 기준으로 정렬해서 `[('A', 30), ('B', 15), ('C', 5)]` 가 아니라 `[('A', 30), ('B', 15), ('C', 5)]`... 어라 우연히 같네요? 데이터가 `{'A': 10, 'B': 50}` 이었다면, 알파벳 순으로는 A가 먼저지만, 숫자 순으로는 B가 먼저 나와야 하니까 결과가 완전히 달라집니다!

---
### ① `sort()` vs `sorted()` (면접 단골)

둘 다 정렬을 해주지만, **"원본을 건드리냐 안 건드리냐"** 의 차이가 엄청납니다.

|**특징**|**sorted(리스트)**|**리스트.sort()**|
|---|---|---|
|**원본 변경**|❌ **(안 바뀜, 안전!)**|✅ **(바뀌어버림, 주의!)**|
|**반환값**|**정렬된 새 리스트**|**None** (없음)|
|**사용 대상**|모든 반복 가능한 것 (튜플, 딕셔너리 등)|오직 **리스트(List)** 만|
|**체이닝**|가능 (`.method()` 계속 연결 가능)|불가능 (값이 없으니까)|

### 초보자의 흔한 실수 (The None Trap)

```python
numbers = [3, 1, 2]

# (X) 이렇게 하면 데이터 다 날아갑니다!
result = numbers.sort()
print(result) # 결과: None 😱

# (O) sort()는 그냥 혼자 실행해야 합니다.
numbers.sort()
print(numbers) # 결과: [1, 2, 3] (원본이 바뀜)

# (O) 안전하게 하려면 sorted()를 쓰세요.
safe_result = sorted([3, 1, 2])
print(safe_result) # 결과: [1, 2, 3]
```


### ② `reverse=True` 깜빡함

- 기본값(`False`)은 **오름차순** (1, 2, 3... / 가, 나, 다...) 입니다.
- **TOP 10**이나 **랭킹**을 구할 때는 무조건 `reverse=True`를 써야 합니다.

>"스파크나 판다스나 결국 파이썬 위에서 돌아가잖아? 그래서 **'데이터 많은 순서대로(빈도수) 정렬해라'** 라는 미션이 떨어지면, 
>반사적으로 **`sorted(..., key=lambda x: x[1], reverse=True)`** 이 공식이 손가락에서 나와야 해. 
>**`key`는 채점 기준표!** 이것만 기억해!"
