##  해결 전략 (Code Before Think)

> `Tree` 테이블에는 각 노드의 `id`와 부모 노드의 `p_id`가 들어 있습니다.
> 각 노드를 아래 3가지 타입 중 하나로 분류해야 합니다.
> **Root**: 부모가 없는 최상위 노드
> **Inner**: 부모도 있고, 자식도 있는 중간 노드
> **Leaf**: 부모는 있지만, 자식은 없는 마지막 노드

1. **타겟 데이터:** (어떤 테이블에서 무엇을 뽑아야 하는가?)
    - `Tree` 테이블
	    - `id`
	    - `p_id`
2. **조건 분석:**
    - `p_id IS NULL`이면 부모가 없으므로 `Root`
    - 현재 `id`가 다른 행의 `p_id`로 등장하면 자식이 있다는 뜻이므로 `Inner`
    - 위 두 조건에 해당하지 않으면 자식이 없는 노드이므로 `Leaf`
3. **사용할 문법:**
   - `CASE WHEN` : 조건별로 타입 분류
   - `IS NULL` : Root 판별
   - `IN (SELECT ...)` : 현재 노드가 누군가의 부모인지 확인
---
## 정답 쿼리 (Solution)

```sql
-- 여기에 작성한 쿼리를 붙여넣으세요.
SELECT id,
    CASE
        WHEN p_id IS NULL THEN 'Root'
        WHEN id IN (
            SELECT p_id
            FROM Tree
            WHERE p_id IS NOT NULL
        ) THEN 'Inner'
        ELSE 'Leaf'
    END AS type
FROM Tree;
```

---
##  오답 노트 & 배운 점 (Retrospective)

-  **내가 실수한 부분:**
    - 처음에는 `id IN (SELECT id FROM Tree WHERE p_id IS NOT NULL)`처럼 생각했다.
    - 찾아야 하는 `Inner`는 “자식이 있는 노드”
    - `id`가 아니라 `p_id` 목록 안에 현재 `id`가 있는지 확인
-  **새로 알게 된 함수/꿀팁:**
	- `IN (SELECT p_id FROM Tree)`는 “현재 값이 부모 목록에 있는가?”를 확인하는 패턴이다.
