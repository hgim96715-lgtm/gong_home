##  해결 전략 (Code Before Think)

> 주어진 데이터를 활용해 ID가 3820인 사용자를 포함해 세 명의 사용자가 친구 관계인 경우를 모두 출력하는 쿼리를 작성해주세요.

## 정답 쿼리 (Solution)

```sql
-- 여기에 작성한 쿼리를 붙여넣으세요.
WITH all_edges AS(
  SELECT user_a_id as u1, user_b_id as u2 FROM edges
  UNION ALL
  SELECT user_b_id as u1, user_a_id as u2 FROM edges 
)

SELECT 
  e1.u1 as user_a_id,
  e1.u2 as user_b_id,
  e2.u2 as user_c_id
FROM all_edges e1
JOIN all_edges e2 ON e1.u2 = e2.u1
JOIN all_edges e3 ON e1.u1 = e3.u1 AND e2.u2 = e3.u2
WHERE e1.u1 < e1.u2 
  AND e1.u2 < e2.u2 
  AND (e1.u1 = 3820 OR e1.u2 = 3820 OR e2.u2 = 3820);
```

##  오답 노트 & 배운 점 (Retrospective)

-  **내가 실수한 부분:**
	- ID가 3820인 사용자가 무조건 `user_a_id`일 것이라고 단정 짓고 `WHERE e.user_a_id = 3820`으로 시작했던 점.
	- 조인을 할 때 앞 테이블의 도착점(`e1.u2`)과 뒤 테이블의 출발점(`e2.u1`)을 연결해야 한다는 개념이 헷갈렸다.
	- `WITH` 구문에서 `UNION`을 쓸 때 양방향 짝꿍(Pair)을 유지하지 않고 단일 컬럼만 합치려고 했던 점.
    
-  **새로 알게 된 함수/꿀팁:**
	- **Self Join으로 꼬리물기**: 테이블에 존재하지 않는 제3의 인물(C)을 찾으려면, 테이블을 여러 번 조인하면서 `e1.u2 = e2.u1` 처럼 다리를 놔주면 된다.
	- 세 명이 모두 친구인지 확인하려면 마지막 조인(`e3`)에서 처음 출발한 사람(`e1.u1`)과 마지막에 도착한 사람(`e2.u2`)이 연결되어 있는지 `AND` 조건으로 묶어서 확인하면 된다.
