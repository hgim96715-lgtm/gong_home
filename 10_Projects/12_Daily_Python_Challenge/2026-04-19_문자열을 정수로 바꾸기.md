## 문제 분석 & 설계 (Code Before Think)

> 새로 생긴 놀이기구는 인기가 매우 많아 줄이 끊이질 않습니다. 이 놀이기구의 원래 이용료는 price원 인데, 놀이기구를 N 번 째 이용한다면 원래 이용료의 N배를 받기로 하였습니다. 즉, 처음 이용료가 100이었다면 2번째에는 200, 3번째에는 300으로 요금이 인상됩니다.  
> 놀이기구를 count번 타게 되면 현재 자신이 가지고 있는 금액에서 얼마가 모자라는지를 return 하도록 solution 함수를 완성하세요.  단, 금액이 부족하지 않으면 0을 return 하세요.

1. **입/출력 분석:** (무엇을 받고, 무엇을 돌려주는가?)
    - **Input**: `price` (int): 놀이기구 1회 이용 요금 / `money` (int): 현재 가지고 있는 금액 /`count` (int): 이용 횟수
    - **Output**: 부족한 금액 (int) - - 단, 부족하지 않으면 `0` 반환
2. **핵심 로직:** (어떤 자료구조나 흐름을 쓸 것인가?)
	- 최종 총 비용과 `money`를 비교하여 부족 금액 계산
	- 반복문을 이용한 누적합 방식
3. **사용할 함수/라이브러리:**
   - `총 금액 > money` → 차액 반환

---
## ✅ 정답 코드 (Solution)

```python
def solution(price, money, count):
    answer = 0
    for p in range(1,count+1):
        answer+=p*price
    return answer-money if answer>money else 0
```

## 더 나은 풀이 (Pythonic Code)

> 다른 사람의 풀이나, 더 간결한 코드(리팩토링)가 있다면?
> 등차수열 합 공식: `n(n+1)/2`  반복문 없이도 누적합 계산 가능 → **시간복잡도 O(1)**


```python
# 다른 사람의 풀이나, 더 간결한 코드(리팩토링)가 있다면?
def solution(price, money, count):
    return max(0,price*(count+1)*count//2-money)
```
