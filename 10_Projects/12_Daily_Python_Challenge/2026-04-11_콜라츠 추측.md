## 문제 분석 & 설계 (Code Before Think)

> 1937년 Collatz란 사람에 의해 제기된 이 추측은, 주어진 수가 1이 될 때까지 다음 작업을 반복하면, 모든 수를 1로 만들 수 있다는 추측입니다. 작업은 다음과 같습니다.

```
1-1. 입력된 수가 짝수라면 2로 나눕니다. 
1-2. 입력된 수가 홀수라면 3을 곱하고 1을 더합니다. 
2. 결과로 나온 수에 같은 작업을 1이 될 때까지 반복합니다. 
```

1. **입/출력 분석:** (무엇을 받고, 무엇을 돌려주는가?)
    - **Input**: 정수 `num`
    - **Output**: `num`이 1이 될 때까지 반복 횟수 (int),단, **500번을 초과하면 -1 반환**
2. **핵심 로직:** (어떤 자료구조나 흐름을 쓸 것인가?)
	- 반복 횟수 `> 500` → 강제 종료 후 `-1`
3. **사용할 함수/라이브러리:**
   - 현재 숫자가 **짝수** → `num = num // 2`
   - 현재 숫자가 **홀수** → `num = num * 3 + 1`

---
## ✅ 정답 코드 (Solution)

```python
def solution(num):
    answer = 0
    while num!=1:
        if num%2:
            num=num*3+1
        else:
            num//=2
        answer+=1
    return answer if answer<=500 else -1
```
