## 문제 분석 & 설계 (Code Before Think)

> 정수 `n`과 문자열 `control`이 주어집니다. `control`은 "w", "a", "s", "d"의 4개의 문자로 이루어져 있으며, `control`의 앞에서부터 순서대로 문자에 따라 `n`의 값을 바꿉니다.
> - "w" : `n`이 1 커집니다. "s" : `n`이 1 작아집니다. "d" : `n`이 10 커집니다. "a" : `n`이 10 작아집니다.
> - 위 규칙에 따라 `n`을 바꿨을 때 가장 마지막에 나오는 `n`의 값을 return 하는 solution 함수를 완성해 주세요.

1. **입/출력 분석:** (무엇을 받고, 무엇을 돌려주는가?)
    - **Input**: `control` : 문자열, `n` : 정수
    - **Output**: 모든 control 문자를 순차적으로 적용한 뒤의 **최종 정수 값 `n`**
2. **핵심 로직:** (어떤 자료구조나 흐름을 쓸 것인가?)
	- 문자열 `control`을 **왼쪽부터 순회(iteration)**
3. **사용할 함수/라이브러리:**
   - `for` 문 (문자열 순회)
   - 조건문 `if / elif`

---
## ✅ 정답 코드 (Solution)

```python
n=0
def solution(n, control):
    for v in control:
        if v=="w":
            n+=1
        elif v=="s":
            n-=1
        elif v=="d":
            n+=10
        elif v=="a":
            n-=10
    return n
```

---
## 오답 노트 & 배운 점 (Retrospective)

-  **내가 실수한 부분 (Syntax/Logic):**
	- 처음에 함수의 입력으로 받은 `n`을 무시하고, 함수 내부에서 `n = 0`으로 다시 초기화함
	- 조건문 분기 자체는 맞았으나, **코드가 길어지고 반복적**이어서 확장성과 가독성이 떨어지고 있음
-  **새로 알게 된 함수/메서드:**
	- `zip()`을 활용하면 **명령어 ↔ 값 매핑 관계를 선언적으로 표현**할 수 있다는 점을 배움
	- 조건문(`if/elif`) 대신 **데이터 구조(dict) + 반복 처리**로 로직을 단순화할 수 있음

## 더 나은 풀이 (Pythonic Code)

> 다른 사람의 풀이나, 더 간결한 코드(리팩토링)가 있다면?


```python
# 다른 사람의 풀이나, 더 간결한 코드(리팩토링)가 있다면?
def solution(n, control):
    key = dict(zip(['w','s','d','a'], [1,-1,10,-10]))
    return n + sum([key[c] for c in control])
```

>`zip` + `dict`는  **조건문을 데이터로 치환**할 때 가장 강력한 패턴이다.
