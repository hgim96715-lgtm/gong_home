## 문제 분석 & 설계 (Code Before Think)

> 랜덤으로 서로 다른 k개의 수를 저장한 배열을 만드려고 합니다. 적절한 방법이 떠오르지 않기 때문에 일정한 범위 내에서 무작위로 수를 뽑은 후, 지금까지 나온적이 없는 수이면 배열 맨 뒤에 추가하는 방식으로 만들기로 합니다.
> 이미 어떤 수가 무작위로 주어질지 알고 있다고 가정하고, 실제 만들어질 길이 `k`의 배열을 예상해봅시다.
> 정수 배열 `arr`가 주어집니다. 문제에서의 무작위의 수는 `arr`에 저장된 순서대로 주어질 예정이라고 했을 때, 완성될 배열을 return 하는 solution 함수를 완성해 주세요.
> 단, 완성될 배열의 길이가 `k`보다 작으면 나머지 값을 전부 -1로 채워서 return 합니다

1. **입/출력 분석:** (무엇을 받고, 무엇을 돌려주는가?)
    - **Input**: `arr` : 무작위로 주어질 정수들이 순서대로 담긴 정수 배열
    - **Output**: `arr`를 앞에서부터 순회하며 **중복되지 않는 수만 순서대로 저장**
2. **핵심 로직:** (어떤 자료구조나 흐름을 쓸 것인가?)

3. **사용할 함수/라이브러리:**
   - `list.append()` : 결과 배열에 값 추가
   - `in` 연산자 : 중복 여부 확인

## ✅ 정답 코드 (Solution)

```python
def solution(arr, k):
    answer = []
    for i,v in enumerate(arr):
        if v not in answer :
            answer.append(v)
            if len(answer)==k:
                break
    while len(answer)<k:
        answer.append(-1)
    return answer
```

## 오답 노트 & 배운 점 (Retrospective)

-  **내가 실수한 부분 (Syntax/Logic):**
	- 최종 결과 배열의 길이가 `k`보다 작을 경우를 고려하지 못했다.
    
-  **새로 알게 된 함수/메서드:**
	- **`while` 문을 이용해 “조건이 만족될 때까지 반복하며 값을 채우는 패턴”** 을 떠올리지 못했다는 점을 깨달았다.

```python
# 다른 사람의 풀이나, 더 간결한 코드(리팩토링)가 있다면?
def solution(arr, k):
    res = list(dict.fromkeys(arr))
    res.extend([-1] * max(0, k - len(res)))
    return res[:k]
```
