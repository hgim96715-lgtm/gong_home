## 문제 분석 & 설계 (Code Before Think)

> 명함 지갑을 만드는 회사에서 지갑의 크기를 정하려고 합니다. 다양한 모양과 크기의 명함들을 모두 수납할 수 있으면서, 작아서 들고 다니기 편한 지갑을 만들어야 합니다. 이러한 요건을 만족하는 지갑을 만들기 위해 디자인팀은 모든 명함의 가로 길이와 세로 길이를 조사했습니다.
> 모든 명함의 가로 길이와 세로 길이를 나타내는 2차원 배열 sizes가 매개변수로 주어집니다. 모든 명함을 수납할 수 있는 가장 작은 지갑을 만들 때, 지갑의 크기를 return 하도록 solution 함수를 완성해주세요.

1. **입/출력 분석:** (무엇을 받고, 무엇을 돌려주는가?)
    - **Input**: 명함의 가로/세로 길이가 담긴 2차원 배열 `sizes`
    - **Output**: 모든 명함을 수납할 수 있는 **가장 작은 지갑의 넓이 (int)**
2. **핵심 로직:** (어떤 자료구조나 흐름을 쓸 것인가?)
	- 각 명함마다 **긴 변을 가로, 짧은 변을 세로로 정렬**
3. **사용할 함수/라이브러리:**
   - `max()`

---
## ✅ 정답 코드 (Solution)

```python
def solution(sizes):
    w=0
    h=0
    for a,b in sizes:
        if a<b:
            a,b=b,a
        w=max(w,a)
        h=max(h,b)
    return w*h
```

-  **내가 실수한 부분 (Syntax/Logic):**
    
-  **새로 알게 된 함수/메서드:**
	- `max(x)` = 각 명함의 긴 변
	- `min(x)` = 각 명함의 짧은 변

# 다른 사람의 풀이나, 더 간결한 코드(리팩토링)가 있다면?
def solution(sizes):
    return max(max(x) for x in sizes) * max(min(x) for x in sizes)
