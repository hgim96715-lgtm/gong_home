## 문제 분석 & 설계 (Code Before Think)

>문자열 `s`가 주어졌을 때, `s`의 각 위치마다 자신보다 앞에 나왔으면서, 자신과 가장 가까운 곳에 있는 같은 글자가 어디 있는지 알고 싶습니다.  
>예를 들어, `s`="banana"라고 할 때,  각 글자들을 왼쪽부터 오른쪽으로 읽어 나가면서 다음과 같이 진행할 수 있습니다.

1. **입/출력 분석:** (무엇을 받고, 무엇을 돌려주는가?)
    - **Input**: 영어 소문자로 이루어진 문자열 `s`
    - **Output**: 이전에 같은 문자가 없으면 `-1` 각 문자마다 가장 가까운 이전 같은 문자의 거리 값을 담은 리스트
2. **핵심 로직:** (어떤 자료구조나 흐름을 쓸 것인가?)
	- 각 문자가 **마지막으로 등장한 인덱스**를 딕셔너리에 저장한다.
	- 현재 문자가 딕셔너리에 없다면 처음 나온 문자이므로 `-1`을 추가
	- 현재 문자가 딕셔너리에 있다면, 현재 인덱스와 이전 등장 인덱스의 차이를 구해서 `answer`에 추가
3. **사용할 함수/라이브러리:**
    - `enumerate()`
    - `append()`

---
## ✅ 정답 코드 (Solution)

```python
def solution(s):
    answer = []
    closer={}
    for i,s1 in enumerate(s):
        if s1 not in closer:
            answer.append(-1)
        else:
            # print(closer)
            answer.append(i-closer[s1])
        closer[s1]=i
    return answer
```

---
## 오답 노트 & 배운 점 (Retrospective)

-  **내가 실수한 부분 (Syntax/Logic):**
    - 처음에 `answer` 리스트에 문자 정보를 저장하려고 했다.
    - 하지만 `answer`는 최종 결과인 `[-1, 2, ...]` 같은 숫자 리스트를 저장하는 용도
    - 문자별 마지막 위치는 따로 딕셔너리에 저장해야 한다.
-  **새로 알게 된 함수/메서드:**
	- “가장 가까운 이전 위치”를 구해야 할 때는 모든 위치를 저장할 필요 없이, **마지막으로 나온 위치만 저장하면 된다.**

## 더 나은 풀이 (Pythonic Code)

> 다른 사람의 풀이나, 더 간결한 코드(리팩토링)가 있다면?


```python
# 다른 사람의 풀이나, 더 간결한 코드(리팩토링)가 있다면?
def solution(s):
    answer = []
    dic = dict()
    for i in range(len(s)):
        if s[i] not in dic:
            answer.append(-1)
        else:
            answer.append(i - dic[s[i]])
        dic[s[i]] = i

    return answer
```
