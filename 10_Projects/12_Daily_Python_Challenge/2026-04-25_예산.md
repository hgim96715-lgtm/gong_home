## 문제 분석 & 설계 (Code Before Think)

> S사에서는 각 부서에 필요한 물품을 지원해 주기 위해 부서별로 물품을 구매하는데 필요한 금액을 조사했습니다. 그러나, 전체 예산이 정해져 있기 때문에 모든 부서의 물품을 구매해 줄 수는 없습니다. 그래서 최대한 많은 부서의 물품을 구매해 줄 수 있도록 하려고 합니다.
> 물품을 구매해 줄 때는 각 부서가 신청한 금액만큼을 모두 지원해 줘야 합니다. 예를 들어 1,000원을 신청한 부서에는 정확히 1,000원을 지원해야 하며, 1,000원보다 적은 금액을 지원해 줄 수는 없습니다.
> 부서별로 신청한 금액이 들어있는 배열 d와 예산 budget이 매개변수로 주어질 때, 최대 몇 개의 부서에 물품을 지원할 수 있는지 return 하도록 solution 함수를 완성해주세요.

1. **입/출력 분석:** (무엇을 받고, 무엇을 돌려주는가?)
    - **Input**: 부서별 신청 금액 리스트 `d`, 전체 예산 `budget`
    - **Output**: 지원 가능한 최대 부서 개수 (정수)
2. **핵심 로직:** (어떤 자료구조나 흐름을 쓸 것인가?)
	- 최대한 많은 부서를 지원하려면 **신청 금액이 적은 부서부터** 지원
	- 배열 `d`를 오름차순 정렬
	- 작은 금액부터 차례대로 예산에서 차감
3. **사용할 함수/라이브러리:**
	- `sort()` : 신청 금액 오름차순 정렬
	- `for` 반복문

---
## ✅ 정답 코드 (Solution)

```python
def solution(d, budget):
    answer=0
    d.sort()
    for d1 in d:
        budget-=d1
        print(d1,budget)
        if budget>=0:
            answer+=1
        elif budget<0:
            return answer
    return answer
```

-  **새로 알게 된 함수/메서드:**

```python
# 다른 사람의 풀이나, 더 간결한 코드(리팩토링)가 있다면?
def solution(d, budget):
    d.sort()
    while budget < sum(d):
        d.pop()
    return len(d)
```
