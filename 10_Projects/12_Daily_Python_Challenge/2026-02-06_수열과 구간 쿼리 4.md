## 문제 분석 & 설계 (Code Before Think)

> 정수 배열 `arr`와 2차원 정수 배열 `queries`이 주어집니다. `queries`의 원소는 각각 하나의 `query`를 나타내며, `[s, e, k]` 꼴입니다.
> 각 `query`마다 순서대로 `s` ≤ `i` ≤ `e`인 모든 `i`에 대해 `i`가 `k`의 배수이면 `arr[i]`에 1을 더합니다.
> 위 규칙에 따라 `queries`를 처리한 이후의 `arr`를 return 하는 solution 함수를 완성해 주세요.

1. **입/출력 분석:** (무엇을 받고, 무엇을 돌려주는가?)
    - **Input**: `arr` : 정수로 이루어진 1차원 배열, `queries` : 2차원 정수 배열
    - **Output**: 모든 `queries`를 **순서대로 처리한 후 변경된 `arr`**
2. **핵심 로직:** (어떤 자료구조나 흐름을 쓸 것인가?
	- 값이 아닌 인덱스를 기준으로 조건을 판단
3. **사용할 함수/라이브러리:**
   - `for` 반복문

---
## ✅ 정답 코드 (Solution)

```python
def solution(arr, queries):
    for q in queries:
        s,e,k=q
        for i,v in enumerate(range(s,e+1)):
            if v%k==0:
                arr[i]+=1
    return arr
```

---
## 오답 노트 & 배운 점 (Retrospective)

-  **내가 실수한 부분 (Syntax/Logic):**
	- `arr[i]`가 아니라 **`i`가 k의 배수** 라는 것을 헷갈려서 계속 틀림
	- “0도 배수에 포함된다”는 개념을 처음에 헷갈렸다 -> 0은 모든 양의 정수의 배수
	- `for i, v in enumerate(range(s, e+1)):` 를 사용했는데,  **이 문제에서는 전혀 필요 없는 구조였다.**`enumerate`의 `i`는 문제에서 말하는 인덱스 `i`와 의미가달랐고, 그때문에 최종 테스트에 `i%k==0` 이라는 실수를 저질러서 최종테스트가 다 실패했다.
	- `for i in range(s, e+1):` 만으로도 충분한 문제였다. 

-  **새로 알게 된 함수/메서드:**
	- 값(`arr[i]`)과 인덱스(`i`)는 완전히 다른 개념

## 더 나은 풀이 (Pythonic Code)

```python
# 다른 사람의 풀이나, 더 간결한 코드(리팩토링)가 있다면?
def solution(arr, queries):
    for s, e, k in queries:
        if k == 0:
            continue  # 0의 배수는 정의되지 않으므로 스킵
        for i in range(s, e + 1):
            if i % k == 0:
                arr[i] += 1
    return arr
```
