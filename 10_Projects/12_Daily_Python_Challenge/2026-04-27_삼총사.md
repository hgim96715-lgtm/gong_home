## 문제 분석 & 설계 (Code Before Think)

> 한국중학교에 다니는 학생들은 각자 정수 번호를 갖고 있습니다. 이 학교 학생 3명의 정수 번호를 더했을 때 0이 되면 3명의 학생은 삼총사라고 합니다. 예를 들어, 5명의 학생이 있고, 각각의 정수 번호가 순서대로 -2, 3, 0, 2, -5일 때, 첫 번째, 세 번째, 네 번째 학생의 정수 번호를 더하면 0이므로 세 학생은 삼총사입니다. 또한, 두 번째, 네 번째, 다섯 번째 학생의 정수 번호를 더해도 0이므로 세 학생도 삼총사입니다. 따라서 이 경우 한국중학교에서는 두 가지 방법으로 삼총사를 만들 수 있습니다.
> 한국중학교 학생들의 번호를 나타내는 정수 배열 `number`가 매개변수로 주어질 때, 학생들 중 삼총사를 만들 수 있는 방법의 수를 return 하도록 solution 함수를 완성하세요.

1. **입/출력 분석:** (무엇을 받고, 무엇을 돌려주는가?)
    - **Input**: 학생들의 정수 번호가 담긴 리스트 `number`
    - **Output**: 세 명의 번호 합이 0이 되는 조합의 개수 (정수)
2. **핵심 로직:** (어떤 자료구조나 흐름을 쓸 것인가?)
	- 같은 학생을 두 번 뽑지 않기 위해 `i < j < k` 조건으로 3중 반복문을 구성
3. **사용할 함수/라이브러리:**
   - `range()`: 인덱스 반복
   - `len()`: 리스트 길이 확인

---
## ✅ 정답 코드 (Solution)

```python
def solution(number):
    answer = 0
    for n1 in range(len(number)):
        for n2 in range(n1+1,len(number)):
            for n3 in range(n2+1,len(number)):
                if number[n1] + number[n2] + number[n3] == 0:
                    answer+=1
    return answer
```

## 오답 노트 & 배운 점 (Retrospective)

-  **내가 실수한 부분 (Syntax/Logic):**
    - 처음에는 값 자체를 3중 반복문으로 순회해서 같은 학생이 여러 번 선택될 수 있었다.
    - 순서만 다른 같은 조합도 여러 번 카운트되는 문제가 있었다.
-  **새로 알게 된 함수/메서드:**
	- 중복 없는 선택은 `for j in range(i+1, ...)` 형태로 해결할 수 있다.
	- `itertools.combinations()`을 사용하면 조합 생성을 직접 구현하지 않아도 된다.

## 더 나은 풀이 (Pythonic Code)

> 다른 사람의 풀이나, 더 간결한 코드(리팩토링)가 있다면?


```python
# 다른 사람의 풀이나, 더 간결한 코드(리팩토링)가 있다면?
def solution (number) :
    from itertools import combinations
    cnt = 0
    for i in combinations(number,3) :
        if sum(i) == 0 :
            cnt += 1
    return cnt
```
