## 문제 분석 & 설계 (Code Before Think)

> 숫자로 이루어진 문자열 `t`와 `p`가 주어질 때, `t`에서 `p`와 길이가 같은 부분문자열 중에서, 이 부분문자열이 나타내는 수가 `p`가 나타내는 수보다 작거나 같은 것이 나오는 횟수를 return하는 함수 solution을 완성하세요.
> 예를 들어, `t`="3141592"이고 `p`="271" 인 경우, `t`의 길이가 3인 부분 문자열은 314, 141, 415, 159, 592입니다. 이 문자열이 나타내는 수 중 271보다 작거나 같은 수는 141, 159 2개 입니다.

1. **입/출력 분석:** (무엇을 받고, 무엇을 돌려주는가?)
    - **Input**: 숫자로 이루어진 문자열 `t`, 비교 대상 문자열 `p`
    - **Output**: 조건을 만족하는 부분문자열의 개수 (정수)
2. **핵심 로직:** (어떤 자료구조나 흐름을 쓸 것인가?)
	- `p`의 길이를 기준으로 `t`를 슬라이싱하며 부분문자열을 만든다
	- 길이가 `len(p)`와 같은 부분문자열만 비교한다.
3. **사용할 함수/라이브러리:**
	- 문자열 슬라이싱 `t[i:i+len(p)]`
	- `range()`


---
## ✅ 정답 코드 (Solution)

```python
def solution(t, p):
    answer = 0
    for t1 in range(0,len(t)):
        res=t[t1:t1+len(p)]
        if len(res)==len(p) and res<=p:
            answer+=1
    return answer
```

---
## 오답 노트 & 배운 점 (Retrospective)

-  **내가 실수한 부분 (Syntax/Logic):**
    - 반복 범위를 `len(t)`까지 돌리면 불필요한 슬라이싱이 발생
-  **새로 알게 된 함수/메서드:**
	- `range(len(t) - len(p) + 1)`로 필요한 구간만 순회 가능하다.

```python
# 다른 사람의 풀이나, 더 간결한 코드(리팩토링)가 있다면?
def solution(t, p):
    answer = 0

    for i in range(len(t) - len(p) + 1):
        if int(p) >= int(t[i:i+len(p)]):
            answer += 1

    return answer
```
