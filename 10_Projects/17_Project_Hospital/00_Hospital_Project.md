## 💡 기획 배경

>응급실 포화라는 안타까운 현실 속에서도, 소방(119)과 병원은 환자를 살리기 위해 끊임없이 수용 여부를 확인하고 실시간으로 병상 데이터를 주고받으며 사투를 벌이고 있습니다. 
>병원 근무와 대학교 소방 실습 시절, 구급대원이 상황실과 연락을 취하면 즉각적으로 수용 가능한 병원을 파악해 내는 이 연계 시스템을 현장에서 직접 경험했습니다.
>"전국 수백 개 병원의 실시간 병상 현황과 수용 가능 여부가, 어떻게 지연 없이 119 지령센터와 구급차 패드로 공유될 수 있을까?" 
>과거에는 그저 신기하게만 바라보았던 이 시스템이, 데이터 엔지니어링을 공부하며 '거대한 실시간 데이터 파이프라인'의 결과물이라는 것을 깨달았습니다.
> 현장에서 생명을 살리는 데 기여하는 이 거대한 데이터의 흐름을 기술적으로 직접 이해하고 구현해 보고 싶다는 강렬한 호기심이 생겼습니다.
> 단순히 데이터를 분석하는 것을 넘어, 실제 국가 응급 의료망에서 데이터가 흐르는 길을 엔지니어의 시각으로 직접 역설계(Reverse Engineering)해보고자 본 프로젝트를 기획했습니다. 실시간 API 데이터를 수집해 메시지 큐로 버퍼링하고, 분산 처리 엔진으로 가공하여 데이터베이스에 적재한 뒤 대시보드로 시각화하는 **엔드투엔드(End-to-End) 파이프라인**을 구축하는 것이 목표입니다.

## 🏗 아키텍처

```
국립중앙의료원 API (5분마다 폴링)
        ↓
Python Producer → Kafka → Spark Streaming → PostgreSQL → Superset
                                    ↑
                          Airflow (배치: 골든타임 분석 / 시간대별 집계)
```

|역할|기술|
|---|---|
|데이터 수집|Python (requests)|
|메시지 큐|Apache Kafka|
|스트림 처리|Apache Spark Streaming|
|데이터 저장|PostgreSQL|
|스케줄링·알림|Apache Airflow|
|시각화|Apache Superset|
|인프라|Docker, Docker Compose|

## 📡 데이터 소스 — 국립중앙의료원 API

**출처:** 국립중앙의료원_전국 응급의료기관 정보 조회 서비스 **공공데이터포털:** [https://www.data.go.kr/data/15000563/openapi.do](https://www.data.go.kr/data/15000563/openapi.do) 
**엔드포인트:** `http://apis.data.go.kr/B552657/ErmctInfoInqireService`
**포맷:** XML **트래픽:** 개발계정 1,000회/일 (운영계정은 활용사례 등록 후 증가 가능)

> 마이페이지-> 데이터활용->Open API -> 활용신청 현황에서 미리 보기 가능 
### 실시간 수집 (Producer 5분마다)

| #   | API            | 설명                         |
| --- | -------------- | -------------------------- |
| ①   | 응급실 실시간 가용병상정보 | 시도별 가용병상 현황                |
| ②   | 중증질환자 수용가능정보   | 지금 받을 수 있나 (스트리밍은 아니지만 동적) |
| ③   | 응급실 및 중증질환 메시지 | 병원 공지 메시지                  |

```
① + ② + ③ hpid 기준으로 병합 → er_realtime 한 row 로 저장
5분마다 3개 API = 288 × 3 = 864회/일 ← 1,000 이내 ✅
```

### 정적 데이터 (Airflow 1일 1회)

| #   | API         | 설명                     |
| --- | ----------- | ---------------------- |
| ④   | 응급의료기관 기본정보 | 전화/주소/위경도/역량(MKioskTy) |

```
④ 하나로 충분한 이유:
  기본정보 API 에 dutyAddr / dutyTel3 / wgs84Lat / wgs84Lon
  + MKioskTy (병원 인증 역량, 정적) 전부 포함
  → 목록정보 / 위치정보 API 별도 불필요

⑤ 응급의료기관 목록정보  → 기본정보 API 로 커버
⑥ 응급의료기관 위치정보  → 기본정보 API 의 wgs84 로 커버
⑦ 외상센터 정보         → 제외 (mkioskty.. 로 커버)
```
