## 📌 랩 목표

Linux 환경에서 관리자들의 필수 역량인 Bash 셸 스크립팅의 기초를 다집니다. 변수 선언, 사용자 입력 처리, 조건문(`if`), 반복문(`for`)을 활용하여 서버의 로그 파일들을 안전하게 백업하는 자동화 스크립트를 직접 구축해 봅니다.

---

## 📖 학습 내용

---

### 🔷 섹션 1 — 간단한 셸 스크립트 작성하기

**개념 요약**

-  **Shebang (`#!/bin/bash`):** 스크립트 파일의 맨 첫 줄에 반드시 들어가야 하는 마법의 주문입니다. 시스템에게 이 파일을 'bash' 프로그램으로 실행하라고 알려주는 역할을 합니다.
- **실행 권한 부여:** 리눅스에서는 텍스트 파일을 만들었다고 해서 바로 실행할 수 없습니다. 반드시 `chmod +x [파일명]` 명령어를 통해 스크립트에 실행(`eXecute`) 권한을 부여해야 합니다.

**명령어 / 문법**

```bash
nano log_manager.sh # #!/bin/bash echo "Log Manager Initialized."
chmod +x ~/project_log_manager.sh
cat ~/project/log_manager.sh
```

**실습 메모**

실무 환경에서 스크립트를 다 짰는데 막상 `./log_manager.sh`를 쳤을 때 `Permission denied(권한 거부)` 에러가 뜬다면 당황하지 마세요. 99%는 파일 생성 후 `chmod +x`를 잊어버렸기 때문입니다.

### 🔷 섹션 2 — 변수와 사용자 입력 추가하기

**개념 요약**

- **변수 선언:** `VARIABLE_NAME="value"` 형태로 선언합니다. **(주의: 타 언어와 달리 `{text}=` 기호 양옆에 절대 공백이 없어야 합니다)**
- **변수 사용:** 선언한 변수의 값을 불러올 때는 이름 앞에 달러 기호(`$`)를 붙여 `$VARIABLE_NAME` 형태로 사용합니다.
- **`read` 명령어:** 스크립트 실행을 잠시 멈추고 사용자의 키보드 입력을 기다립니다. 사용자가 입력하고 Enter를 누르면, 그 값이 지정된 변수(예: `BACKUP_FILENAME`)에 쏙 저장됩니다.

**명령어 / 문법**

```bash
nano ~/project/log_manager.sh

#!/bin/bash 
#echo "Log Manager Initialized."
#LOG_DIR="/home/labex/project/app_logs" 
# echo "Enter the backup filename: " 
#read BACKUP_FILENAME 
#echo "Backing up logs to: $BACKUP_FILENAME"

./log_manager.sh
```

**실습 메모**

셸 스크립팅을 처음 할 때 가장 많이 하는 실수가 바로 띄어쓰기입니다. `LOG_DIR = "/경로/"` 처럼 무심코 띄어쓰기를 하는 순간 bash는 이를 변수 선언이 아니라 명령어로 착각하고 에러를 뱉어냅니다. 변수 할당 시 '공백 금지'는 꼭 기억해야 할 꿀팁입니다.

### 🔷 섹션 3 — 조건부 로직 구현하기

**개념 요약**

 - **`if` 조건문:** 무작정 스크립트를 실행하는 것이 아니라, 특정 조건이 맞을 때만 작업을 수행하도록 안전장치를 마련합니다.
 - **디렉토리 존재 검사 (`-d`):** `[ -d "$LOG_DIR" ]` 연산자는 지정한 경로가 실제로 존재하는 디렉토리인지 확인해 줍니다.
 - **에러 처리 (`exit 1`):** 대상 폴더가 없다면 에러 메시지를 출력(`echo`)하고, `exit 1`을 호출하여 스크립트가 '비정상 종료'되었음을 시스템에 알린 뒤 즉시 스크립트를 끝냅니다.

**명령어 / 문법**

```bash
nanao log_manager.sh
```

```bash title='nano log_manager.sh안 '
!/bin/bash 
echo "Log Manager Initialized." 
LOG_DIR="/home/labex/project/app_logs" 
if [ -d "$LOG_DIR" ]; then 
	echo "Log directory found. Proceeding..." 
	echo "Enter the backup filename: "
	read BACKUP_FILENAME 
	echo "Backing up logs to: $BACKUP_FILENAME" 
else 
	echo "Error: Log directory not found." 
exit 1 
fi
```

**실습 메모**

타겟 폴더도 없는데 무작정 백업 로직이 돌아서 빈 파일만 잔뜩 생성되는 대참사를 막기 위한 필수 방어 로직입니다. 훌륭한 시스템 스크립트는 '기능이 잘 동작하는가'보다 '오류 상황에서 얼마나 안전하게 멈추는가'가 훨씬 중요합니다.

### 🔷 섹션 4 — 파일 작업을 위한 반복문 사용하기

**개념 요약**

- **`for` 반복문:** 대상 디렉토리(`app_logs`) 내에서 특정한 패턴(`.log`로 끝나는 파일)을 가진 모든 파일을 하나씩 꺼내어 순회하는 기능입니다.
- **`cp` 명령어:** 복사 명령어입니다. `cp [원본 파일] [목적지 경로]` 형태로 사용하여, 반복문이 순회할 때마다 대상 로그 파일을 백업 폴더로 하나씩 복사해 넣습니다.

**명령어 / 문법**

```bash
mkdir ~/project/backups

nano ~/project/log_manager.sh
```

```bash title='nano log_manager.sh안 '
!/bin/bash 
echo "Log Manager Initialized." 
LOG_DIR="/home/labex/project/app_logs" 
if [ -d "$LOG_DIR" ]; then 
	echo "Log directory found. Proceeding..." 
	echo "Enter the backup filename: "
	read BACKUP_FILENAME 
	echo "Backing up logs to: $BACKUP_FILENAME" 
	
	for file in $LOG_DIR/*.log; do
		echo "Copied $file"
		cp "$file" "$BACKUP_DIR/"
	done
	echo "Backup complete."
else 
	echo "Error: Log directory not found." 
exit 1 
fi
```

**실습 메모**

이전 `phoenix_project`와 같은 환경에서 구조화된 작업을 할 때, 수십 개의 분산된 로그나 파일들을 일일이 수동으로 옮기는 것은 비효율적입니다. 이렇게 셸 스크립트의 반복문을 이용해 규칙적으로 파일을 필터링하고 백업하는 것이야말로 리눅스 커맨드라인 시스템 관리의 진정한 묘미이자, 엔지니어의 시간과 체력을 아껴주는 강력한 무기입니다.
