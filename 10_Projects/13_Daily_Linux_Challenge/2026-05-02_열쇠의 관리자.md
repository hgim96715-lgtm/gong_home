## 📌 랩 목표

Linux 시스템에서 사용자 계정을 생성, 수정, 삭제하고 비밀번호 및 그룹을 관리하는 전반적인 '사용자 라이프사이클 관리(User Lifecycle Management)'를 실습하여 시스템 보안과 권한 제어의 기초를 다집니다.

---

## 📖 학습 내용


---

### 🔷 섹션 1 — 시스템에 새로운 개발자 온보딩

**개념 요약**

- **명령어 차이:** 친절한 대화형 도구인 `adduser` 대신, 쉘 스크립팅과 자동화에 적합한 표준 명령어 `useradd`를 사용해 계정을 생성합니다.
- **권한:** 계정 생성은 시스템 핵심 권한이므로 반드시 `sudo`를 앞에 붙여야 합니다. (`sudo useradd [사용자명]`)
- **검증:** 생성된 사용자 정보는 시스템의 모든 계정 명부가 담긴 `/etc/passwd` 파일에서 확인할 수 있습니다.

**명령어 / 문법**

```bash
sudo useradd b.smith
grep "b.smith" /etc/passwd
id b.smith
```

**실습 메모**

실무에서는 수십 명의 신규 입사자 계정을 한 번에 생성해야 할 때가 많습니다. 이때 일일이 물어보는 `adduser`보다, 옵션을 주어 묵묵히 제 할 일을 하는 `useradd`가 쉘 스크립트(자동화)를 작성할 때 훨씬 유용하다는 점을 알게 되었습니다.

---

### 🔷 섹션 2 — 새 사용자를 위한 전용 홈 디렉토리 생성

**개념 요약**

-  **홈 디렉토리 필수성:** `useradd` 명령어는 기본적으로 깡통 계정만 만들 뿐 홈 디렉토리(`/home/[사용자명]`)를 만들어주지 않습니다.
- **`-m` (`--create-home`) 옵션:** 이 옵션을 넣어야만 사용자의 독립적인 작업 공간(홈 디렉토리)이 함께 생성됩니다.
- **작동 원리:** 이 옵션을 쓰면 `/etc/skel`(Skeleton, 뼈대) 폴더 안에 있는 기본 환경 설정 파일(`.bashrc` 등)들이 새 사용자의 홈 디렉토리로 자동 복사됩니다.
- **수정:** 옵션 없이 잘못 만들었다면 `sudo userdel`로 깔끔하게 지우고 다시 만드는 것이 좋습니다.

**명령어 / 문법**

```bash
sudo userdel b.smith
sudo useradd -m b.smith(sudo useradd --create-home b.smith)
ls -la /home/
sudo ls -la /home/b.smith/
```

**실습 메모**

리눅스 서버에 접속했는데 프롬프트가 이상하게 깨져 나오거나 파일 저장이 안 된다면 십중팔구 홈 디렉토리가 없거나 권한이 꼬인 것입니다. `/etc/skel`이 초기 세팅의 '거푸집' 역할을 한다는 점은 매우 흥미로운 리눅스의 구조적 특징입니다.

### 🔷 섹션 3 — 새 사용자를 위한 초기 비밀번호 할당

**개념 요약**

-  **`passwd` 명령어:** 사용자 계정의 비밀번호를 설정하거나 변경합니다.
- **관리자 권한:** 다른 사람의 비밀번호를 강제로 설정해 주는 것이므로 `sudo`가 필요합니다. (`sudo passwd [사용자명]`)
- **보안 저장소:** 리눅스는 보안을 위해 비밀번호 자체를 `/etc/passwd`에 두지 않고, 암호화(해시)하여 일반 사용자는 읽을 수 없는 `/etc/shadow` 파일에 철저히 숨겨 보관합니다.


**명령어 / 문법**

```bash
sudo passwd b.smith
sudo grep "^b.smith:" /etc/shadow
```

**실습 메모**

실무 시스템에서는 보안을 위해 정기적으로 패스워드를 교체해야 합니다. `/etc/shadow` 파일을 열어보면 해시 처리된 복잡한 문자열이 보이는데, 이것이 리눅스 계정 보안의 최후의 보루라는 점을 명심해야 합니다.

### 🔷 섹션 4 — 새 개발자를 "developers" 그룹에 추가

**개념 요약**

- **`usermod` 명령어:** 이미 생성된 사용자 계정의 속성(그룹, 홈 디렉토리 등)을 수정할 때 사용합니다.
- **`-aG` 옵션 조합 (매우 중요):**
    - `-G` (Groups): 이 그룹에 사용자를 넣겠다는 뜻입니다.
    - `-a` (Append): **기존 그룹을 유지한 채로** 덧붙이겠다는 뜻입니다.
- **주의사항:** `-a` 옵션을 빼먹고 `-G`만 쓰면, 사용자가 기존에 속해있던 모든 그룹에서 탈퇴되고 오직 새 그룹으로만 덮어쓰기 당합니다.

**명령어 / 문법**

```bash
 sudo usermod -aG developers b.smith
groups b.smith
id b.smith
grep "^developers:" /etc/group
```

**실습 메모**

서버 운영 중 가장 많이 일어나는 대참사 중 하나가 `usermod -G`만 쳐서 관리자(`sudo` 그룹)가 일반 유저로 강등되어 서버 제어권을 잃어버리는 일입니다. 반드시 `-aG`를 습관처럼 붙여 쓰는 것이 서버 관리자의 생존 비결입니다.

### 🔷 섹션 5 — 퇴사하는 직원의 계정을 일시적으로 비활성화

**개념 요약**

- **계정 잠금 (Lock):** 직원이 퇴사했을 때 계정을 아예 지우지 않고 **로그인만 못하게 차단**하는 보안 조치입니다.
- **명령어:** `usermod -L [사용자명]` (또는 `passwd -l`)
- **작동 원리:** 이 명령을 내리면 `/etc/shadow` 파일의 비밀번호 필드 맨 앞에 느낌표(`!`)가 추가됩니다. 리눅스는 `!`를 보면 즉시 인증을 거부합니다.
- **잠금 해제 (Unlock):** `usermod -U [사용자명]` (또는 `passwd -u`)

**명령어 / 문법**

```bash
sudo usermod -L j.doe
sudo grep "^j.doe:" /etc/shadow # j.doe:!:20575:0:99999:7:::
```

**실습 메모**

퇴사자의 계정을 바로 삭제하면 그 사람이 작성해 둔 중요한 소스 코드나 로그 파일들의 소유권(UID)이 이름 없는 '고아 파일'로 변해버려 시스템 권한 관리가 크게 꼬이게 됩니다. 데이터 보존과 보안 침해를 동시에 막기 위한 '계정 잠금'은 인프라 보안 관리의 표준 절차(Standard Operating Procedure)입니다.
