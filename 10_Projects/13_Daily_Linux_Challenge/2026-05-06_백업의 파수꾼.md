## 📌 랩 목표

Linux 환경에서 시스템의 핵심 자산을 식별하고, `tar` 명령어를 이용해 데이터를 압축 백업 및 부분 복원하는 방법을 익힙니다. 또한 `cron` 스케줄러를 활용해 백업 작업을 자동화하는 실무 필수 관리 기술을 실습합니다.

---

## 📖 학습 내용


---

### 🔷 섹션 1 — 백업을 위한 핵심 데이터 식별

**개념 요약**

- **백업 대상 식별:** 애플리케이션 운영에 필수적인 디렉토리(`data`, `config`, `logs`)를 명확히 파악하는 것이 백업의 첫걸음입니다.
- **매니페스트 파일(backup-list.txt) 생성:** 매번 백업할 경로를 길게 입력하는 대신, 백업 대상 목록을 담은 텍스트 파일을 만들어 두면 관리가 훨씬 쉽고 실수를 방지할 수 있습니다.
- 텍스트 편집기 `nano`를 활용해 목록 파일을 작성합니다. (저장 및 종료: `Ctrl+X` → `Y` → `Enter`)

**명령어 / 문법**

```bash
nano backup-list.txt
```

**실습 메모**

- 실무에서는 백업 대상이 수십 개에 달할 수 있습니다. 명령줄 한 줄에 모든 경로를 욱여넣는 것은 오타를 유발하기 쉬우므로, 이렇게 리스트 파일을 따로 만들어 관리하는 것이 훨씬 안전하고 유지보수하기 좋은 '엔지니어의 기본기'입니다.

---

### 🔷 섹션 2 —전체 시스템 백업 아카이브 생성

**개념 요약**

- **`tar` (Tape Archive) 명령어:** 여러 파일과 폴더를 하나의 아카이브 파일로 묶어줍니다. 여기에 `gzip`을 결합하면 용량까지 크게 압축할 수 있습니다
- 핵심 옵션 조합 (`-czvf`):
	- `c` (Create): 새로운 아카이브를 생성합니다.
	- `z` (Zip): `gzip` 포맷으로 압축하여 디스크 공간을 절약합니다.
	- `v` (Verbose): 어떤 파일들이 묶이고 있는지 화면에 자세히 보여줍니다.
	- `f` (File): 저장될 아카이브 파일의 이름을 지정합니다. **(반드시 옵션의 맨 마지막에 위치해야 합니다)**
- **`-T` 옵션:** 직접 경로를 입력하지 않고, 앞서 만든 `backup-list.txt` 파일 안의 목록을 읽어와서 백업을 수행합니다.

**명령어 / 문법**

```bash
tar -czvf backups/system-backup.tar.gz -T backup-list.txt
```

**실습 메모**

실무에서 용량 절약을 위해 `c`(묶기)와 `z`(압축하기)는 거의 항상 세트로 다닙니다. 특히 백업이 잘 진행되고 있는지 눈으로 확인시켜 주는 `v` 옵션은 무거운 작업을 걸어둘 때 꼭 넣는 편입니다.

---

### 🔷 섹션 3 — 백업 무결성 검증

**개념 요약**

- 백업 파일이 성공적으로 만들어졌는지, 그리고 내용물이 온전한지 검증하는 단계입니다.
-  **`t` (List) 옵션:** 압축을 실제로 풀지 않고도 아카이브 안에 어떤 파일들이 들어있는지 **목록만 미리보기**할 수 있는 강력한 옵션입니다.
- 리다이렉션(`>`)을 사용하여 화면에 출력될 그 긴 목록을 `backup-contents.txt`라는 파일로 저장해 검토할 수 있습니다.

**명령어 / 문법**

```bash
tar -tzvf backups/system-backup.tar.gz > backup-contents.txt
cat backup-contents.txt
```

**실습 메모**

현업에서 백업 파일은 '만드는 것'보다 '정상 복원이 가능한지 확인하는 것'이 100배 더 중요합니다. 껍데기만 만들어지고 속은 비어있는 사고를 막기 위해, 생성 직후 `-t` 옵션으로 내용물이 꽉 차 있는지 교차 검증하는 습관을 들여야 합니다.

### 🔷 섹션 4 — 백업에서 파일 복원하기

**개념 요약**

- 실수로 주요 설정 파일(`app.conf`)이 날아간 치명적인 상황을 가정합니다
- **`x` (eXtract) 옵션:** `tar` 아카이브 파일에서 데이터를 추출(복원)합니다.
- **부분 복원:** 아카이브 전체를 복원할 필요 없이, 명령어 맨 뒤에 살리고 싶은 특정 파일의 경로(`config/app.conf`)만 명시하면 **그 파일 단 하나만 쏙 골라서 복구**할 수 있습니다.

**명령어 / 문법**

```bash
rm ~/project/config/app.conf
ls -l ~/project/config/
tar -xzvf backups/system-backup.tar.gz config/app.conf
ls -l ~/project/config/app.conf 
cat ~/project/config/app.conf
```

**실습 메모**

서비스 장애 시 수십 GB짜리 전체 백업 파일을 풀고 있으면 복구 시간(Downtime)이 한없이 길어집니다. 장애 원인이 된 파일 단 한 개만 스나이퍼처럼 콕 집어서 복원하는 이 기술은 장애 발생 시 여러분의 퇴근 시간을 지켜주는 1등 공신입니다.

### 🔷 섹션 5 — 자동 백업 작업 예약

**개념 요약**

- **`cron` 스케줄러:** 백업은 사람의 손을 떠나 정기적이고 자동으로 실행되어야 합니다. `crontab -e`를 사용하여 자동화 일정을 등록합니다.
- **동적 파일명 생성:** 계속 같은 이름으로 덮어씌워지는 것을 막기 위해 `$(date +\%Y-\%m-\%d_\%H-\%M-\%S)` 명령을 조합하여 백업 파일명에 현재 날짜와 시간(타임스탬프)이 자동으로 찍히도록 설정합니다.
- _주의:_ `cron` 작업 안에서 `%` 기호는 줄바꿈으로 인식될 수 있으므로, 반드시 앞에 역슬래시(`\`)를 붙여 이스케이프 처리를 해주어야 문자로 정상 인식됩니다.

**명령어 / 문법**

```bash
crontab -e
* * * * * tar -czf /home/labex/project/backups/backup-$(date +\%Y-\%m-\%d_\%H-\%M-\%S).tar.gz -C /home/labex/project data config logs
ls -l ~/project/backups
```

**실습 메모**

"기억에 의존하는 백업은 이미 망한 백업이다"라는 실무 격언이 있습니다. `cron`을 통한 무인 자동화와, 덮어쓰기 방지를 위한 '날짜 기반 네이밍 룰' 적용은 서버를 운영하는 모든 기업 인프라 환경에서 1순위로 세팅하는 핵심 방어 체계입니다.
