## 📌 랩 목표

Red Hat 계열 Linux 시스템에서 `yum` 패키지 관리자를 활용하여 설치된 패키지를 조회하고, 저장소(Repository)를 탐색하며, 시스템 및 개별 패키지를 안전하게 업데이트하는 방법을 실습하고 이해합니다.

## 📖 학습 내용

---

### 🔷 섹션 1 — 'yum list' 및 'yum deplist'로 설치된 패키지 조사하기

**개념 요약**

- `yum`을 사용하여 특정 패키지의 설치 상태와 버전을 확인하는 방법을 배웁니다.
- `yum list` 명령어를 실행하면 해당 패키지가 시스템에 설치되어 있는지, 그리고 어느 버전인지 알려줍니다. 출력 결과에서 `@` 기호(예: `@anaconda`, `@System`)가 붙어 있다면 이미 설치된 패키지임을 의미합니다.
- Linux의 대부분의 소프트웨어는 단독으로 동작하지 않고 여러 라이브러리와 도구에 의존합니다. 이를 '의존성(Dependency)'이라고 하며, `yum deplist` 명령어를 통해 특정 패키지가 작동하기 위해 필요한 전체 의존성 목록을 확인할 수 있습니다.

**명령어 / 문법**

```bash
sudo yum list bash # bash.x86_64 <version> @anaconda
sudo yum deplist bash
```

**실습 메모**

- 실무에서 특정 프로그램을 설치하거나 실행할 때 "라이브러리가 없다"는 에러를 마주치는 경우가 많습니다. 이때 `deplist`를 활용하면 어떤 의존성 패키지가 누락되었는지 역추적하여 문제를 빠르고 정확하게 해결(Troubleshooting)할 수 있습니다.

### 🔷 섹션 2 —  'yum repolist'로 저장소 및 사용 가능한 패키지 탐색하기

**개념 요약**

- `yum`은 사용자가 패키지 설치를 요청하면, 시스템에 구성된 '저장소(Repository)'에 접속하여 파일을 다운로드하고 설치를 진행합니다.
- `yum repolist` 명령어는 현재 시스템에 연결되어 활성화된 소프트웨어 저장소의 목록을 보여줍니다.
- `yum list available`은 저장소에는 존재하지만 내 시스템에는 아직 설치되지 않은 패키지 목록을 출력합니다.
- 패키지가 너무 많아 검색이 필요할 때 와일드카드(`*`)를 활용합니다. 단, 쉘(Shell)이 와일드카드를 임의로 해석하지 못하도록 작은따옴표(`'kernel*'`)로 감싸서 검색하는 것이 중요합니다.

**명령어 / 문법**

```bash
sudo yum repolist
sudo yum list available 'kernel*'
```

**실습 메모**

- 외부 인터넷이 차단된 '폐쇄망' 환경에서 서버를 구축할 때는 내부 사설망에 로컬 저장소를 만들어야 합니다. 이때 `repolist`를 쳐서 내가 설정한 내부 저장소가 정상적으로 등록되고 패키지 개수가 제대로 뜨는지 확인하는 것이 첫 번째 검증 단계입니다.

### 🔷 섹션 3 — 'yum check-update'로 사용 가능한 업데이트 확인하기

**개념 요약**

- `yum check-update` 명령어는 시스템에 설치된 패키지 중 더 최신 버전이 저장소에 등록되어 있는 패키지가 무엇인지 확인합니다.
- 이 명령어는 실제 다운로드나 설치를 진행하지 않고 단순 조회만 하므로 매우 안전합니다.
- 패키지 정보의 로컬 캐시를 갱신하며, 패키지 이름, 업데이트될 새 버전, 그리고 해당 업데이트를 제공하는 저장소 정보를 나열해 주어 관리자가 사전에 검토할 수 있게 합니다.

**명령어 / 문법**

```bash
sudo yum check-update
```

**실습 메모**

- 라이브 서비스가 돌아가는 운영 서버에서는 단 하나의 패키지 업데이트만으로도 서비스가 중단될 위험이 있습니다. 따라서 `check-update`로 현재 취약점 패치나 주요 업데이트가 있는지 정기적으로 모니터링하고, 안전한 유지보수 계획을 세우는 데 필수적인 명령어입니다

### 🔷 섹션 4 —  'yum update '로 단일 패키지 업데이트 적용하기

**개념 요약**

- 시스템 전체를 업데이트하는 대신, 필요에 따라 특정 패키지 하나만 콕 집어서 업데이트할 수 있습니다.
- 명령어 뒤에 패키지명을 명시하면 해당 패키지와 연관된 의존성만 계산하여 업데이트를 진행합니다.
- (실습 참고: 업데이트 과정을 직접 눈으로 확인하기 위해, 먼저 `yum downgrade` 명령을 사용해 `curl` 패키지를 구버전으로 강제 다운그레이드한 후 업데이트를 진행합니다.)

**명령어 / 문법**

```bash
sudo yum downgrade -y curl
sudo yum update curl
```

**실습 메모**

- 특정 데몬(예: Apache HTTPD, MySQL)에서 치명적인 보안 취약점(CVE)이 발견되었을 때, 전체 시스템을 건드리는 위험을 감수하지 않고 딱 해당 서비스의 패키지만 신속하게 패치해야 할 때 유용하게 쓰입니다.

### 🔷 섹션 5 — 'yum update'로 전체 시스템 업데이트 수행하기

**개념 요약**

- 명령어 뒤에 특정 패키지 이름을 지정하지 않고 `yum update`만 입력하면, 시스템에 설치된 모든 패키지를 대상으로 활성화된 저장소와 버전을 비교해 일괄 업데이트를 시도합니다.
- 수많은 패키지의 의존성을 한 번에 해결한 뒤, 화면에 '트랜잭션 요약(Transaction Summary)'을 출력하여 어떤 파일들이 변경될지 보여줍니다.
- 많은 용량의 데이터가 다운로드되고 설치되므로 시스템 리소스와 네트워크 상황에 따라 시간이 오래 걸릴 수 있습니다.

**명령어 / 문법**

```bash
sudo yum update
```

**실습 메모**

- 이 명령어는 주로 운영체제를 갓 설치한 초기 세팅 단계나, 충분히 테스트가 완료된 개발/스테이징 서버에서 사용합니다. 운영 서버에서 수행할 경우 예상치 못한 호환성 충돌이 일어날 수 있으므로, 반드시 스냅샷이나 데이터 백업을 선행한 후 진행해야 하는 무거운 작업입니다.
