## 📌 랩 목표

Linux 환경에서 원격 접속의 표준인 OpenSSH 서버를 구축하고, 사용자 계정을 생성하여 로컬 및 원격 네트워크 환경에서 안전하게 서버에 접속(SSH)하고 제어하는 방법을 실습합니다.

## 📖 학습 내용


---

### 🔷 섹션 1 — OpenSSH 서버 설치 및 설정

**개념 요약**

- **설치:** 외부 SSH 접속을 받기 위해 `openssh-server` 패키지를 설치합니다. 설치와 동시에 `sshd` 데몬(서비스)이 자동으로 백그라운드에서 실행됩니다.
- **상태 확인:** `systemctl status ssh` 명령어로 서비스가 정상적으로 동작 중인지(`active (running)`) 확인합니다.
- **종료:** 상태 확인 화면에서 빠져나오려면 키보드 `q`를 누릅니다.

**명령어 / 문법**

```bash
sudo apt-get update 
sudo apt-get install -y openssh-server
sudo systemctl status ssh
```

**실습 메모**

- 실무 환경에 투입되어 새로운 서버를 프로비저닝할 때 가장 먼저 세팅하게 되는 것이 바로 SSH입니다. 서비스가 `active` 상태인지 눈으로 직접 확인하는 습관은 장애 발생 시 네트워크 문제인지, 데몬 문제인지 원인을 좁히는 훌륭한 첫걸음이 됩니다.

---

### 🔷 섹션 2 — SSH 실습을 위한 새 사용자 생성

**개념 요약**

- **사용자 생성:** `adduser [사용자명]` 명령어로 새로운 계정을 만듭니다. (비밀번호 지정 필수, 나머지 정보는 Enter로 생략 가능)
- **계정 확인:** `/etc/passwd` 파일을 조회(`grep` 활용)하여 계정이 시스템에 정상 등록되었는지 검증합니다.
- **권한 구조:** 생성된 사용자의 홈 디렉토리(`/home/sshuser`)는 본인과 소속 그룹만 접근 가능하도록 제한적 권한(`drwxr-x---`)이 부여되므로, 다른 관리자가 보려면 `sudo` 권한이 필요합니다.

**명령어 / 문법**

```bash
sudo adduser sshuser
grep sshuser /etc/passwd # sshuser:x:1000:1000:,,,:/home/sshuser:/bin/bash
sudo ls -la /home/sshuser
```

**실습 메모**

- 보안 가이드라인이 엄격한 기업 인프라에서는 보안 사고를 막기 위해 `root` 계정의 직접적인 SSH 접속을 차단합니다. 따라서 용도와 권한에 맞는 별도의 일반 사용자 계정을 생성하여 접속 권한을 부여하는 것이 실무 서버 관리의 기본 원칙입니다.

### 🔷 섹션 3 — SSH 서버의 IP 주소 확인

**개념 요약**

- 원격으로 접속하려면 목적지 서버의 IP 주소를 알아야 합니다.
- **`ip addr`:** 시스템의 모든 네트워크 인터페이스(기본 이더넷 `eth0`, 루프백 `lo`, 도커 등)의 상세 정보를 출력합니다. 여기서 `inet` 항목 옆의 주소가 IP입니다.
-  **`hostname -I`:** 복잡한 정보 없이 서버에 할당된 IP 주소들만 깔끔하게 나열하여 보여줍니다.

**명령어 / 문법**

```bash
ip addr
hostname -I # 172.16.50.3 172.17.0.1 
```

**실습 메모**

- 로컬 환경(`127.0.0.1`)과 외부망/내부망 IP를 확실히 구분하는 것은 네트워크 트러블슈팅의 핵심입니다. 특히 `hostname -I`는 여러 서버를 관리할 때 현재 서버의 IP만 스크립트 변수로 간결하게 뽑아내기 좋아 스크립트 작성 시 자주 애용됩니다.

### 🔷 섹션 4 — 원격 서버에 대화형 SSH 세션 수립

**개념 요약**

- **접속 방법:** `ssh <사용자>@<IP>` 형식으로 명령을 내립니다. (실습에서는 내 PC에서 띄운 서버로 접속하므로 로컬 IP `127.0.0.1` 사용)
- **보안 확인:** 처음 접속하는 서버일 경우 "중간자 공격(MITM)" 방지를 위해 호스트 키(Fingerprint)를 신뢰할 것인지 묻습니다. `yes`를 입력해야 통과됩니다
- **작업 및 종료:** 비밀번호 입력 후 터미널 프롬프트가 원격 서버 이름으로 바뀐 것을 확인합니다. `pwd` 등으로 원격지임을 확인한 뒤, 작업이 끝나면 `exit`로 세션을 종료하고 원래 PC로 돌아옵니다.

**명령어 / 문법**

```bash
ssh sshuser@127.0.0.1
sshuser@127.0.0.1's password:

pwd # /home/sshuser
exit
```

**실습 메모**

- 여러 대의 서버를 띄워두고 작업하다 보면 현재 쉘이 내 로컬인지, 아니면 운영 서버인지 헷갈려 치명적인 명령어(`rm` 등)를 잘못 입력하는 실수가 종종 발생합니다. SSH 접속 직후 프롬프트를 확인하고 `hostname`을 쳐서 위치를 한 번 더 크로스체크하는 것은 훌륭한 엔지니어의 기본 소양

### 🔷 섹션 5 —  SSH 를 통한 원격 단일 명령 실행

**개념 요약**

- 굳이 대화형 쉘로 완전히 접속하지 않고, 원격 서버에서 특정 명령어 딱 하나만 실행하고 결과만 받아올 수 있습니다.
- **방법:** `ssh <사용자>@<IP> "실행할 명령어"` 형태로 명령어 끝에 따옴표(`""`)로 감싼 명령을 추가합니다.
- **장점:** 비밀번호 입력 후 명령어가 즉시 실행되고 결과값만 내 화면에 출력된 뒤 SSH 연결이 **자동으로 종료**됩니다. (`exit`를 칠 필요가 없습니다.)

**명령어 / 문법**

```bash
ssh sshuser@127.0.0.1 "hostname"
ssh sshuser@127.0.0.1 "ls -l /"
```

**실습 메모**

- 이 기능은 '시스템 자동화'의 꽃입니다. 여러 대의 서버 상태(예: 디스크 용량, 구동 중인 서비스 상태)를 점검해야 할 때, 일일이 접속하지 않고 로컬 PC에서 반복문(for) 스크립트를 짜서 원격 명령을 던지면 순식간에 수십 대의 서버 리포트를 수집할 수 있는 강력한 기술입니다.
