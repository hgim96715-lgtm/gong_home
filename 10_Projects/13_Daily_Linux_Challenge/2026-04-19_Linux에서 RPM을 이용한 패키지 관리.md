## 📌 랩 목표

Red Hat 계열 Linux 시스템(CentOS, Fedora, RHEL 등)에서 소프트웨어를 관리하는 `rpm` 패키지 관리자의 강력한 조회, 검증, 삭제 시뮬레이션 및 아카이브 추출 기능을 실습하고 이해합니다.

## 📖 학습 내용

### 🔷 섹션 1 — rpm -qi 및 rpm -qf를 이용한 기본 패키지 정보 조회

**개념 요약**

- `rpm`은 RPM 기반 Linux 배포판에서 소프트웨어를 관리하기 위한 명령줄 유틸리티입니다.
- 패키지 조회 시 `-q` (query) 플래그를 기본으로 사용합니다.
- `-qi` (query information): 특정 패키지의 상세 정보(버전, 설치 날짜, 크기 등)를 가져옵니다.
- `-qf` (query file): 시스템의 특정 파일이 어느 패키지를 설치할 때 생성된 것인지 출처 패키지를 확인합니다.

**명령어 / 문법**

```bash
rpm -qi bash
rpm -qf /usr/bin/bash
```

**실습 메모**

- 시스템을 관리하다 보면 출처나 용도를 알 수 없는 실행 파일이나 설정 파일을 마주칠 때가 있을때,  이때 `rpm -qf`를 사용하면 해당 파일이 어떤 패키지에 의해 설치되었는지 단번에 알 수 있어 트러블슈팅 시간을 크게 단축할 수 있다.

### 🔷 섹션 2 — rpm -qR 및 rpm -qc를 이용한 패키지 의존성 및 설정 파일 목록 확인

**개념 요약**

- `-qR` (query requires): 특정 패키지가 정상적으로 작동하기 위해 필요한 다른 패키지나 라이브러리(의존성) 목록을 보여줍니다. (예: bash 패키지는 glibc 등을 요구함)
- `-qc` (query configfiles): 특정 패키지가 시스템에 설치한 설정 파일들의 목록만 필터링해서 보여줍니다.
- `-qcf`: 특정 파일의 소속 패키지를 찾은 뒤, 해당 패키지의 설정 파일 목록을 한 번에 보여주는 조합도 가능합니다.

**명령어 / 문법**

```bash
rpm -qR bash
rpm -qc bash
rpm -qcf /usr/bin/passwd
/etc/pam.d/passwd
```

**실습 메모**

- 특정 서비스(예: 웹 서버, 데이터베이스)의 설정을 변경해야 하는데 설정 파일 경로가 기억나지 않을 때, 인터넷 검색에 의존하지 않고 `rpm -qc` 명령어로 시스템 내에서 직접 정확하게 찾아낼 수 있는 유용한 기술

### 🔷 섹션 3 —  `rpm -V`를 이용한 패키지 무결성 및 파일 변경 사항 검증

**개념 요약**

- `rpm -V` (verify): 설치된 패키지의 파일들이 원본 상태를 유지하고 있는지 무결성을 검증합니다. 점(`.`)으로 표시되면 원본과 동일하게 정상 통과했음을 의미합니다.
- 파일이 변경된 경우 출력되는 주요 플래그:
	- `S`: 파일 크기 (Size) 변경
	- `5`: MD5 체크섬 (Hash) 변경
	- `T`: 수정 시간 (Modification Time) 변경
	- `c`: 설정 파일 (Configuration file)임을 나타냄
- `-qVf`를 사용하면 특정 파일을 소유한 패키지 전체의 무결성을 검증할 수 있습니다.

**명령어 / 문법**

```bash
sudo dnf install -y lsscsi at
rpm -V lsscsi
echo "labex" | sudo tee -a /etc/at.deny
sudo rpm -V at # S.5....T. c /etc/at.deny
sudo rpm -qVf /etc/at.deny
```

**실습 메모**

- 보안 관리 측면에서 매우 중요한 명령어입니다. 해킹이나 악성코드로 인해 시스템 핵심 바이너리 파일이 변조되었는지 검사하거나, 다른 관리자가 실수로 중요한 설정 파일을 건드렸는지 감사(Audit)할 때 강력하게 활용

### 🔷 섹션 4 — rpm -e --test를 이용한 역의존성 확인 및 패키지 삭제 시뮬레이션

**개념 요약**

- `--whatrequires`: `-qR`의 반대 개념으로, "이 패키지를 요구하는 다른 패키지는 무엇인가?"(역의존성)를 확인합니다.
- `rpm -e --test`: `-e` (erase)로 패키지 삭제를 명령하되, `--test`를 붙여 실제로 파일을 삭제하지 않고 모의 실행(Dry run)만 수행합니다.
- 핵심 시스템 패키지(예: glibc)를 삭제하려 할 경우, 이를 요구하는 수많은 패키지들로 인해 오류가 발생하며 삭제가 차단되는 것을 확인할 수 있습니다.

**명령어 / 문법**

```bash
rpm -q --whatrequires glibc
sudo rpm -e --test glibc
rpm -q --whatrequires lsscsi # no package requires lsscsi
sudo rpm -e --test lsscsi
```

**실습 메모**

- 실무(운영 서버)에서 불필요해 보이는 패키지를 정리할 때 반드시 습관화해야 하는 명령어입니다. `--test` 옵션 없이 삭제를 진행했다가 다른 중요 서비스가 함께 멈춰버리는 대형 사고를 미연에 방지할 수 있습니다.

### 🔷 섹션 5 — rpm2cpio 및 cpio 를 이용한 RPM 패키지 내용물 조사

**개념 요약**

- 시스템에 패키지를 실제로 설치하지 않고도 그 안에 어떤 파일들이 들어있는지 확인할 수 있는 고급 기술입니다.
- `rpm2cpio`: RPM 파일을 읽어서 표준 출력 형태의 cpio 아카이브 포맷으로 변환해 줍니다.
- `cpio -t`: 변환된 아카이브 데이터에서 목차(Table of contents)만 나열하도록 지시합니다. 파이프(`|`)를 통해 두 명령어를 연결하여 사용합니다.

**명령어 / 문법**

```bash
sudo dnf download bash
ls bash-*.rpm
rpm2cpio bash-*.rpm | cpio -t
```

**실습 메모**

- 시스템의 특정 실행 파일이나 라이브러리가 하나만 손상 또는 삭제되었을 때, 패키지 전체를 재설치하면 기존 설정이 날아갈 위험이 있습니다. 이때 이 기술을 응용(`cpio -idv`)하여 RPM 파일에서 필요한 파일 딱 하나만 쏙 추출해내어 복구하는 고급 트러블슈팅이 가능합니다.
