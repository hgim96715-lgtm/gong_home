## 📌 랩 목표

Linux 환경에서 배포된 소스 코드 아카이브를 다운로드하여 압축을 풀고, 시스템 환경에 맞게 구성(`configure`)한 뒤, 직접 컴파일(`make`)하여 설치(`make install`) 및 제거(`make uninstall`)하는 일련의 수동 빌드 파이프라인을 실습하고 이해합니다.

### 🔷 섹션 1 — tar를 사용하여 소스 코드 추출하기

**개념 요약**

- 소프트웨어 소스 코드는 필요한 모든 파일을 하나로 묶고 전체 파일 크기를 줄이기 위해 `.tar.gz`와 같은 압축 아카이브 파일 형태로 배포되는 경우가 많습니다.
- `ls -l` 명령어를 사용하여 파일과 세부 정보를 나열합니다.
- `tar` 명령어를 사용하여 이 아카이브의 내용을 추출합니다. 사용할 옵션은 다음과 같습니다.
	- `-x`: 아카이브에서 파일을 추출(extract) 합니다.
	- `-z`: `.gz` 확장자로 표시되는 `gzip`을 사용하여 아카이브의 압축을 풉니다.
	- `-v`: 상세(verbose) 출력 모드로, 추출되는 모든 파일을 나열합니다.
	- `-f`: 처리할 아카이브의 파일명(filename)을 지정합니다.
- `-v` 옵션을 사용하면 tar가 추출하는 각 파일의 이름을 출력합니다.
- 명령어가 완료되면 소스 코드가 포함된 새 디렉토리가 생성됩니다.


**명령어 / 문법**

```bash
ls -l
tar -zxvf pure-ftpd-1.0.53.tar.gz
ls -l
```

**실습 메모**

- 실무에서 다운로드한 압축 파일을 풀 때 `-v` (verbose) 옵션을 습관화하면 좋습니다. 수많은 파일이 풀리는 과정을 눈으로 확인할 수 있어, 만약 중간에 용량 부족이나 권한 문제로 멈췄을 때 어느 지점에서 오류가 났는지 즉각적으로 파악하기 유리

### 🔷 섹션 2 — ./configure와 make를 사용하여 소프트웨어 구성 및 컴파일하기

**개념 요약**

- 첫 번째 과정은 `configure` 스크립트를 실행하는 것입니다. 이 스크립트는 소프트웨어를 빌드하는 데 필요한 모든 도구, 라이브러리 및 종속성이 시스템에 있는지 확인합니다.
- 스크립트를 실행하려면 이름 앞에 `./`를 입력해야 합니다. 이는 셸에게 현재 디렉토리에서 `configure` 스크립트를 찾으라고 지시하는 것입니다.
- `config.status: creating Makefile` 라인을 확인하세요. 이는 `Makefile`이 성공적으로 생성되었음을 의미합니다.
- `make` 명령어는 `Makefile`을 읽고 컴파일러(예: gcc)를 올바른 옵션으로 실행하여 사람이 읽을 수 있는 소스 코드(`.c` 파일)를 기계가 실행할 수 있는 바이너리 파일로 변환합니다.
- `make` 명령어가 오류 없이 완료되면 소프트웨어가 성공적으로 컴파일된 것입니다. 이제 바이너리 파일을 시스템에 설치할 준비가 되었으며, 다음 단계에서 이를 수행합니다.

**명령어 / 문법**

```bash
cd ~/project/pure-ftpd-1.0.53
./configure
make
```

**실습 메모**

- 소스 컴파일 설치를 할 때 가장 많은 에러를 마주하는 구간이 바로 `./configure` 단계입니다. C 컴파일러(gcc)가 없거나 필수 라이브러리가 누락되었다면 여기서 멈춥니다. 에러 메시지를 꼼꼼히 읽고 부족한 패키지를 `yum`이나 `apt`로 먼저 채워주는 트러블슈팅 능력이 매우 중요

### 🔷 섹션 3 — make install을 사용하여 소프트웨어 설치하기

**개념 요약**

- `make install` 명령어는 이 파일들을 표준 시스템 전체 경로로 복사하여 어디서든 소프트웨어를 실행할 수 있게 합니다.
- 설치 과정은 일반적으로 root 사용자가 소유한 `/usr/local/bin` 및 `/usr/local/sbin`과 같은 디렉토리에 파일을 써야 합니다. 따라서 관리자 권한으로 `make install`을 실행하려면 `sudo` 명령어를 사용해야 합니다.
- 이를 확인하는 간단한 방법은 시스템의 PATH에서 실행 파일을 검색하여 전체 경로를 출력하는 `which` 명령어를 사용하는 것입니다.

**명령어 / 문법**

```bash
sudo make install
which pure-ftpd
```

**실습 메모**

- 패키지 매니저(`apt`, `yum`)로 설치한 프로그램은 보통 `/usr/bin`에 들어가지만, 소스 코드로 직접 수동 설치한 프로그램은 대부분 `/usr/local/bin` 또는 `/usr/local/sbin`에 저장됩니다. 이 경로의 차이를 알아두면 시스템 기본 파일과 내가 수동으로 추가한 파일을 구분하여 관리하기 쉽다.

### 🔷 섹션 4 — make uninstall과 rm을 사용하여 소프트웨어 제거 및 정리하기

**개념 요약**

- `Makefile`을 사용하는 대부분의 잘 작성된 소스 패키지는 `uninstall` 타겟을 제공합니다. 이 타겟은 `make install`의 작업을 되돌려 시스템 디렉토리에 복사된 모든 파일을 제거하도록 설계되었습니다.
- 먼저 여전히 `~/project/pure-ftpd-1.0.53` 디렉토리에 있는지 확인하세요. 설치 과정과 마찬가지로 제거 과정도 관리자 권한이 필요합니다. `sudo`와 함께 `make uninstall` 명령어를 사용합니다.
- 소프트웨어가 제거되었는지 확인하려면 `which` 명령어를 다시 사용합니다.
- 소프트웨어가 제거되었으므로 마지막 단계는 소스 코드 디렉토리 자체를 삭제하는 것입니다. 디렉토리 안에 있는 동안에는 해당 디렉토리를 삭제할 수 없으므로, 먼저 프로젝트 루트 디렉토리로 이동합니다.
- `rm` 명령어에 `-r` (재귀적) 옵션을 사용하여 `pure-ftpd-1.0.53` 디렉토리와 그 안의 모든 내용을 삭제합니다.
- 성공적으로 수행되면 아무런 출력도 나타나지 않습니다. `~/project` 디렉토리의 내용을 나열하여 삭제되었는지 확인할 수 있습니다.

**명령어 / 문법**

```bash
sudo make uninstall
which pure-ftpd
cd ~/project
rm -r pure-ftpd-1.0.53
ls -l
```

**실습 메모**

- `make uninstall` 명령어가 정상적으로 작동하려면 우리가 실습했던 원본 소스 폴더와 그 안의 `Makefile`이 그대로 보존되어 있어야 합니다. 만약 설치 직후에 용량 확보를 위해 소스 폴더를 지워버리면, 나중에 언인스톨을 할 때 수동으로 파일 경로를 찾아가며 지워야 하는 대참사가 발생할 수 있으니 주의
