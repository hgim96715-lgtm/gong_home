## 📌 랩 목표

리눅스 환경에서 `sudo` 권한의 개념을 이해하고, 데이터 보호를 위한 필수 유틸리티인 `tar`를 활용하여 디렉터리 백업(아카이브)을 생성하고 안전한 임시 공간에 복구(추출)하는 전체 프로세스를 실습합니다.

### 🔷 섹션 1 —  현재 환경 이해하기

**개념 요약**

- `sudo`를 사용하는 것은 보안상의 모범 사례입니다. 사용자 컨텍스트를 유지하면서 특정 명령에 대해 일시적으로 승인된 권한을 제공하기 때문입니다. 이 방식은 root 사용자로 완전히 전환하는 것보다 안전하며, 실수로 위험한 명령을 실행할 위험을 줄여줍니다.
- `whoami`와 `pwd`(print working directory) 명령어를 실행하여 현재 사용자 ID 와 위치를 확인합니다.
- 관리자 권한이 필요한 간단한 명령어를 실행하여 `sudo` 권한을 테스트할 수 있습니다. 보통 승인된 권한이 필요한 `/root` 디렉토리를 확인해 보겠습니다.
- `sudo` 명령어는 파일 시스템의 제한된 영역에 접근하는 데 필요한 권한을 일시적으로 부여합니다.

**명령어 / 문법**

```bash
whoami # labex
pwd # /home/labex/project

sudo ls /root
```

**실습 메모**

- `sudo`는 무조건 최고 권한을 부여하는 것이 아니라, '필요한 순간에만 권한을 빌려오는' 안전장치입니다. `whoami`가 사용자 본인(`labex`)으로 나오더라도 `sudo`를 앞에 붙이면 `/root` 디렉터리처럼 굳게 닫힌 문도 열어볼 수 있습니다. 실무에서는 습관적인 root 계정 로그인을 지양하고 항상 `sudo`를 활용해야 합니다.

### 🔷 섹션 2 —  /home 디렉토리의 백업 아카이브 생성

**개념 요약**

-  `tar`(tape archiver) 유틸리티는 Linux 에서 아카이브 파일을 생성, 조회 및 추출하기 위한 표준 도구입니다.
- `sudo tar -cvf ~/backup.tar /home` 형식으로 사용합니다.
- `-c` (create): tar에게 새 아카이브를 생성하도록 지시합니다.
- `-v` (verbose): 아카이브에 추가되는 각 파일을 목록으로 보여줍니다. 작업 진행 상황을 확인하는 데 유용합니다. 화면에 긴 파일 목록이 스크롤되는 것을 볼 수 있습니다.
- `-f` (file): 아카이브의 파일 이름을 지정할 수 있게 합니다. 아카이브 이름이 바로 뒤에 와야 합니다. (`~/backup.tar`)
- `tar: Removing leading '/' from member names`라는 메시지는 정상입니다

**명령어 / 문법**

```bash
sudo tar -cvf ~/backup.tar /home 
```

**실습 메모**

- `-cvf` 옵션은 압축/백업 시 가장 자주 쓰이는 '국룰' 조합입니다. 실행 시 나오는 `Removing leading '/'` 경고 문구는 오류가 아닙니다. 나중에 압축을 풀 때 기존 `/home` 경로를 덮어써버려서 시스템이 망가지는 대참사를 막기 위해, 리눅스가 알아서 절대경로(`/home`)를 상대경로(`home/`)로 변환해 저장하는 아주 똑똑한 보호 메커니즘입니다.

### 🔷 섹션 3 —  backup.tar 파일 생성 확인

**개념 요약**

- 파일을 확인하는 가장 간단한 방법은 전체 경로와 함께 ls(list) 명령어를 사용하거나 홈 디렉토리로 이동하는 것입니다
- sudo를 사용하여 생성했기 때문에 파일 소유자는 root이지만, 접근 가능한 홈 디렉토리에 저장되어 있음을 유의하십시오.

**명령어 / 문법**

```bash
ls ~
ls -l ~/backup.tar # -rw-r--r-- 1 root root 10240 <Date> <Time> /home/labex/backup.tar
```

**실습 메모**

- 파일은 내 홈 디렉터리(`~`) 안에 예쁘게 생성되었지만, 생성할 때 `sudo` 명령어를 빌려 썼기 때문에 파일의 소유권은 내가 아닌 `root`에게 있습니다. 만약 나중에 이 파일을 다른 곳으로 옮기거나 삭제하려고 할 때 "권한 없음(Permission denied)" 에러가 뜬다면, 다시 `sudo`를 써야 한다는 점을 꼭 기억하세요.

### 🔷 섹션 4 — 복구를 위해 /tmp 디렉토리로 이동

**개념 요약**

- `/tmp` 디렉토리는 임시 파일을 위한 Linux 시스템의 표준 위치이며 이 목적에 완벽합니다

**명령어 / 문법**

```bash
cd /tmp 
pwd # /tmp
```

**실습 메모**

- 백업된 파일을 원래 있던 위치(홈 디렉터리)에 바로 풀면, 기존에 멀쩡히 쓰던 파일들까지 덮어씌워져 데이터가 꼬일 위험이 있습니다. 따라서 파일 복구 테스트를 하거나 필요한 파일만 빼낼 때는 시스템 재부팅 시 자동으로 비워지는 임시 놀이터인 `/tmp` 폴더를 활용하는 것이 매우 훌륭한 실무 습관입니다.

### 🔷 섹션 5 — 현재 디렉토리에 백업 아카이브 추출

**개념 요약**

-  아카이브를 추출하는 명령어는 `tar -xvf ~/backup.tar`입니다.
- `-x` (extract): tar에게 아카이브에서 파일을 추출하도록 지시합니다. 이것이 생성 명령어과의 주요 차이점입니다.
- `-v` (verbose): 이전과 마찬가지로 추출되는 각 파일을 나열하여 진행 상황을 보여줍니다
- `-f` (file): 처리할 아카이브 파일의 이름을 지정합니다. 홈 디렉토리에 있는 백업 파일을 참조하기 위해 `~/backup.tar`를 사용합니다.
- 추출 시에는 `sudo`가 필요하지 않습니다.

**명령어 / 문법**

```bash
tar -xvf ~/backup.tar
```

**실습 메모**

- 백업(생성)할 때 `-c`를 썼다면, 복구(추출)할 때는 `-x`로 딱 한 글자만 바꿔주면 됩니다. 현재 우리가 있는 곳은 누구나 자유롭게 읽고 쓸 수 있는 `/tmp` 디렉터리이므로, 압축을 풀 때 굳이 `sudo` 권한을 쓰지 않아도 에러 없이 매끄럽게 압축이 해제됩니다.

### 🔷 섹션 6 — /tmp 에 복구된 파일 확인

**개념 요약**

- 현재 `/tmp` 디렉토리에 있습니다. 새로 생성된 `home` 디렉토리를 보려면 `ls` 명령어를 사용하십시오.
- 존재할 수 있는 다른 임시 파일들 사이에서 `home` 디렉토리가 나열된 것을 볼 수 있어야 합니다.
- 몇 개의 파일을 비교하여 복구된 파일이 원본과 일치하는지 확인할 수 있습니다. 예를 들어, `.bashrc` 파일이 제대로 복구되었는지 확인할 수 있습니다.

**명령어 / 문법**

```bash
ls
ls home #labex
ls home/labex/.bashrc #home/labex/.bashrc
```

**실습 메모**

- 아까 섹션 2에서 `/`(선행 슬래시)를 제거하고 백업했던 그 안전 메커니즘이 여기서 빛을 발합니다. 기존의 실제 `/home` 디렉터리를 덮어쓰지 않고, 현재 위치인 `/tmp` 안에 새로운 `home/labex/...` 폴더 구조를 만들어 안전하게 파일들을 뱉어냈습니다. 숨김 설정 파일인 `.bashrc`까지 완벽하게 복구된 것을 확인했습니다.
