## 📌 랩 목표

- Linux 환경에서 SSH 공개키 인증 방식의 동작 원리를 이해한다.
- 사용자 계정 생성, RSA 키 쌍 생성, 공개키 등록 과정을 직접 실습한다.
- 비밀번호 로그인과 공개키 로그인의 차이를 확인한다
- 안전한 SSH 키 관리 방법(권한, passphrase, authorized_keys)을 익힌다.

### 🔷 섹션 1 — 실습용 사용자 생성 및 RSA 키 쌍 생성

**개념 요약**

- SSH 실습용 계정 `sshuser`를 새로 생성합니다.
- 사용자 생성 시 추가 정보(이름, 방 번호 등)는 선택 사항이므로 Enter로 건너뛸 수 있습니다.
- 키 파일이 해당 사용자 홈 디렉토리에 생성되도록 `sshuser` 계정으로 전환합니다.
- `ssh-keygen -t rsa` 명령어로 RSA 방식의 공개키/개인키 쌍을 생성
- 저장 경로는 기본값 `~/.ssh/id_rsa`를 그대로 사용합니다.
- 개인키 보호를 위해 passphrase로 `sshkey-password`를 설정
- 생성 후 `~/.ssh` 디렉토리에서 키 파일이 정상 생성되었는지 확인합니다.
- `id_rsa`는 개인키로 소유자만 접근 가능해야 하며, `id_rsa.pub`는 공개키로 배포 가능한 파일입니다.

**명령어 / 문법**

```bash
sudo adduser sshuser
sudo su - sshuser
ssh-keygen -t rsa
ls -l ~/.ssh
```

**실습 메모**

- 사용자마다 별도의 SSH 키를 가질 수 있으며 홈 디렉토리에 저장됩니다.
- 개인키는 절대 공유하면 안 되며 유출 시 계정 보안에 큰 문제가 발생합니다.
- `ls -l`로 권한을 확인하는 습관이 중요

### 🔷 섹션 2 — ssh-copy-id 를 사용하여 서버에 공개키 복사하기

**개념 요약**

- `ssh-copy-id`는 공개키를 원격 서버에 자동 등록해 주는 도구
- 이번 실습에서는 `localhost`를 서버처럼 사용하여 자기 자신에게 키를 등록합니다.
- `ssh-copy-id sshuser@localhost` 실행 시 로컬 공개키(`id_rsa.pub`)를 찾아 서버에 복사합니다.
- 공개키는 서버의 `~/.ssh/authorized_keys` 파일에 저장됩니다.
- `authorized_keys`에 등록된 공개키와 개인키가 짝이 맞으면 SSH 로그인 시 인증이 가능합니다.
- `~/.ssh` 디렉토리가 없으면 자동 생성되며 필요한 권한도 함께 설정됩니다.

**명령어 / 문법**

```bash
sudo su - sshuser
ssh-copy-id sshuser@localhost
cat ~/.ssh/authorized_keys
```

**실습 메모**

- `authorized_keys` 파일은 서버 측 로그인 허용 목록 역할을 합니다.
- `ssh-copy-id`를 사용하면 직접 파일을 복사하지 않아도 되어 편리

### 🔷 섹션 3 — 공개키 인증을 이용한 SSH 로그인 테스트

**개념 요약**

- 여전히 `sshuser` 계정 상태인지 확인합니다.
- `ssh sshuser@localhost` 명령어로 SSH 접속을 시도
- 계정 비밀번호 대신 개인키의 passphrase 입력을 요구받습니다.
- 서버가 공개키를 확인하고 클라이언트가 개인키로 인증했음을 의미
- `echo $SSH_CONNECTION`으로 SSH 연결 정보(클라이언트/서버 IP, 포트)를 확인할 수 있습니다.
- `exit` 명령어로 SSH 세션을 종료하고 원래 터미널로 돌아옵니다.

**명령어 / 문법**

```bash
whoami # sshuser
sudo su - sshuser
ssh sshuser@localhost
echo $SSH_CONNECTION
exit
```

**실습 메모**
- 비밀번호 로그인보다 공개키 인증 방식이 더 안전하고 자동화에 유리합니다.
- passphrase는 계정 비밀번호와 다르며 개인키 자체를 보호
