### 🔷 섹션 1 —  파일에서 기본 검색 수행하기

**개념 요약**

- `grep`의 기본 구문은 `grep PATTERN [FILE...]` 형태를 가집니다.
- 지정한 문자열(패턴)이 포함된 행 전체를 화면에 출력합니다.
- 파일 경로를 띄어쓰기로 구분하여 여러 파일에서 동시에 검색할 수도 있습니다. 이 경우 출력 결과 앞에 해당 패턴이 발견된 파일명이 함께 표시됩니다.

**명령어 / 문법**

```bash
grep labex /etc/passwd

sudo grep labex /etc/passwd /etc/shadow /etc/group
```

**실습 메모**

- **권한 주의:** `/etc/shadow`처럼 관리자 권한이 필요한 파일은 일반 사용자로 `grep`을 실행하면 `Permission denied` 에러가 발생하므로 반드시 `sudo`를 붙여야 함.
- **대소문자 구분:** `grep`은 기본적으로 대소문자를 엄격하게 구분함. 대소문자 구분 없이 검색하려면 `-i` 옵션을 추가하면 됨. (예: `grep -i labex`)

### 🔷 섹션 2 —  -n 옵션으로 행 번호 표시하기

**개념 요약**

- `-n` 옵션(number)은 `grep`이 출력의 각 행 앞에 입력 파일의 해당 행 번호를 콜론(`:`)과 함께 붙이도록 지시합니다.
- 스크립트 작성이나 설정 파일 디버깅 시, 문제가 발생한 정확한 위치를 파악하는 데 필수적인 옵션입니다.

**명령어 / 문법**

```bash
sudo grep -n labex /etc/passwd /etc/shadow /etc/group
```

**실습 메모**

- **결과 해석:** 출력 결과가 `/etc/passwd:32:labex...` 형태로 나온다면, `/etc/passwd` 파일의 32번째 줄에 `labex`가 존재한다는 의미임.
- 편집기(예: `vi`)를 열 때 해당 줄 번호로 바로 이동(`vi +32 /etc/passwd`)할 수 있어 작업 속도를 크게 높여줌.

### 🔷 섹션 3 — ^ 및 $ 앵커를 이용한 행 위치 매칭

**개념 요약**

- 앵커(Anchor)는 특정 문자가 아닌 **'위치'** 를 지정하는 기호입니다.
- `^` (캐럿): 행의 맨 시작 부분과 일치합니다. (예: `^root`는 줄이 root로 시작할 때만 매칭)
- `$` (달러 기호): 행의 맨 끝 부분과 일치합니다. (예: `bash$`는 줄이 bash로 끝날 때만 매칭)

**명령어 / 문법**

```bash
# 문자열 'root'가 줄의 어디에 있든 검색
grep root /etc/passwd

# 줄이 'root'로 시작하는 경우만 검색 (일반적으로 root 계정 정보)
grep ^root /etc/passwd

# 줄이 'bash'로 끝나는 경우만 검색 (기본 셸이 bash인 계정들)
grep bash$ /etc/passwd
```

**실습 메모**

- 단순히 `grep root`를 하면 `myroot`, `root123` 등 의도하지 않은 결과가 섞여 나올 수 있음. 앵커를 사용하면 검색의 정확도가 비약적으로 상승함.
- 팁: `^$`를 조합해서 사용하면 '빈 줄(시작과 끝 사이에 아무것도 없음)'을 의미함. `grep -v ^$` 로 사용하면 빈 줄을 모두 제외하고 출력할 수 있음.

### 🔷 섹션 4 — 기본 정규 표현식으로 패턴 매칭하기

**개념 요약**

- **별표 (`*`)**: 바로 앞에 있는 문자가 **0번 이상 반복**되는 것과 일치합니다.
	- 'roo*' 패턴은 `ro` 뒤에 `o`가 없거나(ro), 한 개 있거나(roo), 여러 개 있는(rooo) 행을 모두 검색합니다.
- **마침표 (`.`)**: 새줄 문자를 제외한 **임의의 단일 문자 딱 1개**와 일치합니다.
	- 'ro.' 패턴은 `roa`, `rob`, `ro1`, `ro_` 등 ro 뒤에 정확히 하나의 문자가 오는 행을 검색합니다.

**명령어 / 문법**

```bash
grep 'roo*' /etc/passwd
grep 'ro.' /etc/passwd
```

**실습 메모**

- **흔한 착각 주의:** 리눅스 파일 검색 명령어(`find`나 `ls`)에서 쓰는 `*`(모든 것)와 정규식의 `*`는 완전히 다름! 정규식에서 모든 문자를 표현하고 싶다면 `.*` (임의의 문자 1개가 0번 이상 반복) 형식으로 써야 함.
- 정규식을 사용할 때는 셸(Shell)이 특수문자를 오해하지 않도록 반드시 패턴을 작은따옴표(`' '`)로 감싸주는 습관을 들이는 것이 좋음.

### 🔷 섹션 5 — 복잡한 검색을 위해 확장 정규 표현식 사용하기

**개념 요약**

- `grep -E` (Extended Regular Expression) 옵션을 사용하면 더 직관적이고 강력한 기호(`{}`, `|`, `+`, `?`)를 백슬래시(`\`) 없이 바로 사용할 수 있습니다. (과거에는 `egrep` 명령어를 썼으나, 현재는 `grep -E`가 표준입니다.)
- **중괄호 (`{}`)**: 문자가 나타나야 하는 **정확한 횟수나 범위**를 지정합니다. (예: `o{2}`는 'o'가 연속으로 딱 2번 나옴)
- **교체 (`|`)**: 파이프 기호를 사용하여 **'OR (또는)'** 연산을 수행합니다. 여러 개의 키워드 중 하나라도 일치하면 출력합니다.

**명령어 / 문법**

```bash
# 'o'가 정확히 2번 반복되는 문자열('oo') 검색
grep -E 'o{2}' /etc/passwd

# 'root' 또는 'Root' 문자열이 포함된 행 검색
grep -E 'root|Root' /etc/passwd
```

**실습 메모**

- `|` 연산자는 여러 개의 조건으로 로그를 걸러낼 때 매우 유용함. (예: `grep -E 'ERROR|WARN|FAIL' app.log`)
- `-E` 옵션을 빼먹고 기본 `grep`에서 `{}`나 `|`를 쓰면 그냥 일반 문자로 취급해 버리므로 원하는 결과가 나오지 않음. 확장이 필요할 땐 반드시 `-E`를 챙길 것.
