### 🔷 섹션 1 —  `&` 와 `jobs` 를 이용한 백그라운드 프로세스 시작 및 확인

**개념 요약**

- 리눅스 셸에서는 보통 명령어를 실행하고 해당 작업이 완료될 때까지 기다린 후에야 다시 프롬프트가 나타납니다. 이를 **포그라운드 (foreground)** 에서 프로세스를 실행한다고 합니다.
- 시간이 오래 걸리는 작업의 경우, 터미널에서 다른 명령어를 계속 입력할 수 있도록 **백그라운드 (background)** 에서 실행하는 것이 효율적입니다.
- 명령어를 백그라운드에서 실행하려면 명령어 끝에 앰퍼샌드 (`&`)를 붙이기만 하면 됩니다
- `[1]`: 셸이 각 백그라운드 프로세스에 할당하는 고유한 **작업 ID (Job ID)** 입니다.
- 그 뒤에 오는 숫자: **프로세스 ID (PID)** 입니다.
- `jobs` 명령어를 사용하면 현재 셸 세션의 백그라운드에서 실행 중인 모든 작업을 나열할 수 있습니다.


**명령어 / 문법**

```bash
sleep 300 & #[1] 482
jobs #[1]  + running    sleep 300
```

**실습 메모**

- 대용량 파일 복사나 압축처럼 오래 걸리는 명령을 실행할 때 `&`를 붙이면, 터미널 창이 멈춰있지 않고 바로 다음 명령을 칠 수 있어 작업 효율이 크게 올라간다.
- PID와 Job ID는 다르며, 셸 내부 제어에서는 Job ID가 더 간단하게 쓰임.

### 🔷 섹션 2 — ps 를 이용한 실행 중인 프로세스 조사

**개념 요약**

- `jobs`는 현재 세션으로 범위가 제한되지만, `ps` (process status)는 시스템에서 실행 중인 모든 프로세스를 더 넓은 관점에서 볼 수 있는 강력한 도구입니다.
- 가장 흔히 사용되는 유용한 조합은 **`ps aux`** 입니다.
	- **a:** 모든 사용자의 프로세스를 표시합니다.
	- **u:** 사용자 중심 형식으로 표시합니다 (사용자, CPU%, MEM% 등 포함).
	- **x:** 터미널에 연결되지 않은 프로세스까지 포함합니다.
- 프로세스를 확인하는 또 다른 대중적인 형식은 **`ps -ef`** 입니다.
	- **-e:** 시스템의 모든 프로세스를 선택합니다.
	- **-f:** 부모 프로세스 ID (PPID) 와 같은 유용한 정보를 포함하는 "전체 (full)" 형식으로 리스트를 표시합니다.

**명령어 / 문법**

```bash
ps
ps aux 
ps aux | grep sleep

ps -ef | grep sleep
```

**실습 메모**

- `jobs`가 현재 창의 작업만 보여준다면, `ps`는 윈도우의 작업 관리자처럼 시스템 전체를 보여줌.
- 리소스(CPU, RAM) 사용량을 볼 때는 `aux`가 유리하고, 어떤 프로세스가 어떤 프로세스를 실행시켰는지(부모-자식 관계) 구조를 볼 때는 `-ef`가 더 적합함

### 🔷 섹션 3 — top 을 이용한 시스템 리소스 모니터링

**개념 요약**

- `top`은 시스템 활동을 동적으로 계속 업데이트하여 보여주는 대화형 도구입니다.
- 리소스를 많이 점유하는 프로세스를 즉각적으로 식별하는 데 매우 유용합니다.
- **요약 영역 (Summary area):** 시스템 가동 시간, 작업 수, CPU 부하, 메모리 사용량 등 전반적인 통계를 보여줍니다.
- **프로세스 목록 (Process list):** 개별 프로세스를 보여주며, 기본적으로 CPU 사용량 순으로 정렬됩니다.
- 단축키 (대소문자 구분):
	- **대문자 `M`:** 메모리 사용량 (`%MEM`) 순으로 정렬합니다.
	- **대문자 `P`:** CPU 사용량 (`%CPU`) 순으로 기본 정렬합니다.
	- **방향키:** 위/아래 화살표로 목록을 스크롤 할 수 있습니다.
	- **`q`:** `top` 인터페이스를 빠져나와 명령 프롬프트로 돌아갑니다.

**명령어 / 문법**

```bash
top
```

**실습 메모**

- `ps`가 특정 순간의 스냅샷이라면, `top`은 실시간 스트리밍 방식임. 서버가 갑자기 느려졌을 때 `top`을 켜고 `P`나 `M`을 눌러 자원을 독식하는 범인 프로세스를 가장 빠르게 찾아낼 수 있는 핵심 도구

### 🔷 섹션 4 — fg, bg, Ctrl-Z 를 이용한 작업 제어

**개념 요약**

- **작업 제어 (Job control):** 프로세스를 포그라운드와 백그라운드 사이에서 이동시키거나, 일시 중지 (stop) 및 재개할 수 있는 기능입니다.
- **`fg` (foreground):** 백그라운드 작업을 화면 앞으로 가져옵니다. `fg %작업ID` 형식으로 사용합니다.
- **`Ctrl-Z`:** 현재 포그라운드에서 실행 중인 프로세스에 중단 신호 (SIGTSTP)를 보내 실행을 일시 정지시킵니다. 프로세스가 종료된 것이 아니라 단지 중단(suspended)된 상태입니다.
- **`bg` (background):** 일시 정지된 작업을 백그라운드에서 다시 실행(재개)시킵니다.

**명령어 / 문법**

```bash
sleep 300 &
jobs
fg %1
(이후 `Ctrl-Z` 입력하여 일시 정지)
jobs #[1] + suspended sleep 300
bg %1 # [1]  + 472 continued  sleep 300
```

**실습 메모**

- 명령어 뒤에 `&`를 깜빡하고 엔터를 쳤더라도 당황해서 취소할 필요 없이, `Ctrl-Z`로 잠깐 멈춘 후 `bg`를 입력해 백그라운드로 밀어 넣을 수 있다

### 🔷 섹션 5 — renice 를 이용한 프로세스 우선순위 조정

**개념 요약**

- 리눅스에서 프로세스의 **나이스 (niceness)** 값은 다른 프로세스와 비교하여 CPU 시간을 얼마나 할당받을지를 결정합니다.
- 나이스 값 (NI)은 **-20 (가장 높은 우선순위)** 에서 **+19 (가장 낮은 우선순위)** 사이의 범위를 가집니다. (기본값은 보통 0)
- 나이스 값이 높을수록 다른 프로세스에 CPU 시간을 더 잘 양보한다는 의미에서 "나이스 (친절)"하다고 표현합니다.
- 일반 사용자는 자신이 소유한 프로세스의 나이스 값만 높일 수 있습니다. 즉, 우선순위를 낮춰 남에게 양보하는 것만 가능하며, 우선순위를 높이려면 관리자 권한이 필요합니다.

> **`-20` (가장 불친절함 / 이기적):** "내가 무조건 먼저야!" ➔ **가장 높은 우선순위**
> **`0` (기본값):** "보통 수준으로 할게." ➔ **일반적인 우선순위**
> **`+19` (가장 친절함 / 이타적):** "다들 먼저 하세요, 난 나중에 할게." ➔ **가장 낮은 우선순위**
> 이유 때문에 일반 사용자는 자신의 프로세스를 양보하게(숫자를 플러스로 올리게) 만들 수는 있지만, 이기적으로 자원을 독점하려는 행위(숫자를 0 미만인 음수로 내리는 것)는 시스템 전체를 느려지게 할 수 있으므로 오직 관리자(root) 권한으로만 가능하게 막아둔 것입니다.

**명령어 / 문법**

```bash
ps aux | grep sleep
ps -o pid,ni,cmd -p <YOUR_PID> # ps -o  pid,ni,cmd -p 670, 기본 나이스 값 (`NI`) 이 5 로 표시됩니다.
renice -n 10 -p <YOUR_PID> # renice -n 10 -p 670 # 670 (process ID) old priority 5, new priority 10
ps -o pid,ni,cmd -p 670
```

**실습 메모**

- 숫자 값이 작을수록 덜 친절해서 CPU를 독차지하고(우선순위 높음), 숫자가 클수록 친절해서 양보를 많이 함(우선순위 낮음). 무거운 작업을 돌릴 때 다른 중요한 시스템 작업에 방해되지 않도록 매너 있게 나이스 값을 올려두는 습관이 필요

### 🔷 섹션 6 —  `kill` 을 이용한 프로세스 종료

**개념 요약**

- 더 이상 필요하지 않거나 오작동하는 프로세스를 수동으로 중단시킬 때 `kill` 명령어를 사용합니다.
- `kill` 명령어는 지정된 프로세스에 신호 (signal)를 보냅니다.
- **SIGTERM (기본값):** 프로세스에 종료를 정중하게 요청하여 프로세스가 정리 작업 후 정상 종료되도록 합니다.
- 대상 지정은 PID 번호를 사용하거나, `%작업ID` 형식을 사용할 수 있습니다.
- **SIGKILL (신호 번호 9):** 프로세스가 응답하지 않을 때 사용하는 강력한 신호입니다. 프로세스에 정리할 기회를 주지 않고 즉시 강제 종료시킵니다. (예: `kill -9 %1`). 최후의 수단으로 사용해야 합니다.

**명령어 / 문법**

```bash
jobs
kill %1  (또는 kill <PID>)
kill -9 %1 # (강제 종료 필요 시)
jobs # 프로세스가 완전히 사라졌는지 확인하기 위해 `jobs` 명령어를 다시 실행합니다.
ps aux | grep sleep
```

**실습 메모**

- `kill`은 무조건 죽이는 게 아니라 "종료 신호"를 보내는 명령어
- 아예 먹통이 되어서 말을 듣지 않을 때만 `kill -9`로 강제로 날려버려야 데이터 손상을 막을 수 있다.
