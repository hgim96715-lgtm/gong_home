### 🔷 섹션 1 — 로컬 변수와 환경 변수의 생성 및 차이점 이해

**개념 요약**

```
로컬 변수   = 현재 셸에서만 존재 / env 에 안 나옴
환경 변수   = export 하면 env 에 등록 / 자식 셸에 전달됨

확인 방법:
  env | grep 변수명  → 환경 변수면 나옴 / 로컬이면 안 나옴
```

**명령어 / 문법**

```bash
# 로컬 변수
flower=rose
echo $flower       # rose
env | grep flower  # 아무것도 안 나옴 ← 환경 변수 아님

# 환경 변수
export nut=almond
echo $nut          # almond
env | grep nut     # nut=almond ← 환경 변수로 등록됨
```

**실습 메모**

### 🔷 섹션 2 — 자식 셸에서의 변수 및 별칭 상속 테스트

**개념 요약**

```
자식 셸 = 현재 셸 안에서 새로 실행한 셸
  bash 에서 zsh 입력하면 zsh 가 자식 셸

변수 상속:
  로컬 변수 (flower) → 자식 셸에 전달 안 됨
  환경 변수 (nut)    → 자식 셸에 전달됨

alias 상속:
  alias 는 자식 셸에 전달 안 됨
  zsh 로 들어가면 ldetc 명령어 없어짐
  → .bashrc / .zshrc 에 등록해야 영구 적용
```

**명령어 / 문법**

```bash
# 현재 셸 PID 확인
echo $$            # 123 (현재 bash PID)

# 자식 셸 시작
zsh

# 프로세스 확인
ps -f              # 부모(bash) + 자식(zsh) 둘 다 보임

# 변수 확인
echo $flower       # 빈 값 ← 로컬 변수라 전달 안 됨
echo $nut          # almond ← 환경 변수라 전달됨

# 별칭 선언 (부모 셸로 돌아가서)
exit               # 자식 셸 종료
alias ldetc='ls -ld /etc'
alias ldetc        # ldetc='ls -ld /etc' 확인
ldetc              # drwxr-xr-x 1 root root 4096 ...

# 자식 셸에서 별칭 확인
zsh
ldetc              # zsh: command not found: ldetc ← 전달 안 됨
```

**실습 메모**

```
ps -f 로 부모/자식 프로세스 관계 눈으로 확인
$$ 가 바뀌는 것 = 새로운 셸(새 프로세스) 진입 확인 방법
```

### 🔷 섹션 3 — set -o allexport 자동 변수 내보내기

**개념 요약**

```
set -o allexport  →  이후 선언 변수 전부 자동 export (on)
set +o allexport  →  자동 export 해제 (off)

-o = on  /  +o = off  ← 헷갈리기 쉬움!
```

**명령어 / 문법**

```bash
set -o allexport         # 자동 export 켜기

MY_VAR=hello             # export 없이도 환경 변수 됨
env | grep MY_VAR        # MY_VAR=hello 나옴

set +o allexport         # 자동 export 끄기

# 현재 옵션 상태 확인
set -o | grep allexport  # on / off 확인
```

**실습 메모**

### 🔷 섹션 4 — .bashrc 파일을 이용한 셸 옵션 영구 적용

**개념 요약**

```
.bashrc = 터미널 열 때마다 자동 실행되는 설정 파일
여기에 alias / export 넣으면 매번 자동 적용
수정 후 source ~/.bashrc 해야 즉시 반영
```

**명령어 / 문법**

```bash
# .bashrc 열기
vim ~/.bashrc
nano ~/.bashrc

# 추가 예시
export MY_VAR=hello
alias ll='ls -la'
alias ldetc='ls -ld /etc'

# 즉시 반영
source ~/.bashrc
. ~/.bashrc          # source 단축키
```

**실습 메모**

## ⚡ 핵심 명령어 한눈에 정리

|명령어|의미|
|---|---|
|`변수=값`|로컬 변수 선언|
|`export 변수=값`|환경 변수 선언|
|`env \| grep 변수명`|환경 변수 등록 여부 확인|
|`echo $$`|현재 셸 PID|
|`ps -f`|프로세스 계층 확인|
|`alias 이름='명령어'`|별칭 선언|
|`unalias 이름`|별칭 해제|
|`set -o allexport`|자동 export 켜기|
|`set +o allexport`|자동 export 끄기|
|`source ~/.bashrc`|설정 파일 즉시 반영|

## 🔴 헷갈렸던 것 / 새로 알게 된 것

```
alias 는 자식 셸에 전달 안 됨
  → zsh 들어가면 ldetc 없어짐 (command not found)
  → .bashrc / .zshrc 에 등록해야 영구 적용

set -o vs set +o
  -o = on (켜기)
  +o = off (끄기) ← 직관과 반대라 헷갈림

echo $$ 로 PID 확인
  자식 셸 들어가면 PID 바뀜 = 새 프로세스 진입 확인
```
