## 📌 랩 목표

표준 출력 (stdout), 표준 에러 (stderr), 표준 입력 (stdin) 이라는 세 가지 표준 스트림을 조작하여 명령어의 데이터 흐름을 제어하는 방법을 살펴봅니다.

### 🔷 섹션 1 — 연산자를 이용한 표준 출력 리다이렉션

**개념 요약**
```text
`>` 연산자는 파일이 존재하지 않으면 새로 생성합니다.
`>` 연산자는 아무런 경고 없이 기존 내용을 모두 덮어씁니다.
기존 파일 내용은 사라지고 새로운 내용으로 대체됩니다.
```

**명령어 / 문법**

```bash
echo "Hello from LabEx" > hello.txt
cat hello.txt

ls -l > file_list.txt
cat file_list.txt

echo "This is new content." > file_list.txt
cat file_list.txt

```

**실습 메모**

```text
 > 는 덮어쓰기라서 기존 내용이 사라짐
 echo 없이 문자열만 쓰면 command not found 발생
```

---

### 🔷 섹션 2 —  `>>` 연산자를 이용한 표준 출력 추가

**개념 요약**

```text
`>>` 연산자는 기존 파일에 내용을 추가합니다.
로그 파일을 만들 때 자주 사용됩니다.
`date` 명령어를 사용하여 타임스탬프를 추가해 보겠습니다.
```

**명령어 / 문법**

```bash
echo "Hello, this is the first line." > greetings.txt
cat greetings.txt # Hello, this is the first line.
echo "This is the second line, appended." >> greetings.txt
cat greetings.txt # Hello, this is the first line. This is the second line, appended.

date > activity.log
date >> activity.log
cat activity.log # Sat Apr 5 08:35:34 AM CST 2026 Sat Apr 5 08:35:42 AM CST 2026 
```

**실습 메모**

```text
 >> 는 기존 내용 유지 + 뒤에 추가
  로그 파일 만들 때 필수
```

### 🔷 섹션 3 —   `2>` 연산자를 이용한 표준 에러 리다이렉션

**개념 요약**

```text
stderr를 리다이렉션하려면 파일 디스크립터 2를 사용합니다.
`>` 는 stdout(1)의 줄임 표현입니다.
stderr는 기본적으로 파일에 저장되지 않습니다.
```

**명령어 / 문법**

```bash
ls non_existent_file # ls: cannot access 'non_existent_file': No such file or directory

ls non_existent_file 2> error.log
cat error.log. # ls: cannot access 'non_existent_file': No such file or directory

ls another_fake_file 2>> error.log
cat error.log # ls: cannot access 'non_existent_file': No such file or directory ls: cannot access 'another_fake_file': No such file or directory
```

**실습 메모**
```text
 > 만 쓰면 에러는 저장 안 됨
 에러는 반드시 2> 사용해야 함
```

### 🔷 섹션 4 —    표준 출력과 표준 에러를 단일 파일로 리다이렉션하기

**개념 요약**

```text
stdout과 stderr를 하나의 파일로 합칠 수 있습니다.
`2>&1`은 stderr를 stdout과 같은 위치로 보냅니다.
`&>` 는 이를 간단하게 표현한 문법입니다.
```

**명령어 / 문법**

```bash
ls -l /etc/passwd non_existent_file # ls: cannot access 'non_existent_file': No such file or directory -rw-r--r-- 1 root root 1916 Jul 18 2024 /etc/passwd

ls -l /etc/passwd non_existent_file > output_only.txt # ls: cannot access 'non_existent_file': No such file or directory

cat output_only.txt # -rw-r--r-- 1 root root 1916 Jul 18 2024 /etc/passwd

ls -l /etc/passwd non_existent_file &> combined.log
cat combined.log # ls: cannot access 'non_existent_file': No such file or directory -rw-r--r-- 1 root root 1916 Jul 18 2024 /etc/passwd


ls -l /etc/passwd non_existent_file > combined_traditional.log 2>&1
cat combined_traditional.log # ls: cannot access 'non_existent_file': No such file or directory -rw-r--r-- 1 root root 1916 Jul 18 2024 /etc/passwd
```

**실습 메모**

```text
stdout만 저장하면 에러는 화면에 출력됨
 &> 또는 2>&1로 둘 다 저장 가능
```

### 🔷 섹션 4 —     `tee` 를 이용한 출력 분할 및 `<` 를 이용한 표준 입력 리다이렉션

**개념 요약**

```text
tee는 출력을 화면과 파일에 동시에 출력합니다.
-a 옵션을 사용하면 파일에 내용을 추가합니다.

< 는 입력을 파일에서 가져오도록 합니다.
```

**명령어 / 문법**

```bash
ls /etc/ | tee etc_listing.txt # 전체 디렉토리 목록이 출력
cat etc_listing.txt # 


echo "banana" > items.txt echo "apple" >> items.txt echo "cherry" >> items.txt
sort < items.txt # apple banana cherry
cat < items.txt # banana apple cherry
```

**실습 메모**


```text
- tee는 출력 + 저장 동시에 가능
- < 는 입력 방향을 바꾸는 것
```

## ⚡ 핵심 명령어 한눈에 정리

```bash
command > file
command >> file
command 2> file
command 2>> file
command &> file
command > file 2>&1
command | tee file
command < file
```

## 🔴 헷갈렸던 것 / 새로 알게 된 것

```text
1. 문자열만 쓰면 명령어로 인식됨 → echo 필요
2. > 는 덮어쓰기, >> 는 추가
3. 에러는 따로 존재 → 2> 사용
4. stdout + stderr 합칠 수 있음
5. 리다이렉션은 순서 중요
6. tee는 출력 복제
7. < 는 입력 방향 변경
```
