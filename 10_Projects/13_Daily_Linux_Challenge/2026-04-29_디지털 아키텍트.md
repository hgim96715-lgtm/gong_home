## 📌 랩 목표

Linux 환경에서 프로젝트 디렉토리를 생성하고, 파일을 이동/복사/정리하며, 오래된 데이터를 압축 및 삭제하는 전반적인 파일 시스템 관리 명령어를 실습하여 실무적인 디렉토리 관리 능력을 기릅니다.

## 📖 학습 내용

### 🔷 섹션 1 — 프로젝트 디렉토리 구조 설정

**개념 요약**

- `mkdir` (Make Directory): 새로운 디렉토리(폴더)를 생성하는 명령어입니다. 띄어쓰기로 구분하여 여러 디렉토리를 한 번에 생성할 수 있습니다.
- `cd` (Change Directory): 작업할 디렉토리 경로로 이동합니다.
-  `ls -F`: 현재 디렉토리의 내용을 출력할 때, 디렉토리 이름 뒤에 슬래시(`/`)를 붙여주어 일반 파일과 시각적으로 쉽게 구분할 수 있게 해주는 옵션입니다.

**명령어 / 문법**

```bash
cd ~/project/phoenix_project
mkdir src config docs
ls -F
```

**실습 메모**

실무에서는 코딩을 시작하기 전 프로젝트 루트 폴더에 소스코드(`src`), 설정파일(`config`), 문서(`docs`) 등을 명확히 분리해 두는 것이 협업과 유지보수를 위한 필수적인 첫 단추

---

### 🔷 섹션 2 — 프로젝트 파일 이동 및 정리

**개념 요약**

- `mv` (Move): 파일이나 디렉토리를 다른 위치로 이동시키는 명령어입니다.
- 파일을 이동시킬 뿐만 아니라, 같은 경로 안에서 지정할 경우 **파일 이름을 변경**하는 데에도 사용됩니다.
- **기본 문법:** `mv [원본 파일명] [대상 경로]`

**명령어 / 문법**

```bash
mv main_app.py src
mv config.json config
mv README.md docs
ls -F
```

**실습 메모**

윈도우에서는 마우스로 드래그 앤 드롭을 하지만, 리눅스 환경에서는 `mv` 명령어로 빠르고 정확하게 파일들을 재배치합니다. 배포 스크립트를 작성할 때 빌드된 결과물을 특정 폴더로 밀어 넣을 때 매우 자주 쓰입니다.

---

### 🔷 섹션 3 — 중요 설정 파일 백업

**개념 요약**

-  `cp` (Copy): 파일이나 디렉토리의 복사본을 만드는 명령어입니다.
- 설정 파일이나 중요한 코드를 수정하기 전에, 원본을 보존하기 위해 `.bak`(백업) 확장자를 붙여 복사본을 만들어 두는 것이 안전한 작업 방식입니다.
- **기본 문법:** `cp [원본 파일명] [복사될 파일명]`

**명령어 / 문법**

```bash
cd config
cp config.json config.json.bak
ls
```

**실습 메모**

서버 환경이나 앱의 설정 파일을 수정하기 전에 복사본을 남기는 것은 인프라 엔지니어의 0순위 철칙입니다. 혹시라도 설정을 잘못 건드려 시스템이 다운되더라도, 원본으로 즉각 롤백(Rollback)할 수 있는 생명줄이 됩니다.

### 🔷 섹션 4 — 팀 공유 리소스 재구성

**개념 요약**

- `mv` 명령어는 단일 파일뿐만 아니라 **디렉토리 전체**를 이동시킬 때도 동일한 문법으로 작동합니다.
-  디렉토리를 이동시키면 그 내부에 들어있는 모든 파일과 하위 폴더까지 누락 없이 대상 경로로 함께 이동됩니다.
- **기본 문법:** `mv [원본_디렉토리] [대상_디렉토리]`

**명령어 / 문법**

```bash
mv ~/project/shared_docs ~/project/phoenix_project/docs/
ls ~/project/phoenix_project/docs/
```

**실습 메모**

별도의 폴더 복사/삭제 명령어를 조합할 필요 없이, `mv` 명령어 하나만으로 무거운 디렉토리 구조를 한 번에 재배치할 수 있어 시스템 디렉토리 리팩토링 시 매우 효율적입니다.

### 🔷 섹션 5 — 오래된 로그 파일 아카이브 및 삭제

**개념 요약**

- `tar` (Tape Archive): 여러 파일과 폴더를 하나의 파일(아카이브)로 묶어주는 강력한 리눅스 도구입니다. 주로 `gzip`과 결합하여 디스크 용량을 줄이는 **압축** 용도로 쓰이며 `.tar.gz` 확장자를 가집니다.
- `rm` (Remove): 파일이나 디렉토리를 삭제하는 명령어입니다. 와일드카드(`*`)를 활용해 특정 패턴의 파일을 일괄 삭제할 수 있습니다.
- **tar의 주요 옵션 (플래그):**
	- `c`: 새로운 아카이브 생성 (Create)
	- `z`: gzip을 사용하여 압축 (Zip)
	- `f`: 생성할 아카이브의 파일 이름 지정 (항상 옵션 맨 끝에 와야 함)
	- `t`: 압축을 풀지 않고 파일 내부 목록만 확인 (lisT)

**명령어 / 문법**

```bash
cd ~/project/logs
tar -czf old_logs.tar.gz *_2023-*.log
tar -tzf old_logs.tar.gz
rm *_2023-*.log
ls
```

**실습 메모**

서버를 장기간 운영하면 텍스트로 된 로그 파일들이 기가바이트(GB) 단위로 쌓여 디스크 공간 부족(100%) 장애를 유발합니다. 오래된 로그를 압축하여 보관하고 원본을 날리는 이 작업은 '로그 로테이션(Log Rotation)' 시스템의 가장 핵심적인 작동 원리입니다.
