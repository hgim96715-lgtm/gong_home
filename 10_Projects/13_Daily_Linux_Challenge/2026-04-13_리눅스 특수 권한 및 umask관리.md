### 🔷 섹션 1 —  환경 준비 및 디렉터리 접근

**개념 요약**

- 실습을 위한 작업 디렉터리를 생성하고, 특정 그룹(`research`)만이 접근할 수 있도록 소유 그룹과 권한을 초기 세팅

**명령어 / 문법**

```bash
sudo groupadd research
mkdir ~/project/RandD
sudo chgrp research ~/project/RandD
chmod 770 ~/project/RandD
ls -ld ~/project/RandD
cd ~/project/RandD
```

**실습 메모**

- `chgrp` 명령어는 `chown :그룹명` 과 동일한 역할을 하여 파일이나 디렉터리의 소유 그룹만을 변경할 때 유용
- 렉터리에 `770` 권한을 부여함으로써, 소유자와 `research` 그룹 외의 기타 사용자(Others)는 디렉터리 접근(`cd`) 및 내용 확인(`ls`)을 원천 차단

### 🔷 섹션 2 —  `umask 027`을 이용한 기본 파일 권한 제한

**개념 요약**

- `umask`(User file creation mode mask)는 새 파일이나 디렉터리가 생성될 때 기본적으로 부여되는 권한에서 **'제거할'** 권한을 지정합니다.
- 파일 생성 시 리눅스의 기본 최대 권한은 `666`(rw-rw-rw-)이며, 디렉터리는 `777`(rwxrwxrwx)입니다.
- `umask`를 `027`로 설정하면, 새로 생성되는 파일은 `666 - 027 = 640`(rw-r-----) 권한을 갖게 됩니다.

**명령어 / 문법**

```bash
umask 027
umask
```

**실습 메모**

- `umask`는 덧셈/뺄셈의 개념이라기보다 '마스킹(가리기)' 개념에 가깝습니다. 즉, 허용하지 않을 권한 비트를 빼앗는 원리입니다.
- `027`의 의미는 소유자의 권한은 그대로 두고(`0`), 그룹은 쓰기 권한을 빼앗고(`2`), 기타 사용자는 모든 권한을 빼앗는다(`7`)는 뜻

### 🔷 섹션 3 — 파일 생성 및 새로운 umask 권한 확인

**개념 요약**

- `umask` 명령어만 단독으로 치면 일시적으로 세션에만 적용되지만, `~/.bashrc`에 등록하면 로그인할 때마다 영구적으로 적용됩니다.

**명령어 / 문법**

```bash
touch schedule.odt
ls -l schedule.odt # -rw-r----- 1 labex research 0 Dec 12 10:35 schedule.odt
```

**실습 메모**

- 확인 결과 파일 권한이 `640`으로 생성되었습니다. 이는 파일을 만든 본인(`labex`)은 읽고 쓸 수 있지만, 같은 `research` 그룹원은 읽기만 가능하고, 그 외의 사용자는 아예 접근 불가함을 의미합니다. 보안이 한층 강화된 기본 설정입니다.

### 🔷 섹션 4 — chmod 1771을 이용한 디렉터리 스티키 비트 설정

**개념 요약**

- 일반적으로 디렉터리에 쓰기(`w`) 권한이 있으면, 해당 디렉터리 안의 모든 파일을 **(자신이 만든 파일이 아니더라도)** 삭제할 수 있습니다.
- 이러한 문제를 방지하기 위해 공용 디렉터리(예: `/tmp`)에 **스티키 비트 (Sticky Bit)** 를 설정합니다.
- 스티키 비트가 설정된 디렉터리에서는 오직 **'파일의 소유자'** 또는 **'디렉터리의 소유자(보통 root)'** 만이 파일을 삭제하거나 이름을 변경할 수 있습니다.
- 4자리 8진수 표기법을 사용하여 맨 앞자리에 `1`을 붙여 스티키 비트를 설정합니다. (`chmod 1771`)

**명령어 / 문법**

```bash
cd ..
chmod 1771 RandD # 
```

**실습 메모**

- 다수의 사용자가 협업하는 R&D(연구개발) 디렉터리 같은 곳에서 누군가 실수로 다른 사람의 결과물을 지우는 대참사를 막기 위한 필수적인 안전장치

### 🔷 섹션 5 —  `ls -l`을 이용한 스티키 비트 권한 확인

**개념 요약**

- `ls -l` 명령어를 통해 디렉터리의 권한 마지막 자리에 스티키 비트가 제대로 적용되었는지 시각적으로 확인
- 마지막 부분의 `t` 기호가 스티키 비트를 나타냅니다.
- 소문자 `t`는 기타 사용자의 실행 권한(`x`)이 있는 상태에서 스티키 비트가 설정되었음을 의미하고, 대문자 `T`는 실행 권한 없이 스티키 비트만 설정되었음을 의미합니다.

**명령어 / 문법**

```bash
ls -ld RandD # drwxrwx--t 2 labex research 4096 Dec 12 10:45 RandD
```

**실습 메모**
