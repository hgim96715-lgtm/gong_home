### 🔷 섹션 1 — mkdir, rmdir, rm 을 이용한 디렉토리 구조 생성 및 삭제

**개념 요약**

- **mkdir**: "make directory"의 약자.
- **-p 옵션**: 상위 디렉토리까지 한꺼번에 생성 (parents).
- **ls -lR**: 상세 정보(`-l`)와 재귀적 표시(`-R`)를 함께 사용.
- **rm -r**: 디렉토리 내부까지 삭제 (데이터 영구 삭제 주의).


**명령어 / 문법**

```bash
mkdir cars
ls -ld cars #drwxr-xr-x 2 labex labex 4096 May 20 10:30 cars

rmdir cars
ls -ld cars # ls: cannot access 'cars': No such file or directory


mkdir -p pastry/pies/cakes
ls -lR pastry

rm -r pastry
ls -ld pastry
```

**실습 메모**

- `mkdir -p`를 사용하면 중간 단계의 디렉토리가 없어도 오류 없이 한 번에 계층 구조를 만들 수 있어 편리함.
    
- `rm -r`은 비어있지 않은 디렉토리를 지울 때 필수적이지만, 복구가 불가능하므로 경로 확인이 매우 중요함.

### 🔷 섹션 2 — vi 와 ln -s 를 이용한 파일 및 심볼릭 링크 생성


**개념 요약**

- **vi 편집기**: 명령 모드와 입력 모드(`i`)가 있음. 저장 후 종료는 `Esc` -> `ZZ`
- **ln -s**: 심볼릭 링크(Symbolic Link) 생성. 원본의 '경로'를 가리킴.
- **특징**: `ls -l` 시 파일 유형이 `l`로 표시되며 `fileb -> filea`와 같이 포인터가 나타남.

**명령어 / 문법**

```bash
vi filea
ls filea # 파일 이름 filea
cat filea # This is filea. 방금 입력한 텍스트

ln -s filea fileb
ls -il file[ab]
# 131075 -rw-r--r-- 1 labex labex 14 May 20 10:40 filea 131076 lrwxrwxrwx 1 labex labex 5 May 20 10:41 fileb -> filea

cat fileb  # This is filea.
```

**실습 메모**

- 심볼릭 링크 `fileb`는 원본 `filea`와 다른 별개의 아이노드 번호를 가짐.
- `ls -l` 실행 시 권한 앞글자가 `l`로 표시되는 것을 통해 링크 파일임을 식별할 수 있음.

### 🔷 섹션 3 — ln 과 아이노드를 이용한 하드 링크 생성 및 분석

**개념 요약**
- **아이노드(inode)**: 파일 시스템 내 파일의 고유 식별 번호.
- **하드 링크**: 원본과 **동일한 아이노드**를 공유함 (`-s` 옵션 없이 사용).
- **특징**: 일반 파일(`-`)로 표시되며, 원본과 완벽히 동일한 데이터를 가짐.

**명령어 / 문법**

```bash
ln filea filec
cat filea cat fileb cat filec

ls -il file[a-c]
# 131075 -rw-r--r-- 2 labex labex 14 May 20 10:40 filea 131076 lrwxrwxrwx 1 labex labex 5 May 20 10:41 fileb -> filea 131075 -rw-r--r-- 2 labex labex 14 May 20 10:40 filec
```

**실습 메모**

- 하드 링크를 생성하면 `ls -l` 결과에서 링크 카운트(권한 뒤 숫자)가 증가함.
- 하드 링크는 원본과 똑같은 아이노드를 가리키기 때문에, 시스템 입장에서는 두 파일이 같은 데이터의 다른 이름일 뿐임.

### 🔷 섹션 4 — 원본 파일 수정 후 링크 동작 관찰

**개념 요약**

- **원본 삭제 시**:
	- **심볼릭 링크(`fileb`)**: 대상이 사라져 '깨진 링크'가 됨. `cat` 실행 시 오류 발생.
	- **하드 링크(`filec`)**: 원본 이름이 사라져도 데이터(아이노드)는 유지되므로 정상 작동.
- **원본 재생성 시**:
	- **심볼릭 링크(`fileb`)**: 이름이 같은 새 파일이 생기면 다시 연결됨.
	- **하드 링크(`filec`)**: 새 원본은 다른 아이노드를 가지므로, 기존 하드 링크와는 연결되지 않음.

**명령어 / 문법**

```bash
rm filea
ls -l file[a-c]
# ls: cannot access 'filea': No such file or directory lrwxrwxrwx 1 labex labex 5 May 20 10:41 fileb -> filea -rw-r--r-- 1 labex labex 14 May 20 10:40 filec

cat filec # This is filea.


## 다시 생성
vi filea  > This is the new filea. 
ls -il file[a-c] #131080 -rw-r--r-- 1 labex labex 20 May 20 11:05 filea 131076 lrwxrwxrwx 1 labex labex 5 May 20 10:41 fileb -> filea 131075 -rw-r--r-- 1 labex labex 14 May 20 10:40 filec

cat filea cat fileb cat filec
#This is the new filea. This is the new filea. This is filea.
```

**실습 메모**

- 심볼릭 링크는 '이름'을 따라가고, 하드 링크는 '데이터(아이노드)'를 따라간다
- 원본을 지웠다 다시 만들었을 때, 하드 링크는 여전히 과거의 데이터 상태를 보존하고 있음.

### 🔷 섹션 5 — cp 와 mv 를 이용한 파일 복사 및 이동

**개념 요약**

- **cp (copy)**: 복제본 생성. **새로운 아이노드**가 할당됨.
- **mv (move)**: 이름 변경 또는 이동. 동일 파일 시스템 내에서는 **아이노드가 유지**됨.
- **-i 옵션**: 덮어쓰기 전 확인 메시지를 띄우는 대화형(interactive) 옵션

**명령어 / 문법**

```bash
rm filea fileb filec
touch source_file
ls -i source_file # 131081 source_file

cp source_file copied_file
ls -i source_file copied_file
# 131082 copied_file 131081 source_file

mv source_file renmaed_file
ls -i renamed_file
# 131081 renamed_file

mkdir move_destinatiion
mv renamed_file move_destination/
# / 필수 
ls -i move_destination/renamed_file
# 131081 move_destination/renamed_file

cp -i copied_file move_destination/renamed_file
cp: overwrite 'move_destination/renamed_file'?
```

**실습 메모**

- `cp`는 데이터 전체를 새로 쓰기 때문에 아이노드가 변하지만, `mv`는 파일 시스템 상의 주소록(이름)만 고치는 것이라 아이노드가 변하지 않음.
- 이동할 때 목적지 경로 끝에 `/`를 붙여 디렉토리임을 명시하는 것이 안전함.
