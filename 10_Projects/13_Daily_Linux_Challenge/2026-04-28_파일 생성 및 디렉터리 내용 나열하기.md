## 📌 랩 목표

Linux 시스템의 기본 정보를 확인하는 핵심 명령어들을 익히고, 현재 접속 중인 사용자 식별부터 시스템 성능 모니터링, 그리고 수집한 정보를 텍스트 파일로 저장하여 보고서를 작성하는 과정을 실습합니다.

### 🔷 섹션 1 — 첫 로그인 및 환경 점검

**개념 요약**

- **사용자 확인:** `whoami` 명령어를 사용하여 현재 쉘을 실행 중인 사용자가 누구인지 확인
- **커널 확인:** `uname` 명령어를 사용하여 현재 운영체제의 핵심인 커널(Kernel) 이름을 확인합니다.

**명령어 / 문법**

```bash
whoami # labex
uname #Linux 
```

**실습 메모**

### 🔷 섹션 2 —시스템 정보 및 가동 시간 확인

**개념 요약**

- `uname -a` 옵션을 사용하면 커널 버전, 호스트 이름, 아키텍처(x86_64 등) 및 빌드 날짜를 포함한 상세 시스템 정보를 한 번에 볼 수 있습니다.
- `uptime` 명령어는 시스템이 마지막으로 부팅된 후 얼마나 지났는지, 현재 접속자 수, 그리고 시스템 부하 수치(Load Average)를 보여줍니다.
	- 시스템 평균 부하 (Load Average):  오른쪽의 세 숫자는 각각 최근 **1분, 5분, 15분** 동안의 평균 CPU/IO 부하를 의미합니다.

**명령어 / 문법**

```bash
uname -a # Linux 69f156a6633c710a0c90b341 5.4.0-162-generic #179-Ubuntu SMP Mon Aug 14 08:51:31 UTC 2023 x86_64 x86_64 x86_64 GNU/Linux

uptime # 08:56:59 up 236 days, 16:23,  0 users,  load average: 0.12, 0.21
```

**실습 메모**

- `uptime`의 부하 수치를 통해 서버가 현재 과부하 상태인지 판단할 수 있습니다. 1분 부하 수치가 15분 부하 수치보다 현저히 높다면 현재 서버에 갑작스러운 작업량이 몰리고 있다는 신호로 해석할 수 있습니다.

### 🔷 섹션 3 — 사용자 및 그룹 세부 정보 수집

**개념 요약**

- **식별자 조회:** `id` 명령어로 사용자의 고유 번호(UID)와 기본 그룹 번호(GID), 그리고 소속된 모든 그룹 목록을 확인합니다.
- `id` 명령어를 인자 없이 실행하면 기본적으로 현재 사용자의 정보를 보여줍니다.
- `uid`: 사용자 이름과 매칭되는 고유 번호입니다. (보통 일반 사용자는 1000번대 이상)
- `gid`: 사용자가 생성될 때 할당되는 기본 그룹입니다.
- `groups`: 사용자가 추가로 소속된 그룹들로, 특히 `sudo` 그룹에 속해 있다면 관리자 권한을 행사할 수 있습니다.

**명령어 / 문법**

```bash
id # uid=5000(labex) gid=5000(labex) groups=5000(labex),27(sudo),121(ssl-cert),5002(public)
```

**실습 메모**

### 🔷 섹션 4 — 실시간 시스템 성능 모니터링

**개념 요약**

- `top`은 윈도우의 '작업 관리자'와 같은 도구로, CPU, 메모리 사용량 및 실행 중인 프로세스 목록을 실시간으로 업데이트하여 보여줍니다.
- 화면 구성:
	- **상단:** 가동 시간, 부하(Load), 전체 프로세스 상태(실행/대기/좀비), CPU 및 메모리 점유율 요약.
	- **하단:** 각 프로세스별 PID, 사용자, 자원 소비량 정보.
-  **종료:** `q` 키를 누르면 즉시 명령 프롬프트로 복귀합니다.

**명령어 / 문법**

```bash
top
q
```

**실습 메모**

- `top` 화면에서 특정 프로세스가 CPU를 100% 가깝게 쓰고 있다면 무한 루프나 오류가 발생했을 가능성이 높습니다.

### 🔷 섹션 5 — 시스템 상태 보고서 생성

**개념 요약**

- `touch`: 빈 보고서 파일을 생성합니다.
- `>>`: **추가(Append) 연산자**입니다. 기존 파일 내용 뒤에 새로운 명령어 실행 결과를 덧붙입니다. (단일 `>`는 덮어쓰기이므로 주의)
- `cat`: 최종적으로 생성된 보고서 파일의 내용을 한 번에 읽어 확인합니다.

**명령어 / 문법**

```bash
touch system_report.txt
whoami >>system_report.txt
uname -a >>system_report.txt
uptime >>system_report.txt
cat system_report.txt
```

**실습 메모**
