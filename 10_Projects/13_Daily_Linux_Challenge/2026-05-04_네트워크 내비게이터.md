## 📌 랩 목표

Linux 환경에서 네트워크 인터페이스의 상태를 확인하고, IP 주소를 검증하며, 외부 통신 및 포트 대기 상태를 점검하는 필수 네트워크 진단 방법을 익힙니다. 또한 UFW를 활용해 서버의 기본 방화벽 규칙을 안전하게 구성하는 능력을 기릅니다.

---

## 📖 학습 내용

---

### 🔷 섹션 1 — 네트워크 인터페이스 상태 확인

**개념 요약**

- **`ip addr` 명령어:** 현대 리눅스 시스템에서 네트워크 인터페이스의 물리적 상태와 할당된 IP 주소를 한눈에 보여주는 표준 명령어입니다.
- 주요 확인 항목:
	- `lo` (Loopback): 서버 자기 자신을 가리키는 가상 인터페이스입니다.
	- `eth0` (Ethernet): 외부와 통신하는 메인 네트워크 카드입니다.
	- **상태 표시:** 네트워크 카드가 정상적으로 켜져 있고 활성화된 상태라면 `<UP>`이라는 플래그가 표시되어야 합니다.

**명령어 / 문법**

```bash
ip addr
```

**실습 메모**

원격 접속이 안 될 때 가장 먼저 의심해야 할 부분입니다. 인터페이스가 'DOWN' 상태라면 소프트웨어 설정 문제가 아니라, 물리적인 네트워크 연결이나 커널의 드라이버 로드 문제일 확률이 높다는 것을 빠르게 판단할 수 있습니다.

---

### 🔷 섹션 2 — IP 주소 구성 확인

**개념 요약**

- **`ifconfig` 명령어:** `ip` 명령어 이전부터 널리 쓰여온 고전적인 네트워크 검사 도구입니다. (`net-tools` 패키지에 포함되어 있습니다.)
- **IPv4 주소 찾기:** `ifconfig` 입력 시 출력되는 메인 네트워크 인터페이스(예: `eth0`) 항목에서 **`inet`** 필드 옆에 적힌 숫자가 바로 해당 서버가 통신에 사용하는 실제 IP 주소입니다.

**명령어 / 문법**

```bash
ifconfig
```

**실습 메모**

최신 클라우드 서버들은 `ip` 명령어를 권장하지만, 금융 데이터 서비스 기업이나 보수적인 레거시 인프라 환경의 서버를 다룰 때는 `ifconfig`가 기본 도구인 경우가 많습니다. 환경에 구애받지 않도록 두 가지 명령어 모두 능숙하게 다룰 줄 아는 것이 중요합니다.

---

### 🔷 섹션 3 — 원격 호스트 연결 테스트

**개념 요약**

- **`ping` 명령어:** ICMP 패킷을 목적지로 보내어 서버가 외부 인터넷과 정상적으로 통신할 수 있는지(살아있는지) 테스트하는 완벽한 도구입니다.
- **`-c` (count) 옵션:** 리눅스의 `ping`은 윈도우와 달리 중단 명령(`Ctrl+C`)을 내리기 전까지 **무한히** 패킷을 보냅니다. 따라서 `-c [숫자]` 옵션을 붙여 전송할 패킷의 개수를 제한하는 것이 안전합니다.

**명령어 / 문법**

```bash
 ping -c 3 8.8.8.8
```

**실습 메모**

쉘 스크립트를 작성하여 서버의 상태를 주기적으로 체크(Health Check)하는 로직을 짤 때 `-c` 옵션은 필수입니다. 횟수를 제한하지 않으면 백그라운드에서 `ping`이 무한히 돌면서 시스템 리소스를 낭비하게 됩니다.

### 🔷 섹션 4 — 열린 네트워크 포트 검사

**개념 요약**

- **`ss` (Socket Statistics) 명령어:** 현재 서버에서 어떤 포트가 열려있고 통신을 대기 중인지 빠르게 파악하는 현대적인 도구입니다
- 필수 플래그 (옵션) 조합 (`ss -tlnp`):
	- `-t` (TCP): TCP 소켓만 필터링해서 보여줍니다.
	- `-l` (Listening): 통신을 대기(리스닝) 중인 소켓만 보여줍니다.
	- `-n` (Numeric): 서비스 이름 대신 명확한 숫자로 포트 번호를 표기합니다.
	- `-p` (Process): 해당 포트를 점유하고 있는 실제 프로세스의 이름을 보여줍니다.

**명령어 / 문법**

```bash
ss -tlnp | grep 8000
```

**실습 메모**

- 애플리케이션을 실행했는데 접속이 안 될 때 가장 먼저 쳐야 하는 명령어입니다. 8000번 포트가 안 보인다면 애플리케이션 자체가 죽어있는 것이고, 포트는 보이는데 외부에서 접속이 안 된다면 다음에 배울 '방화벽' 문제일 확률이 높습니다. 트러블슈팅의 기준점이 되는 강력한 도구입니다.

### 🔷 섹션 5 — 기본 방화벽 규칙 구성

**개념 요약**

-  **`ufw` (Uncomplicated Firewall):** 리눅스의 복잡한 방화벽 규칙을 초보자도 쉽게 다룰 수 있게 해주는 사용자 친화적인 방화벽 관리 도구입니다. (네트워크 보안 설정이므로 항상 `sudo`가 필요합니다.)
- 작업 순서의 중요성:
	- 웹/포털 서비스 트래픽 허용 (`allow 8000`)
	- 서버 원격 관리를 위한 SSH 트래픽 허용 (`allow ssh` 또는 `allow 22`)
	- 방화벽 기능 최종 활성화 (`enable`)
	- **적용 상태 확인 (`status`):** 방화벽이 정상적으로 켜졌는지(Active), 그리고 허용한 포트 규칙들이 누락 없이 잘 등록되었는지 최종 점검합니다

**명령어 / 문법**

```bash
sudo ufw allow 8000 
sudo ufw allow ssh
sudo ufw enable
sudo ufw status
```

**실습 메모**

다수의 학생이나 사용자가 접속하는 에듀테크 서비스 포털을 오픈할 때 방화벽 설정은 타협할 수 없는 필수 사항입니다. **단, 가장 주의할 점이 있습니다.** 방화벽을 켜기(`enable`) 전에 **반드시 SSH 포트를 먼저 열어주어야 합니다.** 이 순서를 잊으면 방화벽이 켜지는 순간 내 원격 접속까지 차단되어 버려 서버와 영영 이별하게 되는 대참사가 발생합니다. 작업을 마친 후 `sudo ufw status`를 습관처럼 입력하여 내가 뚫어놓은 구멍이 안전하게 적용되었는지 더블체크하는 것이 사고를 예방하는 가장 좋은 방법입니다.
