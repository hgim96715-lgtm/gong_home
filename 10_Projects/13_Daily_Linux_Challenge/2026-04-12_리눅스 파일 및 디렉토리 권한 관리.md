### 🔷 섹션 1 — 작업 공간 준비 및 파일 생성

**개념 요약**

- 실습을 위한 독립적인 디렉토리를 생성하고, 권한 테스트를 위한 더미(dummy) 파일을 준비

**명령어 / 문법**

```bash
mkdir RandD
cd RandD
touch design_doc.odt
ls -l
```

**실습 메모**

- `touch` 명령어로 빈 파일을 빠르게 생성할 수 있습니다.
- 파일 생성 직후 `ls -l`을 통해 기본적으로 부여되는 소유자(현재 로그인한 계정)와 기본 권한(umask 설정에 따름)을 확인하는 습관을 들이는 것이 좋습니다.

### 🔷 섹션 2 —  `chown` 명령어로 파일 소유권 변경

**개념 요약**

- 소유권을 변경하는 데 사용되는 명령어는 `chown`입니다.
-  `-m` 플래그는 새 사용자의 홈 디렉토리를 생성하는 데 사용되며, 이는 일반적인 관례입니다. 이 작업 역시 `sudo`가 필요합니다.
- `chown`의 구문은 `chown 사용자:그룹 파일이름`입니다.

**명령어 / 문법**

```bash
sudo groupadd research
sudo useradd -m student1

sudo chown student1:research design_doc.odt
ls -l. # -rw-rw-r-- 1 student1 research 0 Jun 26 10:36 design_doc.odt
```

**실습 메모**

- 사용자 및 그룹 관리는 시스템 전반에 영향을 미치므로 반드시 `sudo` 권한이 필요합니다.
- `chown` 명령어에서 콜론(`:`)을 사용하면 소유자와 그룹을 한 번의 명령으로 동시에 변경할 수 있어 효율적입니다. 데이터 파이프라인이나 서버 환경을 세팅할 때 서비스 전용 계정으로 소유권을 넘기는 데 자주 사용됩니다.

### 🔷 섹션 3 —  `chmod` 명령어로 파일 권한 수정

**개념 요약**

-  `chmod`(모드 변경) 명령어를 사용하여 파일을 읽고, 쓰고, 실행할 수 있는 대상을 제어하는 방법
-  `rw-`: 소유자인 `student1`은 읽기 (**r**ead) 및 쓰기 (**w**rite) 권한을 가집니다.
- `rw-`: 그룹인 `research`는 이미 읽기 및 쓰기 권한을 가지고 있습니다.
- `r--`: 기타 사용자는 읽기 권한만 가집니다.
- 8진수(숫자) 표기법을 사용한 `chmod`로 빠르고 직관적으로 권한을 설정할 수 있습니다. (r=4, w=2, x=1)

**명령어 / 문법**

```bash
sudo chmod 660 design_doc.odt
ls -l # -rw-rw---- 1 student1 research 0 Jun 26 10:36 design_doc.odt
```

**실습 메모**

- `660`을 적용함으로써 소유자와 그룹 멤버 외의 '기타 사용자(Others)'의 권한을 완전히 차단(`0`)했습니다.
- 숫자 표기법은 각 자리수가 소유자(User) - 그룹(Group) - 기타(Others)를 의미한다는 점을 기억해야 계산이 빠릅니다 (4+2=6).

### 🔷 섹션 4 —  디렉토리 액세스 권한 제어

**개념 요약**
- 여기서 `-d` 플래그가 매우 중요합니다. 이는 `ls`가 디렉토리의 내용이 아니라 디렉토리 자체에 대한 정보를 나열하도록 지시합니다.
- 디렉토리에서의 권한은 파일과 다르게 동작합니다. `x`(실행) 권한이 있어야만 해당 디렉토리로 진입(`cd`)할 수 있습니다.

**명령어 / 문법**

```bash
cd ..
ls -ld RandD # drwxrwxr-x 2 labex labex 28 Jun 26 10:36 RandD

sudo chown :research RandD
chmod 770 RandD

ls -ld RandD # drwxrwx--- 2 labex research 28 Jun 26 10:36 RandD

```

**실습 메모**

- 디렉토리의 소유 그룹만 변경할 때는 `chown :새그룹명` 형태로 사용자 부분을 생략할 수 있습니다. (`chgrp` 명령어와 동일한 효과)
- `770` 권한 부여를 통해, 허가된 그룹(`research`) 멤버만 이 R&D 폴더의 흐름에 접근하고 내부 파일을 확인할 수 있도록 물리적인 차단벽을 세웠습니다.
