## 📌 랩 목표

데비안/우분투(Debian/Ubuntu) 계열 리눅스의 핵심 패키지 관리 도구인 APT(Advanced Package Tool)를 활용하여, 서버에 필요한 소프트웨어를 설치, 조회, 삭제하고 불필요한 찌꺼기 파일을 정리하는 시스템 관리 능력을 기릅니다.

---

## 📖 학습 내용

---

### 🔷 섹션 1 — 시스템 패키지 리포지토리 업데이트

**개념 요약**

-  **`apt update` 명령어:** 실제 프로그램을 설치하는 것이 아니라, "설치 가능한 소프트웨어 목록(인덱스)"을 최신 상태로 새로고침(동기화)하는 명령어입니다.
- 패키지 목록을 건드리는 시스템 핵심 작업이므로 반드시 관리자 권한(`sudo`)이 필요합니다.

**명령어 / 문법**

```bash
sudo apt update
```

**실습 메모**

- 실무에서 `apt install`을 쳤는데 "패키지를 찾을 수 없습니다(Unable to locate package)"라는 에러가 뜬다면 십중팔구 이 `update` 과정을 생략했기 때문입니다. 무언가를 설치하기 전에는 숨 쉬듯 자연스럽게 쳐야 하는 기본 명령어입니다.

---

### 🔷 섹션 2 — 필수 소프트웨어 패키지 설치

**개념 요약**

- **`apt install` 명령어:** 원하는 소프트웨어 패키지를 서버에 설치합니다.
- **`-y` 옵션 (자동 승인):** 설치 과정에서 묻는 "디스크 공간을 차지하는데 정말 설치하시겠습니까? (Y/n)" 질문에 자동으로 "Y"를 입력해 주어, 중간 멈춤 없이 쾌적하게 설치를 진행합니다.

**명령어 / 문법**

```bash
sudo apt install -y neofetch
```

**실습 메모**

쉘 스크립트를 짜서 여러 서버에 자동으로 동일한 프로그램들을 설치(자동화)할 때 `-y` 옵션은 절대 빠져선 안 됩니다. 이 옵션이 없으면 스크립트가 혼자 설치하다가 사용자 입력을 기다리며 영원히 멈춰버리는 대참사가 일어납니다.

---

### 🔷 섹션 3 — 설치된 패키지 버전 확인

**개념 요약**

- **`apt show` 명령어:** 특정 패키지의 버전, 용량, 설명, 의존성 등 상세한 명세서를 출력합니다.
- 출력된 결과 중 **`Version:`** 항목을 통해 현재 설치되어 있거나 설치될 정확한 버전 번호를 확인할 수 있습니다.
- (참고) `dpkg -s [패키지명]` 명령어를 통해서도 패키지의 설치 상태와 버전을 확인할 수 있습니다.

**명령어 / 문법**

```bash
apt show neofetch #Package: neofetch Version: 7.1.0-3
```

**실습 메모**

운영 서버와 테스트 서버의 환경을 똑같이 맞출 때, 혹은 특정 버전의 프로그램에서 치명적인 보안 취약점(해킹 위험)이 발견되었다는 뉴스를 들었을 때 우리 서버가 안전한지 검증하기 위해 가장 먼저 사용하는 명렁어입니다.

### 🔷 섹션 4 — 불필요한 소프트웨어 패키지 제거

**개념 요약**

- **`apt remove` 명령어:** 시스템에서 더 이상 사용하지 않는 특정 소프트웨어를 삭제합니다.
- 사용하지 않는 패키지를 방치하면 아까운 디스크 용량을 낭비할 뿐만 아니라, 잠재적인 해킹 통로(보안 취약점)가 될 수 있으므로 즉각 제거해야 합니다.

**명령어 / 문법**

```bash
sudo apt remove figlet -y
```

**실습 메모**

"언젠가 또 쓰겠지" 하는 마음으로 남겨둔 프로그램들이 모여 서버를 무겁게 만들고 보안을 위협합니다. 최소 권한, 최소 설치의 원칙에 따라 안 쓰는 도구는 가차 없이 지우는 것이 훌륭한 엔지니어의 미덕입니다.

### 🔷 섹션 5 — 패키지 의존성 관리

**개념 요약**

- **고립된 의존성(Orphaned Dependencies):** 메인 프로그램을 설치할 때 함께 딸려온 '보조 패키지'들이, 메인 프로그램을 지운 후에도 길을 잃고 남아 쓸데없이 용량을 차지하는 현상입니다.
- **`apt autoremove` 명령어:** 시스템을 스캔하여 더 이상 어떤 프로그램에서도 쓰지 않는 이런 불필요한 찌꺼기 패키지들을 알아서 찾아 일괄 삭제해 줍니다.

**명령어 / 문법**

```bash
sudo apt autoremove -y
```

**실습 메모**

`remove`가 단순한 방 청소라면, `autoremove`는 구석구석 쌓인 먼지와 쓰레기를 버리는 완벽한 뒷정리입니다. 실무에서 서버 디스크 용량이 꽉 차서(100% Full) 시스템이 멈췄을 때, 이 명령어 한 줄만 쳐도 꽤 많은 용량을 확보해 급한 불을 끌 수 있는 경우가 많습니다.
