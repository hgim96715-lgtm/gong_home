
# 🎨 인터파크 전시 트렌드 분석 데이터 마트

> **인터파크 크롤링 → PostgreSQL(Raw) → dbt(Transform) → 데이터 마트 → Superset** ELT 패러다임 기반 전시회 트렌드 분석 파이프라인

---

## 💡 기획 배경

옛날부터 전시회 보러 다니는 걸 정말 좋아해서, 주말마다 인터파크에서 전시 정보를 찾아보곤 합니다. 그러다 문득 궁금해졌습니다.

> **"요즘 사람들은 어떤 전시를 많이 볼까?"** **"지역마다 전시 분포가 다를까?"** **"가격대별로 어떤 전시가 인기 있을까?"** **"계절마다 전시 트렌드가 바뀔까?"** 

사람들의 전시 관심도는 어떻게 높을까? 등 옛날부터 전시회 분석 분포도에 대한 수치가 있으면 좋겠다고 느꼈습니다. 
인터파크 전시 데이터를 직접 수집하고 분석하면, 전시 시장의 트렌드를 데이터로 확인해볼 수 있겠다는 생각이 들었습니다. 직접 해볼수 있다니 기획 하면서도 즐겁습니다.

기존 프로젝트(열차, 병원)에서는 **ETL** 방식으로 Spark에서 데이터를 변환했다면, 이번 프로젝트에서는 **ELT** 방식으로 Raw 데이터를 먼저 적재하고 **dbt**로 Transform하는 현대적인 데이터 엔지니어링 패러다임을 경험해보고자 합니다.

또한, 요즘 트렌드가 ETL에서 ELT 방식으로 나아가고 있다는 말을 많이 들었습니다. 직접 두 방식을 모두 경험해보고 차이점을 체감하고 싶었습니다.

---
## ETL vs ELT 비교

| 항목               | ETL (기존 프로젝트) | ELT (이번 프로젝트)   |
| ---------------- | ------------- | --------------- |
| **Transform 위치** | Spark (외부 엔진) | dbt (DB 내부 SQL) |
| **Raw 데이터**      | 변환 후 적재       | 먼저 적재 후 변환      |
| **장점**           | 대용량 분산 처리     | 유연한 변환, 버전 관리   |
| **도구**           | Kafka + Spark | 크롤링 + dbt       |
| **적합한 상황**       | 실시간 스트리밍      | 배치 분석, 데이터 마트   |
|                  |               |                 |

## 기술 스택

| 역할        | 기술                                | 선택 이유                 |
| --------- | --------------------------------- | --------------------- |
| 데이터 수집    | Python + BeautifulSoup + Requests | 정적 페이지 크롤링            |
| Raw 저장소   | PostgreSQL 16                     | 기존 경험 활용, dbt 호환      |
| Transform | dbt Core                          | SQL 기반, 버전 관리, 문서화 자동 |
| 시각화       | Apache Superset 3.1               | 기존 경험 활용              |
| 스케줄링      | Apache Airflow 2.9                | dbt 실행 자동화            |
| 컨테이너      | Docker Compose                    | 재현 가능한 환경             |

## BeautifulSoup vs Selenium

|항목|BeautifulSoup|Selenium|
|---|---|---|
|JavaScript 렌더링|❌ 불가|✅ 가능|
|클릭/스크롤|❌ 불가|✅ 가능|
